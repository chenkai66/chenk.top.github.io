1. Two Sum

Problem

Given an integer array nums and an integer target, return the indices of the two numbers such that they add up to target. You may assume each input has exactly one solution, and you may not use the same element twice.

Approach

Use a hash map from value ‚Üí index. As you scan the array, for each num check whether target - num has appeared before. If yes, you‚Äôve found the answer; otherwise store num with its index and continue.

Where this pattern shows up

  • E-commerce: quickly find two items whose prices meet a budget constraint.
  • FinTech: match buy/sell orders or detect paired transactions under constraints.

2. Longest Consecutive Sequence

Problem

Given an unsorted array of integers, find the length of the longest consecutive elements sequence. The algorithm must run in (O(n)) time.

Approach

Put all numbers into a hash set for (O(1)) membership checks. For each number, only start counting if it is the beginning of a run (i.e., num - 1 is not in the set). Then extend forward (num + 1, num + 2, ‚Ä¶) to measure the run length. This ensures each number is visited a constant number of times overall.

Code

Where this pattern shows up

  • Social/engagement analytics: longest streak of consecutive active days.
  • Bioinformatics (analogy): detecting longest consecutive runs after mapping events to integers.

3. Group Anagrams

Problem

Given an array of strings, group the anagrams together. An anagram is a word formed by rearranging the letters of another word.

Approach

Use a hash map from a canonical representation ‚Üí list of original strings. A common canonical key is the sorted characters of the string (or a 26-length frequency signature). All anagrams share the same key, so they naturally land in the same bucket.

Code

Where this pattern shows up

  • Text processing: clustering normalized word forms to improve indexing.
  • Data cleaning: deduplicating noisy variants by grouping by canonical keys.
