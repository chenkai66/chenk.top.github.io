大模型工程（四）：Post-training —— SFT、DPO、RLHF、RLAIF

SFT、DPO、RLHF、RLAIF 各自具体在优化什么，奖励模型在哪里失败，KL 约束的作用，LoRA vs 全量微调那场争论，以及 2026 年生产里实际跑的 post-training 配方。

预训练出来的 base model 能续写文本，不会跟随指令、不会拒绝有害请求、不会保持人设——那些都是 post-training 行为。Post-training 也是论文承诺和生产级模型差距栖身的地方。这一篇讲每个 post-training 算法实际在优化什么、为什么大多数奖励模型微妙地坏掉、以及 2026 年真正跑得通的配方。

四阶段栈

现代 LLM post-training 大致是：

  1. SFT（supervised fine-tuning）on instruction data。教模型回答格式和基础指令遵循。
  2. 偏好优化（DPO、IPO、KTO 或 RLHF）。教模型在两个都合理的回复里挑人（或代理）偏好的那个。
  3. Online RL（RLHF、RLAIF 或 RLVR——可验证奖励）。对着奖励模型或程序化检查器更激进地调。可选，非推理模型越来越多直接跳。
  4. 专项阶段：tool-use SFT、长上下文 SFT、安全红队、constitutional AI 轮次。

OpenAI、Anthropic、Google 仍跑 “SFT → 偏好 DPO → RLHF/RLAIF” 类似的栈。DeepSeek-R1 和 o1 系列加了 RLVR（数学/代码这种正确性能用程序判的可验证奖励 RL）作为主信号。这是 2024-2025 年 post-training 最大的单一变化。

SFT：人们低估了它

SFT 就是 “给模型 10 万到 100 万 instruction-response pair，训 next-token prediction loss，loss mask 只盖 response”。那个 mask 很关键——你不希望模型学会预测用户的提问，只学预测助手的回答。

# SFT 的 loss masking
def sft_loss(logits, labels, response_mask):
    # logits: [B, T, V], labels: [B, T], response_mask: [B, T] in {0, 1}
    shifted_logits = logits[:, :-1, :].contiguous()
    shifted_labels = labels[:, 1:].contiguous()
    shifted_mask   = response_mask[:, 1:].contiguous().float()
    loss_per_token = F.cross_entropy(
        shifted_logits.view(-1, shifted_logits.size(-1)),
        shifted_labels.view(-1),
        reduction="none",
    ).view(shifted_labels.shape)
    return (loss_per_token * shifted_mask).sum() / shifted_mask.sum().clamp(min=1)

SFT 成败的两件事：

数据质量，不是量。 LIMA（Meta，2023）证明 1000 条精挑细选的数据能追上 5 万条平庸的。Tulu-3 mix（AllenAI，2024）精修到约 20 万条。Qwen3 SFT mix 接近 100 万但带重过滤。质量高的话，10 万之后剂量-反应曲线显著平。

格式一致性。 一半 SFT 数据带 “Sure, I’ll help!” 开头一半不带，模型就学不出稳定的开头。SFT 数据全是 Markdown 标题但测试 prompt 是普通散文，输出会错位。前处理要狠规范化格式。

一个出乎意料的失败模式：训了太多短回复，模型就拒绝写长的。模型会学到你数据上的条件长度分布。想要能写 2000 字的模型，SFT mix 至少要有 5-10 % 长样本。我们有过一个 Qwen3-7B 微调死活不肯写超过 800 token，因为 SFT mix 全是 Q&A。

DPO：不要奖励模型的偏好优化

经典 RLHF（InstructGPT，2022）配方是：在人类偏好上训奖励模型，再用 PPO 把策略对着这个奖励模型推。能跑，但实现痛苦——单独的奖励模型、value head、GAE、advantage normalization、KL 惩罚，训练还不稳。

DPO（Direct Preference Optimization），Rafailov 等 2023，把奖励模型干掉了。洞察是：你能从偏好关系导出策略的闭式解，得到的 loss 就是 log 概率比上的二元交叉熵：

$$ \mathcal{L}_{\text{DPO}} = -\mathbb{E}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w | x)}{\pi_{\text{ref}}(y_w | x)} - \beta \log \frac{\pi_\theta(y_l | x)}{\pi_{\text{ref}}(y_l | x)}\right)\right] $$

$y_w$ 是被选中的回复，$y_l$ 是被拒绝的，$\pi_{\text{ref}}$ 是冻住的 SFT 模型作参考，$\beta$ 控制策略允许漂多远。没有奖励模型，没有 PPO，每个偏好对一次前向 + 一次反向。

# DPO loss 10 行
def dpo_loss(policy_chosen_logps, policy_rejected_logps,
             ref_chosen_logps, ref_rejected_logps, beta=0.1):
    pi_logratios  = policy_chosen_logps - policy_rejected_logps
    ref_logratios = ref_chosen_logps   - ref_rejected_logps
    logits = beta * (pi_logratios - ref_logratios)
    return -F.logsigmoid(logits).mean()

2026 年 DPO 是主流，两个原因。一是它有效——HuggingFace alignment-handbook ↗ 报告 DPO 在 AlpacaEval 上跟 PPO 持平，单位质量成本更低。二是它只有前向训练循环，跟 FSDP/LoRA 栈干净集成。

会踩的坑：

  • β 调起来很影响。 β 太低（0.01）→ 策略漂移，base 能力退化。β 太高（1.0）→ 学不动。生产 run 大多在 β=0.1 到 0.3。
  • 参考模型漂移。 连续 DPO 两次，你的参考变成了上次 DPO 的模型，原始的没了。SFT checkpoint 一定要存，每次偏好 pass 都拿它做 ref。
  • 偏好数据质量。 合成偏好（“GPT-4 选了 A 不是 B”）容易造，但带 teacher 偏置。混 20 % 以上的人类偏好能防止坍缩。

值得知道的 DPO 变体：IPO（不同 loss 形式，避免在嘈杂偏好上过拟合）、KTO（用绝对 “好/坏” 标签代替成对——从生产日志收更容易）、SimPO（彻底丢掉参考模型，更简单但质量低些）。默认还是 DPO。

RLHF 和 PPO：为什么还有人用

虽然 DPO 在通用 post-training 占主导，几个前沿实验室仍然在最后阶段用 PPO-based RLHF：

  • 更擅长对抗用例。 PPO 能持续惩罚训练中发现的新失败模式。DPO 只能从你已经收集到的偏好里学。
  • 奖励 shaping。 你能加辅助奖励（长度惩罚、格式遵循、拒绝校准）——这些塞不进成对偏好里。
  • 迭代 RL。 Anthropic 的 constitutional AI（CAI）概念上是迭代 RLAIF——生成、批评、偏好、再训。

# 单训练 step（重度简化）
prompts = sample_prompts(batch)
responses = policy.generate(prompts)         # rollout
rewards   = reward_model(prompts, responses) # 每 token 一个标量
values    = value_head(prompts, responses)
advantages = compute_gae(rewards, values, gamma=1.0, lam=0.95)
for ppo_epoch in range(4):
    new_logps = policy.logprobs(prompts, responses)
    ratio = (new_logps - old_logps).exp()
    clip_loss = -torch.min(
        ratio * advantages,
        ratio.clamp(1 - 0.2, 1 + 0.2) * advantages
    ).mean()
    kl_loss = beta_kl * (new_logps - ref_logps).mean()
    total = clip_loss + kl_loss
    total.backward()
    optimizer.step()

KL 项是关键。没有它 PPO 会 hack 奖励模型——生成奖励高但跟连贯文本不沾边的东西。Anthropic 的 Sleeper Agents 论文（2024）记录了几个有意思的 reward hacking 模式，包括 “永远以 ‘Yes I can help with that!’ 结尾”，因为奖励模型学到了顺从开头预示高奖励。

RLVR：可验证奖励的 RL

DeepSeek-R1（2025 年 1 月）出货的模型，推理能力几乎全来自 RLVR：在数学题上训，奖励就是 “最终答案对不对”——程序判，不是学出来的模型。没有学习的奖励模型就没有 reward hacking。

数学和代码有 ground truth，所以这套能跑。模型生成长 chain-of-thought，末尾抽答案，checker（Python interpreter、sympy、单测）告诉你对不对。对，+1；不对，0 或 -1。对着这个 PPO/GRPO。

GRPO（Group Relative Policy Optimization），DeepSeek-V3 和 R1 用的，把 value head 也丢了——不学 value function，对同一个 prompt 采 $G$ 个回复，组内算相对 advantage：

$$ A_i = \frac{r_i - \text{mean}(r_1, ..., r_G)}{\text{std}(r_1, ..., r_G)} $$

比 PPO 简单很多，可验证奖励场景下效果好。

当前前沿（Qwen3-Reasoning、GPT-5-thinking、Claude-4.5-thinking、Gemini-3-Thinking）的推理能力主信号都是 RLVR 风格训练。base 经过 SFT、通用行为的 DPO，再在数学/代码/逻辑上狠跑 RLVR。这就是 thinking 模型在数学/代码上巨幅领先、对话上只略好的原因。

LoRA vs 全量微调

永恒的争论。2026 年的诚实答案：

  • 窄任务 SFT（风格迁移、单领域）：LoRA 赢。同样质量、1/10 的 GPU 内存、训得更快、容易合并或切换。rank 16-64 适用大多数任务。
  • 从根本扩展能力的 SFT（长上下文、新语种）：要全量。LoRA 改不动模型基础表征。
  • DPO：LoRA 适用。DPO 不需要大改模型。
  • RL（PPO/GRPO）：前沿实验室默认全量。最近有工作（LoRA-RL，2025）显示 LoRA 调好能追上 PPO，但脆弱。

config = LoraConfig(

    lora_dropout=0.05,
    bias="none",
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",

    task_type="CAUSAL_LM",
)

print(model.print_trainable_parameters())
# trainable params: 167,772,160 || all params: 7,408,357,376 || trainable%: 2.27

两个实战要点：

  • 所有线性投影都加，不只 QKV。 LoRA 原论文只加 QKV；后续工作（QLoRA、LongLoRA）显示 MLP 投影也必须加，质量才行。
  • DoRA（weight-Decomposed LoRA） 在同样 rank 下持续比 LoRA 好，额外开销可忽略。trainer 支持就用。

一个可上线的 instruction-tuned 模型配方

2026 年我要出一个生产模型，配方是：

  1. 挑最强的开源 base（Qwen3-32B-Base、LLaMA-3.3-70B-Base 或 DeepSeek-V3-Base，看预算）。
  2. SFT 5 万到 20 万精修样本。混通用 instruction（Tulu-3、OpenHermes）、领域专项（你的数据）、几千条长样本。预算够走全量；不够 LoRA r=64。
  3. DPO 1 万到 5 万偏好对。混 Anthropic HH、UltraFeedback、再加一些来自强 judge 模型的合成偏好。β=0.1，1-2 epoch。
  4. 可选安全 pass：对红队 prompt 收 1-5K 拒绝偏好；再 DPO 一遍 β=0.3 更激进。
  5. 可选推理 RL：数学/代码重要的话，几千道可验证题上 GRPO RLVR 跑 1-2K step。

这套大致能落到一个生产质量助手该有的邮编。剩下是评估（第 10 篇）和部署（第 5、12 篇）。

小结与下一篇

SFT 比人们承认的重要、对数据卫生比对量更敏感。DPO 替代了大部分偏好工作里的 PPO，但要持续适配新失败模式时 PPO 仍是对的工具。RLVR——可验证奖励上的 RL——是 2024 年后推理能力的解锁。生产微调多数 LoRA 赢；前沿实验室的 post-DPO RL 阶段全量。
