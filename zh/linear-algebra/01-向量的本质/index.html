线性代数（一）：向量的本质——不仅仅是箭头

向量无处不在 -- 从 GPS 导航到 Netflix 推荐。本章从空间中的箭头出发，构建直觉，一路推进到抽象向量空间，涵盖加法、数乘、内积、范数，以及线性性为何如此重要。

为什么要学向量？

物理学家说“力”，数据科学家说“特征”，游戏开发者说“速度”，量子理论家说“态”。表面上看，这些是不同的领域、不同的语言，但背后其实都指向同一个数学对象：向量。

这不是偶然。向量是数学中最灵活的基本构件，能够描述任何可以相加、可以缩放的东西。一旦你意识到这个模式，就会发现它无处不在。

入门课程通常会给出两种对向量的定义：

  • 一个有长度和方向的箭头。
  • 一组有序的数字。

这两种说法都没错，但也都不完整。完整的理解是这样的：向量就是任何存在于向量空间中的对象——一个支持加法和数乘操作的集合。箭头和数字列表是最常见的例子，但函数、信号、多项式、量子态同样也是合法的向量。

本章会一步步带你从具体到抽象，沿着以下四个层次展开：

  1. 几何视角：可以画出来的箭头。
  2. 数值视角：可以用来计算的数字列。
  3. 结构视角：内积——将几何与代数连接起来的关键。
  4. 公理视角：让这一切适用于任意向量的规则。

你将学到什么

  • 如何用几何（箭头）和数值（列表）两种方式理解向量
  • 向量的三种基本运算：加法、数乘、减法
  • 点积——代数与几何如何共同定义“对齐”的含义
  • 衡量向量大小的几种方法（范数），以及为什么需要多种方式
  • 向量空间的公理化定义——为什么函数也可以是向量

前置知识

  • 中学代数
  • 平面直角坐标系
  • 勾股定理

一、几何视角：向量就是箭头

1.1 从走路指令到向量

站在公园中央（当作原点）。朋友说：“向东走 4 步，再向北走 3 步。”

这句指令就是一个向量。我写成：

$$\vec{v} = \begin{pmatrix} 4 \\ 3 \end{pmatrix},$$

其中$4$表示东向（$x$分量），$3$表示北向（$y$分量）。

$$\|\vec{v}\| = \sqrt{4^2 + 3^2} = 5.$$$$\theta = \arctan\!\left(\frac{3}{4}\right) \approx 37^\circ.$$

所以一个向量把长度和方向这两个几何信息打包成了一个整体。

1.2 平移不变性：向量没有固定起点

初学者常常会惊讶于这一点：向量不在乎起点在哪。无论"4 东、3 北"是从公园中心开始，还是从东北角开始，都是同一个向量。只有方向和长度才是它的本质属性。

速度是最好的例子。一艘以 20 节速度向东航行的船，不管是在太平洋中央还是靠近西班牙海岸，速度向量都一样。位置每分钟都在变，但速度向量始终不变。

正因为如此，画向量时我通常把它放在原点——这是为了方便，不是必须的。

1.3 向量加法：三种等价方式

给定两个向量$\vec{a}$和$\vec{b}$，它们的和$\vec{a} + \vec{b}$可以用三种方式理解，结果完全一样。

首尾相接。 先沿$\vec{a}$走，再沿$\vec{b}$走，从起点到终点画一条箭头，这就是和。这是我们直觉上理解位移叠加的方式。

平行四边形。 把两个向量从同一点画出，补全平行四边形，从公共点出发的对角线就是和。物理学家叠加力的时候常用这种方法。

$$\begin{pmatrix} 3 \\ 1 \end{pmatrix} + \begin{pmatrix} 1 \\ 2.5 \end{pmatrix} = \begin{pmatrix} 4 \\ 3.5 \end{pmatrix}.$$

几何方法用来理解，分量公式用来计算。两者是一回事。

1.4 数乘：拉伸、压缩、反向

把向量$\vec{v}$乘以一个数$c$（标量），就是沿它所在直线重新缩放。结果分成四种情况：

-$c > 1$：在原方向上拉长。 -$0 < c < 1$：在原方向上缩短。 -$c = 0$：变成零向量。 -$c < 0$：反向，并按$|c|$缩放。

非零向量的所有数乘构成一条经过原点的直线——这就是第 2 章要讲的**张成（span）**的雏形。

import numpy as np

开车的例子能让这个概念更直观。如果$\vec{v}$是当前速度，那么$2\vec{v}$是加速到两倍快，$0.5\vec{v}$是减速到一半，$-\vec{v}$则是同样的车速来个掉头。

1.5 向量减法：从这里到那里

$$\vec{b} - \vec{a} \;=\; \text{从 } A \text{ 指向 } B \text{ 的位移向量}.$$

游戏引擎几乎每一帧都在用这个操作：玩家到目标的方向、子弹飞行的方向、两个航点之间的偏移——都是位置向量的差。

二、数值视角：向量作为数据

2.1 超越二维和三维

$$\vec{v} = \begin{pmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{pmatrix} \in \mathbb{R}^n.$$

虽然我无法想象 100 维的箭头，但所有操作——加法、缩放、点积、范数——都完全适用。这就是从“看得见的几何”到“只能计算的几何”的桥梁。

2.2 同一个对象，三种视角

这五个数$\{25.3,\,65.0,\,1013,\,15.2,\,45\}$，可以用三种不同的方式理解，但它们本质上是同一个向量。

物理学视角。 物理空间中的一支箭头，比如速度或力。

计算机科学视角。 一个特征向量：一组测量值，用来描述一个样本——这里表示某一时刻的天气状况（温度、湿度、气压、风速、云量）。

数学视角。 一列实数，属于$\mathbb{R}^5$。

三、内积：几何与代数的交汇点

内积（在$\mathbb{R}^n$中也叫点积）是本章最核心的运算。它隐藏了一个小奇迹：两个看似毫无关联的定义，最终却得出相同的结果。

3.1 两种定义，一个运算

$$\vec{a} \cdot \vec{b} \;=\; \sum_{i=1}^{n} a_i b_i \;=\; a_1 b_1 + a_2 b_2 + \cdots + a_n b_n.$$$$\vec{a} \cdot \vec{b} \;=\; \|\vec{a}\|\,\|\vec{b}\|\,\cos\theta,$$

其中$\theta$是$\vec{a}$和$\vec{b}$之间的夹角。

代数定义告诉你怎么算，几何定义告诉你它的意义：内积衡量了两个向量的方向一致性。方向相同，结果是大的正数；垂直时，结果为零；方向相反，结果是大的负数。

3.2 投影：最佳近似

$$\operatorname{proj}_{\vec{b}}\vec{a} \;=\; \frac{\vec{a}\cdot\vec{b}}{\vec{b}\cdot\vec{b}}\,\vec{b}.$$

这是*$\vec{b}$所在直线上离$\vec{a}$最近的点*。这不是巧合——最小二乘回归、主成分分析、信号滤波等许多"最佳线性近似"问题，背后都是这个思想。

import numpy as np

a = np.array([3, 4])
b = np.array([1, 0])
proj = (np.dot(a, b) / np.dot(b, b)) * b
print(proj)   # [3. 0.]

3.3 正交：独立性的几何表达

当$\vec{a} \cdot \vec{b} = 0$时，两个向量正交（即垂直），记作$\vec{a} \perp \vec{b}$。几何上，这意味着它们的夹角是 90°。为什么这值得专门命名？

  • 几何角度： 正交方向互不干扰——沿一个方向移动，不会影响在另一个方向上的投影。
  • 统计角度： 不相关的随机变量对应正交向量（协方差$= 0$）。
  • 计算角度： 正交基能把高维问题分解成多个独立的一维问题。数值线性代数里几乎所有高效算法都依赖正交性。

3.4 Cauchy–Schwarz 不等式

$$|\vec{a}\cdot\vec{b}| \;\leq\; \|\vec{a}\|\,\|\vec{b}\|,$$

等号成立当且仅当其中一个向量是另一个向量的标量倍。简单说：点积永远不会超过两向量长度的乘积。它是$|\cos\theta| \leq 1$的代数体现，也是整个分析学中最常用的不等式之一。

四、范数：多种测量“大小”的方法

我熟悉的“长度”公式 $\sqrt{x_1^2+\cdots+x_n^2}$ 是衡量向量大小的一种方式，叫 $L^2$ 范数。但这不是唯一的方法，选哪种范数会直接影响问题的几何特性。

4.1 最常用的三种范数

范数公式直观理解
$L^2$（欧几里得）$\sqrt{\sum_i x_i^2}$直线距离
$L^1$（曼哈顿）$\sum_i \lvert x_i \rvert$城市街区距离，只能沿街道走
$L^\infty$（最大值）$\max_i \lvert x_i \rvert$最差分量

为什么是三种？因为不同问题关注的重点不同：

-$L^2$ 光滑、可微（原点除外），还具有旋转不变性。它是默认选择，也是点积的最佳搭档。
-$L^1$ 有棱角。这种不光滑的特性反而是个优势：它会让优化过程倾向于把多个坐标压到零。这就是 LASSO 回归和压缩感知的核心思想。
-$L^\infty$ 适合只关心最差分量的场景，比如鲁棒控制、误差上界和最大误差保证。

4.2 单位球：范数的几何特征

要直观理解一个范数，可以画出它的 单位球——所有长度为 1 的向量组成的集合。三种范数对应三种截然不同的形状：

$L^1$ 的单位球是菱形，坐标轴上的尖角正是它产生稀疏解的原因。在优化过程中，最优解往往落在尖角上，而尖角意味着某些坐标正好为零。

有个让人放心的定理叫 范数等价性，它告诉我们：在有限维空间中，所有范数在定性上是一致的。一个数列在某种范数下收敛，当且仅当它在其他范数下也收敛。数值可能不同，但拓扑结构不会变。

五、抽象视角：从公理出发的向量空间

到目前为止，我一直把“向量”理解成“箭头”或者“一列数字”。但数学家发现了一件有趣的事：许多看似完全不同的东西——比如箭头、多项式、连续函数、随机变量、量子态——都遵循同一套规则。于是，他们把这些规则总结成了一个定义。

5.1 定义

一个向量空间$V$（定义在域$\mathbb{F}$上，通常是$\mathbb{R}$或$\mathbb{C}$）是一个集合，支持两种运算：

  • 向量加法：$\vec{u} + \vec{v} \in V$，
  • 数乘：$c\,\vec{v} \in V$，

并且满足十条公理（加法的交换律和结合律、零向量和逆元的存在性、分配律、数乘的结合律以及单位元）。这些公理是细节，核心思想其实很简单：

只要能相加、能被数乘，并且符合代数规则的东西，就是向量。

5.2 出人意料的向量空间

连续函数。 区间$[0,1]$上的所有连续函数构成一个向量空间。加法是逐点相加，$(f+g)(x) = f(x) + g(x)$；数乘也是逐点进行，$(cf)(x) = c\,f(x)$；零向量是恒等于 0 的常函数。这个空间是无穷维的——可以把函数看作一个在每个$x \in [0,1]$处都有一个分量的向量。

多项式。 次数不超过$n$的多项式构成一个$(n+1)$维向量空间，自然基是$\{1, x, x^2, \ldots, x^n\}$。

矩阵。 所有$m \times n$矩阵构成一个$mn$维向量空间。矩阵加法和数乘都是逐元素操作。

量子态。 一个量子态是复 Hilbert 空间中的单位向量。所谓的“叠加态”，其实就是穿了燕尾服的向量加法。

5.3 抽象的好处在哪里？

一旦在公理层面证明了一个定理，它就能用在任何满足这些公理的地方：

  • Cauchy–Schwarz 不等式适用于列向量、函数空间，也适用于随机变量（这时它变成了协方差不等式$|\operatorname{Cov}(X,Y)| \le \sigma_X \sigma_Y$）。一个定理，横跨三个领域。
  • 傅里叶分析的核心是“把函数分解为正交基向量的线性组合”——这和我们将在第 7 章对列向量做的操作完全一致。
  • 量子力学的本质就是在无穷维内积空间中研究物理。

这就是线性代数的最大价值：学会一套结构，终身受用。

6.3 GPS 定位归结为向量方程

$$\|\vec{x} - \vec{s}_i\| \;=\; d_i.$$

在二维平面上，两个方程会给出两个交点，第三个方程确定唯一解。在三维空间中，需要四颗卫星（第四颗用于修正接收机时钟偏差）。就这么简单——价值数十亿美元的系统，最终归结为几个向量方程。

七、常见误区

“向量必须从原点开始。” 不对。向量具有平移不变性，从原点画只是方便而已。

“向量就是一组数字。” 只对了一半。一组数字是向量的一种表示方法。连续函数、多项式和量子态也是向量，但它们无法用有限的数字列表来表示。

“点积和叉积差不多。” 完全不同。点积接收两个向量，返回一个标量；叉积（仅在$\mathbb{R}^3$中存在，$\mathbb{R}^7$中有类似形式）接收两个向量，返回一个向量。输入相同，输出完全不同。

“零向量有方向。” 错误。零向量是唯一没有明确方向的向量，这也是为什么像$\vec{v}/\|\vec{v}\|$这样的公式需要特殊处理的原因。

八、本章总结

概念核心思想
向量既有大小又有方向的对象——更广义地说，是向量空间中的一个元素
加法首尾相连、平行四边形法则或分量相加——三种方法等价
数乘沿同一直线拉伸、压缩或反向
内积衡量"对齐程度"；代数与几何在此一致
投影最优的一维近似——最小二乘法的原型
范数度量"大小"；不同范数体现不同侧重点
向量空间满足加法和数乘规则的任意集合——包括函数空间

核心启示

向量不是一支箭头，也不是一列数字。它是一种模式——一套简洁而有力的规则，描述了"能加能乘"的事物。一旦我开始留意，就会发现这种模式无处不在：图像、信号、量子态、金融组合、神经网络参数、概率分布中都能找到它的身影。线性代数研究的就是这种模式。

7.5 数值稳定性：向量运算在代码中出问题的时候

我想分享一个很多人容易忽视的坑：教科书上的公式 $\cos\theta = \vec{u}\cdot\vec{v} / (\|\vec{u}\|\,\|\vec{v}\|)$ 并不适合用来计算两个向量之间的夹角。下面用一段代码来说明问题：

import numpy as np
u = np.array([1.0, 1e-8])
v = np.array([1.0, 0.0])
c = np.dot(u, v) / (np.linalg.norm(u) * np.linalg.norm(v))
print(c)            # 0.9999999999999999
print(np.arccos(c)) # 1.4901161193847656e-08  -- 正确结果
u2 = np.array([1.0, 0.0])
v2 = np.array([1.0, 1e-9])
c2 = np.dot(u2, v2) / (np.linalg.norm(u2) * np.linalg.norm(v2))
print(np.arccos(c2)) # 0.0  -- 错了，真实角度是 1e-9

问题出在哪里？norm 函数内部需要对向量分量平方，$10^{-9}$ 平方后变成 $10^{-18}$，这已经低于双精度浮点数在接近 1 时的分辨能力（大约 $2.22\times 10^{-16}$）。结果 $c$ 被四舍五入成精确的 $1$，而 arccos(1) 的结果就是 $0$。

$$\theta = \operatorname{atan2}\bigl(\|\vec{u}\times\vec{v}\|,\ \vec{u}\cdot\vec{v}\bigr).$$

atan2 在处理接近 $0$ 或接近 $\pi$ 的角度时都能保持高精度。同样的道理也适用于其他场景：

  • Gram-Schmidt 正交化要用 modified 版本或者 Householder 反射，不要用经典版本；
  • 计算方差时要用 Welford 在线公式，而不是 $E[X^2]-(E[X])^2$；
  • Softmax 要先减去最大值，否则 $e^x$ 很容易溢出。

总结一下：这一章学到的代数在 $\mathbb{R}$ 上是精确的，但每一行代码实际运行在 $\mathbb{F}_{64}$ 中，这是一个只有约 $2^{64}$ 个有理数的有限集合。抵消、上溢和舍入不是边缘情况，它们直接决定了模型能不能正常训练。

7.6 np.dot 到底在做什么

当你对两个一维数组调用 np.dot(u, v) 时，numpy 并不会运行 Python 循环。它会直接调用 BLAS Level 1 的底层函数——通常是 OpenBLAS、MKL 或 Apple Accelerate 提供的 cblas_ddot，这些代码是用手工优化的 C 或汇编写的。对于长度为 $n$ 的向量，这个函数做了以下几件事：

  1. 检查内存对齐和步长。如果两个数组都是连续存储的，就走快速路径。
  2. 把循环拆成每组 4 或 8 个元素，利用 SIMD 指令（x86 上的 AVX2/AVX-512，ARM 上的 NEON），让一条 CPU 指令并行完成 4 或 8 个双精度数的乘加操作。
  3. 用多个寄存器分别累加部分结果，打破浮点加法的依赖链，最后再合并。这种方法比简单的顺序累加快 4 到 8 倍，而且——有点反直觉——结果在比特级别上会不一样，因为浮点加法不满足结合律。

性能开销：$\Theta(n)$ 次浮点运算，$\Theta(n)$ 次内存读取。在现代笔记本上，处理长度为 $10^6$ 的向量时，你应该能看到大约 5-10 GFLOPS 的性能，耗时在 0.1 毫秒左右。如果你测得的速度明显更慢，那说明 numpy 没有走 BLAS 的快速路径——通常是因为切片不连续（比如 u[::2]）或者数据类型是 object。

同样的逻辑也适用于更高维度的操作。对两个 $n \times n$ 矩阵执行 A @ B 时，调用的是 BLAS Level 3 的 dgemm 函数，它使用分块算法把数据尽量保留在 L1/L2 缓存中。这就是为什么手写三重循环的 Python 代码比 numpy 慢上千倍——不是因为 Python 单次乘法慢，而是因为它无法像 dgemm 那样高效利用缓存。

所以，当这一章说“内积就是 $\sum u_i v_i$”时，数学上没错。但在实际操作中，np.dot 是我见过的最高度优化的代码之一。相信它，别自己折腾。

九、接下来的内容

第 2 章：线性组合与向量空间 会探讨几个很自然的问题：

  • 怎么用少量向量构建出整个空间？（张成）
  • 什么时候某些向量是多余的？（线性无关）
  • 描述一个空间所需的最小工具集是什么？（基和维数）

这三个概念把“向量”这个静态的东西，变成了“坐标系”这种动态的结构。
