自然语言处理（十一）：多模态大模型

九月 24, 2025 Updated 5月 06, 2026 — min read — words

多模态大模型深度解析：CLIP的对比学习、BLIP-2的Q-Former桥接架构、LLaVA的视觉指令微调、Whisper语音识别、GPT-4V能力图谱以及MMBench/MME/MMMU评测体系——配可运行代码。

人不会只靠一种感官理解世界。我们看图表时读文字，听语调时读表情，看截图时讨论代码 bug。纯文本语言模型对这些又聋又瞎。多模态大语言模型（Multimodal LLM, MLLM） 试图填补这一鸿沟：将图像、音频、视频对齐到与语言模型相同的表示空间。

过去四年的发展可以用三步概括清楚。CLIP（2021）证明，从网上抓取 4 亿对带噪图文，就足以学到一个统一向量空间，使图像和文本通过余弦相似度互相检索。BLIP-2（2023）降低了对齐成本：冻结一个强力视觉编码器，再冻结一个强力 LLM，仅训练它们之间约 1.88 亿参数的"Q-Former"。LLaVA / GPT-4V（2023–2024）把桥接简化为一行线性投影，再加上视觉指令微调——将图像 embedding 当作词 embedding 输入给 LLM，再用 GPT-4 生成的视觉对话数据微调。最终，单个模型既能读图表、调试截图，又能像聊天助手一样描述照片。

本文沿着这条主线深入解析多模态 NLP：对比对齐背后的数学原理、可扩展的架构设计、音频侧的故事（Whisper）、GPT-4V 的能力边界，以及如何正确解读 MMBench / MME / MMMU 等评测——全部附上可运行代码。

¶视觉-语言模型

¶CLIP：对比视觉-语言对齐

CLIP（Radford 等人，OpenAI 2021）同时训练两个编码器。图像编码器 fIf_IfI​（ViT-L/14 或 ResNet）将图片 xxx 映射为向量 I=fI(x)∈R512I = f_I(x) \in \mathbb{R}^{512}I=fI​(x)∈R512。文本编码器 fTf_TfT​（12 层 Transformer）将文本 ttt 映射为同维度向量 T=fT(t)T = f_T(t)T=fT​(t)。两者均做 L2 归一化，因此 I⋅TI \cdot TI⋅T 即为余弦相似度。训练目标是让匹配对的相似度尽可能高，其他尽可能低。

具体而言，对于一个 batch 中的 NNN 对图文，定义相似度矩阵 sij=(Ii⋅Tj)/τs_{ij} = (I_i \cdot T_j)/\tausij​=(Ii​⋅Tj​)/τ，CLIP 最小化对称的 InfoNCE 损失：

L=−12N∑i=1N[log⁡esii∑jesij+log⁡esii∑jesji].\mathcal{L} = -\frac{1}{2N}\sum_{i=1}^{N}\Bigg[\log\frac{e^{s_{ii}}}{\sum_{j} e^{s_{ij}}} + \log\frac{e^{s_{ii}}}{\sum_{j} e^{s_{ji}}}\Bigg]. L=−2N1​i=1∑N​[log∑j​esij​esii​​+log∑j​esji​esii​​].

两项分别是图→文与文→图的分类损失：以批内 NNN 个候选做 softmax，对角线即为正确答案。温度 τ≈0.07\tau \approx 0.07τ≈0.07 是可学习标量，初始化为 log⁡(1/τ)=log⁡100\log(1/\tau) = \log 100log(1/τ)=log100 并裁剪以防崩溃。batch 越大负样本池越难：CLIP 训练 4 亿图文对时使用了跨上百块 GPU 的 32 768 batch。

这一目标天然导出三个性质：

  1. 零样本分类：把每个候选标签写成 "a photo of a {label}"，挑余弦相似度最高的那个即可，无需任何标注。
  2. 跨模态检索：图→文与文→图检索退化为点积索引（FAISS / ScaNN）。
  3. 生成模型条件信号：DALL·E 2、Stable Diffusion 的文本编码器、各种 image-to-image 系统都用 CLIP embedding 做监督。

干净的猫照片通常会在 "a photo of a cat" 上输出 >95%——而模型从未见过任何 "cat" 监督样本。

CLIP 的失败模式。CLIP 类似词袋：它会混淆 "a red cube on a blue sphere" 和 "a blue cube on a red sphere"。它不擅长计数（两个 vs. 三个）、细粒度类别（约克夏 vs. 诺里奇梗），也不擅长读图中文字（需要带 OCR 的模型）。中文场景请用 Chinese-CLIP 或通义千问 VL——原始 WIT 数据集压倒性以英文为主。

¶BLIP-2：连接冻结的视觉与语言

CLIP 只对齐不生成。BLIP-2（Li 等人，ICML 2023）解决生成问题，同时将预训练成本降到极低。图像编码器（ViT-g/14, 10 亿参数）和 LLM（OPT-2.7B/6.7B 或 FlanT5）均冻结，只训练桥接模块——Q-Former。

Q-Former 是一个 12 层 Transformer，关键设计有两点：

  • 32 个可学习的 query embedding 作为输入——这与图像 token 无关，就是 32 个可训练向量。
  • 每层都有交叉注意力层，让 query 关注冻结 ViT 输出的 patch 特征。query 因此从冻结的视觉特征中主动抽取与语言相关的信息。

经过 12 层后，32 个 query 变成 32 个"软视觉 token"，再经线性投影对齐到 LLM 的 embedding 维度，以前缀形式拼接到文本。LLM 将其视为普通词 embedding。

BLIP-2 分两阶段训练：

  1. 视觉-语言表示学习。Q-Former 只对接冻结的 ViT，用三个损失联合训练：图文对比（ITC，类似 CLIP）、图文匹配（ITM，硬负样本上的二分类）、图像条件文本生成（ITG，captioning 损失）。这一阶段教会 query "看懂"视觉内容。
  2. 视觉到语言生成。Q-Former 输出接到冻结 LLM，用标准语言模型损失在图像条件文本上训练。这一阶段教会 query "说 LLM 的语言"。

总可训练参数约 1.88 亿——OPT-6.7B 的约 1.4%。然而 BLIP-2 在 VQA-v2 与图像描述上的零样本表现可与端到端训练的更大模型相当。

¶LLaVA：视觉指令微调

BLIP-2 巧妙地预训练了一个桥。LLaVA（Liu 等人，NeurIPS 2023）做了更激进的事：将桥换成单层线性投影（或 2 层 MLP），专注于视觉指令跟随数据。

流程出乎意料地简单。取一个冻结的 CLIP ViT-L/14，得到 256 个 patch 特征 Zv∈R256×1024Z_v \in \mathbb{R}^{256 \times 1024}Zv​∈R256×1024，过一个可学习的投影器 WWW 映射到 LLM 的 4096 维 embedding 空间：

Hv=W⋅Zv.H_v = W \cdot Z_v. Hv​=W⋅Zv​.

将 HvH_vHv​ 与用户文本 token embedding 拼接喂给 Vicuna，自回归预测助手回答。训练分两阶段：

  1. 特征对齐。只训练 WWW，数据是 558K 图文对（LAION/CC/SBU）。LLM 完全不变——只让投影器学会将 CLIP 特征映射到 LLM 能读懂的空间。
  2. 端到端视觉指令微调。解冻 LLM，与 WWW 联合训练，数据是 158K 由 GPT-4 从 COCO 描述与边界框生成的图像-指令对。GPT-4 被引导生成三类对话：详细描述、对话问答、复杂推理。

单层线性投影最多 ~17M 参数，但在 MMBench 上 LLaVA-1.5（MLP-2 连接器）拿到 80.0——在自家擅长的基准上击败了更重的 Q-Former 设计。经验：对指令跟随 MLLM 来说，数据质量与端到端微调比连接器设计更重要。

¶视觉-语言任务：VQA、描述、检索

三类经典任务支配着评测和大部分实际应用：

  • 视觉问答（VQA）。给图像和问题，输出答案。数据集：VQA-v2、OK-VQA（开放知识）、GQA（组合）、TextVQA（需要 OCR）。现代指令微调 MLLM 用单模型即可处理。
  • 图像描述（Captioning）。生成自然语言描述。数据集：COCO Captions、NoCaps、Flickr30K。num_beams=3 配 length_penalty>1.0 通常在细节与流畅性间取得不错平衡。
  • 跨模态检索。给文本查询返回最相关图像（反之亦然）。退化为对归一化 CLIP embedding 做最近邻——生产系统用 FAISS-IVF 或 ScaNN，可扩展到十亿级。

¶音频：Whisper

语音是第二大非文本模态，Whisper（Radford 等人，ICML 2023）是开源主力。它基于普通的 encoder-decoder Transformer 架构，在 68 万小时互联网弱监督多语言音频数据上训练。单模型同时完成转写、翻译、语种识别和语音活动检测。

流水线：

  1. 将音频重采样为 16 kHz 单声道，切成 30 秒 chunk。短的补静音，长的滑窗。
  2. 将 chunk 转为 80 通道 log-mel 频谱，窗长 25 ms、跳长 10 ms——每 30 秒 chunk 得到 3000 帧。
  3. 使用两层 stride-2 Conv1D + GELU 降采样得到 1500 个音频 token，送入编码器（不同尺寸 12–32 层自注意力）。
  4. 解码器以自回归方式生成，前缀加上控制 token：<|startoftranscript|> <|en|> <|transcribe|> <|notimestamps|>。将 <|transcribe|> 替换为 <|translate|>，同一模型即可输出外语音频的英文翻译。

模型尺寸：tiny (39M), base (74M), small (244M), medium (769M), large (1.55B)。多数生产场景用 small 或 medium GPU 推理可达 0.1–0.3× 实时延迟，常见英文 WER < 10%。重口音、code-switching 或强噪声场景推荐使用 large-v3。

实战要点。处理长录音时务必关闭 condition_on_previous_text——否则前段一旦出现幻觉会级联放大。需要批吞吐时，请使用 faster-whisper（CTranslate2 后端，4–8× 加速）。说话人分割（who-spoke-when）可搭配 pyannote.audio。

¶GPT-4V 与前沿 MLLM 全景

GPT-4V（及其后续 GPT-4o、GPT-4.1）将图像视为与文本同等重要的输入模态，开箱即用，支持多种任务：

  • OCR 与文档理解：解析发票、表格、表单和截图。
  • 图表：提取数值、总结趋势、回答比较问题。
  • 示意图与流程图：解读 UML、系统图和手绘白板内容。
  • 代码与 UI：从代码截图中调试问题，根据渲染页面建议 CSS 修改。
  • 视觉推理：比较多张图片、分步解答照片中的考题、推断场景安全性。

detail 参数（"low" | "high" | "auto"）在成本与分辨率之间权衡——选择 "high" 时，图像会被切分为 512px 的子块，使用约 10 倍的视觉 token。

值得关注的开源替代方案：LLaVA-1.6（基线扎实、易于微调）、通义千问 VL-Max（中文与 OCR 表现突出）、InternVL（grounding 能力强）、CogVLM（感知能力优异）。这些模型可本地部署，既降低成本，又满足隐私合规需求。

MLLM 普遍存在的薄弱点（跨厂商一致）：

  • 计数：超过 5 个对象时表现不稳定。
  • 细粒度空间推理：例如「哪支笔离杯子更近？」。
  • 低分辨率密集文字：难以处理截图中的小字号内容。
  • 物体幻觉：诱导性 prompt 可能让模型描述不存在的对象（如要求「描述这只狗」，但实际没有狗）。

¶视频理解

视频是 4D 张量（时、高、宽、通道），逐帧处理会导致 token 数量爆炸。2024 年的主流方法如下：

  1. 均匀采样帧：在剪辑中均匀选取 N∈{8,16,32}N \in \{8, 16, 32\}N∈{8,16,32} 帧。
  2. 每帧视觉编码：冻结 ViT 编码，可选搭配时间池化适配器（VideoLLaMA、Video-LLaVA），先在投影前混合相邻帧。
  3. 拼接：将帧 token（或池化后的帧 token）拼接到 LLM 上下文中，通常添加时序位置编码以标明顺序。

针对长视频（几小时），采用粗到细策略：先用 captioner 总结小窗口，再用 LLM 在文字摘要上检索定位需要细看的几秒。

像 Gemini 1.5 Pro 这样的模型支持原生视频输入，通过长上下文注意力流式处理小时级视频。

¶MLLM 评测：MMBench、MME、MMMU、SEED、POPE

基准选择比想象中更重要。每个基准侧重不同方面，单一榜单的数字容易造成误导。

基准 测什么 形式 备注
MMBench 20 个能力维度：感知、推理、OCR、细粒度识别 4 选 1 选择题 "循环评测"轮换选项顺序，避免字母偏置。
MME 14 个子任务：感知 (10) + 认知 (4) 是/否问题 总分为各项准确率之和（最高 2800）。"全说 yes" 容易作弊。
MMMU 30 个学科大学级别题（医学、工程、艺术） 开放式 + 选择题 2024 最难——前沿模型仍未到 60%。
SEED-Bench 空间关系、实例计数、场景理解、时序 选择题 包含视频子集（SEED-Bench v2）。
POPE 物体幻觉探测 物体存在与否 真实与虚构物体配对，暴露幻觉。
TextVQA / DocVQA OCR 重的阅读理解 开放式 验证视觉塔能否真"读"。
RefCOCO 短语 grounding（返回 bbox） 区域预测 空间精度诊断。

合理的评测报告至少包括：一个感知基准（MME、SEED）、一个推理基准（MMMU）、一个幻觉基准（POPE），以及一个针对实际部署场景的领域内测试集。

¶多模态应用工程经验

一些上线后才学到的硬经验：

  • 预处理要狠。将图像缩放到模型原生分辨率（CLIP: 224，LLaVA-1.5: 336，GPT-4V high-detail: 768）。更大的输入不会带来更好的效果，只会浪费视觉 token。
  • 缓存图像 embedding。CLIP 编码的成本远低于每次查询都重新编码同一张图像——存储 512 维向量并复用即可。
  • 量化 LLM 那一半。对 LLaVA-1.5 的语言模型部分采用 4-bit AWQ/GPTQ，VRAM 从 14 GB 降至约 5 GB，MMBench 损失 < 1 分。
  • prompt 要扎根。措辞至关重要。"What color is the car?" 没问题；"What color is the car if there is one?" 在无车图上能将幻觉降低约 40%。永远允许模型显式回答"不知道"。
  • 当心 OCR 漂移。许多 MLLM 会先 OCR 图中文字，再“信任”它——照片角落的对抗性文字可能劫持回答。安全敏感部署必须清洗输入。

¶常见问题

Q：CLIP / BLIP / LLaVA 怎么选？
CLIP 适合检索、零样本分类，以及任何需要固定 embedding 的系统。BLIP-2 更适合冻结 LLM、算力有限时的 VQA 和 captioning。LLaVA 风格指令微调 MLLM，则用于对话式、指令驱动的多模态交互。

Q：BLIP-2 为什么训练参数那么少？
昂贵的部分——10 亿参数的 ViT 和 70 亿参数的 LLM——保持冻结。仅 1.88 亿参数的 Q-Former 在两者之间桥接。两阶段课程让桥接更专注：第一阶段扎根视觉，第二阶段对齐到 LLM 的词表。

Q：微调需要多少数据？
预训练 MLLM（如 LLaVA、Qwen-VL）的领域适配，通常 5K–50K 高质量图像-指令对即可，配合 LLM 的 LoRA 和投影器全量训练。从零开始预训练则需 10⁸–10⁹ 对。

Q：如何把 VRAM 压到 24 GB 以内？
三件事配合：LLM 的 4-bit 量化（bitsandbytes 或 AWQ）、注意力投影上的 LoRA、梯度检查点。7B LLaVA 在单张 RTX 4090 上微调毫无压力。

Q：怎样诚实地评测？
选择一个与用例对齐的主基准（如文档场景选 DocVQA）、一个幻觉基准（POPE）、一个独立的内部测试集。至少报告三个基准——单一榜单数字会掩盖 trade-off。

Q：CLIP 能用在中文上吗？
原始 OpenAI CLIP 95% 是英文数据。中文场景建议使用 Chinese-CLIP（OFA-Sys）、通义千问 VL 或 Wukong-CLIP。它们 API 兼容，可即插即用。
