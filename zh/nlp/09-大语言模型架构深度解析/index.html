自然语言处理（九）：大语言模型架构深度解析

九月 16, 2025 Updated 5月 06, 2026 — min read — words

拆解现代 LLM 的内部结构：Pre-norm + RMSNorm + SwiGLU + RoPE + GQA、KV Cache 机制、FlashAttention 的 IO 调度、稀疏 MoE，以及 INT8/INT4 量化。

2017 年，Transformer 论文画出了一个 block。如今，每一款生产级 LLM 依然沿用它的轮廓，但内部几乎每个零件都已更换：post-norm 换成 pre-norm，LayerNorm 换成 RMSNorm，GELU 换成 SwiGLU，正弦位置编码换成旋转位置编码（RoPE），多头注意力演进为分组查询注意力（GQA），稠密 FFN 在部分模型中被稀疏 MoE 替代。更重要的是，主导推理性能的数据结构——KV Cache——甚至没有出现在原论文中。

本文按照"实现/训练/部署时真正起作用的顺序"梳理这些改动：先剖析现代 decoder block，接着讨论为长上下文付出代价的 KV Cache，然后分析位置编码方式（RoPE / ALiBi），研究让 cache 缩小的注意力布局（GQA / MQA），探讨加速注意力计算的 IO 友好内核（FlashAttention），最后讲解"长大但不增加每 token 计算量"（MoE）以及"压小但不掉精度"（量化）的方法。

¶你将学到什么

  • 现代 block 布局：pre-norm、RMSNorm、SwiGLU、RoPE、GQA，每一项替换背后的动机。
  • KV Cache 机制：将前缀注意力从 O(n2)O(n^2)O(n2) 的重复计算优化为每步 O(n)O(n)O(n)，代价是什么，内存开销又是多少。
  • 位置编码：sinusoidal、RoPE、ALiBi 三种方案的不同设计思路。
  • 注意力变种：MHA、MQA、GQA 在 70B 规模模型上的具体缓存差异。
  • FlashAttention：通过 IO 感知调度让 tile 留在 SRAM，避免实例化 n×nn \times nn×n 分数矩阵。
  • MoE：top-kkk 稀疏路由让总参数增加，但每 token 的 FLOPs 保持不变。
  • 量化：FP16 → INT8 → INT4 的 GPTQ/AWQ 路线，以及精度与显存的权衡。

¶前置知识

  • 掌握基础 Transformer block（第四篇 — 注意力机制与 Transformer）。
  • 理解预训练模型概念（第五篇 — BERT 与预训练模型）。
  • 能读懂 PyTorch 代码和基础线性代数。

¶三大家族：encoder-only、decoder-only、encoder-decoder

进入现代 block 之前，先回顾为何几乎所有通用 LLM 都收敛到了 decoder-only。三种架构的差异仅在于注意力 mask：

家族 Mask 预训练目标 强项 代表
Encoder-only 双向 掩码语言模型（MLM） 语义理解 BERT、RoBERTa、DeBERTa
Decoder-only 因果（下三角） 下一 token 预测（LM） 生成、ICL GPT、LLaMA、Qwen、Mistral
Encoder-Decoder 双向 encoder + 因果 decoder + cross-attn 去噪 / span corruption seq-to-seq T5、BART、FLAN-T5

Decoder-only 在 scaling 上胜出有两个原因：第一，所有任务都能转为"预测下一个 token"，单一目标和数据格式支持无限扩展；第二，因果 mask 让前缀缓存（KV Cache）成本极低——encoder-decoder 还需缓存 cross-attention，而 encoder-only 根本无法生成。如今提到"LLM"，几乎默认指代带这些改动的 decoder-only 模型。

¶现代 decoder block

LLaMA 风格的 block（LLaMA、LLaMA-2、Mistral、Qwen、Yi、DeepSeek……）相比 2017 年的原始版本改动了五处。单看每一处影响都不显著，但叠加后造就了一个"无需 warmup 调参即可稳定训练、支持长上下文扩展、推理速度更快"的模型。

1. Pre-norm 替代 post-norm。 原版公式为 Norm(x + Sublayer(x))，现代版改为 x + Sublayer(Norm(x))。残差路径变为干净的恒等映射，深层堆叠时梯度尺度更稳定，同时省去了 Transformer 中著名的 warmup 学习率调整。

2. RMSNorm 替代 LayerNorm。 LayerNorm 减去均值再除以标准差，RMSNorm 则仅除以均方根，不涉及均值计算，也无需 bias：

RMSNorm(x)=x1d∑ixi2+ε⋅g.\mathrm{RMSNorm}(x) = \frac{x}{\sqrt{\frac{1}{d}\sum_i x_i^2 + \varepsilon}} \cdot g. RMSNorm(x)=d1​∑i​xi2​+ε​x​⋅g.

减少一次归约操作，少一组参数，质量未见可测损失。

3. SwiGLU 替代 GELU。 标准 FFN 结构是 Linear → GELU → Linear，SwiGLU 增加了一条 gating linear 分支：

SwiGLU(x)=(Swish(W1x)⊙(W3x))W2.\mathrm{SwiGLU}(x) = \big(\mathrm{Swish}(W_1 x) \odot (W_3 x)\big) W_2. SwiGLU(x)=(Swish(W1​x)⊙(W3​x))W2​.

逐元素乘法为 FFN 引入了乘性交互，在相同参数量下 perplexity 通常提升 1–2%。为了平衡参数预算，隐藏维度按 2/32/32/3 缩减。

4. RoPE 替代学习式绝对位置。 位置信息不再加在 embedding 上，而是在注意力计算时对 Q/K 进行旋转，下一节详细展开。

5. GQA / MQA 替代 MHA。 多个 query 头共享一个 KV 头，下下节详细展开。

¶KV Cache：为长上下文买单的数据结构

自回归解码每次只生成一个新 token。朴素实现中，新位置的注意力需要"所有历史 token 的 K 和 V"——如果每次都重新计算投影，单步复杂度是 O(n)O(n)O(n)，总复杂度达到 O(n2)O(n^2)O(n2)。KV Cache 的核心洞察是：这些投影仅依赖于历史 token，属于确定性函数，只需计算一次并存储即可。

这一机制成立基于两个前提：

  • 模型是因果的，已存入 cache 的位置不会因后续 token 改变；
  • K、V 投影对输入是线性的，缓存投影结果与重新计算等价。

代价并不低：cache 显存随序列长度线性增长，成为 decode 阶段的主要内存开销。以 LLaMA-2-70B 的形状为例（80 层、64 KV 头、head_dim=128、fp16）：

2⋅80⋅64⋅128⋅2 B=2.6 MB / token,2 \cdot 80 \cdot 64 \cdot 128 \cdot 2\text{ B} = 2.6\text{ MB / token}, 2⋅80⋅64⋅128⋅2 B=2.6 MB / token,

在 32K 上下文中，仅 cache 就占用 84 GB——比模型权重还大。这正是 GQA、MQA 和 PagedAttention（vLLM）诞生的根本原因。

¶位置编码：sinusoidal、RoPE、ALiBi

自注意力机制本身没有顺序：输入重新排列后，输出只是相应地重新排列。三种主流方案在三个不同位置注入位置信息。

Sinusoidal 绝对位置（Vaswani 等，2017）。 在 embedding 层将固定的 sin⁡/cos⁡\sin/\cossin/cos 向量加到 token 上。位置信息需要通过后续每一层线性投影传递，外推到训练长度之外时效果很差。

RoPE — 旋转位置嵌入（Su 等，2021）。 不在 embedding 上添加，而是在计算注意力时旋转 Q 和 K。把 ddd 维 head 切分为 d/2d/2d/2 个二维平面，平面 iii 的频率为 θi=10000−2i/d\theta_i = 10000^{-2i/d}θi​=10000−2i/d，位置 mmm 将第 iii 个平面旋转角度 mθim\theta_imθi​。关键恒等式：

⟨Rmq,  Rnk⟩=⟨q,  Rn−mk⟩,\langle R_m q,\; R_n k \rangle = \langle q,\; R_{n-m} k \rangle, ⟨Rm​q,Rn​k⟩=⟨q,Rn−m​k⟩,

点积仅依赖于相对偏移 n−mn - mn−m。这就是 RoPE 能够外推到比训练长度更长的上下文的原因，也是所有现代模型（LLaMA、Qwen、Mistral、Yi、DeepSeek、GPT-NeoX）都采用它的原因。

ALiBi — 线性偏置注意力（Press 等，2021）。 完全不使用位置嵌入，在 softmax 前的分数上增加一个按头不同的线性惩罚 −mh⋅∣i−j∣-m_h \cdot |i - j|−mh​⋅∣i−j∣。mhm_hmh​ 较小的头关注全局，mhm_hmh​ 较大的头关注局部。原论文中 ALiBi 的外推能力优于 RoPE，但因未注入相对相位，在知识密集型长上下文 benchmark 上通常表现不如 RoPE。BLOOM、MPT 曾使用它。

工程实践表明：RoPE 配合 NTK-aware scaling、YaRN、position interpolation 等后训练技巧，可以让训练长度为 4K 的模型微调至 32K–128K，已成为绝对主流。

¶注意力变种：MHA → GQA → MQA

标准多头注意力（MHA）中，每个 query 头都有独立的 K、V 投影。KV Cache 的大小与 KV 头数成正比，这正是上一节计算得出 84 GB 的原因。以下是两种折中方案：

  • MQA — 多查询注意力（Shazeer，2019）： 所有 query 头共享一个 KV 头，cache 缩小 HHH 倍。但 K/V 投影失去头多样性，在复杂任务上质量显著下降。
  • GQA — 分组查询注意力（Ainslie 等，2023）： LLaMA-2-70B、Mistral 等模型普遍采用的中间方案。将 HqH_qHq​ 个 query 头分成 GGG 组，每组共享一个 KV 头。LLaMA-2-70B 使用 Hq=64H_q = 64Hq​=64、G=8G = 8G=8，cache 缩小 8 倍，质量却与 MHA 持平。

LLaMA-2-70B 形状下的具体数字（fp16，每 token）：

  • MHA，64 KV 头 → 2.56 MB/token → 32K 上下文 80 GB；
  • GQA-8，8 KV 头 → 0.32 MB/token → 32K 上下文 10 GB；
  • MQA，1 KV 头 → 0.04 MB/token → 32K 上下文 1.25 GB。

GQA-8 几乎实现了 MQA 的全部内存收益，同时质量基本无损——这正是近期开源权重模型默认采用它的原因。

¶FlashAttention：相同的数学，IO 感知的调度

朴素注意力内核会将 S=QK⊤S = QK^\topS=QK⊤ 整个 n×nn \times nn×n 分数矩阵写入 HBM（GPU 主显存），在 HBM 上运行 softmax，再乘以 VVV。当 n=8192n=8192n=8192、使用 fp16 时，仅 SSS 一个矩阵每个 head 每层就占用 128 MB，绝大多数流量被反复读写浪费。

FlashAttention（Dao 等，2022）完成的是同一份数学，但采用不同的调度：

  1. 将 QQQ、KKK、VVV 切分为行/列 tile，使其能够放入每个 SM 上约 192 KB 的 SRAM；
  2. 在 SRAM 内一次只计算一个 SSS tile；
  3. 使用在线 softmax：维护逐行的最大值 mmm 和分母 ℓ\ellℓ，每个 tile 增量更新部分输出，整行 SSS 无需存在；
  4. 只将最终 OOO 写回 HBM。完整的 SSS 始终不会出现。

结果是数学上完全等价（仅浮点归约顺序不同）的注意力，HBM 流量从 O(n2)O(n^2)O(n2) 降至 O(n)O(n)O(n)，当 n≥2048n \geq 2048n≥2048 时墙钟提速 2–4 倍，显存最高节省 8 倍。FlashAttention-2 进一步优化 warp 级任务划分，性能可达到 A100 FP16 峰值的约 70%。

记住：FlashAttention 不改变模型计算的内容，只调整 GPU 上的算子顺序。序列长度超过 ~1K 时一律应该启用。

¶混合专家：长大但不增加每 token 计算量

稠密 Transformer 的参数和 FLOPs 主要集中在 FFN。MoE 将单个 FFN 替换为 NNN 个"专家"FFN，再加一个微型 router，对每个 token选择 top-kkk 个专家执行：

y=∑i∈TopK(Wgx)gi(x) Ei(x),g(x)=softmax(Wgx).y = \sum_{i \in \mathrm{TopK}(W_g x)} g_i(x)\, E_i(x),\qquad g(x) = \mathrm{softmax}(W_g x). y=i∈TopK(Wg​x)∑​gi​(x)Ei​(x),g(x)=softmax(Wg​x).

每 token 的 FFN FLOPs 随 kkk（通常为 2）增长，而非随 NNN 增长。因此，8 专家模型的 FFN 参数大约是稠密模型的 8 倍，但 FFN 计算仅增加 2 倍。Mixtral 8×7B 总参数 47B，每 token 只激活约 13B，获得 70B 量级的质量、13B 量级的推理成本。

代价体现在其他方面：

  • 显存。 所有专家必须常驻，即使只运行 kkk 个。Mixtral 8×7B fp16 需 ~94 GB，比稠密 70B 还大；INT4 后才能装进 24 GB。
  • 负载均衡。 未干预的 router 会塌缩到少数偏爱的专家。真实训练需加入辅助负载均衡损失（Shazeer 2017、Switch Transformer），并添加小幅 router 噪声以鼓励探索。
  • All-to-all 通信。 多卡训练时，每个 token 需跑到目标专家上再返回，对互联拓扑极为敏感。

（生产实现会用 grouped GEMM 和容量限制 bucket 并行调度 token 到专家，上面的循环仅为清晰展示。）

¶量化：每个权重少几比特

现代 LLM 权重通常用 BF16 训练（每参数 2 字节），70B 模型需要 140 GB 显存，单张 A100 80GB 或 H100 80GB 无法容纳。量化将每个权重替换为低比特整数，并附加一组 per-block scale，以小幅精度损失换取显存节省。

对称 INT8。 选择一个 per-tensor（或 per-channel）scale s=max⁡∣w∣/127s = \max|w| / 127s=max∣w∣/127，存储 w^=round(w/s)∈[−127,127]\hat w = \mathrm{round}(w / s) \in [-127, 127]w^=round(w/s)∈[−127,127]。计算时恢复 w≈s⋅w^w \approx s \cdot \hat ww≈s⋅w^。显存减少一半，INT8 tensor core 的吞吐量大致翻倍。

INT4 + GPTQ（Frantar 等，2022）。 直接将每个权重独立量化到 4 bit 会导致精度大幅下降。GPTQ 改为按列量化，每完成一列后，使用一小批校准数据估计的 Hessian 修正剩余未量化列，以补偿当前列的舍入误差。结果表明，7B 以上模型在 4 bit 量化后 perplexity 损失小于 1%。

INT4 + AWQ（Lin 等，2023）。 AWQ 发现，权重通道中仅有约 1% 是"显著"的（由大激活值驱动）。只要保护这部分通道（通过 per-channel scaling，并非真正保留 fp16），即可恢复绝大部分精度。AWQ 比 GPTQ 计算快得多，成为当下许多量化开源 checkpoint 的默认方案。

精度 字节/参数 LLaMA-2-7B 权重 LLaMA-2-70B 权重 典型 PPL 增量
FP16 / BF16 2.0 13.5 GB 140 GB 参考基准
INT8（RTN） 1.0 6.7 GB 70 GB <0.5%
INT4（GPTQ / AWQ） 0.5 3.4 GB 35 GB <2%
INT3（高级算法） 0.375 2.5 GB 26 GB 3–6%

激活通常仍保留 FP16/BF16，因为它们的分布具有重尾特性（LLM.int8()，Dettmers 2022 发现少量"离群特征"承载了大部分激活能量，必须保留高精度）。仅权重 INT4 是当前推理的最佳折中。

¶拼起来：高吞吐推理栈

这些组件可以自由组合。一个现代的 vLLM 服务栈将以下内容整合到一起：

  • LLaMA 风格 decoder block（pre-norm + RMSNorm + SwiGLU + RoPE + GQA）；
  • 每层 KV Cache，但采用固定大小的页布局（PagedAttention），上下文增删不再产生碎片；
  • 预填和解码均使用 FlashAttention 内核；
  • 权重 INT4 量化（AWQ / GPTQ）节省显存；
  • 连续批处理：某条请求结束后空出的 slot 立即被新提示填满，无需等待批中最慢的请求。

实测结果显示，单张 A100 80GB 运行 7B AWQ 模型能达到 3000–5000 tokens/s 的聚合吞吐，单张 H100 运行 Mixtral 8×7B AWQ 能达到 >1000 tokens/s——这两个数字在 2017 年原版 block 的基础上是难以想象的。

¶常见问题

为什么选择 decoder-only 而非 encoder-decoder？
因果 decoder-only 模型只需更换提示，就能充当分类器、翻译器或对话机。而 encoder-decoder 需要额外的 cross-attention 参数，训练流程更复杂，生成时也无法利用廉价的前缀缓存。此外，decoder-only 更适合扩展，因为语料中的每个 token 都是训练信号。

RoPE 还是 ALiBi？
工程实践中几乎都选择 RoPE：它为模型提供真正的相对相位信息，配合 NTK / YaRN 等后训练缩放技术，可将上下文从 4K 扩展到 32K–128K。ALiBi 的优势在于无需微调即可外推，但代价是 in-distribution 性能较弱。目前主流开源权重全部采用 RoPE。

FlashAttention 是否会改变模型输出？
不会。FlashAttention 是精确注意力算法，仅浮点归约顺序不同，与朴素内核的数值差异远低于训练噪声。

MoE 是否总是比稠密模型便宜？
计算成本更低，但显存占用更高。如果显存受限（例如单张 24GB 显卡推理），量化的稠密模型通常优于 MoE；如果多卡显存充足但 decode FLOPs 是瓶颈，MoE 更具优势。

7B 模型 INT4 量化损失多少精度？
使用 GPTQ 或 AWQ 配合约 128 条校准样本，perplexity 增量通常小于 2%，绝大多数下游任务的质量肉眼难以分辨。70B 模型的损失通常小于 1%。对于 7B 以下模型，量化难度显著增加。

为什么 KV Cache 是长上下文的瓶颈？
KV Cache 大小为 2⋅L⋅Hkv⋅dh⋅T⋅字节数2 \cdot L \cdot H_{kv} \cdot d_h \cdot T \cdot \text{字节数}2⋅L⋅Hkv​⋅dh​⋅T⋅字节数。70B 模型在 32K 上下文中使用 MHA 时，cache 占用约 80 GB，超过权重大小。GQA 将其降至约 10 GB，PagedAttention 又将碎片率压低至小于 5%。两者结合才让长上下文服务能在普通硬件上高效运行。
