自然语言处理（五）：BERT与预训练模型

八月 31, 2025 Updated 5月 06, 2026 — min read — words

BERT 如何让双向预训练成为 NLP 的默认范式：从架构到 80/10/10 掩码规则，到微调技巧，再到 RoBERTa/ALBERT/ELECTRA 全家桶，并附完整 HuggingFace 代码。

2018 年 10 月，Google 推出 BERT，在 11 个 NLP 基准测试中一举刷新纪录。方法极其简洁：使用一个 Transformer 编码器，根据上下文左右两侧预测随机遮盖的词；预训练完成后，针对各类下游任务进行轻量微调即可。在 BERT 出现之前，每个任务都需要从零开始训练模型；BERT 之后，“先预训练，再微调”成为整个领域的标准范式。

过去几年里，如果你用过情感分析 API、能理解意图的搜索引擎，或者稍微靠谱的客服机器人，背后大概率是 BERT 或其某个改进版本在发挥作用。

¶预训练-微调范式是怎么火起来的

BERT 之前，每个 NLP 任务都从随机初始化模型起步，用少量标注数据从零训练。这种方式昂贵（算力成本高）、浪费（任务间不共享知识），而且脆弱（数据不足时模型性能波动大）。整个领域如何摆脱这个困境，可以归结为四个里程碑式的工作。

¶一段简短的演进史

Word2Vec（2013）：从无标注文本中学习静态词向量。问题在于，"bank"在 river bank 和 bank account 中得到的向量完全相同——上下文无法改变一个词的含义。

ELMo（2018 年初）：通过双向 LSTM 生成上下文相关的向量，最后对每一层的隐藏状态加权求和：

ELMok=γ∑j=0Lsj hk,j\text{ELMo}_k = \gamma \sum_{j=0}^{L} s_j \, h_{k,j} ELMok​=γj=0∑L​sj​hk,j​

其中 hk,jh_{k,j}hk,j​ 是第 jjj 层在位置 kkk 的隐藏状态，sjs_jsj​ 是可学习的 softmax 权重。ELMo 的结果证明了"上下文相关的预训练表示"对几乎所有下游任务都有显著提升——但它仍是 RNN，训练慢且难以并行。

GPT-1（2018 年 6 月）：首次将 Transformer 用于规模化预训练，目标函数是从左到右的标准语言模型：

P(w1,…,wn)=∏i=1nP(wi∣w1,…,wi−1)P(w_1, \ldots, w_n) = \prod_{i=1}^{n} P(w_i \mid w_1, \ldots, w_{i-1}) P(w1​,…,wn​)=i=1∏n​P(wi​∣w1​,…,wi−1​)

GPT-1 表现已经很强，但它是单向的：读到 "the bank is closed" 时，模型在 "bank" 这个位置还未看到 "closed"，因此无法利用 "closed" 消除"银行 / 河岸"的歧义。

BERT（2018 年 10 月）：突破点是将预训练目标改造成允许每个位置同时看到双向上下文。这一个改动直接带来了横扫所有任务的性能提升。

¶为什么这个范式重要

整个流程分为两步：

  1. 预训练：在海量无标注文本（书籍、维基百科、整个互联网）上用自监督任务训练。这一步成本高，但只需做一次。
  2. 微调：在每个下游任务上添加一个小型"任务头"，用小学习率端到端微调几轮。

收益显而易见：

  • 数据效率高：底座模型已掌握语法和不少常识，每个任务只需几百到几千条标注即可取得不错的结果。
  • 通用性强：同一个底座既能处理分类，也能完成标注、抽取、句对任务。
  • 基线强大：朴素的 BERT 微调往往就能超越过去复杂的定制化架构。

¶BERT 的架构

BERT 是原始 Transformer 的编码器部分，堆叠 12 层或 24 层。没有解码器，没有 causal mask，也不做自回归生成——它就是一叠双向自注意力层，将 token 序列转化为"懂上下文"的向量。

¶输入表示：三个嵌入相加

每个 token 的输入向量是三个同维度嵌入的和：

Inputi=Ewitok+Esiseg+Eipos\text{Input}_i = E^{\text{tok}}_{w_i} + E^{\text{seg}}_{s_i} + E^{\text{pos}}_{i} Inputi​=Ewi​tok​+Esi​seg​+Eipos​

  • Token 嵌入：WordPiece 子词 id，词表大小 30K。
  • Segment 嵌入：第一句用 EAE_AEA​，第二句用 EBE_BEB​。这让 BERT 不改架构就能处理 NLI、QA 等句对任务。
  • Position 嵌入：从 0 到 511 每个绝对位置对应一个可学习向量。（不同于原始 Transformer 的正余弦位置，BERT 自己学习。）

两个特殊 token 是整个协议的关键：

  • [CLS] 放在输入开头。所有层走完后，它的隐藏状态被当作整段序列的"池化摘要"，送入分类头。
  • [SEP] 用于分隔句子 A 和 B，也标记输入结束。

¶双向自注意力

每一层编码器内部，多头自注意力让每个 token 都能看到所有其他 token：

Attention(Q,K,V)=softmax ⁣(QK⊤dk)V\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V Attention(Q,K,V)=softmax(dk​​QK⊤​)V

关键点在于：QQQ、KKK、VVV 都来自同一段输入（自注意力），并且没有 causal mask（双向）。因此，位置 3 上的 "bank" 在一次前向中能同时整合左边的 "river" 和右边的 "is closed"。

¶预训练目标

BERT 的预训练结合了两个自监督任务。第一个任务广为人知，第二个任务后来被证明可有可无。

¶掩码语言建模（MLM）

对每条输入序列，随机选择 15% 的位置。选中的位置按以下概率处理：

  • 80% 的概率替换为 [MASK]；
  • 10% 的概率替换为词表中随机的一个 token；
  • 10% 的概率保持不变。

模型的任务是预测每个被选中位置的原始词：

LMLM=−∑i∈Mlog⁡P(wi∣x~)\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(w_i \mid \tilde{x}) LMLM​=−i∈M∑​logP(wi​∣x~)

其中 M\mathcal{M}M 是被选中位置的集合，x~\tilde{x}x~ 是经过损坏的输入。

为什么采用 80/10/10 的比例？ 这个比例经过精心设计，专门用来规避两种失败模式：

  • 如果只使用 [MASK]，模型在微调时将无法见到 [MASK]（下游真实输入中不会出现），导致训练与推理之间产生分布失配。
  • 如果只使用随机 token，模型会变得不信任任何输入，转而过度依赖远处信息。
  • 保留 10% 不变，是为了迫使模型即使看到一个“看似正常”的词，也不能直接复制输出——必须真正通过上下文进行判断。

MLM 是让 BERT 实现“干净”双向建模的关键：要从左右两侧预测被遮掩的词，编码器必须在每个位置融合整段上下文。

¶下一句预测（NSP）

加入 NSP 的初衷是让 BERT 学习句间关系，以便迁移到 NLI、QA 等任务。每条训练样本将两个句子打包成 [CLS] A [SEP] B [SEP]，标签通过抛硬币决定：

  • 50% 的样本中，B 是语料中紧跟 A 的下一句（标签 IsNext）；
  • 50% 的样本中，B 是从其他文档随机抽取的句子（标签 NotNext）。

最后一层的 [CLS] 向量通过一个 Linear+softmax 头预测标签：

P(IsNext)=softmax(W h[CLS]+b)P(\text{IsNext}) = \text{softmax}(W \, h_{\text{[CLS]}} + b) P(IsNext)=softmax(Wh[CLS]​+b)

预训练的总损失是 MLM 损失与 NSP 损失之和。

这里有一段后来被打脸的历史：RoBERTa、ALBERT 等后续工作发现 NSP 的作用微乎其微，去掉甚至替换它反而效果更好。后面变体那一节会再讨论。

¶预训练语料

BERT 的训练数据包括 BooksCorpus（约 8 亿词）和 英文维基百科（约 25 亿词），总计约 33 亿词。从 2026 年的视角来看，这个数量可以忽略不计——现代 LLM 动辄上万亿 token——但在当时已经足以将基线提升到一个新高度。

¶WordPiece 分词

BERT 不处理整词，而是采用 WordPiece 子词方案。这种方案在两个极端之间找到了平衡：

  • 整词词表要覆盖真实语料需要几百万词，推理时还难免遇到 OOV；
  • 纯字符词表虽小，但模型每次都需要从字符拼出"词义"，效率很低。

WordPiece 的策略是贪心地合并那些"合并后能最大化训练语料似然"的字符对，最终形成约 30K 个 token 的词表。分词时，将每个词切分为词表中能匹配的最长片段；位于词内部的片段统一加 ## 前缀，作为"延续"标记：

这种方式既避免了 OOV（任何内容都能拆解为已知子词，最坏情况退化到单字符），又保留了常用词作为单 token 的高效性。

¶把 BERT 微调到下游任务

微调的核心思想是：底座几乎不变，换个头就能换任务。

¶文本分类

整句级标签（情感、垃圾邮件、意图识别）的做法是：把输入过一遍 BERT，取最终的 [CLS] 向量，再过一个线性层。

底层发生的事：

  1. 分词，前后加上 [CLS] 和 [SEP]；
  2. 过 12 层 Transformer 编码器；
  3. 取 [CLS] 的隐藏状态（BERT-Base 是 768 维）；
  4. 过一个 768 → num_labels 的线性层。

¶BERT 家族：RoBERTa、ALBERT、ELECTRA

BERT 是起点，不是终点。两年内涌现出一批从不同角度改进 BERT 的变体。

¶RoBERTa（Facebook，2019）：把训练流程做对就行

Liu 等人发现 BERT 其实是训练不足的。RoBERTa 一行架构都没改，只调了配方：

  • 去掉 NSP：没用，甚至还可能有害。
  • 动态掩码：每个 epoch 重新生成掩码，而不是预处理时一次性固定下来。
  • 更大的 batch：从 256 提到 8K。
  • 多得多的数据：在 BooksCorpus + Wikipedia 之外加上 CC-News、OpenWebText、Stories（约 1600 亿 token，对比 BERT 的 33 亿）。

仅这些改动就让 GLUE 涨了 2-3 分，而架构跟 BERT-Large 一模一样。

¶ALBERT（Google，2019）：把参数压一压

ALBERT 用更少的独立参数达到接近的效果：

  • 嵌入分解：把 V×HV \times HV×H 的嵌入矩阵拆成 V×EV \times EV×E 和 E×HE \times HE×H，其中 E≪HE \ll HE≪H。token 嵌入活在低维空间里，再投到隐藏维度。
  • 跨层参数共享：所有 Transformer 层共享同一套权重，深度不再随参数量线性放大。
  • 句子顺序预测（SOP） 取代 NSP：给两句相邻的句子，判断它们的顺序是否被颠倒过。这比"是否随机配对"难得多，信号也更有用。

ALBERT-xxlarge 大约 2.35 亿独立参数（对比 BERT-Large 的 3.4 亿），但在 GLUE 上能持平甚至超过。

¶ELECTRA（Google，2020）：每个 token 都参与训练

MLM 只在 15% 的位置上有损失，剩下 85% 的位置纯属"路过"，浪费。ELECTRA 把 MLM 替换成替换 token 检测（RTD）：

  1. 一个小型的生成器（一个迷你 MLM）把一些 token 换成"看起来像但不是原版"的词；
  2. 一个更大的判别器逐 token 地判断：这个 token 是原文的，还是被换过的？
  3. 训练完之后，只保留判别器。

因为判别器在每个位置上都有损失，ELECTRA 用更少的算力就能达到 BERT 级别——ELECTRA-Small 只用四分之一的训练时间就赶上了 BERT-Base。

¶BERT 做不到的事

了解它的边界与了解它的能力同样重要。

  • 成本：1.1 亿到 3.4 亿参数，加上二次方注意力，实时推理变得困难。需要通过蒸馏（DistilBERT、TinyBERT）或量化来压缩。
  • 不能生成：BERT 是双向 encoder-only 模型，无法完成合理的自回归解码。生成文本的任务需要借助 GPT 系列的 decoder 模型——下一篇会详细说明。
  • 512 token 上限：位置嵌入仅学到 0-511。处理长文档时，必须依靠滑窗、层级聚合，或者直接切换架构（Longformer、BigBird）。
  • 以英语为中心：原始 BERT 仅在英文数据上训练。多语言 BERT（mBERT）覆盖了 100 多种语言，但在每种具体语言上的表现都不如该语言的专用模型（BERT-Chinese、CamemBERT 等）。

¶常见问题

为什么用 [CLS] 做分类？ 它固定在每条输入的第 0 位置，注意力机制天然会让它在最后一层聚合整段序列的信息。预训练阶段的 NSP 目标也在显式训练 [CLS] 承担"序列摘要"的功能。

选 BERT 还是 RoBERTa？ 如果追求更高分数，且有多余几个小时的 GPU 时间，选择 RoBERTa；如果需要最大的生态、最多的教程和最丰富的 checkpoint，BERT 仍是更稳妥的基线。

变体怎么选？ 通用场景选 BERT，冲分用 RoBERTa，参数量受限（移动端、嵌入式）选 ALBERT，算力不足时用 ELECTRA。

中文怎么办？ 中文有比 mBERT 更专业的选项：BERT-wwm-ext 和 MacBERT 是常用基线。直接从 bert-base-chinese 开始通常也够用，关键看任务对中文细粒度（字 vs 词）的敏感程度。

能拿来做句向量吗？ 直接平均 BERT token 向量得到的句向量效果一般。若要做相似度检索，请使用 Sentence-BERT（在 BERT 上通过 Siamese 对比训练得到的变体）。

¶核心要点

  • BERT 采用双向预训练（通过 MLM），使每个 token 在一次前向计算中就能感知完整上下文。
  • 80/10/10 的掩码比例避免了训练与推理分布失配，同时迫使模型即便输入"看似正常"也要依赖上下文。
  • 预训练-微调范式让昂贵的预训练成本分摊到无数下游任务上；微调只需添加一个小头、使用小学习率并训练几个 epoch。
  • RoBERTa、ALBERT、ELECTRA 共同表明：配方（数据、掩码方式、参数共享、训练目标）和架构同等重要。
  • BERT 擅长理解类任务，但不擅长生成——生成任务需参考下一篇 GPT。
