自然语言处理（四）：注意力机制与Transformer

从 Seq2Seq 的瓶颈到 Attention Is All You Need，建立缩放点积注意力、多头注意力、位置编码和因果掩码的直觉，并用 PyTorch 从零搭一个完整 Transformer。

#注意力机制#NLP#Transformer
Read in English →

2017 年 6 月，Google 的八位研究者发了一篇标题相当大胆的论文：Attention Is All You Need。论文里提出的 Transformer 架构干脆把循环结构整个扔掉了——没有 LSTM，没有 GRU，也不再从左到右一个一个地读句子。取而代之，序列里的每个 token 都可以通过一个数学操作直接看到其他所有 token：缩放点积注意力。

这一个设计决定，解锁了 GPU 上的大规模并行训练，顺手解决了困扰 RNN 几十年的长距离依赖问题，并成为 BERT、GPT、T5、LLaMA、Claude 以及今天几乎所有大模型的底座。把这一篇读懂，本系列后面的内容基本就是同一主题的不同变奏。

从"带注意力的 RNN"到完整的 Transformer 这条路并不长，但每一步都不能跳。我们慢慢走。

你将学到什么

  • 固定大小的上下文向量为什么会让朴素 Seq2Seq 在处理长句子时崩溃，注意力机制又是如何解决这个问题的
  • Bahdanau 和 Luong 注意力：从传统注意力到自注意力的过渡
  • Query / Key / Value 的抽象概念、缩放点积注意力，以及 $\sqrt{d_k}$ 的作用到底是什么
  • 多头注意力的直观理解：为什么要并行运行多个“视角”
  • 正弦位置编码和可学习位置编码的对比
  • 因果掩码、残差连接和 LayerNorm 的作用
  • 一个完整的、从零开始实现的 PyTorch Transformer，可以直接在笔记本上运行
  • BERT、GPT 和 T5 如何用相同的模块完成不同的任务

前置知识：第三篇（RNN 和 Seq2Seq），基本线性代数（矩阵乘法、softmax），以及对 PyTorch 的基本熟悉程度。

1. 驱动注意力机制的瓶颈问题

回想一下第三篇提到的基础编解码器。编码器 RNN 按顺序逐个读取源句子的 token，最后把所有信息压缩成一个固定大小的向量 $c = h_T^{\text{enc}}$。解码器只用这个向量生成目标序列。

举个例子，翻译句子 “The cat that chased the mouse that ate the cheese was very tired” 到法语。编码器需要把猫、老鼠、奶酪、追逐、吃、疲惫以及它们之间的语法关系，统统塞进 512 个数字里。解码器则要靠这 512 个数字还原整句话，而且不能再回头看源句。

这种方法会因为两个原因失效：

  • 信息容量不足。一个固定向量无法无损地存储任意长度的序列。实验表明，朴素 Seq2Seq 的 BLEU 分数在句子长度超过 30 个词后急剧下降。
  • 缺乏选择性聚焦。解码器生成法语单词 “cat” 时，应该关注源句中的“猫”，而不是“奶酪”。但静态上下文向量对所有内容一视同仁，无法区分重点。

打个比方：让我背下一段话，然后合上书凭记忆翻译；再对比一下专业译者，他们把原文摊开，每写一个词前都回头看一眼相关部分。注意力机制就是让神经网络学会后者的工作方式。

2. Bahdanau 注意力（2015）：每一步都回头看

Bahdanau、Cho 和 Bengio 在论文 Neural Machine Translation by Jointly Learning to Align and Translate 中提出了第一个被广泛使用的注意力机制。核心思想很简单：不要固定一个上下文向量，而是在每个解码步骤动态计算所有编码器隐藏状态的加权组合。

假设解码器在第 $t$ 步的隐藏状态是 $s_{t-1}$，编码器隐藏状态是 $h_1, \ldots, h_n$。整个过程分为四步：

第一步：打分
用一个小的前馈网络计算每个编码器状态与当前解码状态的相关性分数：

$$e_{tj} = \mathbf{v}^\top \tanh(W_s s_{t-1} + W_h h_j)$$

第二步：归一化
对所有分数做 softmax，把它们变成一个概率分布，总和为 1：

$$\alpha_{tj} = \frac{\exp(e_{tj})}{\sum_{k=1}^{n} \exp(e_{tk})}$$

第三步：生成上下文向量
上下文向量是编码器状态的加权平均：

$$c_t = \sum_{j=1}^{n} \alpha_{tj}\, h_j$$

第四步：更新解码器状态
RNN 同时接收上下文向量和上一步的输出，更新状态：

$$s_t = \text{RNN}(s_{t-1}, [c_t; y_{t-1}])$$

上面那张图展示了这些 $\alpha_{tj}$ 在一个英译法小例子中的表现。每一行的概率加起来等于 1，亮色格子正好对应语言学上的对齐关系（Le ↔ The、chat ↔ cat、tapis ↔ mat）。最重要的是，我并没有告诉模型这些对齐关系。它们是在训练过程中为了降低翻译损失自己学到的。从这一刻起，注意力不再只是一个提升性能的技巧，而是变成了一个有语言学意义且可解释的对象。

3. Luong 注意力：更简单的打分函数

几个月后，Luong、Pham 和 Manning 提出了比 Bahdanau 小型前馈打分器更简单的替代方案：

名称打分函数备注
点积$e_{tj} = s_t^\top h_j$最快，但要求维度匹配
通用$e_{tj} = s_t^\top W h_j$可学习的双线性形式，支持维度不匹配
拼接$e_{tj} = \mathbf{v}^\top \tanh(W [s_t; h_j])$基本等同于 Bahdanau 的方法

点积的形式是两年后 Transformer 使用的打分方式的鼻祖。Luong 还引入了 局部注意力，只在对齐点周围 $2D+1$ 的窗口内计算注意力，大幅降低了长序列的计算成本。

4. 关键一跃：彻底抛弃循环结构

Bahdanau 和 Luong 的注意力机制是建立在 RNN 基础上的。虽然它们能加速收敛、提升效果，但 RNN 依然把计算串成了一条线：必须先处理第 $t$ 个 token，才能处理第 $t+1$ 个。在一块有几千个核心的 GPU 上，这种设计简直是暴殄天物。

Vaswani 等人提出了一个简单直接的问题：如果只用注意力机制呢？ 如果每个 token 都可以直接关注其他所有 token，我们就能获得两大好处：

  1. 完全并行化。所有位置可以通过一次矩阵乘法同时计算。
  2. 路径长度恒定。任意两个 token 之间永远只隔一次操作，不管序列有多长。梯度消失问题？再见吧。

实现这一点的核心机制就是自注意力：不再是解码器去看编码器的状态，而是同一个序列里的每个 token 都去关注序列中的所有 token（包括自己）。

一个直观的例子

来看这句话："The animal didn’t cross the street because it was too tired."（那只动物没有过马路，因为它太累了。）

要正确表示 it，模型必须知道 it 指的是 animal 而不是 street。在自注意力中，it 的新表示是句子中所有其他 token 表示的加权和。一个训练得当的注意力头会给 animal 高权重，给 street 低权重，然后用这个加权和来精炼 it 的含义。

Query、Key、Value：三位一体

对每个 token 的嵌入 $x_i$，我学习三组线性投影：

$$q_i = W_Q\, x_i, \qquad k_i = W_K\, x_i, \qquad v_i = W_V\, x_i$$

这三个角色可以类比一次字典查询：

  • Query $q_i$：我在找什么？——这是当前 token 提出的问题。
  • Key $k_i$：我有什么？——用来判断这个 token 是否与问题相关。
  • Value $v_i$：如果真的关注我，你能得到什么信息？

为位置 $i$ 计算注意力时，我会用 $q_i$ 和所有 $k_j$ 打分，再决定从每个 $v_j$ 中读取多少信息。

缩放点积注意力，分四步走

把所有 query、key、value 堆成矩阵 $Q, K, V \in \mathbb{R}^{n \times d_k}$，公式如下：

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V$$

对照上图的四个步骤：

  1. 打分：$Q K^\top$ 生成一个 $n \times n$ 的矩阵，第 $(i, j)$ 个元素是点积 $q_i \cdot k_j$，值越大表示越相关。
  2. 缩放：每个元素都除以 $\sqrt{d_k}$。
  3. softmax：沿每一行做 softmax，将缩放后的分数转化为概率分布，总和为 1。第 $i$ 行告诉我：“要计算位置 $i$ 的输出，请按这个比例混合各个 value。”
  4. 加权求和：与 $V$ 相乘，得到每个位置的新表示。

为什么要除以 $\sqrt{d_k}$？

这是面试中最常被问到的细节。假设 $q$ 和 $k$ 的每一维独立、均值为 0、方差为 1，那么：

$$\text{Var}(q \cdot k) = \text{Var}\!\left(\sum_{i=1}^{d_k} q_i k_i\right) = d_k$$

当 $d_k = 64$ 时，点积的标准差会达到 8。把这种量级的数喂给 softmax，分布会被推得非常尖锐，几乎变成 one-hot 向量。除了最大值的位置，其他位置的梯度基本为 0，训练直接卡住。

除以 $\sqrt{d_k}$ 把方差拉回 1，让 softmax 保持在有意义的工作区间。一行代码，效果却出奇地大。

5. 多头注意力

单个注意力操作只能提供一种加权视角。但语言中同时存在多种结构：主谓一致、指代关系、句法依赖、语义相似性、位置邻近等。一个注意力头根本无法处理所有这些结构。

多头注意力通过并行运行 $h$ 个注意力操作来解决这个问题，每个注意力头都有自己的投影矩阵：

$$\text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V)$$

然后将所有头的输出拼接起来，再通过一个线性层：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W^O$$

在原始论文中，$d_{\text{model}} = 512$，$h = 8$，因此每个头的 $d_k = d_v = 512 / 8 = 64$。每层的总计算量和使用一个大头时相当，但模型可以分工合作。后续研究发现，不同的头会专注于不同的关系，比如句法依赖和指代关系。

因果掩码

在自回归解码器中，训练时不能让位置 $i$ 看到位置 $j > i$ 的信息，否则模型会直接抄答案。我用加性掩码来实现这一点：

$$\text{MaskedAttention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}} + M\right) V$$

其中，当 $j \le i$ 时，$M_{ij} = 0$（允许）；当 $j > i$ 时，$M_{ij} = -\infty$（禁止）。$-\infty$ 经过 softmax 后会变成精确的 0——图中右侧的上三角区域被完全抹除。

正是这个技巧，让 GPT 类模型在一次前向传播中就能计算整个序列的 loss，但在推理时表现得像逐个 token 生成一样。

6. 位置编码：让顺序回归

自注意力机制是置换不变的。如果打乱输入 token 的顺序，输出也会跟着打乱，而注意力权重本身完全不变。比如"猫吃鱼"和"鱼吃猫"在模型眼里是一回事——这显然不对。

解决方法是在输入进入第一层之前，给每个 token 的嵌入加上一个位置编码 $\text{PE}(\text{pos}) \in \mathbb{R}^{d_{\text{model}}}$。

正弦编码

原始 Transformer 使用了一种固定的（不可学习的）正弦方案：

$$PE_{(\text{pos},\, 2i)} = \sin\!\left(\frac{\text{pos}}{10000^{2i / d_{\text{model}}}}\right), \qquad PE_{(\text{pos},\, 2i+1)} = \cos\!\left(\frac{\text{pos}}{10000^{2i / d_{\text{model}}}}\right)$$

上图左侧是编码矩阵的热力图，右侧展示了几个具体维度的波形：低维振荡快（精细位置），高维振荡慢（粗略位置）。这种组合为每个位置生成了独一无二的指纹。由于 $\sin$ 和 $\cos$ 满足简单的线性关系，模型理论上可以学会"往前三个位置"这样的相对偏移。

可学习的位置嵌入

另一种方法是把位置当作一个标准的嵌入表，形状为 $(\text{max\_len}, d_{\text{model}})$。BERT 和 GPT-2 都采用了这种方法。它更简单，实际训练效果也稍好一些，但有一个缺点：无法外推到训练时没见过的更长序列。

为什么用相加而不是拼接？

相加保留了 $d_{\text{model}}$ 的完整维度，既包含内容信息，也包含位置信息，后续的 $W_Q, W_K, W_V$ 投影可以自行学习如何分离它们。拼接要么会增加维度，要么会挤占内容嵌入的容量，得不偿失。

现代大模型已经很少使用原始的正弦编码了。RoPE（旋转位置编码）、ALiBi（注意力分数线性偏置）、NoPE 等方法都是为了解决长度外推问题，现在已经成为生产环境中的标配。不过，原始的正弦编码仍然是一个清晰易懂的基线方案。

7. 完整的 Transformer 架构

Transformer 就是用一堆编码器层堆起来，再接一堆解码器层。两边的设计都用了三个核心结构：残差连接包裹子层、LayerNorm 和 dropout。

编码器层

每个编码器层有 $N$ 层，每层包含两个子模块：

  1. 多头自注意力，处理源序列。
  2. 逐位置前馈网络，对每个位置单独计算。
$$\text{FFN}(x) = \max(0,\, x W_1 + b_1)\, W_2 + b_2$$

FFN 先把维度扩展到 $d_{\text{ff}} = 4 \cdot d_{\text{model}}$（通常是 2048），然后再投影回去。这个模块集中了模型的大部分参数，也是 token 级非线性处理的核心。每个子模块都用下面的方式封装：

$$\text{output} = \text{LayerNorm}(x + \text{Sublayer}(x))$$

残差连接让梯度可以直接从任意一层流回前面的任意一层。这种设计在深度达到 6、12、24 或 96 时尤为重要。

解码器层

每个解码器层也有 $N$ 层，每层包含三个子模块：

  1. 掩码多头自注意力，只看已经生成的目标 token（因果掩码）。
  2. 交叉注意力：query 来自解码器，key 和 value 来自编码器的最后一层输出。这是连接源和目标的关键桥梁。
  3. 前馈网络，和编码器里的完全一致。

上图把编码器和解码器并排画出来，还特别标出了交叉注意力的连接。

整体流程

源 token -> 嵌入 + 位置编码 -> [编码器层 x N] -> 编码器输出
                                                       |
                                                 K, V  |
                                                       v
目标 token -> 嵌入 + 位置编码 -> [解码器层 x N] -> 线性 -> softmax -> 概率

基础版 Transformer（$N = 6$、$d_{\text{model}} = 512$、$h = 8$、$d_{\text{ff}} = 2048$）大约有 6500 万参数。GPT-3 的做法很简单：把 $N$、$d_{\text{model}}$ 和 $h$ 都放大，去掉编码器，然后用整个互联网的数据训练。

import math
import torch
import torch.nn as nn
import torch.nn.functional as F

def scaled_dot_product_attention(query, key, value, mask=None):
    """公式：softmax(QK^T / sqrt(d_k)) V。

    输入形状：
        query: (batch, heads, seq_q, d_k)
        key:   (batch, heads, seq_k, d_k)
        value: (batch, heads, seq_k, d_v)
        mask:  可广播到 (batch, heads, seq_q, seq_k)，0 表示屏蔽
    输出：
        output:  (batch, heads, seq_q, d_v)
        weights: (batch, heads, seq_q, seq_k)
    """
    d_k = query.size(-1)
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))
    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, value), weights

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model: int, num_heads: int, dropout: float = 0.1):
        super().__init__()
        assert d_model % num_heads == 0
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        # 每个角色用一个大矩阵，后面再切分成多头
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
        self.dropout = nn.Dropout(dropout)

    def _split(self, x):
        b, t, _ = x.size()
        return x.view(b, t, self.num_heads, self.d_k).transpose(1, 2)

    def _merge(self, x):
        b, _, t, _ = x.size()
        return x.transpose(1, 2).contiguous().view(b, t, self.d_model)

    def forward(self, q, k, v, mask=None):
        Q, K, V = self._split(self.W_q(q)), self._split(self.W_k(k)), self._split(self.W_v(v))
        out, weights = scaled_dot_product_attention(Q, K, V, mask)
        return self.W_o(self.dropout(self._merge(out))), weights

class PositionalEncoding(nn.Module):
    def __init__(self, d_model: int, max_len: int = 5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).float().unsqueeze(1)
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        self.register_buffer("pe", pe.unsqueeze(0))

    def forward(self, x):
        return x + self.pe[:, : x.size(1), :]

class FeedForward(nn.Module):
    def __init__(self, d_model: int, d_ff: int = 2048, dropout: float = 0.1):
        super().__init__()
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        return self.linear2(self.dropout(F.relu(self.linear1(x))))

class EncoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.attn = MultiHeadAttention(d_model, num_heads, dropout)
        self.ffn = FeedForward(d_model, d_ff, dropout)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.drop1 = nn.Dropout(dropout)
        self.drop2 = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        attn_out, _ = self.attn(x, x, x, mask)
        x = self.norm1(x + self.drop1(attn_out))
        x = self.norm2(x + self.drop2(self.ffn(x)))
        return x

class DecoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, num_heads, dropout)
        self.cross_attn = MultiHeadAttention(d_model, num_heads, dropout)
        self.ffn = FeedForward(d_model, d_ff, dropout)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)
        self.drop1 = nn.Dropout(dropout)
        self.drop2 = nn.Dropout(dropout)
        self.drop3 = nn.Dropout(dropout)

    def forward(self, x, enc_out, src_mask=None, tgt_mask=None):
        sa, _ = self.self_attn(x, x, x, tgt_mask)
        x = self.norm1(x + self.drop1(sa))
        ca, _ = self.cross_attn(x, enc_out, enc_out, src_mask)
        x = self.norm2(x + self.drop2(ca))
        x = self.norm3(x + self.drop3(self.ffn(x)))
        return x

class Transformer(nn.Module):
    def __init__(self, src_vocab, tgt_vocab, d_model=512, num_heads=8,
                 num_layers=6, d_ff=2048, max_len=5000, dropout=0.1):
        super().__init__()
        self.d_model = d_model
        self.src_emb = nn.Embedding(src_vocab, d_model)
        self.tgt_emb = nn.Embedding(tgt_vocab, d_model)
        self.pos_enc = PositionalEncoding(d_model, max_len)
        self.encoder = nn.ModuleList(
            [EncoderLayer(d_model, num_heads, d_ff, dropout) for _ in range(num_layers)]
        )
        self.decoder = nn.ModuleList(
            [DecoderLayer(d_model, num_heads, d_ff, dropout) for _ in range(num_layers)]
        )
        self.proj = nn.Linear(d_model, tgt_vocab)
        self.dropout = nn.Dropout(dropout)

        for p in self.parameters():
            if p.dim() > 1:
                nn.init.xavier_uniform_(p)

    @staticmethod
    def make_pad_mask(seq, pad_idx=0):
        return (seq != pad_idx).unsqueeze(1).unsqueeze(2)

    @staticmethod
    def make_causal_mask(size):
        return ~torch.triu(torch.ones(size, size, dtype=torch.bool), diagonal=1)

    def encode(self, src, src_mask=None):
        x = self.dropout(self.pos_enc(self.src_emb(src) * math.sqrt(self.d_model)))
        for layer in self.encoder:
            x = layer(x, src_mask)
        return x

    def decode(self, tgt, enc_out, src_mask=None, tgt_mask=None):
        x = self.dropout(self.pos_enc(self.tgt_emb(tgt) * math.sqrt(self.d_model)))
        for layer in self.decoder:
            x = layer(x, enc_out, src_mask, tgt_mask)
        return x

    def forward(self, src, tgt, src_mask=None, tgt_mask=None):
        enc_out = self.encode(src, src_mask)
        dec_out = self.decode(tgt, enc_out, src_mask, tgt_mask)
        return self.proj(dec_out)

9. 自注意力 vs. RNN vs. CNN

为什么 Transformer 能彻底取代 RNN 和 CNN，成为序列建模的主流？答案就在每层的三个关键指标里：

架构单层计算量串行操作数最大路径长度
自注意力$O(n^2 \cdot d)$$O(1)$$O(1)$
RNN（LSTM/GRU）$O(n \cdot d^2)$$O(n)$$O(n)$
CNN（核 $k$）$O(k \cdot n \cdot d^2)$$O(1)$$O(\log_k n)$

自注意力的计算复杂度是 $O(n^2)$，但它带来了恒定的路径长度和完全并行化的能力。对于常见的序列长度（$n < 1000$），现代硬件擅长矩阵乘法，这种权衡非常划算。当序列特别长（几万 token）时，像 FlashAttention、Longformer、Performer 和 Mamba 这样的高效变体模型才显得更有吸引力。

10. 三种 Transformer 的工业应用

最初的 Transformer 是编码器-解码器架构。后来，两种广泛使用的变体各自砍掉了一半：

家族架构预训练目标擅长任务示例模型
仅编码器编码器栈掩码语言建模分类、命名实体识别、检索、问答BERT、RoBERTa、DeBERTa
仅解码器解码器栈下一个 token 预测（因果）文本生成、对话、代码、推理GPT、LLaMA、Claude
编码器-解码器双栈架构片段还原 / seq2seq翻译、摘要生成、结构化任务T5、BART、mT5

仅解码器模型在大模型竞赛中占据了主导地位。原因很简单：下一个 token 预测在原始网页文本上扩展性极佳，并且几乎把所有任务都统一成了文本生成。

11. 常见问题

为什么解码器需要掩码，而编码器不需要？
编码器处理的是完整的源句子，目标就是双向地看全上下文。解码器则是逐个生成 token，如果训练时不挡住未来的 ground-truth token，模型就会偷懒直接抄答案。像 BERT 这种纯编码器模型之所以是双向的，正是因为没有因果掩码。

显存的二次方开销从哪来？
主要来自那个 $n \times n$ 的注意力分数矩阵，还没做任何降维之前。比如 $n = 4000$、$h = 16$ 个头，float16 精度下，单层的这个缓冲区就已经超过 500 MB。FlashAttention 的核心思想是永远不生成完整的矩阵，而是分块流式计算 softmax。

为什么 $d_{\text{ff}} = 4 \cdot d_{\text{model}}$？
这是经验得出的。2017 年觉得 4 倍是个合理的选择，后来就一直沿用。最近的一些工作（比如 PaLM 和 LLaMA）有时会调整这个比例，或者把 ReLU 换成 SwiGLU 或 GeGLU 来获得一点边际提升。

Pre-LN 还是 Post-LN？
原论文用的是 Post-LN（LayerNorm 放在残差之后）。但现在的实现几乎都用 Pre-LN（LayerNorm 放在子层之前），深层网络下训练更稳定，也不需要复杂的学习率 warmup。

现代大模型还用正弦位置编码吗？
基本不用了。主流大模型现在用 RoPE（LLaMA、GPT-NeoX）或 ALiBi（BLOOM）。这些方法能自然外推到更长的上下文，并且通过旋转或加性偏置无缝融入多头注意力机制。

12. 核心要点

  • 朴素 Seq2Seq 处理长输入时会失败，因为单个上下文向量容量太小。注意力机制让解码器能够动态访问每个编码器的状态。
  • 自注意力去掉了循环结构：每个位置在 $O(1)$ 步内就能看到其他所有位置。
  • 缩放点积注意力公式很简单：$\text{softmax}(QK^\top / \sqrt{d_k}) V$。这里的 $\sqrt{d_k}$ 是训练稳定性和梯度崩溃之间的关键。
  • 多头注意力并行执行多个小规模注意力计算，让不同的头专注于不同的关系。
  • 位置编码把纯注意力丢失的顺序信息补回来了。
  • Transformer 块 = （多头注意力 + 前馈网络），每部分都用残差连接和 LayerNorm 包起来。堆叠 $N$ 层，就这么简单。
  • BERT、GPT、T5 分别是这个模板的三种变体：仅编码器、仅解码器、编码器-解码器。
