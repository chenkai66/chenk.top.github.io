自然语言处理（四）：注意力机制与Transformer

八月 27, 2025 Updated 5月 06, 2026 — min read — words

从 Seq2Seq 的瓶颈到 Attention Is All You Need，建立缩放点积注意力、多头注意力、位置编码和因果掩码的直觉，并用 PyTorch 从零搭一个完整 Transformer。

2017 年 6 月，Google 的八位研究者发表了一篇标题大胆的论文：Attention Is All You Need。论文提出的 Transformer 架构彻底抛弃了循环结构——没有 LSTM，没有 GRU，也不再从左到右逐字读取句子。取而代之的是，序列中的每个 token 都能通过一个数学操作直接感知其他所有 token：缩放点积注意力。

这一设计决策释放了 GPU 上的大规模并行训练潜力，顺带解决了 RNN 长期以来的长距离依赖难题，并成为 BERT、GPT、T5、LLaMA、Claude 以及当下几乎所有大模型的基础。理解这篇论文后，本系列后续内容基本就是同一主题的不同变奏。

从“带注意力的 RNN”到完整的 Transformer，这条路不长，但每一步都不可或缺。我们一步步来。

¶你将学到什么

  • 固定大小的上下文向量为何让朴素 Seq2Seq 在长句子上失效，注意力机制如何解决问题
  • Bahdanau 和 Luong 注意力——通向自注意力的必经之路
  • Query / Key / Value 抽象、缩放点积注意力，以及 dk\sqrt{d_k}dk​​ 的实际作用
  • 多头注意力的直观理解：为何需要并行运行多个"视角"
  • 正弦位置编码与可学习位置编码的对比
  • 因果掩码、残差连接和 LayerNorm 的原理
  • 一个能在笔记本上运行的、从零实现的 PyTorch Transformer
  • BERT、GPT、T5 如何用同一套组件构建出三种不同模型

前置知识：第三篇（RNN 与 Seq2Seq），基本线性代数（矩阵乘法、softmax），以及少量 PyTorch 经验。

¶1. 让人头疼的瓶颈问题

回想第三篇提到的朴素编解码器：编码器 RNN 按顺序逐个读取源句子的 token，将所有信息压缩成一个固定大小的向量 c=hTencc = h_T^{\text{enc}}c=hTenc​；解码器仅依靠这一个向量生成整个目标序列。

假设需要翻译这样一句话："那只追逐了吃奶酪的老鼠的猫非常疲惫"。编码器必须把猫、老鼠、奶酪、追逐、吃、疲惫，以及它们之间的语法关系全部塞进 512 个数字里。解码器接下来要靠这 512 个数字还原整句话，而且不能再参考源句。

这种设计存在两种崩溃方式，原因各不相同：

  • 信息容量有限。一个固定向量无法容纳任意长度的序列。实际测试中，朴素 Seq2Seq 的 BLEU 分数在句子长度超过 30 个词时开始急剧下降。
  • 缺乏选择性聚焦能力。生成英文的 "cat" 时，模型应该关注源句中的"猫"，而非"奶酪"。但静态上下文向量对所有源词同等对待。

用一个比喻来说明：让你背下一段话，然后合上书凭记忆翻译出来；或者让你把原文摊开，每写一个译词前都回头看一眼相关部分。注意力机制做的就是后者，只不过是在神经网络中实现的。

¶2. Bahdanau 注意力（2015）：每一步都回头看一眼

Bahdanau、Cho 和 Bengio 在 Neural Machine Translation by Jointly Learning to Align and Translate 中提出了第一个被广泛使用的注意力机制。核心思想只有一句话：不用固定的上下文向量，每个解码步骤重新计算所有编码器状态的加权和。

设解码器在第 ttt 步的隐藏状态为 st−1s_{t-1}st−1​，编码器隐藏状态为 h1,…,hnh_1, \ldots, h_nh1​,…,hn​，过程分为四步：

第 1 步——打分。一个小型前馈网络为每个编码器状态计算相关性分数：

etj=v⊤tanh⁡(Wsst−1+Whhj)e_{tj} = \mathbf{v}^\top \tanh(W_s s_{t-1} + W_h h_j) etj​=v⊤tanh(Ws​st−1​+Wh​hj​)

第 2 步——归一化。沿 jjj 做 softmax，将分数转化为加起来等于 1 的概率分布：

αtj=exp⁡(etj)∑k=1nexp⁡(etk)\alpha_{tj} = \frac{\exp(e_{tj})}{\sum_{k=1}^{n} \exp(e_{tk})} αtj​=∑k=1n​exp(etk​)exp(etj​)​

第 3 步——求上下文向量。它是编码器状态的凸组合：

ct=∑j=1nαtj hjc_t = \sum_{j=1}^{n} \alpha_{tj}\, h_j ct​=j=1∑n​αtj​hj​

第 4 步——更新解码器：

st=RNN(st−1,[ct;yt−1])s_t = \text{RNN}(s_{t-1}, [c_t; y_{t-1}]) st​=RNN(st−1​,[ct​;yt−1​])

上面那张图展示了这些 αtj\alpha_{tj}αtj​ 在英译法小例子中的表现。每一行的值加起来等于 1，亮格子正好对应语言学上的对齐关系（Le ↔ The、chat ↔ cat、tapis ↔ mat）。关键在于：没人告诉模型这些对齐关系，它们是模型为了降低翻译损失自己学到的。从这一刻起，注意力不再只是性能优化技巧，而是具备了语言学意义和可解释性。

¶3. Luong 注意力：简化打分函数

几个月后，Luong、Pham 和 Manning 提出了几种更简单的打分方法：

名称 打分函数 备注
点积 etj=st⊤hje_{tj} = s_t^\top h_jetj​=st⊤​hj​ 计算最快，但要求两边维度一致
通用 etj=st⊤Whje_{tj} = s_t^\top W h_jetj​=st⊤​Whj​ 加入一个可学习矩阵，解决维度不一致问题
拼接 etj=v⊤tanh⁡(W[st;hj])e_{tj} = \mathbf{v}^\top \tanh(W [s_t; h_j])etj​=v⊤tanh(W[st​;hj​]) 基本延续 Bahdanau 的思路

点积方法正是两年后 Transformer 全面普及时所采用的打分方式的雏形。此外，Luong 还提出了 局部注意力：仅在对齐点附近 2D+12D+12D+1 个位置计算注意力，大幅减少长序列的计算开销。

¶4. 关键一跃：彻底扔掉循环结构

Bahdanau 和 Luong 的注意力机制仍然建立在 RNN 基础之上。这种设计虽然加速了训练收敛，也提升了效果，但 RNN 本质上还是串行计算：必须完成第 ttt 步才能开始第 t+1t+1t+1 步。在一块拥有几千个核心的 GPU 上，这种串行化是对计算资源的极大浪费。

Vaswani 等人提出了一个显而易见的问题：如果只用注意力呢？ 如果每个 token 都能直接关注其他所有 token，我们就能获得两个巨大优势：

  1. 完全并行。所有位置的计算可以在一次矩阵乘法中同时完成。
  2. 路径长度恒定。任意两个 token 之间永远只隔一次操作，无论序列有多长。100 个时间步上的梯度消失问题？再见。

实现这一目标的核心机制是自注意力：不再是解码器查看编码器状态，而是同一个序列中的每个 token 都去查看序列中的所有 token（包括自身）。

¶一个直觉例子

来看这个句子："The animal didn't cross the street because it was too tired."（那只动物没有过马路，因为它太累了。）

要正确表示 it，模型需要知道 it 指的是 animal 而不是 street。在自注意力机制中，it 的新表示是其他所有 token 表示的加权和；一个训练好的注意力头会给 animal 高权重、给 street 低权重，然后用这个加权和来精炼 it 的含义。

¶Query、Key、Value：三位一体

对每个 token 的嵌入 xix_ixi​，我们学习三组线性投影：

qi=WQ xi,ki=WK xi,vi=WV xiq_i = W_Q\, x_i, \qquad k_i = W_K\, x_i, \qquad v_i = W_V\, x_i qi​=WQ​xi​,ki​=WK​xi​,vi​=WV​xi​

这三个角色恰好对应一次字典查询：

  • Query qiq_iqi​：我在找什么？——这个位置正在问的问题。
  • Key kik_iki​：我里面有什么？——用来判断“这个 token 跟你要找的相关吗”。
  • Value viv_ivi​：要是真的关注我，你能拿到什么信息？

为位置 iii 计算注意力，就是用 qiq_iqi​ 跟所有的 kjk_jkj​ 打分，再决定从每个 vjv_jvj​ 中读取多少信息。

¶缩放点积注意力，分四步走

将所有的 query、key、value 堆叠成矩阵 Q,K,V∈Rn×dkQ, K, V \in \mathbb{R}^{n \times d_k}Q,K,V∈Rn×dk​：

Attention(Q,K,V)=softmax ⁣(QK⊤dk)V\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V Attention(Q,K,V)=softmax(dk​​QK⊤​)V

对照上图的四个面板：

  1. 打分：QK⊤Q K^\topQK⊤ 生成一个 n×nn \times nn×n 的矩阵，第 (i,j)(i, j)(i,j) 个元素是点积 qi⋅kjq_i \cdot k_jqi​⋅kj​，值越大表示越相关。
  2. 缩放：每个元素除以 dk\sqrt{d_k}dk​​。
  3. softmax：沿行做 softmax，把缩放后的分数转换为加起来等于 1 的概率分布。第 iii 行表明：“要计算位置 iii 的输出，请按这个比例混合各个 value”。
  4. 加权求和：与 VVV 相乘，得到每个位置的新表示。

¶为什么要除以 dk\sqrt{d_k}dk​​？

这是面试中出现频率最高的细节。假设 qqq 和 kkk 的每一维独立、均值为 0、方差为 1，那么：

Var(q⋅k)=Var ⁣(∑i=1dkqiki)=dk\text{Var}(q \cdot k) = \text{Var}\!\left(\sum_{i=1}^{d_k} q_i k_i\right) = d_k Var(q⋅k)=Var(i=1∑dk​​qi​ki​)=dk​

当 dk=64d_k = 64dk​=64 时，点积的标准差达到 8。将这种量级的数值输入 softmax，分布会被推得非常尖锐，几乎变成 one-hot 向量；除了最大值位置，其他位置的梯度基本为 0，训练直接卡死。

除以 dk\sqrt{d_k}dk​​ 把方差拉回 1，让 softmax 保持在有意义的工作区间。一行代码，效果出奇地显著。

¶5. 多头注意力

单个注意力操作只能提供一种加权视角。然而，语言中同时存在多种结构：主谓一致、指代消解、句法依赖、语义相似、位置邻近……一个头显然忙不过来。

多头注意力的解决方法是并行运行 hhh 套注意力机制，每套机制都有独立的投影矩阵：

headi=Attention(QWiQ,KWiK,VWiV)\text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V) headi​=Attention(QWiQ​,KWiK​,VWiV​)

将所有头的输出拼接起来后，再通过一个线性层：

MultiHead(Q,K,V)=Concat(head1,…,headh) WO\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W^O MultiHead(Q,K,V)=Concat(head1​,…,headh​)WO

原论文中 dmodel=512d_{\text{model}} = 512dmodel​=512、h=8h = 8h=8，因此每个头的 dk=dv=512/8=64d_k = d_v = 512 / 8 = 64dk​=dv​=512/8=64。每层的总计算量与使用一个大头几乎相同，但模型可以分工合作：后续探针实验表明，不同头确实学会了关注不同的关系，比如句法依赖和指代。

¶因果掩码

在自回归解码器中，训练时不能让位置 iii 看到位置 j>ij > ij>i 的内容，否则模型会直接抄答案。通过加性掩码强制实现这一点：

MaskedAttention(Q,K,V)=softmax ⁣(QK⊤dk+M)V\text{MaskedAttention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}} + M\right) V MaskedAttention(Q,K,V)=softmax(dk​​QK⊤​+M)V

其中 Mij=0M_{ij} = 0Mij​=0（允许，j≤ij \le ij≤i）或 Mij=−∞M_{ij} = -\inftyMij​=−∞（禁止，j>ij > ij>i）。经过 softmax，−∞-\infty−∞ 会变为精确的 0——上图右侧的上三角被干净地抹平了。

正是这个小技巧，让 GPT 类模型能在一次前向传播中计算整个序列的 loss，但在推理时表现得像逐个 token 输出一样。

¶6. 位置编码：把"顺序"找回来

自注意力是置换等变的。输入 token 打乱后，输出也会同步打乱，而注意力权重完全不变。这就导致模型无法区分"猫吃鱼"和"鱼吃猫"——显然有问题。

解决方法是在第一层之前，为每个 token 的嵌入加上一个位置编码 PE(pos)∈Rdmodel\text{PE}(\text{pos}) \in \mathbb{R}^{d_{\text{model}}}PE(pos)∈Rdmodel​。

¶正弦编码

原版 Transformer 使用了一种固定（不学习）的正弦方案：

PE(pos, 2i)=sin⁡ ⁣(pos100002i/dmodel),PE(pos, 2i+1)=cos⁡ ⁣(pos100002i/dmodel)PE_{(\text{pos},\, 2i)} = \sin\!\left(\frac{\text{pos}}{10000^{2i / d_{\text{model}}}}\right), \qquad PE_{(\text{pos},\, 2i+1)} = \cos\!\left(\frac{\text{pos}}{10000^{2i / d_{\text{model}}}}\right) PE(pos,2i)​=sin(100002i/dmodel​pos​),PE(pos,2i+1)​=cos(100002i/dmodel​pos​)

上图左侧是整个编码矩阵的热力图，右侧将几个具体维度绘制成波形：低维振荡快（精细位置），高维振荡慢（粗略位置）。这种设计为每个位置生成独一无二的指纹。同时，sin⁡\sinsin 和 cos⁡\coscos 满足简单的线性恒等式，模型理论上可以学到"往后三个位置"这样的相对偏移。

¶可学习的位置嵌入

另一种方法是将位置视为标准的嵌入表，形状为 (max_len,dmodel)(\text{max\_len}, d_{\text{model}})(max_len,dmodel​)。BERT 和 GPT-2 都采用这种方法——更简单，实际效果通常更好；唯一的缺点是训练时未见过的更长序列无法外推。

¶为什么是相加而不是拼接？

相加保留了 dmodeld_{\text{model}}dmodel​ 的完整维度，供"内容"和"位置"两部分使用，后续的 WQ,WK,WVW_Q, W_K, W_VWQ​,WK​,WV​ 投影会自行学会分离它们。拼接要么增加维度，要么挤占内容容量，得不偿失。

如今主流大模型已基本弃用原版正弦编码：RoPE（旋转位置编码）、ALiBi（注意力分数线性偏置）、NoPE 等新方法主要为了解决长度外推这一难题。原版正弦方案更多作为清晰的入门基线存在。

¶7. 完整的 Transformer 架构

一个 Transformer 就是一组编码器层连接一组解码器层。两边共享三个核心设计：残差连接包裹子层、LayerNorm 和 dropout。

¶编码器层

NNN 个编码器层，每层包含两个子层：

  1. 多头自注意力，作用在源序列上。
  2. 逐位置前馈网络，对每个位置独立计算：

FFN(x)=max⁡(0, xW1+b1) W2+b2\text{FFN}(x) = \max(0,\, x W_1 + b_1)\, W_2 + b_2 FFN(x)=max(0,xW1​+b1​)W2​+b2​

FFN 先将维度扩展到 dff=4⋅dmodeld_{\text{ff}} = 4 \cdot d_{\text{model}}dff​=4⋅dmodel​（通常为 2048），再投影回原维度。这里是模型参数的主要集中地，也是 token 级非线性处理的核心场所。每个子层都被如下结构包裹：

output=LayerNorm(x+Sublayer(x))\text{output} = \text{LayerNorm}(x + \text{Sublayer}(x)) output=LayerNorm(x+Sublayer(x))

残差连接让梯度能从任意一层直接流向之前的任意一层，这对深度为 6、12、24 或 96 的网络至关重要。

¶解码器层

NNN 个解码器层，每层包含三个子层：

  1. 掩码多头自注意力，仅关注已生成的目标 token（通过因果掩码实现）。
  2. 交叉注意力：query 来自解码器，key 和 value 来自编码器最后一层的输出。这是连接源序列和目标序列的关键桥梁。
  3. 前馈网络，与编码器中的完全一致。

¶9. 自注意力 vs. RNN vs. CNN

Transformer 为什么能彻底取代 RNN 和 CNN 在序列建模中的地位？答案藏在这三个数字里（每层）：

架构 单层计算量 串行步数 任意两点最大路径长度
自注意力 O(n2⋅d)O(n^2 \cdot d)O(n2⋅d) O(1)O(1)O(1) O(1)O(1)O(1)
RNN（LSTM/GRU） O(n⋅d2)O(n \cdot d^2)O(n⋅d2) O(n)O(n)O(n) O(n)O(n)O(n)
CNN（核 kkk） O(k⋅n⋅d2)O(k \cdot n \cdot d^2)O(k⋅n⋅d2) O(1)O(1)O(1) O(log⁡kn)O(\log_k n)O(logk​n)

自注意力的代价是 O(n2)O(n^2)O(n2) 的计算量，但回报是恒定的路径长度和完全并行的能力。对于常见的序列长度（n<1000n < 1000n<1000），以及擅长处理大矩阵乘法的现代硬件来说，这笔账非常划算。当序列长度达到几万 token 时，FlashAttention、Longformer、Performer 以及 Mamba 类的状态空间模型才开始显现吸引力。

¶10. 三种工业 Transformer 变体

原版 Transformer 是编码器-解码器架构。两个广泛使用的变体各自砍掉了一半：

家族 架构 预训练目标 擅长 代表
仅编码器 编码器栈 掩码语言建模 分类、NER、检索、QA BERT、RoBERTa、DeBERTa
仅解码器 解码器栈 下一个 token 预测（因果） 生成、对话、写代码、推理 GPT、LLaMA、Claude
编码器-解码器 两个都有 片段还原 / seq2seq 翻译、摘要、有结构的任务 T5、BART、mT5

仅解码器路线最终在大模型竞赛中胜出，原因是"下一个 token 预测"在原始网页文本上扩展得极其漂亮，并且能将几乎所有任务统一成"文本生成"。

¶11. 常见问题

为什么只有解码器需要掩码，编码器不用？
编码器处理完整源句，天然双向；解码器逐个生成 token，训练时若不遮挡未来信息，模型会直接抄答案。BERT 这类仅编码器模型是双向的，正因为它没有因果掩码。

O(n2)O(n^2)O(n2) 的显存到底花在哪儿？
主要花在 n×nn \times nn×n 的注意力分数矩阵上，且是在聚合之前。当 n=4000n = 4000n=4000、h=16h = 16h=16 个头、float16 精度时，单层这个缓冲区就占用 500 MB 以上。FlashAttention 的核心思想是永远不显式物化这个矩阵，而是分块流式计算 softmax。

为什么 dff=4⋅dmodeld_{\text{ff}} = 4 \cdot d_{\text{model}}dff​=4⋅dmodel​？
这是经验值。2017 年认为 4 倍是个不错的选择，便沿用至今。近期工作（如 PaLM、LLaMA）有时调整这个比例，或者将 ReLU 替换为 SwiGLU、GeGLU，以获取边际收益。

Pre-LN 还是 Post-LN？
原论文采用 Post-LN（LayerNorm 在残差之后）。如今的实现几乎全是 Pre-LN（LayerNorm 在子层之前），深层网络下训练更稳定，也不太需要精心设计的学习率 warmup。

今天的大模型还在用正弦位置编码吗？
基本不用了。主流大模型使用 RoPE（LLaMA、GPT-NeoX）或 ALiBi（BLOOM），因为它们能自然外推到更长的上下文，并通过旋转或加性偏置无缝嵌入多头注意力机制中。

¶12. 核心要点

  • 朴素 Seq2Seq 在处理长输入时失效，原因是单一上下文向量容量不足。注意力机制让解码器动态访问每个编码器状态。
  • 自注意力移除了循环结构：任意两个位置之间仅需 O(1)O(1)O(1) 步操作。
  • 缩放点积注意力只有一个公式：softmax(QK⊤/dk)V\text{softmax}(QK^\top / \sqrt{d_k}) Vsoftmax(QK⊤/dk​​)V。其中 dk\sqrt{d_k}dk​​ 决定了训练是否稳定以及梯度是否会崩掉。
  • 多头注意力并行运行多个小注意力模块，不同头分工捕捉不同关系。
  • 位置编码找回了纯注意力丢失的顺序信息。
  • Transformer 块 = 多头注意力 + 前馈网络，每部分外加残差连接和 LayerNorm。堆叠 NNN 层，仅此而已。
  • BERT、GPT、T5 分别是该架构的三种变体："仅编码器""仅解码器""编码器-解码器"。

接下来两篇将深入探讨 BERT 和 GPT——理解架构后，重点在于巧妙的预训练目标与规模扩展。
