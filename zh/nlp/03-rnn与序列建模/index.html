自然语言处理（三）：RNN与序列建模

RNN、LSTM、GRU 如何通过记忆处理序列。从第一性原理推导梯度消失，用 PyTorch 实现字符级文本生成器和 Seq2Seq 翻译器。

#NLP#RNN#深度学习#LSTM
Read in English →

打开翻译软件、用滑动键盘打字、对手机口述备忘——每一项功能背后，都需要一个模型按顺序消费一串 token，再产出另一串。前馈网络把每个输入当成孤立的样本，但语言天生就是有顺序的：要理解"猫坐在垫子上"里"垫子"的含义，你必须知道前面所有词的语境。循环神经网络（RNN）的解决方式是维护一个隐藏状态，每读一个 token 就更新一次。这个隐藏状态，就是网络对过去内容的"持续摘要"，也就是它的记忆。

本文从零开始把循环网络这一族架构串起来。先讲最朴素的 RNN，推导它为什么记不住超过十几个 token，再看 LSTM 和 GRU 用门控机制如何破局，最后用 PyTorch 跑通一个英法翻译器。读完后你会理解从 RNN 走向注意力机制和 Transformer 的真正动因。

你将学到什么

  • RNN 怎么通过循环连接和参数共享记住信息
  • 梯度消失和梯度爆炸的根本原因推导
  • LSTM 的三个门（遗忘门、输入门、输出门）和细胞状态如何解决长距离依赖问题
  • GRU 是 LSTM 的简化版，什么时候该用哪个
  • 双向 RNN 和堆叠 RNN 如何提升每个 token 的表示能力
  • Seq2Seq 编码器-解码器架构的瓶颈，以及为什么注意力机制是必然选择
  • 文本生成和翻译的 PyTorch 实现代码

前置知识：本系列第 1-2 部分（分词与词嵌入），以及基础 PyTorch（张量、nn.Module、训练循环）。

一、核心思想：循环与参数共享

每个时间步 $t$，RNN 接收输入 $x_t$ 和前一个隐藏状态 $h_{t-1}$，然后生成新的隐藏状态和输出：

$$h_t = \tanh(W_h h_{t-1} + W_x x_t + b), \qquad y_t = W_y h_t + b_y.$$

图中最关键的地方是，每一步的箭头都一样——矩阵 $W_h$、$W_x$、$W_y$ 在所有位置共享。这个设计直接带来了三个好处：

  • 跨位置泛化：在第 3 个位置学到的规律，可以直接用到第 30 个位置，因为同一组权重处理了所有位置。
  • 参数数量固定：模型大小和序列长度无关，处理 10 个 token 和 1 万个 token 的存储开销完全相同。
  • 支持变长输入输出：公式里没有任何地方限制 $T$ 是 5 还是 500。

可以想象成网络每次读一个词，每读一个词就更新对整句话的理解。第 $t$ 步的隐藏状态，就是对 $x_1, \dots, x_t$ 这段历史的固定大小的可学习总结。

二、梯度消失问题

训练一开始，问题就来了。为了计算梯度，我需要把网络按时间展开，然后运行反向传播，这就是所谓的沿时间反向传播（BPTT）。第 $T$ 步的损失对第 $t$ 步隐藏状态的梯度，是一串雅可比矩阵的连乘积：

$$\frac{\partial h_T}{\partial h_t} \;=\; \prod_{k=t}^{T-1} \frac{\partial h_{k+1}}{\partial h_k}.$$

每个雅可比因子大约是 $W_h^{\top}\,\mathrm{diag}(\tanh'(\cdot))$。因为 $\tanh' \le 1$，这个因子的谱范数不会超过 $W_h$ 的最大奇异值，记作 $\lambda$。把 $T-t$ 个这样的因子相乘，结果就是：

$$\left\| \frac{\partial h_T}{\partial h_t} \right\| \;\lesssim\; \lambda^{\,T-t}.$$

两种情况很明显，左图也清楚地展示了它们：

  • 如果 $\lambda < 1$，梯度范数会指数级衰减。超过 10–20 步后，数值上基本归零。优化器完全看不出第 $t$ 个 token 对第 $T$ 步的损失有影响，模型自然学不到这种依赖。
  • 如果 $\lambda > 1$，梯度会爆炸——权重更新直接失控，训练一步就发散。

右图举了个具体例子：“那只猫，它坐在垫子上、还大声呼噜，很开心”。主语"猫"和谓语"很"之间隔了十个词。基础 RNN 根本传不了那么远的梯度，所以学不会这种主谓一致。

实际中怎么解决？ 梯度裁剪能搞定爆炸问题——把全局梯度范数限制在 5.0 这样的阈值内。但裁剪对梯度消失没用。真正解决问题的办法是重新设计循环结构，让梯度有一条不会衰减的传播路径。LSTM 和 GRU 就是靠引入这条路径来解决问题的。

三、长短期记忆网络（LSTM）

LSTM（Hochreiter & Schmidhuber, 1997）用一个带门控的单元替换了简单的循环单元。这个单元不仅维护隐藏状态 $h_t$，还显式地保留了一条长期记忆 $C_t$。

三个门

记 $[h_{t-1}, x_t]$ 为上一隐藏状态和当前输入的拼接结果，所有门都共享这一输入。

遗忘门——决定丢弃哪些长期记忆：

$$f_t = \sigma(W_f [h_{t-1}, x_t] + b_f).$$$$ i_t = \sigma(W_i [h_{t-1}, x_t] + b_i), \qquad \tilde{C}_t = \tanh(W_C [h_{t-1}, x_t] + b_C). $$$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t.$$$$ o_t = \sigma(W_o [h_{t-1}, x_t] + b_o), \qquad h_t = o_t \odot \tanh(C_t). $$

这里 $\sigma$ 是 sigmoid 函数（一个软性的 0–1 开关），$\odot$ 表示逐元素乘法。每个门的作用是一个可学习的、位置相关的决策：忘掉这个，写入那个，露出这些。

为什么这解决了梯度消失问题

$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t.$$

求导后得到 $\partial C_t / \partial C_{t-1} = f_t$，这是一个逐元素的标量，取值范围在 $[0,1]$。如果遗忘门的值接近 1，连乘 $\prod f_k$ 也会保持接近 1——梯度可以沿着细胞状态几乎无损地传递，哪怕跨越几百步。相比之下，基础 RNN 的更新通过 $W_h$ 是乘法形式，容易导致梯度逐渐归零。LSTM 把全局共享的 $W_h$ 替换成了可学习、随时间变化的 $f_t$，仅此一点改动，就让它能够建模长上下文。

传送带类比

可以把细胞状态想象成一条贯穿整个序列的传送带。遗忘门像是一个工人，负责取下不再需要的物品；输入门是另一个工人，负责放上新物品；输出门则是一扇窗，决定外部世界（网络的其他部分）此刻能看到什么。第 3 步放上去的物品，可以一路安稳地传送到第 300 步。

四、门控循环单元（GRU）

$$ z_t = \sigma(W_z [h_{t-1}, x_t]), \qquad r_t = \sigma(W_r [h_{t-1}, x_t}), $$$$ \tilde{h}_t = \tanh(W [r_t \odot h_{t-1}, x_t]), \qquad h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t. $$

重置门 $r_t$ 决定过去信息对候选值的影响有多大。更新门 $z_t$ 在旧状态和新候选之间做线性插值。当 $z_t \approx 0$ 时，GRU 直接把 $h_{t-1}$ 复制到下一步，这和 LSTM 的细胞状态高速公路保梯度的机制是一样的。

LSTM vs. GRU

对比维度LSTMGRU
门数量3 个（遗忘、输入、输出）2 个（重置、更新）
是否有独立细胞状态有（$C_t$）无（只有 $h_t$）
参数量约为基础 RNN 的 $4\times$约为基础 RNN 的 $3\times$（比 LSTM 少 25% 左右）
长序列表现在很多基准上略占优相当
训练速度较慢较快

经验法则：优先用 GRU。它训练更快，超参更少，多数任务上和 LSTM 的精度差距可以忽略不计。如果序列特别长，或者任务明确需要更大容量（比如部分语音任务），再换成 LSTM。

五、双向 RNN

很多任务中，未来的信息和过去一样重要。比如句子“他说这道菜不好吃”——如果看不到“不”，从左到右读的模型会把“好吃”误判为正面情感。

$$ \overrightarrow{h}_t = \mathrm{RNN}_\text{fwd}(x_t, \overrightarrow{h}_{t-1}), \qquad \overleftarrow{h}_t = \mathrm{RNN}_\text{bwd}(x_t, \overleftarrow{h}_{t+1}), $$$$ h_t = \big[\overrightarrow{h}_t \,;\, \overleftarrow{h}_t\big]. $$

这样一来，每个位置的表征都能看到前后两个方向的上下文。

适合用在：命名实体识别、词性标注、机器翻译的编码器等场景。只要输入数据是完整的，就可以放心使用。

不适合用在：流式处理或自回归生成任务。因为反向循环需要未来的 token，而逐个生成时这些 token 还不存在。

六、堆叠 RNN

$$ h_t^{(1)} = \mathrm{RNN}^{(1)}(x_t,\, h_{t-1}^{(1)}), \qquad h_t^{(2)} = \mathrm{RNN}^{(2)}(h_t^{(1)},\, h_{t-1}^{(2)}). $$

实际使用中，低层通常学习局部模式，比如字符 n-gram、词边界和形态学特征，而高层则负责捕捉句法结构和长距离语义。对于大多数 NLP 任务，2 到 4 层是最佳选择；如果再加深，残差连接就变得必不可少，否则优化过程容易不稳定。

七、序列到序列模型

Seq2Seq 架构（Sutskever 等人，2014）将输入序列映射为长度不同的输出序列，典型例子就是机器翻译。它用了两个 RNN：

  • 编码器读取整个输入，将其压缩成一个上下文向量 $c = h_T^{\text{enc}}$。
  • 解码器逐个生成输出 token，每一步都依赖 $c$ 和已经生成的 token： $$ s_t = \mathrm{RNN}_\text{dec}(y_{t-1}, s_{t-1}), \qquad P(y_t \mid y_{瓶颈问题。 整个输入——可能包含 50 个词——都要塞进固定大小的向量 $c$。短句子还好办，长句子就麻烦了，编码器根本装不下。Sutskever 的原论文提到，输入超过 30 个 token 后，BLEU 分数会急剧下降；而把源句倒过来处理反而能改善结果，这其实已经暗示了瓶颈是核心问题。

这个瓶颈直接催生了注意力机制，也就是第 4 部分的内容：与其让解码器只靠一个 $c$ 勉强支撑，不如在每一步都让它回顾所有编码器的隐藏状态。

class CharRNN(nn.Module):
    def __init__(self, vocab_size, hidden_size, num_layers):
        super().__init__()
        self.hidden_size = hidden_size
        self.num_layers = num_layers
        self.embedding = nn.Embedding(vocab_size, hidden_size)
        self.lstm = nn.LSTM(hidden_size, hidden_size, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, x, hidden):
        embedded = self.embedding(x)
        out, hidden = self.lstm(embedded, hidden)
        out = self.fc(out.reshape(-1, self.hidden_size))
        return out, hidden

    def init_hidden(self, batch_size, device):
        h0 = torch.zeros(self.num_layers, batch_size, self.hidden_size).to(device)
        c0 = torch.zeros(self.num_layers, batch_size, self.hidden_size).to(device)
        return (h0, c0)

def train(model, text_data, epochs=100, seq_length=50, batch_size=16, lr=0.002):
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model = model.to(device)
    criterion = nn.CrossEntropyLoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    data = [char_to_idx[ch] for ch in text_data]

    for epoch in range(epochs):
        model.train()
        hidden = model.init_hidden(batch_size, device)
        total_loss = 0
        n_batches = len(data) // (seq_length * batch_size)

        for _ in range(n_batches):
            starts = np.random.randint(0, len(data) - seq_length - 1, batch_size)
            inputs = torch.LongTensor([data[s:s+seq_length] for s in starts]).to(device)
            targets = torch.LongTensor([data[s+1:s+seq_length+1] for s in starts]).to(device)

            hidden = tuple(h.detach() for h in hidden)        # 截断 BPTT
            outputs, hidden = model(inputs, hidden)
            loss = criterion(outputs, targets.reshape(-1))

            optimizer.zero_grad()
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), 5.0)   # 防梯度爆炸
            optimizer.step()
            total_loss += loss.item()

        if (epoch + 1) % 20 == 0:
            print(f"Epoch {epoch+1}/{epochs}, Loss: {total_loss/n_batches:.4f}")

这里用了两个 RNN 的小技巧：在 batch 之间 detach 隐藏状态，截断 BPTT，避免梯度回传到整个训练语料；梯度裁剪，防止梯度爆炸。

def generate(model, start_str, length=200, temperature=0.8):
    device = next(model.parameters()).device
    model.eval()
    hidden = model.init_hidden(1, device)
    input_seq = [char_to_idx[ch] for ch in start_str]
    generated = start_str

    with torch.no_grad():
        for idx in input_seq[:-1]:                     # 先把隐藏状态"喂热"
            x = torch.LongTensor([[idx]]).to(device)
            _, hidden = model(x, hidden)

        x = torch.LongTensor([[input_seq[-1]]]).to(device)
        for _ in range(length):
            output, hidden = model(x, hidden)
            probs = torch.softmax(output.squeeze() / temperature, dim=0).cpu().numpy()
            char_idx = np.random.choice(len(probs), p=probs)
            generated += idx_to_char[char_idx]
            x = torch.LongTensor([[char_idx]]).to(device)

    return generated

model = CharRNN(vocab_size, hidden_size=128, num_layers=2)
train(model, text, epochs=100)
print(generate(model, "Deep learning", length=200))

温度在 softmax 前对 logits 做缩放：$P(w) = \mathrm{softmax}(\text{logits}/T)$。低温（约 0.5）让分布变尖，输出保守且容易重复；高温（1.5 以上）让分布变平，输出更有创意但也更容易胡言乱语。$T=0.8$ 是个常见的折中。

import torch
import torch.nn as nn
import random

pairs = [
    ("hello", "bonjour"), ("good morning", "bon matin"),
    ("thank you", "merci"), ("goodbye", "au revoir"),
    ("how are you", "comment allez vous"),
    ("i love you", "je t aime"), ("welcome", "bienvenue"),
]

SOS, EOS = 0, 1

class Vocab:
    def __init__(self):
        self.word2idx = {"<SOS>": SOS, "<EOS>": EOS}
        self.idx2word = {SOS: "<SOS>", EOS: "<EOS>"}
        self.n_words = 2

    def add_sentence(self, sentence):
        for word in sentence.split():
            if word not in self.word2idx:
                self.word2idx[word] = self.n_words
                self.idx2word[self.n_words] = word
                self.n_words += 1

src_vocab, tgt_vocab = Vocab(), Vocab()
for en, fr in pairs:
    src_vocab.add_sentence(en)
    tgt_vocab.add_sentence(fr)

class Encoder(nn.Module):

        super().__init__()
        self.embedding = nn.Embedding(vocab_size, hidden_size)
        self.lstm = nn.LSTM(hidden_size, hidden_size, batch_first=True)

    def forward(self, x):
        return self.lstm(self.embedding(x))

class Decoder(nn.Module):
    def __init__(self, hidden_size, vocab_size):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, hidden_size)
        self.lstm = nn.LSTM(hidden_size, hidden_size, batch_first=True)
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, x, hidden):
        out, hidden = self.lstm(self.embedding(x), hidden)
        return self.fc(out.squeeze(1)), hidden

def train_seq2seq(encoder, decoder, pairs, epochs=500, lr=0.01):
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    encoder, decoder = encoder.to(device), decoder.to(device)
    enc_opt = torch.optim.Adam(encoder.parameters(), lr=lr)
    dec_opt = torch.optim.Adam(decoder.parameters(), lr=lr)
    criterion = nn.CrossEntropyLoss()

    for epoch in range(epochs):
        total_loss = 0
        random.shuffle(pairs)
        for src_sent, tgt_sent in pairs:
            src_ids = [src_vocab.word2idx[w] for w in src_sent.split()] + [EOS]
            tgt_ids = [tgt_vocab.word2idx[w] for w in tgt_sent.split()] + [EOS]

            src_t = torch.LongTensor([src_ids]).to(device)
            tgt_t = torch.LongTensor(tgt_ids).to(device)

            enc_opt.zero_grad(); dec_opt.zero_grad()

            _, hidden = encoder(src_t)                     # 上下文向量
            dec_input = torch.LongTensor([[SOS]]).to(device)
            loss = 0

            for i in range(len(tgt_ids)):
                output, hidden = decoder(dec_input, hidden)
                loss += criterion(output, tgt_t[i:i+1])
                dec_input = tgt_t[i:i+1].unsqueeze(0)      # teacher forcing
            loss.backward()
            enc_opt.step(); dec_opt.step()
            total_loss += loss.item() / len(tgt_ids)

        if (epoch + 1) % 100 == 0:
            print(f"Epoch {epoch+1}/{epochs}, Loss: {total_loss/len(pairs):.4f}")

def translate(encoder, decoder, sentence):
    device = next(encoder.parameters()).device
    encoder.eval(); decoder.eval()
    with torch.no_grad():
        src_ids = [src_vocab.word2idx.get(w, 0) for w in sentence.split()] + [EOS]
        _, hidden = encoder(torch.LongTensor([src_ids]).to(device))
        dec_input = torch.LongTensor([[SOS]]).to(device)
        words = []
        for _ in range(20):
            output, hidden = decoder(dec_input, hidden)
            token = output.argmax(dim=-1).item()
            if token == EOS:
                break
            words.append(tgt_vocab.idx2word[token])
            dec_input = torch.LongTensor([[token]]).to(device)
    return ' '.join(words)

hidden_size = 256
encoder = Encoder(src_vocab.n_words, hidden_size)
decoder = Decoder(hidden_size, tgt_vocab.n_words)
train_seq2seq(encoder, decoder, pairs)
for s in ["hello", "thank you", "good morning"]:
    print(f"{s} -> {translate(encoder, decoder, s)}")

注意事项。 这个实现非常简单，只会过拟合一个小短语表，用的是贪婪解码，没有注意力机制和束搜索。它的作用仅仅是帮助理解编码器-解码器的数据流。如果要做真正的翻译系统，还需要加入注意力机制（第四部分）、束搜索、子词（BPE）分词，以及用验证集实现早停。

十、它们的实际差距有多大

这两张图清楚地说明了问题。在长依赖任务中，基础 RNN 的训练损失很快停滞在高位，而 LSTM 和 GRU 则持续下降——梯度高速公路确实发挥了作用。随着序列长度增加，基础 RNN 的精度急剧下滑，LSTM 和 GRU 的表现则相对平稳。在大多数情况下，LSTM 和 GRU 的差距很小，这也是我通常选择 GRU 作为默认选项的原因。

注意力机制预告

$$ \alpha_{tj} = \frac{\exp(\mathrm{score}(s_t, h_j))}{\sum_k \exp(\mathrm{score}(s_t, h_k))}, \qquad c_t = \sum_j \alpha_{tj}\, h_j. $$

上下文向量现在变成了编码器状态的动态加权和。这正是从 RNN 迈向 Transformer 的关键一步，我会在第 4 部分详细讲。

常见问题

为什么基础 RNN 用 tanh 而不是 ReLU？

tanh 的输出范围是 $[-1, 1]$，能让隐藏状态在时间维度上保持有界。ReLU 在正方向没有上限，循环计算时很容易指数级爆炸。LSTM 的门控用 sigmoid（软性的 0–1 开关），候选值用 tanh（零中心，既能加也能减，灵活调整细胞状态）。

什么是 teacher forcing，它有什么缺点？

训练时，我会把真实的上一个 token 作为解码器输入，而不是模型自己的预测。这样能让训练初期更稳定，但会导致训练和推理不一致——推理时解码器必须处理自己生成的（带噪声的）输出，而这些输出在训练阶段从未出现过。常用的办法是 scheduled sampling：随着训练推进，逐步增加使用模型自身预测的概率。

温度对生成有什么影响？

温度会在 softmax 前对 logits 进行缩放：$P(w) = \mathrm{softmax}(\text{logits}/T)$。低温（比如 0.5）会让分布更尖锐、更保守；高温（比如 1.5）会让分布更平滑、更有创意，但也更容易出错。贪婪解码可以看作是 $T \to 0$ 的极限情况。

Transformer 出现后，RNN 还有用吗？

Transformer 在离线 NLP 基准测试中已经全面领先，但 RNN 在某些场景下依然有价值：(i) 参数量有限；(ii) 真正的流式/在线推理（不需要对整个前缀重新计算注意力）；(iii) 每一步的内存占用固定，不随序列长度增加。RNN 在时间序列预测和端侧语音模型中仍然常见。而且，注意力机制可以理解为“如果把 LSTM 的遗忘门换成对所有历史状态的可学习回顾，会怎样？”从这个角度看，理解 RNN 是理解 Transformer 的最佳起点。

核心要点

  • RNN 用循环连接和参数共享处理带记忆的序列——每个时间步都用同一组权重。
  • 基础 RNN 搞不定长序列，因为雅可比连乘 $\prod \partial h_{k+1} / \partial h_k$ 会指数级衰减或爆炸。
  • LSTM 加了个细胞状态高速公路，通过遗忘门、输入门和输出门控制，让梯度有条不会消失的路径。
  • GRU 把 LSTM 简化成两个门加一个状态，通常用比 LSTM 少 25% 的参数就能达到差不多的精度。
  • 双向和堆叠变体分别扩展了单个位置的上下文范围和网络深度。
  • Seq2Seq 编码器-解码器能实现序列到序列的映射，但卡在单一上下文向量 $c$ 上——这个瓶颈正是第 4 部分讲注意力机制的原因。
