自然语言处理（三）：RNN与序列建模

八月 23, 2025 Updated 5月 06, 2026 — min read — words

RNN、LSTM、GRU 如何通过记忆处理序列。从第一性原理推导梯度消失，用 PyTorch 实现字符级文本生成器和 Seq2Seq 翻译器。

打开翻译软件、用滑动键盘打字、对手机口述备忘——每一项功能背后，都有一个模型在按顺序消费一串 token，再生成另一串。前馈网络将每个输入视为孤立的样本，但语言天生有顺序："猫坐在垫子上"中"垫子"的含义，依赖于前面所有词的语境。循环神经网络（RNN）通过维护一个隐藏状态来解决这个问题，每读取一个 token 就更新一次这个状态。隐藏状态是网络对过去内容的"持续摘要"，也是它的记忆。

本文从零开始梳理循环网络这一族架构。先讲解最基础的 RNN，推导它为何记不住超过十几个 token；接着看 LSTM 和 GRU 如何用门控机制突破限制；最后用 PyTorch 实现一个英法翻译器。读完后，你会明白从 RNN 走向注意力机制和 Transformer 的真正原因。

¶你将学到什么

  • RNN 如何通过循环连接和参数共享保持记忆
  • 从第一性原理推导梯度消失与梯度爆炸的原因
  • LSTM 的三个门（遗忘门、输入门、输出门）与细胞状态高速公路如何解决长距离依赖问题
  • GRU 作为 LSTM 的简化版，选择时的权衡点
  • 双向 RNN 和堆叠 RNN 如何增强每个位置的表征能力
  • Seq2Seq 编码器-解码器架构的设计、瓶颈，以及注意力机制为何成为必然选择
  • 文本生成与翻译任务的 PyTorch 实现方法

前置知识：本系列第 1-2 部分（分词与词嵌入），以及基础 PyTorch（张量、nn.Module、训练循环）。

¶一、核心思想：循环与参数共享

每个时间步 ttt，RNN 接收输入 xtx_txt​ 和上一时刻的隐藏状态 ht−1h_{t-1}ht−1​，计算新的隐藏状态和输出：

ht=tanh⁡(Whht−1+Wxxt+b),yt=Wyht+by.h_t = \tanh(W_h h_{t-1} + W_x x_t + b), \qquad y_t = W_y h_t + b_y. ht​=tanh(Wh​ht−1​+Wx​xt​+b),yt​=Wy​ht​+by​.

图中最关键的细节是，同样的箭头在每个时间步重复出现——矩阵 WhW_hWh​、WxW_xWx​、WyW_yWy​ 在所有位置共用同一份参数。这一设计带来三个显著优势：

  • 位置间泛化：第 3 个位置学到的模式，可以直接用在第 30 个位置，因为同一组权重处理过两边。
  • 参数量恒定：模型大小与序列长度无关，处理 10 个 token 和 1 万个 token 占用的存储完全相同。
  • 支持变长：公式中没有任何地方依赖序列长度 TTT 是 5 还是 500。

形象地说，可以把网络看作逐词阅读，每读一个词就更新对整句话的"理解"。第 ttt 步的隐藏状态，就是 x1,…,xtx_1, \dots, x_tx1​,…,xt​ 这段历史的固定大小的可学习摘要。

¶二、梯度消失问题

从训练说起。计算梯度时，需将网络沿时间维度展开后进行反向传播，这一方法称为"沿时间反向传播"（BPTT）。第 TTT 步的损失对第 ttt 步隐藏状态的梯度，表现为一连串雅可比矩阵的乘积：

∂hT∂ht  =  ∏k=tT−1∂hk+1∂hk.\frac{\partial h_T}{\partial h_t} \;=\; \prod_{k=t}^{T-1} \frac{\partial h_{k+1}}{\partial h_k}. ∂ht​∂hT​​=k=t∏T−1​∂hk​∂hk+1​​.

每个雅可比因子近似为 Wh⊤ diag(tanh⁡′(⋅))W_h^{\top}\,\mathrm{diag}(\tanh'(\cdot))Wh⊤​diag(tanh′(⋅))。由于 tanh⁡′≤1\tanh' \le 1tanh′≤1，其谱范数受 WhW_hWh​ 的最大奇异值（记作 λ\lambdaλ）限制。将 T−tT-tT−t 个这样的因子相乘，结果为：

∥∂hT∂ht∥  ≲  λ T−t.\left\| \frac{\partial h_T}{\partial h_t} \right\| \;\lesssim\; \lambda^{\,T-t}. ​∂ht​∂hT​​​≲λT−t.

由此产生两种情况，左图已清晰展示：

  • 若 λ<1\lambda < 1λ<1，梯度范数指数级衰减。大约 10–20 步以外，数值上几乎归零。优化器无法感知第 ttt 个 token 对第 TTT 步损失的贡献，模型自然学不到这种依赖。
  • 若 λ>1\lambda > 1λ>1，梯度则指数级爆炸——一次更新可能直接让权重失控，训练随即发散。

右图进一步说明了这一点。以句子"那只猫，它坐在垫子上、还大声呼噜，很开心"为例，主语"猫"与谓语"很"之间隔了十个词。基础 RNN 难以将梯度传递如此远的距离，因此无法学会这种主谓对应关系。

实践中如何应对？ 梯度裁剪（将全局梯度范数限制在 5.0 等阈值）能缓解梯度爆炸，但对梯度消失无能为力。真正解决问题需要重新设计循环结构，为梯度提供一条不会衰减的传播路径。LSTM 和 GRU 的核心贡献，正是引入了这条路径。

¶三、长短期记忆网络（LSTM）

LSTM（Hochreiter & Schmidhuber, 1997）用带门控的单元替换了简单的循环单元，并显式维护一条长期记忆 CtC_tCt​，与隐藏状态 hth_tht​ 并行运转。

¶三个门

记 [ht−1,xt][h_{t-1}, x_t][ht−1​,xt​] 为上一隐藏状态与当前输入的拼接，所有门共享这同一份输入。

遗忘门——决定从长期记忆中丢弃哪些内容：

ft=σ(Wf[ht−1,xt]+bf).f_t = \sigma(W_f [h_{t-1}, x_t] + b_f). ft​=σ(Wf​[ht−1​,xt​]+bf​).

输入门与候选值——决定写入哪些新信息：

it=σ(Wi[ht−1,xt]+bi),C~t=tanh⁡(WC[ht−1,xt]+bC).i_t = \sigma(W_i [h_{t-1}, x_t] + b_i), \qquad \tilde{C}_t = \tanh(W_C [h_{t-1}, x_t] + b_C). it​=σ(Wi​[ht−1​,xt​]+bi​),C~t​=tanh(WC​[ht−1​,xt​]+bC​).

细胞状态更新——结合旧记忆与新内容：

Ct=ft⊙Ct−1+it⊙C~t.C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t. Ct​=ft​⊙Ct−1​+it​⊙C~t​.

输出门——决定从细胞中提取哪部分作为新的隐藏状态：

ot=σ(Wo[ht−1,xt]+bo),ht=ot⊙tanh⁡(Ct).o_t = \sigma(W_o [h_{t-1}, x_t] + b_o), \qquad h_t = o_t \odot \tanh(C_t). ot​=σ(Wo​[ht−1​,xt​]+bo​),ht​=ot​⊙tanh(Ct​).

其中 σ\sigmaσ 是 sigmoid（一个 0–1 的软开关），⊙\odot⊙ 是逐元素乘法。每个门都是一次"看情况"的可学习决策：这条丢、那条写、这块露出来。

¶为什么这就解决了梯度消失

整张图最关键的部分是顶端那条细胞状态线。它的更新是加法：

Ct=ft⊙Ct−1+it⊙C~t.C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t. Ct​=ft​⊙Ct−1​+it​⊙C~t​.

求导得到 ∂Ct/∂Ct−1=ft\partial C_t / \partial C_{t-1} = f_t∂Ct​/∂Ct−1​=ft​，这是一个逐元素的标量，取值范围在 [0,1][0,1][0,1] 之间。只要遗忘门保持在 1 附近，连乘 ∏fk\prod f_k∏fk​ 就能维持在 1 附近——梯度可以沿着细胞状态高速公路几乎无损地传递，哪怕跨越几百步。这与基础 RNN 经过 WhW_hWh​ 的乘法更新形成鲜明对比，后者一路相乘最终归零。LSTM 把"全局共享的 WhW_hWh​"换成"可学习、随时间变化的 ftf_tft​"，仅凭这一改动，就让模型能够建模长上下文。

¶传送带类比

把细胞状态想象成一条贯穿整个序列的传送带。遗忘门是一个工人，负责取下不再需要的物品；输入门是另一个工人，负责放上新物品；输出门则是一扇窗，决定外部世界（网络的其他部分）此刻能看到什么。第 3 步放上去的物品，可以一路安稳地走到第 300 步。

¶四、门控循环单元（GRU）

GRU（Cho 等人，2014）保留了门控思想，但设计更简单。它将遗忘门和输入门合并为一个更新门，去掉独立的细胞状态，直接对 hth_tht​ 操作：

zt=σ(Wz[ht−1,xt]),rt=σ(Wr[ht−1,xt]),z_t = \sigma(W_z [h_{t-1}, x_t]), \qquad r_t = \sigma(W_r [h_{t-1}, x_t]), zt​=σ(Wz​[ht−1​,xt​]),rt​=σ(Wr​[ht−1​,xt​]),

h~t=tanh⁡(W[rt⊙ht−1,xt]),ht=(1−zt)⊙ht−1+zt⊙h~t.\tilde{h}_t = \tanh(W [r_t \odot h_{t-1}, x_t]), \qquad h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t. h~t​=tanh(W[rt​⊙ht−1​,xt​]),ht​=(1−zt​)⊙ht−1​+zt​⊙h~t​.

重置门 rtr_trt​ 控制候选值计算时引入多少过去信息。更新门 ztz_tzt​ 在旧状态和新候选之间做线性插值——当 zt≈0z_t \approx 0zt​≈0 时，GRU 将 ht−1h_{t-1}ht−1​ 原样复制到下一步。这与 LSTM 通过细胞状态高速公路保留梯度的方式一致。

¶LSTM vs. GRU

对比维度 LSTM GRU
门数量 3 个（遗忘、输入、输出） 2 个（重置、更新）
是否有独立细胞状态 有（CtC_tCt​） 无（只有 hth_tht​）
参数量 约为基础 RNN 的 4×4\times4× 约为基础 RNN 的 3×3\times3×（比 LSTM 少 25% 左右）
长序列表现 在很多基准上略占优 相当
训练速度 较慢 较快

经验法则：优先选择 GRU。它训练更快、超参数更少，多数任务中与 LSTM 的精度差异在噪声范围内。如果序列特别长，或任务明确需要更大容量（如部分语音任务），再切换到 LSTM。

¶五、双向 RNN

很多任务中，未来的信息和过去同样重要。比如"他说这道菜不好吃"——如果不看到"不"，从左往右读的模型会把"好吃"误判为正面情感。

双向 RNN（Schuster & Paliwal, 1997）运行两条独立的循环，然后拼接它们的状态：

h→t=RNNfwd(xt,h→t−1),h←t=RNNbwd(xt,h←t+1),\overrightarrow{h}_t = \mathrm{RNN}_\text{fwd}(x_t, \overrightarrow{h}_{t-1}), \qquad \overleftarrow{h}_t = \mathrm{RNN}_\text{bwd}(x_t, \overleftarrow{h}_{t+1}), ht​=RNNfwd​(xt​,ht−1​),ht​=RNNbwd​(xt​,ht+1​),

ht=[h→t ; h←t].h_t = \big[\overrightarrow{h}_t \,;\, \overleftarrow{h}_t\big]. ht​=[ht​;ht​].

每个位置的表征都能同时看到双向的上下文。

适用场景：命名实体识别、词性标注、机器翻译的编码器——只要能一次性拿到完整输入，就可以使用。

不适用场景：流式或自回归生成。反向那一遍需要未来的 token，但生成时这些 token 尚未产生。

¶六、堆叠 RNN

加深网络同样有效：堆叠多层 RNN 能让每一层基于前一层的输出逐步构建特征：

ht(1)=RNN(1)(xt, ht−1(1)),ht(2)=RNN(2)(ht(1), ht−1(2)).h_t^{(1)} = \mathrm{RNN}^{(1)}(x_t,\, h_{t-1}^{(1)}), \qquad h_t^{(2)} = \mathrm{RNN}^{(2)}(h_t^{(1)},\, h_{t-1}^{(2)}). ht(1)​=RNN(1)(xt​,ht−1(1)​),ht(2)​=RNN(2)(ht(1)​,ht−1(2)​).

经验表明，低层学习局部模式（字符 n-gram、词边界、形态学特征），高层捕捉句法结构和长距离语义依赖。在多数 NLP 任务中，2–4 层已足够；更深的网络需要引入残差连接，否则优化过程容易失去稳定性。

¶七、序列到序列模型

Seq2Seq 架构（Sutskever 等人，2014）将输入序列映射为长度不同的输出序列，典型应用是机器翻译。它由两个 RNN 组成：

  • 编码器读取整段输入，将其压缩为上下文向量 c=hTencc = h_T^{\text{enc}}c=hTenc​。
  • 解码器逐 token 生成输出，每一步都依赖 ccc 和已生成的 token：

st=RNNdec(yt−1,st−1),P(yt∣y<t,x)=softmax(Wost).s_t = \mathrm{RNN}_\text{dec}(y_{t-1}, s_{t-1}), \qquad P(y_t \mid y_{<t}, x) = \mathrm{softmax}(W_o s_t). st​=RNNdec​(yt−1​,st−1​),P(yt​∣y<t​,x)=softmax(Wo​st​).

瓶颈问题。 整段输入——可能包含多达 50 个词——必须挤进固定大小的向量 ccc。短句尚可应付，长句则难以容纳。Sutskever 的原论文指出，当输入超过约 30 个 token 后，BLEU 分数迅速下降；而倒置源句竟能改善结果，这本身便暗示了瓶颈才是核心问题。

这个瓶颈正是注意力机制的直接诱因，也就是第 4 部分的内容：与其让解码器仅依赖一个 ccc，不如在每一步都回顾所有编码器隐藏状态。

¶十、它们实际差距有多大

上面这两幅图把整件事讲清楚了。在长依赖任务上，基础 RNN 的训练损失早早卡住，而 LSTM 和 GRU 持续下降——梯度高速公路确实在起作用。随着序列变长，基础 RNN 的精度断崖式下跌，LSTM 和 GRU 则平缓退化。LSTM 与 GRU 之间的差距在大多数场景里都很小，这也是为什么 GRU 是个合理的默认选项。

¶注意力机制预告

编码器将所有信息压缩到单一向量 ccc 中。对于长句子，这个瓶颈容易丢失信息。注意力机制让解码器在每一步都能访问所有编码器隐藏状态，权重通过学习获得：

αtj=exp⁡(score(st,hj))∑kexp⁡(score(st,hk)),ct=∑jαtj hj.\alpha_{tj} = \frac{\exp(\mathrm{score}(s_t, h_j))}{\sum_k \exp(\mathrm{score}(s_t, h_k))}, \qquad c_t = \sum_j \alpha_{tj}\, h_j. αtj​=∑k​exp(score(st​,hk​))exp(score(st​,hj​))​,ct​=j∑​αtj​hj​.

上下文向量从此变为编码器状态的随时间变化的加权和。这正是从 RNN 通向 Transformer 的关键桥梁，第 4 部分会深入探讨。

¶常见问题

为什么基础 RNN 用 tanh 而不是 ReLU？ tanh 输出范围是 [−1,1][-1, 1][−1,1]，能让隐藏状态在时间维度上保持有界。ReLU 在正方向无上界，循环作用下容易指数级爆炸。LSTM 的门使用 sigmoid（软 0–1 开关），候选值使用 tanh（零中心，可以对细胞状态做加法也可以做减法）。

什么是 teacher forcing，它有什么副作用？ 训练时将真实的上一步 token 作为解码器输入，而不是模型自己的预测。这能稳定早期训练，但带来了训练-推理不匹配——推理时解码器必须使用自身（带噪声的）输出，而它在训练中从未见过这种输入。常用缓解方法是 scheduled sampling：训练过程中逐步提高"使用模型自身预测"的概率。

温度对生成的影响？ 温度在 softmax 前对 logits 做缩放：P(w)=softmax(logits/T)P(w) = \mathrm{softmax}(\text{logits}/T)P(w)=softmax(logits/T)。低温（0.5）让分布变尖、更保守；高温（1.5）让分布变平、更有创意但也更容易出错。贪婪解码相当于 T→0T \to 0T→0 的极限。

Transformer 之后 RNN 还有意义吗？ 在离线 NLP 基准上，Transformer 已经全面占优，但 RNN 在以下场景仍然有用：(i) 参数预算非常紧张；(ii) 真正的流式/在线推理（不需要每来一个 token 就重新对整个前缀做注意力）；(iii) 单步内存恒定、不随序列长度增长。它们在时间序列预测和端侧语音模型中依然常见。而且——注意力机制本身就可以理解为"如果把 LSTM 的遗忘门换成对全部历史状态的可学习回望，会怎样？"——理解 RNN 是理解 Transformer 最直接的路径。

¶核心要点

  • RNN 借助循环连接和参数共享处理带记忆的序列——同一组权重作用于每个时间步。
  • 基础 RNN 在长序列上失效，原因是雅可比连乘 ∏∂hk+1/∂hk\prod \partial h_{k+1} / \partial h_k∏∂hk+1​/∂hk​ 会指数级缩小（或爆炸）。
  • LSTM 引入加法式的细胞状态高速公路，通过遗忘门、输入门和输出门控制，为梯度提供一条不消失的路径。
  • GRU 将 LSTM 简化为两个门和一个状态，通常以比 LSTM 少 25% 的参数达到相近的精度。
  • 双向和堆叠变体分别扩展了每个位置的上下文范围、增加了网络深度。
  • Seq2Seq 编码器-解码器实现序列到序列的映射，但受限于单一上下文向量 ccc——这一瓶颈直接推动了注意力机制（第 4 部分）的发展。
