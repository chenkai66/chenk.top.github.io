自然语言处理（一）：NLP入门与文本预处理

八月 15, 2025 Updated 5月 06, 2026 — min read — words

从第一性原理出发的 NLP 入门：梳理四个时代的脉络，亲手搭建从清洗到向量化的完整流水线，把分词、TF-IDF、n-gram 与分布式表示背后的数学讲清楚。

每次你向通义千问提问、让 GitHub Copilot 补全代码，或者打开 Google 翻译——实际上都在调用一个耗费七十年搭建的技术栈。自然语言处理（NLP）研究的核心，是教会机器读取、评分、改写并生成人类语言。有趣的是，现代体系的底层仍然大量依赖几十年前发明的一小批预处理工具。

本系列的第一篇聚焦两件事。第一，绘制地图：这门学科从何而来，如今覆盖哪些领域，为何工具演化成当前的模样。第二，亲手搭建最底层的部分——清洗、分词、标准化、特征提取，所有代码均可直接复用。读完后，你会得到一条可复用的预处理流水线，更重要的是，你会清楚每一步何时该用、何时反而会悄悄破坏信号。

¶你将学到什么

  • NLP 的四个范式，以及每次替换背后的技术原因
  • 讲清楚“分词”：字符级、词级、子词级，以及 BPE 为何占优
  • 使用 NLTK、spaCy、scikit-learn 搭建一条可配置的预处理流水线
  • BoW 和 TF-IDF 背后的数学原理，以及如何解读这两个矩阵
  • Zipf 定律、n-gram 语言模型，以及 one-hot 表示的局限性
  • 一张速查表：不同任务需要哪些预处理，哪些可以跳过

前置要求：熟练编写 Python，对 NumPy / pandas 有基本了解，无需任何 NLP 背景。

¶1. NLP 的四个时代

NLP 的发展不是平稳前进，而是跳跃式演进。每次飞跃都源于一种新的语言表示方法。理清这条主线，挑选工具时就会更有直觉：规则系统在窄域填表场景中依然胜过神经网络；统计方法在搜索排序中仍是主力；嵌入表示则主导了几乎所有其他场景。

¶1.1 符号主义时代（1950s — 1980s 末）

早期系统把语言当作逻辑题来解。1966 年的 ELIZA 使用手写正则匹配用户输入，并重新拼接捕获组；1970 年的 SHRDLU 借助手写文法解析“积木世界”中的指令。这些系统在自己的小领域内精度很高，但一旦超出范围就立刻失效——换个同义词或打个错字，规则就崩溃了。事后总结，教训是：人类语言的表面形式过于丰富，无法靠枚举穷尽。

¶1.2 统计革命（1990s）

转折点是一个朴素但锋利的发现：不用写规则，让机器从数据中估计概率即可。最经典的例子是 bigram 模型：

P(wt∣wt−1)=count(wt−1,wt)count(wt−1)P(w_t \mid w_{t-1}) = \frac{\text{count}(w_{t-1}, w_t)}{\text{count}(w_{t-1})} P(wt​∣wt−1​)=count(wt−1​)count(wt−1​,wt​)​

仅这一个公式就支撑起了 IBM 的统计机器翻译、第一代真正可用的语音识别，以及概率词性标注。隐马尔可夫模型（HMM）将这一思想推广到隐状态，概率上下文无关文法（PCFG）进一步将其扩展到句法分析。特征仍需人工设计，但规则已能通过学习获得。

¶1.3 深度学习时代（2013 — 2016）

2013 年，Mikolov 等人提出的 Word2Vec 展示了一个惊人现象：训练一个小型神经网络预测上下文词，得到的词向量竟然具备“做算术”的能力——

king⃗−man⃗+woman⃗≈queen⃗\vec{\text{king}} - \vec{\text{man}} + \vec{\text{woman}} \approx \vec{\text{queen}} king​−man+woman≈queen​

词从此不再是孤立的 ID，而是存在于一片连续空间中，相似度可通过余弦距离轻松计算。RNN 和 LSTM 随后登场，让模型能够沿序列传递上下文，终于学会了顺序，而不仅仅是基于一袋词的统计。

¶1.4 Transformer 革命（2017 — 至今）

2017 年，《Attention Is All You Need》用自注意力机制取代了循环结构：

Attention(Q,K,V)=softmax ⁣(QK⊤dk)V\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V Attention(Q,K,V)=softmax(dk​​QK⊤​)V

这一改变带来了两个工程上的重大突破。其一，模型在序列各位置上完全并行，训练规模随 GPU 内存而非序列长度扩展。其二，任意两个 token 都能直接互相注意，长程依赖问题得以一次性解决。BERT、GPT 以及今天的所有大模型，都是它的直系后代。

时代 时间 核心思想 被什么瓶颈打破
符号主义 1950 — 1980s 手写规则与文法 表面形式无法穷举
统计学习 1990s — 2010s 从语料估概率 人工特征工程触顶
深度学习 2013 — 2016 端到端学稠密表示 循环结构慢、不并行
Transformer 2017 — 现在 全序列自注意力 （仍在探索中）

洞察：每次范式更替都解决了上一代的瓶颈，但并未抛弃前人的成果。如今 LLM 使用的分词器仍是统计学习的产物；你的检索系统底层很可能依然挂着一个 TF-IDF 作为兜底方案。

¶3. 预处理流水线总览

进入任何模型之前，原始文本必须转化为数值特征。标准流水线包含六步，每一步都是在"用信息换取规整性"之间做出权衡。

最常见的错误是机械地执行所有步骤。正确思路是：每一步只去除下游无法处理的噪声，其余信息一律保留。后续每一步都会围绕这条原则展开。

¶3.1 环境准备

¶4. 第一步——文本清洗

从网页抓取的文本通常混杂 HTML 标签、URL 链接和各种控制字符。清洗的目标是去除这些明显的噪音，同时保留语义。

激进清洗的代价。上述函数直接删掉了数字和标点符号。在主题建模任务中，这没有问题，因为数字通常是噪音；但对以下任务来说，这种做法并不合适：

  • 情感分析——!!! 和 ?! 是情感的重要信号。
  • 命名实体识别——"Apple Inc." 必须保留句点和大写。
  • 金融 NLP——$29.99 本身就是关键内容。

因此，不要试图用一套通用清洗规则应对所有场景，规则设计应贴合具体任务。

¶5. 第二步——分词

分词将文本切分成模型最终处理的最小单元。切分粒度——字符、词、子词——直接影响词表大小、序列长度，以及模型面对未见词时的表现。

¶5.1 词级分词

NLTK 把 Dr. 视为一个 token，把标点单独切出，把缩写 Isn't 拆成 Is 和 n't。这些判断基于硬编码的英文惯例，因此词级分词在跨语言场景中表现脆弱。中文通常使用 jieba、THULAC、HanLP 等基于词典和模型的工具。

¶5.2 句子分词

NLTK 的 Punkt 模型从数据中学习：哪些句号表示句子结束，哪些只是缩写的一部分。

¶5.3 子词分词（BPE）

GPT、BERT、Llama、Claude 等现代模型不按“词”切分，而是采用子词分词，几乎都基于字节对编码（Byte-Pair Encoding，BPE）的某种变体。算法逻辑简单：

  1. 初始词表仅包含单个字符。
  2. 统计语料中所有相邻字符对的出现频率。
  3. 合并最高频的一对，生成新符号。
  4. 重复上述步骤，直到词表达到目标大小（通常为 3 万到 10 万）。

BPE 在工程中的重要性体现在以下几点：

  • 罕见词可拆解——unbelievable 被切为 un + believ + able，每部分都在其他地方见过。
  • 词表有上限——五万规模的子词表足以覆盖大多数英文文本和代码。
  • 支持多语言迁移——只要训练语料是多语言的，同一套分词器可以同时处理英文、法文、中文。

¶6. 第三步——标准化

标准化将同一个词的不同表面形式归并为一种，缩小词表规模，提升匹配精度。代价是丢失部分信息，因此需要主动权衡，避免一开始就全量应用。

¶6.1 小写化

小写化对搜索和主题建模有帮助，但会影响命名实体识别（如公司 Apple 和水果 apple 被混为一谈），也会干扰任何依赖大小写作为强调信号的任务。

¶6.2 词干提取 vs 词形还原

词干提取（stemming） 使用确定性规则去掉后缀。速度快、粗糙且有时出错：

easili 并非真实单词——Porter 词干提取只关注能否匹配，不关心结果是否美观。

词形还原（lemmatization） 借助词典和词性信息，返回真实的词典原形：

维度 词干提取 词形还原
速度 微秒级 毫秒级（含词性标注）
输出 可能不是真词 一定是词典原形
准确率 较低 较高
适用场景 搜索 / 信息检索 自然语言理解 / 问答

一个实用的默认选择：优先使用词形还原，除非运行高吞吐量的检索系统且延迟预算非常紧张。

¶7. 第四步——停用词与 Zipf 定律

停用词是一类高频但语义稀薄的封闭类词，例如英文的 the、is、at，中文的"的"、"了"、"是"。去掉这些词，词表能缩减三分之一，信号会更集中在内容词上。

为什么少数几个词占据主导？因为 Zipf 定律：在自然语言语料中，一个词的频率大致和它的排名成反比。

f(rank)∝1rankf(\text{rank}) \propto \frac{1}{\text{rank}} f(rank)∝rank1​

排名前十的词，通常能占到整个语料 25% — 30% 的 token。这就是分布的"头部"，绝大多数是停用词。"长尾"部分——成千上万只出现一两次的词——才是语义最浓的地方，也是模型最难处理的部分。正因如此，子词分词显得格外重要。

什么时候去除停用词：

  • 去——词袋模型、主题建模、搜索倒排索引。
  • 不去——情感分析（not good 和 good 差别巨大）、问答（虚词承载提问语气）、任何能自学习权重的深度模型。

¶8. 第五步——把 token 变成向量

模型只认数字。词袋（Bag-of-Words）和 TF-IDF 是两个经典编码方法，至今仍是大部分检索系统的根基，也是任何新任务都该先跑一遍的基线。

¶8.1 One-hot 表示 vs 分布式表示

讲 BoW 之前，先看为什么最朴素的编码不够用。One-hot 给每个词分配一个唯一的索引，对应位置打 1，其余全是 0。任何两个词的向量都是正交的——也就是说，编码本身根本不带相似度信息。

分布式表示——我们会在第二篇里亲手训出来——把含义压进一个稠密向量里，相关的词彼此靠近。BoW 和 TF-IDF 介于两者之间：每个词仍然占一个维度，但维度上填的不再是 0/1 而是频率。

¶8.3 TF-IDF

TF-IDF 给那种"在本文档里频繁、在整个语料里罕见"的词加权——这是一个朴素但好用的"对当前文档重要、对所有文档不通用"的启发式：

TF-IDF(t,d)=TF(t,d)⋅IDF(t)\text{TF-IDF}(t, d) = \text{TF}(t, d) \cdot \text{IDF}(t) TF-IDF(t,d)=TF(t,d)⋅IDF(t)

IDF(t)=log⁡ ⁣1+N1+df(t)+1\text{IDF}(t) = \log\!\frac{1 + N}{1 + \text{df}(t)} + 1 IDF(t)=log1+df(t)1+N​+1

其中 NNN 是文档总数，df(t)\text{df}(t)df(t) 是包含词 ttt 的文档数。+1 是平滑项，保证某个词在所有文档里都出现（或都不出现）时 IDF 仍有定义。

¶9. 第六步——n-gram 语言模型

切分 token 后，可以建模"它们如何相互跟随"。n-gram 模型将一句话拆解为一串条件概率的乘积：

P(w1,…,wT)=∏t=1TP(wt∣wt−n+1,…,wt−1)P(w_1, \ldots, w_T) = \prod_{t=1}^{T} P(w_t \mid w_{t-n+1}, \ldots, w_{t-1}) P(w1​,…,wT​)=t=1∏T​P(wt​∣wt−n+1​,…,wt−1​)

bigram 使用一个词作为上下文，trigram 使用两个，以此类推。

这里的权衡十分明显：

  • n 越大，上下文越丰富，困惑度（perplexity，可粗略理解为模型有效"分支数"，越低越好）就越低。
  • n 越大，参数量也急剧膨胀，而绝大多数上下文几乎不会出现。设 VVV 为词表大小，trigram 模型最多有 V3V^3V3 个参数，其中大部分在训练时从未见过。这就是稀疏性问题，也是统计 NLP 时代的核心难题。

平滑技术（如 Laplace 平滑、Kneser-Ney 平滑）通过重新分配概率质量给"未见过的 n-gram"来缓解这一问题。现代神经语言模型则直接绕开这个问题——通过嵌入实现上下文之间的参数共享，这正是通往第二篇的桥梁。

¶12. 速查表：什么任务做哪些预处理

任务 分词 标准化 停用词 特征
搜索 / 信息检索 词级 词干提取 去除 TF-IDF
情感分析 词级 / 子词 词形还原 保留 TF-IDF 或词嵌入

经验法则。模型容量越大，数据越多，预处理就越少。深度模型会自己学习标准化，过度预处理反而会破坏它本来能利用的信号；经典 ML 依赖精心的特征工程；LLM 更倾向于原始文本。

¶核心要点

  • 预处理与任务紧密相关：搜索需要强标准化，神经网络则要保留原始文本。
  • 子词分词（BPE、WordPiece、SentencePiece）已成为现代默认选择，因为它能固定词表并处理未见过的词。
  • TF-IDF 依然是必要的基线。如果 TF-IDF + 逻辑回归基线胜过你的复杂模型，那复杂模型一定存在问题。
  • Zipf 定律解释了停用词去除对经典模型有效的原因，也说明了长尾词难以学习的缘由。
  • 少即是多。过度预处理会损害学习表示的模型，始终要用实验来评估效果。
