机器学习数学推导（十九）：神经网络与反向传播

神经网络如何学习？本文推导前向传播的矩阵形式、反向传播的链式法则逐层推导、梯度消失/爆炸的数学分析、以及 Xavier 和 He 初始化的方差保持策略。

本文概览

单个感知机解决不了 XOR 问题。但如果用非线性激活函数把多个感知机堆叠起来，就能得到一个通用函数逼近器。接下来的问题是：这样的网络如何从数据中学习？答案就是反向传播——一种高效的链式法则应用方法。它通过一次反向遍历，复用中间结果，成为过去四十年所有深度学习库的核心引擎。搞清楚它的数学原理后，还能顺便解答两个问题：为什么深层网络会出现梯度消失或爆炸，以及为什么权重初始化的选择远比表面看起来重要。

你会学到：

  1. 感知机：模型、学习规则和收敛定理。
  2. 多层网络的前向传播矩阵形式。
  3. 反向传播：逐层推导链式法则，明确需要存储哪些量。
  4. 梯度消失与梯度爆炸的根源——从雅可比矩阵的乘积中看本质。
  5. Xavier 和 He 初始化：方差保持的推导过程，以及各自的适用场景。

预备知识： 微积分（链式法则、偏导数）、线性代数（矩阵乘法）、基础概率。

1. 感知机：一切的起点

1.1 模型

给定输入 $\mathbf{x}\in\mathbb{R}^d$、权重 $\mathbf{w}\in\mathbb{R}^d$ 和偏置 $b\in\mathbb{R}$，感知机的计算公式是：

$$z = \mathbf{w}^{T}\mathbf{x} + b, \qquad \hat{y} = \operatorname{sign}(z) = \begin{cases} +1 & z \geq 0,\\ -1 & z < 0.\end{cases}$$

从几何上看，方程 $\mathbf{w}^{T}\mathbf{x} + b = 0$ 定义了一个超平面，将输入空间分成两部分。$\hat y$ 的值表示点落在哪一侧。

1.2 学习算法

如果一个样本 $(\mathbf{x}_i, y_i)$ 被错分，即满足 $y_i(\mathbf{w}^{T}\mathbf{x}_i + b) \leq 0$，Rosenblatt 提出如下更新规则：

$$\mathbf{w} \leftarrow \mathbf{w} + \eta\, y_i\, \mathbf{x}_i, \qquad b \leftarrow b + \eta\, y_i.$$

这实际上是在感知机损失函数 $\sum_{i \in M} -y_i(\mathbf{w}^{T}\mathbf{x}_i + b)$ 上执行随机次梯度下降，其中 $M$ 是当前被错分的样本集合。

1.3 收敛定理

定理（Novikoff, 1962）。 如果数据线性可分，也就是说存在 $\mathbf{w}^{*}$ 和 $\gamma > 0$，使得对所有 $i$ 都有 $y_i\,\mathbf{w}^{*\,T}\mathbf{x}_i \geq \gamma$，那么感知机最多会在以下步数内收敛：

$$\frac{\|\mathbf{w}^{*}\|^{2}\, R^{2}}{\gamma^{2}}, \qquad R = \max_i \|\mathbf{x}_i\|.$$

证明的核心思路是：一方面，$\mathbf{w}_k^{T}\mathbf{w}^{*}$ 至少以线性速度增长；另一方面，$\|\mathbf{w}_k\|$ 的增长速度至多是 $\sqrt{k}$。结合这两点，就能得出更新步数 $k$ 的上限。

1.4 XOR 问题

四个点 $(0,0)\!\to\!0$、$(0,1)\!\to\!1$、$(1,0)\!\to\!1$、$(1,1)\!\to\!0$ 并不是线性可分的——没有任何一条直线能把对角线上的两组点分开。Minsky 和 Papert 于 1969 年指出这一点后，连接主义研究陷入了长达十余年的低谷。直到多层网络出现，这一问题才得以解决：通过引入两条超平面，并对其输出进行组合，即可实现对 XOR 的准确分类。

2. 多层网络与前向传播

2.1 网络结构

前馈神经网络由仿射变换和逐元素非线性交替组成：

  • 输入层： $\mathbf{h}^{(0)} = \mathbf{x}\in\mathbb{R}^{d_0}$。
  • 隐藏层： $\mathbf{h}^{(l)}\in\mathbb{R}^{d_l}$，$l = 1, \ldots, L-1$。
  • 输出层： $\mathbf{h}^{(L)} = \hat{\mathbf{y}}\in\mathbb{R}^{d_L}$。

2.2 前向传播（矩阵形式）

第 $l$ 层的计算分为两步：

$$\mathbf{z}^{(l)} = \mathbf{W}^{(l)}\,\mathbf{h}^{(l-1)} + \mathbf{b}^{(l)}, \tag{1}$$$$\mathbf{h}^{(l)} = \sigma\!\left(\mathbf{z}^{(l)}\right), \tag{2}$$

其中 $\mathbf{W}^{(l)}\in\mathbb{R}^{d_l\times d_{l-1}}$，$\mathbf{b}^{(l)}\in\mathbb{R}^{d_l}$。前向传播时，我会缓存每一层的 $\mathbf{h}^{(l-1)}$ 和 $\mathbf{z}^{(l)}$，因为反向传播需要用到它们。

2.3 激活函数全家福

函数公式导数备注
Sigmoid$\sigma(z) = \dfrac{1}{1+e^{-z}}$$\sigma(z)\bigl(1-\sigma(z)\bigr)$输出范围在 $(0,1)$；两端饱和导致梯度消失
Tanh$\tanh(z)$$1 - \tanh^{2}(z)$零中心化，比 Sigmoid 更好，但仍然会饱和
ReLU$\max(0, z)$$\mathbb{1}[z > 0]$正区间梯度恒为 1；可能产生死神经元
Leaky ReLU$\max(\alpha z, z)$，$\alpha\!\approx\!0.01$$1$ 或 $\alpha$解决了 ReLU 的死神经元问题
GELU$z\,\Phi(z)$平滑、近 $0$ 处呈钟形Transformer 的默认选择
Swish$z\,\sigma(z)$$\sigma(z) + z\,\sigma(z)(1-\sigma(z))$自门控、平滑

图的右侧清楚地展示了这一点：Sigmoid 的导数最大值只有 $0.25$，而 ReLU 在激活区间的导数恒等于 $1$。这一看似微小的导数特性差异，直接导致了梯度传播行为的根本不同。

2.4 通用逼近定理

定理（Cybenko 1989；Hornik 1991）。 对于紧集上的任意连续函数 $f$ 和任意 $\varepsilon > 0$，总存在一个单隐层网络

$$g(\mathbf{x}) = \sum_{j=1}^{M} v_j\, \sigma\!\left(\mathbf{w}_j^{T}\mathbf{x} + b_j\right)$$

满足 $\|f - g\|_\infty < \varepsilon$。上图正是这个定理的实际演示：一个仅含 64 个单元的 ReLU 单隐层网络，可以轻松拟合一段平滑的正弦波、一条绝对值折线，甚至一个不连续的阶跃函数。

注意。 这只是一个存在性结果：它没有说明 $M$ 需要多大，也没有保证梯度下降一定能找到合适的 $g$。实际中，深度远比宽度高效——一个深度为 $L$ 的网络能够表达需要宽度 $\Omega(2^{L})$ 的浅层网络才能表示的函数（Telgarsky, 2016）。

3. 反向传播：链式法则的大规模应用

3.1 损失函数

回归任务一般用均方误差：

$$\mathcal{L} = \tfrac{1}{2}\,\bigl\|\hat{\mathbf{y}} - \mathbf{y}\bigr\|^{2}.$$

分类任务则常用 softmax 加交叉熵：

$$\mathcal{L} = -\sum_{c} y_c \log \hat{y}_c, \qquad \hat{\mathbf{y}} = \operatorname{softmax}(\mathbf{z}^{(L)}).$$

3.2 核心思想

目标是计算每一层的 $\partial\mathcal{L}/\partial\mathbf{W}^{(l)}$ 和 $\partial\mathcal{L}/\partial\mathbf{b}^{(l)}$。如果每个参数都单独从输出回溯一遍，复杂度会达到 $\mathcal{O}(P^2)$（$P$ 是参数总数）。反向传播的核心在于一个关键观察：所有梯度共享同一条从输出到当前层的路径。每层只需计算一个误差信号 $\boldsymbol{\delta}^{(l)}$，然后通过简单的外积操作就能得到所有参数梯度。复杂度直接降到 $\mathcal{O}(P)$。

3.3 误差信号的递推公式

定义第 $l$ 层的误差信号为：

$$\boldsymbol{\delta}^{(l)} \;=\; \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(l)}}. \tag{3}$$

输出层（$l = L$，softmax + 交叉熵）。 推导一下 $\partial\hat{\mathbf{y}}/\partial \mathbf{z}^{(L)}$，再结合 $\sum_c y_c = 1$ 化简，结果非常简洁：

$$\boldsymbol{\delta}^{(L)} \;=\; \hat{\mathbf{y}} - \mathbf{y}. \tag{4}$$

预测值减去真实值——这就是 softmax 加交叉熵成为分类任务标配的主要原因之一。

隐藏层（反向递推）。 对于 $l < L$，将 $\mathbf{z}^{(l+1)} = \mathbf{W}^{(l+1)}\sigma(\mathbf{z}^{(l)}) + \mathbf{b}^{(l+1)}$ 代入链式法则，得到：

$$\boldsymbol{\delta}^{(l)} \;=\; \bigl(\mathbf{W}^{(l+1)\,T}\boldsymbol{\delta}^{(l+1)}\bigr) \,\odot\, \sigma'\!\left(\mathbf{z}^{(l)}\right). \tag{5}$$

简单来说：把上一层的误差用转置权重矩阵传递回来，再用本层激活函数的导数逐元素调整。

3.4 参数梯度

有了所有 $\boldsymbol{\delta}^{(l)}$ 后，参数梯度就很容易算出来：

$$\frac{\partial \mathcal{L}}{\partial \mathbf{W}^{(l)}} \;=\; \boldsymbol{\delta}^{(l)}\, \mathbf{h}^{(l-1)\,T}, \tag{6}$$$$\frac{\partial \mathcal{L}}{\partial \mathbf{b}^{(l)}} \;=\; \boldsymbol{\delta}^{(l)}. \tag{7}$$

权重梯度是输出端误差信号和输入端激活值的外积，而激活值在前向时已经缓存好了。

def backprop(X, y, weights, biases, activations):
    """一次前向加一次反向传播。"""
    L = len(weights)

    # 前向传播 —— 缓存激活值和预激活值
    h = [X]
    z = []
    for l in range(L):
        z_l = weights[l] @ h[-1] + biases[l]
        z.append(z_l)
        h.append(activations[l](z_l))

    # 反向传播 —— 假设输出层是 softmax + 交叉熵
    delta = h[-1] - y
    grad_W = [None] * L
    grad_b = [None] * L
    for l in range(L - 1, -1, -1):
        grad_W[l] = delta @ h[l].T
        grad_b[l] = delta.sum(axis=1, keepdims=True)
        if l > 0:
            delta = (weights[l].T @ delta) * activations[l].deriv(z[l - 1])
    return grad_W, grad_b

整个算法本质上只有三步：一次外积、一次求和、一次转置矩阵-向量乘法加激活导数门控。现代 autograd 框架替我们管理了中间变量，但底层跑的就是这套递推逻辑。

4. 梯度消失与梯度爆炸

4.1 梯度消失问题

从第 $L$ 层开始，按照公式 (5) 逐层展开到第 $1$ 层，可以发现第 $1$ 层的梯度是 $L-1$ 个雅可比类因子的连乘积：

$$\frac{\partial \mathcal{L}}{\partial \mathbf{W}^{(1)}} \;\propto\; \prod_{l=2}^{L} \Bigl[\mathbf{W}^{(l)\,T}\, \operatorname{diag}\bigl(\sigma'(\mathbf{z}^{(l-1)})\bigr)\Bigr].$$

对于 sigmoid 激活函数，$\sigma'(z)\leq 0.25$。如果每个权重矩阵的谱范数接近 $1$，那么第 $1$ 层梯度的大小大约是 $0.25^{L-1}$。当网络深度 $L=20$ 时，这个值约为 $3.6\times 10^{-12}$——梯度更新在数值上几乎为零，浅层参数完全无法学习。

4.2 梯度爆炸问题

反过来，如果权重矩阵的谱范数大于 $1$，且激活函数导数没有强收缩效应，同样的连乘积会指数增长。几百次迭代后就会导致数值溢出。RNN 特别容易出现这个问题，因为按时间展开时，同一个权重矩阵会被重复乘 $T$ 次。

4.3 曲线解读

这张图用实验验证了上述现象：将一个单位梯度信号反向传播到随机初始化的网络中，观察不同深度下的梯度变化。Sigmoid（使用 Xavier 初始化）每增加一层，梯度就衰减一个数量级；ReLU + He 初始化基本保持在健康范围内；而 ReLU 如果不做缩放，梯度则会以类似的速度爆炸。注意，y 轴是对数坐标——每条网格线代表十倍的变化。

4.4 解决方案

问题方案原理
梯度消失使用 ReLU 激活函数当 $z>0$ 时，$\sigma'(z) = 1$，避免指数衰减
梯度消失添加残差连接$\mathbf{h}^{(l)} = \mathbf{h}^{(l-1)} + F(\mathbf{h}^{(l-1)})$，梯度通过恒等路径绕过非线性部分
梯度消失批归一化逐层稳定激活值的分布
梯度爆炸梯度裁剪当 $\|\mathbf{g}\| > c$ 时，按 $\mathbf{g}\!\leftarrow\!(c/\|\mathbf{g}\|)\,\mathbf{g}$ 缩放梯度
两者兼顾合理初始化让激活值和梯度的方差在各层之间保持稳定（详见下一节）

5. 权重初始化策略

5.1 为什么初始化很重要

  • 全零初始化。 同一层的每个神经元计算的是同一个函数，收到的梯度也完全相同，网络无法打破对称性。
  • 太大。 预激活值落在激活函数的饱和区域，梯度直接消失。
  • 太小。 激活值逐渐趋近于零，信号在前向传播中彻底丢失。

我的目标很明确：让激活值和梯度的方差在每一层都保持稳定。具体怎么定尺度？看这一层的 fan-in 和激活函数。

5.2 Xavier（Glorot）初始化

假设第 $l$ 层的一个神经元，fan-in 是 $n_{\text{in}}$，fan-out 是 $n_{\text{out}}$：

$$z_j = \sum_{i=1}^{n_{\text{in}}} w_{ji}\, h_i.$$

如果 $w_{ji}$ 和 $h_i$ 独立且均值为零，那么

$$\operatorname{Var}(z_j) = n_{\text{in}}\cdot \operatorname{Var}(w)\cdot \operatorname{Var}(h).$$

前向传播时要求 $\operatorname{Var}(z) = \operatorname{Var}(h)$，可以推出 $\operatorname{Var}(w) = 1/n_{\text{in}}$；反向传播时同理，得到 $\operatorname{Var}(w) = 1/n_{\text{out}}$。Glorot 和 Bengio（2010）取了个折中方案：

$$\operatorname{Var}(w) = \frac{2}{n_{\text{in}} + n_{\text{out}}}, \qquad w \sim \mathcal{U}\!\left(-\sqrt{\tfrac{6}{n_{\text{in}}+n_{\text{out}}}},\; \sqrt{\tfrac{6}{n_{\text{in}}+n_{\text{out}}}}\right). \tag{8}$$

最适合 Sigmoid 和 Tanh 这类对称激活函数。

5.3 He 初始化

ReLU 的特点在于：期望上有一半的单元会被置零，这直接让 $h$ 的方差贡献减半。He 等人（2015）因此把方差放大两倍：

$$\operatorname{Var}(w) = \frac{2}{n_{\text{in}}}, \qquad w \sim \mathcal{N}\!\left(0,\, \tfrac{2}{n_{\text{in}}}\right). \tag{9}$$

最适合 ReLU 及其变体。

5.4 总结表

激活函数初始化方法$\operatorname{Var}(w)$
Sigmoid / TanhXavier$\dfrac{2}{n_{\text{in}} + n_{\text{out}}}$
ReLUHe$\dfrac{2}{n_{\text{in}}}$
Leaky ReLU（$\alpha$）He（修正版）$\dfrac{2}{(1 + \alpha^{2})\, n_{\text{in}}}$

上面提到的梯度图中间那条绿色曲线（ReLU + He），就是这套理论的最佳实验验证：不管网络多深，它始终能保持接近 $1$ 的梯度规模。

6. 损失曲面的特性

即使是再小的网络，损失曲面也是高度非凸的：两个深谷、一个鞍点、一条窄脊。图中展示了 SGD 从糟糕的初始点出发的轨迹。有两个关键点需要记住：

  1. 局部最优通常已经够用。 网络越宽，经验损失的极小值往往集中在深度相近的平坦区域（Choromanska et al., 2015）。“必须找到全局最优"的想法是从凸优化时代遗留下来的，在深度学习中并不适用。
  2. 高维空间中鞍点远多于局部最优。 在 $\mathbb{R}^d$ 中，随机临界点的 Hessian 矩阵所有 $d$ 个特征值同号的概率极低。SGD 自带的噪声正好帮助我们逃离这些鞍点（Dauphin et al., 2014）。

这两点解释了为什么简单的一阶方法在深度学习中依然占据主导地位，即使没有凸性保证也无妨。

import torch, torch.nn as nn

x = torch.randn(32, 784, requires_grad=False)
y = torch.randint(0, 10, (32,))

net = nn.Sequential(
    nn.Linear(784, 128), nn.ReLU(),
    nn.Linear(128, 10),
)
loss_fn = nn.CrossEntropyLoss()

logits = net(x)            # 前向传播：构建计算图
loss = loss_fn(logits, y)
loss.backward()            # 反向传播：应用链式法则，将梯度填充到每个 Parameter 的 .grad

for p in net.parameters():
    print(p.shape, p.grad.norm().item())

调用 loss.backward() 时，实际上完成了三件事。第一，前向传播过程中，所有 requires_grad=True 的张量都会记录生成它的操作——这就是所谓的 tape。第二，反向传播时，backward 按照拓扑逆序遍历 tape，每个节点都知道如何将传入的余切量与局部 Jacobian 相乘（如果存在更高效的 vector-Jacobian product，则不会显式构造 Jacobian）。第三，梯度是累加到 .grad 上的，而不是覆盖原有值——这就是为什么训练循环开始时总要调用 optimizer.zero_grad()。

使用 PyTorch 时有两个常见坑需要注意。第一，loss.backward() 默认会释放计算图，如果尝试调用两次，会抛出 RuntimeError: Trying to backward through the graph a second time。如果确实需要保留计算图，可以传入 retain_graph=True。第二，在参与 autograd 的张量上进行 in-place 操作时，如果反向传播需要用到原值，梯度会被悄悄破坏。错误信息 one of the variables needed for gradient computation has been modified by an inplace operation 是除 shape 不匹配之外最常见的训练问题。

8. 常见误解

学生在学习反向传播时，通常会陷入五个误区。

“反向传播就是梯度下降。” 错了。反向传播的作用是计算 $\nabla_{\boldsymbol\theta} L$，而 SGD、Adam 和 L-BFGS 这些优化器才是使用梯度的工具。你可以算完梯度后什么都不做，也可以用有限差分代替反向传播来计算梯度，再用梯度下降优化。如果把这两者混为一谈，换优化器时就会一头雾水。

“梯度消失是因为网络太深。” 半对半错。深度确实放大了每层的问题：如果每层局部 Jacobian 的谱范数 $< 1$，那么 $L$ 层相乘后梯度会按 $\rho^L$ 衰减。ResNet 并没有改变深度，而是通过加入恒等跳跃连接，让局部 Jacobian 的特征值强制接近 1。问题的关键不是深度，而是 Jacobian 的谱特性。

“ReLU 完全解决了饱和问题。” 只在正半轴成立。如果一个 ReLU 单元的输入始终为负，它的梯度永远是 0——这就是所谓的 dead ReLU。在实际训练中，前几个 epoch 就有 5%-15% 的 ReLU 单元“死亡”，再也无法恢复。Leaky ReLU 和 GELU 的出现就是为了缓解这个问题。

“参数越多越容易过拟合。” 在现代深度学习中，这个观点并不成立。参数量比样本量多 10 倍的网络仍然能很好地泛化，这就是所谓的 double descent 现象：测试误差在插值阈值附近达到峰值，随后随着模型宽度增加反而下降。经典的偏差-方差权衡（第 20 篇）对超参数化模型来说已经不够用了。

“初始化只影响训练速度。” 错误。如果初始化太差，深层 ReLU 网络的预激活要么完全归零，要么彻底饱和，无论怎么训练都救不回来。He 初始化（方差 $2/n_{\text{in}}$）并不是为了加速优化，而是为了让网络能够被训练的基本前提。

9. 练习题

练习 1（链式法则）。 假设 $y = \sigma(wx + b)$，其中 $\sigma(z) = 1/(1+e^{-z})$，求 $\partial y/\partial w$。

解答。 直接计算得 $\partial y/\partial w = \sigma'(z)\,x = \sigma(z)\bigl(1 - \sigma(z)\bigr)\,x$。

练习 2（梯度消失）。 为什么 sigmoid 容易导致梯度消失，而 ReLU 不会？

解答。 sigmoid 的导数 $\sigma'(z)$ 最大值是 0.25，经过 $L$ 层后梯度最多衰减到 $0.25^{L}$，呈指数级下降。ReLU 在激活区域的导数恒为 1，因此不会出现这种问题。

练习 3（批归一化）。 BatchNorm 是如何帮助训练的？

解答。 它把每层的预激活值重新调整为接近零均值和单位方差，从而稳定了梯度的大小，允许使用更大的学习率。此外，mini-batch 带来的噪声还能起到轻微正则化的作用。

练习 4（Dropout 在测试时）。 训练时使用 $p = 0.5$ 的 dropout，测试时需要做什么调整？

解答。 测试时保留所有神经元，并将权重乘以 $(1 - p) = 0.5$，以保证输出期望一致。现代框架通常用反向 dropout：训练时直接将激活值除以 $(1-p)$，测试时无需额外操作。

练习 5（Xavier）。 为什么 Xavier 初始化选择 $\operatorname{Var}(w) = 2/(n_{\text{in}} + n_{\text{out}})$？

解答。 前向传播要求 $n_{\text{in}}\operatorname{Var}(w) = 1$，反向传播要求 $n_{\text{out}}\operatorname{Var}(w) = 1$。取 $\operatorname{Var}(w) = 2/(n_{\text{in}} + n_{\text{out}})$ 是一种折中方案，能同时近似满足这两个条件。
