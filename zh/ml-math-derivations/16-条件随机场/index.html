机器学习数学推导（十六）：条件随机场

CRF 为什么在序列标注任务上压 HMM 一头？本文从零推导线性链 CRF——势函数与特征函数、前向后向算法、对数似然梯度（经验期望减模型期望）、Viterbi 解码，以及现代 BiLSTM-CRF 的整合方式。

这一篇要讲什么

命名实体识别、词性标注、信息抽取——这些任务都要求给序列中的每个元素打上标签。HMM（第十五篇 ）用生成式方法解决问题，建模联合分布 $P(\mathbf{X},\mathbf{Y})$。但为了简化计算，它假设每个观测值只依赖于自己的隐状态。这显然不符合实际情况。比如，bank 是名词还是动词，取决于前一个词、后一个词、词缀、大小写、词典匹配等多重特征。而 HMM 根本无法利用这些复杂的重叠特征。

条件随机场（CRF） 完全放弃了生成式的思路，直接建模条件概率 $P(\mathbf{Y}\mid\mathbf{X})$。既然不再需要为 $\mathbf{X}$ 编造生成故事，就可以随意加入任意多的、相互重叠的特征。

这篇你会学到：

  1. 为什么 CRF 的判别式建模在标注任务中胜过 HMM 的生成式建模
  2. 转移特征和状态特征如何定义 CRF 的评分机制
  3. 前向后向算法的新用途——计算 $Z(\mathbf{X})$ 和边缘概率
  4. 对数似然的梯度为何等于 “经验特征计数 − 模型期望特征计数”
  5. Viterbi 解码：如何找到得分最高的标签序列

前置知识： 概率论基础（条件概率、贝叶斯公式）、HMM（第十五篇 ）、矩阵表示

1. 从 HMM 到 CRF：从生成式到判别式

1.1 HMM 强加给你的两个假设

HMM 建模观测和标签的联合概率：

$$P(\mathbf{X}, \mathbf{Y}) = P(y_1)\prod_{t=2}^{T} P(y_t\mid y_{t-1})\prod_{t=1}^{T} P(x_t\mid y_t)$$

预测时再用贝叶斯反推 $\mathbf{Y}^* = \arg\max_{\mathbf{Y}} P(\mathbf{Y}\mid\mathbf{X})$。这套联合概率分解能成立，靠的是两个简化假设：

  • 观测独立性：$P(x_t\mid y_{1:T}, x_{\setminus t}) = P(x_t\mid y_t)$，每个观测只看自己的标签。
  • 一阶马尔可夫：$P(y_t\mid y_{1:t-1}) = P(y_t\mid y_{t-1})$，每个标签只看上一个标签。

马尔可夫假设大体可以接受，真正受限的是观测独立性假设——它直接排除了“看后一个词”“看后缀”“查词典”等自然且高效的特征。

1.2 CRF 的核心想法：直接建模 $P(\mathbf{Y}\mid\mathbf{X})$

$$P(\mathbf{Y}\mid\mathbf{X}) \;=\; \frac{1}{Z(\mathbf{X})}\exp\!\left(\sum_{t=1}^{T}\Psi_t(y_{t-1}, y_t, \mathbf{X})\right)$$

上图把两者的结构差异画了出来。HMM（上半）里每个 $x_t$ 是 $y_t$ 的子节点，模型必须解释观测；为了让联合可分解，每个 $x_t$ 还只能依赖自己的 $y_t$。CRF（下半）则把整条 $\mathbf{X}$ 画成一条共享的"上下文带"，每个团 $(y_{t-1}, y_t, \mathbf{X})$ 都允许观察整条 $\mathbf{X}$ 的任意函数。我们从来不显式建模 $P(\mathbf{X})$，所以也就不需要对观测做任何独立性假设。

得到的好处：

  • 任意重叠特征：词形、前缀、后缀、大小写、相邻词、词典命中可以同时上，不需要为重复计数受罚。
  • 特征工程更自由：全局特征也能直接接入。
  • 实证效果更强：在标准序列标注任务上，CRF 通常比 HMM 高 5–10 个 F1 点。

付出的代价： 配分函数 $Z(\mathbf{X})$ 要对所有 $L^T$ 条标签序列求和，每轮训练都要重算它和它的梯度。所幸前向后向算法可以把这件事压到 $O(TL^2)$。

1.3 从 HMM 经 MEMM 到 CRF

$$P(\mathbf{Y}\mid\mathbf{X}) = \prod_{t=1}^{T} P(y_t\mid y_{t-1}, \mathbf{X})$$

MEMM 已经是判别式的，也允许任意 $\mathbf{X}$ 特征，但它在每个位置局部归一化。这带来了著名的**标注偏置（label bias）**问题：转移选项较少的状态，由于局部 softmax 的候选状态更少，容易忽略观测信息而过度分配概率。

CRF 用全局归一化根治了这个问题：分母里那个唯一的 $Z(\mathbf{X})$ 强制所有路径在同一个分母下竞争。一句话总结：

  • HMM——生成式，局部归一化，观测独立。
  • MEMM——判别式，局部归一化，有标注偏置。
  • CRF——判别式，全局归一化，无标注偏置。

1.4 生成式 vs 判别式的更广视角

跳出序列模型，这套权衡是普适的。生成式模型把参数花在 $P(\mathbf{X})$ 上——左图那张平滑的紫色密度——若仅关注决策边界，这部分参数预算实为冗余。判别式模型直接把所有参数压在条件决策边界上——右图那条锐利的对角等高线。朴素贝叶斯 vs 逻辑回归、HMM vs CRF，是同一组权衡的不同尺度版本。

2. 线性链 CRF 的数学框架

2.1 基本定义

输入是观测序列 $\mathbf{X} = (x_1, x_2, \dots, x_T)$。
输出是标签序列 $\mathbf{Y} = (y_1, y_2, \dots, y_T)$，其中每个 $y_t \in \mathcal{Y} = \{1, 2, \dots, L\}$。

$$P(\mathbf{Y}\mid\mathbf{X}) = \frac{1}{Z(\mathbf{X})}\prod_{t=1}^{T}\Psi_t(y_{t-1}, y_t, \mathbf{X}) \tag{1}$$

这里：

  • $\Psi_t(y_{t-1}, y_t, \mathbf{X}) > 0$ 是位置 $t$ 的势函数。
  • $Z(\mathbf{X}) = \sum_{\mathbf{Y}'} \prod_t \Psi_t(y'_{t-1}, y'_t, \mathbf{X})$ 是归一化所有标签序列的配分函数。

为了方便，我约定 $y_0$ 是一个特殊的 START 符号，这样第一个转移就有明确的起点。

2.2 特征函数分解

CRF 用两类特征函数来参数化势函数：

$$t_k(y_{t-1}, y_t, \mathbf{X}, t),\qquad k = 1, \dots, K_1$$


例子：$t_1 = \mathbb{1}[y_{t-1}=\text{B-PER},\, y_t=\text{I-PER}]$，表示“人名连续”。

$$s_l(y_t, \mathbf{X}, t),\qquad l = 1, \dots, K_2$$


例子：$s_1 = \mathbb{1}[y_t=\text{B-LOC},\, x_t \text{首字母大写}]$，表示“地名开头且单词首字母大写”。

上图以句子 “Barack Obama visited New York” 在位置 $t=4$（“New” → B-LOC）为例展开。状态特征面板列出了当 B-LOC 在该位置激活时，$\mathbf{X}$ 的所有相关信号（如首字母大写、下一个词是 York、后缀为 ew）。转移特征面板则记录了与该决策一致的所有 (前标签, 当前标签) 配置。这些特征可以自由重叠，这正是 CRF 相比 HMM 的强大之处。

$$\Psi_t(y_{t-1}, y_t, \mathbf{X}) = \exp\!\left(\sum_k\lambda_k\, t_k(y_{t-1}, y_t, \mathbf{X}, t) + \sum_l\mu_l\, s_l(y_t, \mathbf{X}, t)\right)$$

2.3 统一参数化

$$\mathbf{f}(y_{t-1}, y_t, \mathbf{X}, t) = (t_1, \dots, t_{K_1}, s_1, \dots, s_{K_2})^\top,\quad \mathbf{w} = (\lambda_1, \dots, \lambda_{K_1}, \mu_1, \dots, \mu_{K_2})^\top$$$$\Psi_t(y_{t-1}, y_t, \mathbf{X}) = \exp\!\big(\mathbf{w}^\top\mathbf{f}(y_{t-1}, y_t, \mathbf{X}, t)\big) \tag{2}$$$$\mathbf{F}(\mathbf{Y}, \mathbf{X}) = \sum_{t=1}^{T}\mathbf{f}(y_{t-1}, y_t, \mathbf{X}, t)$$$$P(\mathbf{Y}\mid\mathbf{X}) = \frac{\exp\!\big(\mathbf{w}^\top\mathbf{F}(\mathbf{Y}, \mathbf{X})\big)}{Z(\mathbf{X})},\quad Z(\mathbf{X}) = \sum_{\mathbf{Y}'}\exp\!\big(\mathbf{w}^\top\mathbf{F}(\mathbf{Y}', \mathbf{X})\big) \tag{3}$$

2.4 矩阵形式

$$[\mathbf{M}_t(\mathbf{X})]_{i,j} = \exp\!\big(\mathbf{w}^\top\mathbf{f}(y_{t-1}=i, y_t=j, \mathbf{X}, t)\big)$$$$Z(\mathbf{X}) = \mathbf{1}^\top\!\left(\prod_{t=1}^{T}\mathbf{M}_t(\mathbf{X})\right)\!\mathbf{1}$$

计算复杂度是 $T$ 次 $L\times L$ 矩阵乘法，时间复杂度为 $O(TL^2)$，和前向后向算法同阶。

3. 前向后向算法：CRF 版本

3.1 前向递推

定义前向变量 $\alpha_t(j)$，表示所有在位置 $t$ 以标签 $j$ 结尾的部分路径的未归一化得分总和。

$$\alpha_1(j) = \Psi_1(y_0 = \text{START},\, y_1 = j,\, \mathbf{X})$$$$\alpha_t(j) = \sum_{i=1}^{L} \alpha_{t-1}(i) \cdot \Psi_t(y_{t-1}=i,\, y_t=j,\, \mathbf{X}) \tag{4}$$$$Z(\mathbf{X}) = \sum_{j=1}^{L} \alpha_T(j)$$

简单来说，$\alpha_t(j)$ 收集了所有长度为 $t$、终点是标签 $j$ 的路径得分。递推公式的意思是：要到达 $t$ 时刻的 $j$，必须从 $t-1$ 时刻的某个标签 $i$ 转移过来。

复杂度：$O(TL^2)$。相比暴力解法的 $O(L^T)$，这个算法让问题从不可行变成了可行。

3.2 后向递推

类似地，定义后向变量 $\beta_t(i)$，表示从位置 $t$ 标签 $i$ 开始到序列末尾的所有部分路径的未归一化得分总和。递推从 $t = T$ 反向进行到 $t = 1$，复杂度同样是 $O(TL^2)$。

3.3 用 $\alpha$ 和 $\beta$ 计算边缘概率

完成前向和后向两轮计算后，边缘概率可以直接得到：

$$P(y_t = j \mid \mathbf{X}) = \frac{\alpha_t(j) \cdot \beta_t(j)}{Z(\mathbf{X})}$$$$P(y_{t-1} = i, y_t = j \mid \mathbf{X}) = \frac{\alpha_{t-1}(i) \cdot \Psi_t(i, j, \mathbf{X}) \cdot \beta_t(j)}{Z(\mathbf{X})} \tag{5}$$

trellis 图清楚地展示了这一点：每个格点 $(t, j)$ 接收来自左侧的前向蓝色箭头和来自右侧的后向紫色箭头。两者相乘再除以 $Z(\mathbf{X})$，就得到了该位置的边缘概率。下一节梯度公式中会用到这些值。

3.4 数值稳定性：log 空间

$$\log\!\sum_i e^{x_i} = \max_i x_i + \log\!\sum_i e^{x_i - \max_i x_i}$$

这和 softmax 中常用的数值稳定技巧完全一致。

4. 参数学习：极大似然

4.1 目标函数

$$\ell(\mathbf{w}) = \sum_{n=1}^{N} \log P(\mathbf{Y}^{(n)} \mid \mathbf{X}^{(n)}; \mathbf{w}) = \sum_{n=1}^{N}\!\left[\mathbf{w}^\top \mathbf{F}(\mathbf{Y}^{(n)}, \mathbf{X}^{(n)}) - \log Z(\mathbf{X}^{(n)})\right] \tag{6}$$

第一项是关于 $\mathbf{w}$ 的线性项，计算起来很简单；难点全在 $\log Z$ 上。

$$\ell_{\text{reg}}(\mathbf{w}) = \ell(\mathbf{w}) - \tfrac{\lambda}{2} \|\mathbf{w}\|^2$$

4.2 梯度：经验减期望

$$\nabla_{\mathbf{w}} \ell = \sum_{n=1}^{N}\!\left[\underbrace{\mathbf{F}(\mathbf{Y}^{(n)}, \mathbf{X}^{(n)})}_{\text{经验特征计数}} - \underbrace{\mathbb{E}_{P(\mathbf{Y}'\mid \mathbf{X}^{(n)})}\!\big[\mathbf{F}(\mathbf{Y}', \mathbf{X}^{(n)})\big]}_{\text{模型期望特征计数}}\right] \tag{7}$$

这就是所有最大熵模型梯度的共同形态：特征在数据里实际出现的次数，减去当前模型认为它该出现的次数。训练过程就是把模型的期望往经验值上推；收敛时两者完全一致（最大熵条件）。

4.3 把期望降到 $O(TL^2)$

$$\mathbb{E}\big[\mathbf{F}(\mathbf{Y}, \mathbf{X})\big] = \sum_{t=1}^{T} \sum_{i,j} P(y_{t-1}=i, y_t=j \mid \mathbf{X}) \cdot \mathbf{f}(i, j, \mathbf{X}, t) \tag{8}$$

右侧的 pair-marginal 就是之前 (5) 中算过的那些，复杂度为 $O(TL^2)$。所以，一次前向后向算法就能同时得到 $\log Z$ 和所有梯度分量。

4.4 优化：L-BFGS

目标函数是凹的（加了 L2 后严格凹），任何一阶方法都能收敛到全局最优。实际工程中，L-BFGS 是 CRF 训练的标准选择：

  • 它是拟牛顿法，近似逆 Hessian 矩阵，收敛速度接近二阶。
  • “limited-memory” 版本只保留最近 $m$ 次梯度差，即使特征空间有上百万维也能高效处理。
  • 通常 50–200 次迭代就能收敛，每次迭代对每条训练序列跑一次前向后向算法。

参考实现：scipy.optimize.fmin_l_bfgs_b。

5. Viterbi 解码

5.1 解码问题

$$\mathbf{Y}^* = \arg\max_{\mathbf{Y}} P(\mathbf{Y} \mid \mathbf{X}) = \arg\max_{\mathbf{Y}} \mathbf{w}^\top \mathbf{F}(\mathbf{Y}, \mathbf{X})$$

分母 $Z(\mathbf{X})$ 和 $\mathbf{Y}$ 无关，解码时完全不用计算它。

5.2 动态规划

$$\delta_t(j) = \max_{i \in \{1,\dots,L\}}\!\left[\delta_{t-1}(i) + \mathbf{w}^\top \mathbf{f}(i, j, \mathbf{X}, t)\right] \tag{9}$$$$\psi_t(j) = \arg\max_{i}\!\left[\delta_{t-1}(i) + \mathbf{w}^\top \mathbf{f}(i, j, \mathbf{X}, t)\right]$$

到达序列末尾 $T$ 时，取 $y_T^* = \arg\max_j \delta_T(j)$，然后通过 $y_{t-1}^* = \psi_t(y_t^*)$ 回溯，就能得到完整的最优标签序列。

上面这张 trellis 图很直观：浅灰色箭头表示所有可能的转移；橙色箭头是每一步的最大值路径；高亮节点连起来就是回溯路径。比如句子 “He left New York yesterday .” 被解码成 O O B-LOC I-LOC O O。复杂度仍然是 $O(TL^2)$。

6. 深度学习时代的 CRF：BiLSTM-CRF

$$\text{输入} \xrightarrow{\text{Embedding}} \text{BiLSTM} \xrightarrow{\text{发射得分}} \text{CRF 层} \xrightarrow{\text{Viterbi}} \text{标签}$$
  • BiLSTM（或者 Transformer encoder）负责学习每个 token 的上下文表示，用可学习的特征替代了传统手工设计的状态特征。
  • CRF 层维护一个可学习的标签转移矩阵 $A_{ij}$。这个矩阵很重要，因为像“I-PER 必须紧跟 B-PER，不能接 O”这样的约束是序列级别的，单个 token 的 softmax 根本无法表达，而 CRF 的转移矩阵正好能学到这些约束。
  • 训练通过负对数似然目标函数端到端优化整个模型。
  • 推理依然使用 Viterbi 算法。

import torch
import torch.nn as nn

class BiLSTM_CRF(nn.Module):
    def __init__(self, vocab_size, tag_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_dim // 2,
                            bidirectional=True, batch_first=True)
        self.hidden2tag = nn.Linear(hidden_dim, tag_size)
        # CRF 转移矩阵：transitions[i, j] 表示从 j 转移到 i 的得分
        self.transitions = nn.Parameter(torch.randn(tag_size, tag_size))

    def forward_score(self, emissions):
        """通过前向算法在 log 空间中计算 log Z(X)。"""
        T, L = emissions.shape
        alpha = emissions[0]  # (L,)
        for t in range(1, T):
            scores = (alpha.unsqueeze(0)
                      + self.transitions
                      + emissions[t].unsqueeze(1))
            alpha = torch.logsumexp(scores, dim=1)
        return torch.logsumexp(alpha, dim=0)

    def gold_score(self, emissions, tags):
        """计算真实标签序列的得分。"""
        score = emissions[0, tags[0]]
        for t in range(1, len(tags)):
            score += (self.transitions[tags[t], tags[t - 1]]
                      + emissions[t, tags[t]])
        return score

    def neg_log_likelihood(self, sentence, tags):
        emissions = self.hidden2tag(
            self.lstm(self.embedding(sentence))[0].squeeze(0)
        )
        return self.forward_score(emissions) - self.gold_score(emissions, tags)

损失函数就是 $\log Z(\mathbf{X}) - \text{score}(\mathbf{Y}_{\text{gold}}, \mathbf{X})$，也就是公式 (6) 在单条样本上的负值。反向传播会自动计算“经验减期望”的梯度——链式法则在前向算法中自然推导出公式 (7)。

端到端 NER 与置信度

上图展示了训练好的 CRF 在推理时的实际输出，例句是“Apple CEO Tim Cook visited Beijing last week”：

  • 上半部分：Viterbi 解码得到的 BIO 标签序列。三组连续的 B/I 被识别为实体跨度（ORG、PER、LOC），虚线框标出的就是要传递给下游消费者的实体范围。
  • 下半部分：每个 token 的边缘概率 $P(y_t\mid\mathbf{X})$，由前向后向算法计算得出，可以作为校准良好的置信度分数。注意，这和 Viterbi 路径概率不是一回事——边缘概率是逐位置独立计算的，常用于主动学习、拒答策略或下游贝叶斯聚合。

7. 练习题

练习 1：CRF 和 HMM 的特征能力对比。 在词性标注任务中，解释为什么 CRF 可以使用“下一个词是动词”作为特征，而 HMM 不行。

解答： HMM 的观测独立性假设限制了 $P(x_t \mid y_t)$ 只能查看当前的 $x_t$。CRF 没有这种限制，它的特征函数 $\mathbf{f}(y_{t-1}, y_t, \mathbf{X}, t)$ 可以利用整个观测序列 $\mathbf{X}$。因此，“$x_{t+1}$ 是动词”这样的特征在 CRF 中完全合法。

练习 2：设计特征模板。 为命名实体识别设计三个特征函数。

解答：
（1）转移特征：$\mathbb{1}[y_{t-1} = \text{B-ORG},\, y_t = \text{I-ORG}]$（机构名延续）。
（2）状态特征：$\mathbb{1}[y_t = \text{B-LOC},\, x_t \text{首字母大写}]$（地名开头字母大写）。
（3）上下文特征：$\mathbb{1}[y_t = \text{B-PER},\, x_{t-1} = \text{Mr.}]$（前一个词是称谓）。

练习 3：前向算法复杂度。 假设 CRF 的 $T = 100$、$L = 50$，计算前向算法需要的操作次数，并与暴力法比较。

解答： 每个时间步计算 $L$ 个前向值，每个值对 $L$ 个前驱求和，复杂度为 $O(TL^2) = O(100 \cdot 50^2) = O(2.5 \times 10^5)$。暴力法枚举所有路径，复杂度为 $O(L^T) = O(50^{100}) \approx 10^{170}$。动态规划比暴力法快了大约 165 个数量级。

练习 4：梯度的物理意义。 解释 $\nabla_\mathbf{w}\ell = \mathbf{F}_{\text{empirical}} - \mathbb{E}_{\text{model}}[\mathbf{F}]$ 的含义，并说明收敛时的条件。

解答： 如果某个特征在训练数据中出现的次数超过模型预期，权重会被调高；如果模型高估了某个特征，权重会被调低。收敛时（梯度为零），经验特征计数和模型期望特征计数完全一致。这就是最大熵条件：在所有满足经验特征均值的分布中，CRF 选择熵最大的那个。

练习 5：CRF Viterbi 和 HMM Viterbi 的区别。 两者都用 Viterbi 算法，唯一的本质差别是什么？

解答： trellis 和 $O(TL^2)$ 的动态规划递推公式完全相同。差别在于每条边的得分：HMM 使用局部概率 $\log P(x_t \mid y_t) + \log P(y_t \mid y_{t-1})$；CRF 使用内积 $\mathbf{w}^\top \mathbf{f}(y_{t-1}, y_t, \mathbf{X}, t)$，其中特征向量可以依赖整个观测序列 $\mathbf{X}$ 的任意函数。

练习 6：MEMM 为什么有标注偏置而 CRF 没有。 用一个二状态小例子说明局部归一化为什么会忽略观测，并解释全局归一化为什么不会。

解答： 假设状态 $A$ 只有一条出边（到 $B$），而状态 $C$ 有 50 条出边。局部 softmax 归一化后，$A \to B$ 的概率永远是 1，因为模型在 $A$ 处没有不确定性，完全忽略了观测。CRF 的分母 $Z(\mathbf{X})$ 是对整条路径求和，因此经过 $A$ 的高分路径必须胜过所有经过 $C$ 的路径——观测证据正是通过这一比较被引入的。
