引子。 朴素贝叶斯假设特征在给定类别时条件独立。这个假设虽然方便，但并不完全真实——它让我能用一次数据遍历就完成训练，但在几乎所有的 UCI 基准测试中，基于树结构和小型概率图的分类器总能稳定地高出几个百分点。这部分内容会沿着“依赖关系”的轴线展开，从“完全独立”（朴素贝叶斯）到“全联合分布”，重点介绍实际应用中的三个关键点：SPODE、TAN 和 AODE。把这种因子分解的思想推广到一般形式，就得到了贝叶斯网络。

你将学到

  • 完全条件独立假设为什么会失效，放松这个假设时参数数量如何增长。
  • SPODE：所有特征共享一个“超父”节点。
  • TAN：每个特征单独选择一个父节点，选择依据是基于条件互信息的最大生成树（Chow-Liu 算法）。
  • AODE：对所有符合条件的超父节点取平均值，消除模型选择引入的方差。
  • 贝叶斯网络：DAG 因子分解、d-分离、马尔可夫毯、变量消除以及联合树算法。
  • 实用决策规则：如何在 NB、TAN、AODE 和完整 BN 中做出选择。

预备知识

  • 联合分布、条件独立性、互信息
  • 生成树、树结构上的动态规划
  • 熟读 第 9 篇：朴素贝叶斯

1. 为什么要放松独立性假设？

1.1 那个「方便的谎言」

朴素贝叶斯的核心公式是这样的：

$$P(\mathbf{x}\mid c_k) \;=\; \prod_{j=1}^{d} P(x^{(j)}\mid c_k).$$

它的核心假设是：给定类别后，特征之间互不相关。但这个假设在真实数据中几乎总是站不住脚。

  • 文本领域：即使只看「机器学习文章」这一类，短语「machine learning」的出现频率也远高于 $P(\text{machine})\,P(\text{learning})$ 的预测值。
  • 医学领域：在「流感」病例中，发热 和 咳嗽 依然高度相关，因为它们共享同一条生理通路。
  • 图像领域：无论属于哪一类，相邻像素的值通常都非常接近。

如果条件独立假设被严重违反，后验概率 $P(c\mid\mathbf{x})$ 就会变得过度自信：虽然最终胜出的类别没错，但置信度会被高估得很离谱，候选类之间的排序也会变得混乱。

1.2 模型复杂度与效果的权衡

模型允许的依赖关系参数量准确率
朴素贝叶斯无$O(K d)$基线水平
单依赖模型（SPODE / TAN / AODE）每个特征最多依赖 1 个其他特征$O(K d S)$更高
$k$-依赖模型每个特征最多依赖 $k$ 个其他特征$O(K d S^{k})$再高一点
完全联合分布所有可能的依赖$O(K S^{d})$最优，但不可行

这里 $S$ 是每个特征的平均取值数，$K$ 是类别数。经验上，$k=1$ 是最佳选择：相比朴素贝叶斯，参数量只多了 $S$ 倍，但准确率提升却是整个系列里最大的一步。

2. 单依赖估计（ODE）

2.1 SPODE：一个超父节点

选一个特征 $x^{(p)}$，称为「超父」，让其他所有特征都同时依赖类别和这个超父：

$$P(\mathbf{x}\mid c_k) \;=\; P(x^{(p)}\mid c_k)\prod_{j\neq p} P(x^{(j)}\mid c_k,\,x^{(p)}).$$

如何理解这个模型。 朴素贝叶斯假设「给定类别后，所有特征相互独立」；SPODE 则认为「除了类别外，还有一个共同的隐变量影响所有特征」。比如在垃圾邮件检测中，这个隐变量可能是单词 free：它是否出现会影响 click、offer、winner 等词的概率。

带拉普拉斯平滑的最大似然估计公式如下：

$$\hat{P}(x^{(j)}{=}v\mid c_k,\,x^{(p)}{=}u) \;=\; \frac{\#\{x^{(j)}{=}v,\,c_k,\,x^{(p)}{=}u\}+\alpha} {\#\{c_k,\,x^{(p)}{=}u\}+\alpha\,S_j}.$$

如何选择超父。 常用两个标准：

$$p^{*} \;=\; \arg\max_{p}\, I(X^{(p)};Y), \qquad p^{*} \;=\; \arg\max_{p}\,\sum_{j\neq p} I(X^{(j)};Y\mid X^{(p)}).$$

第一个标准挑出与 $Y$ 最相关的特征；第二个标准挑出条件化后能最好解释其他特征的那个。这两个都是启发式方法——后面会提到 AODE 和 TAN，正是因为单靠这一步选择不够可靠。

2.2 AODE：平均所有合格的超父

核心思想。 不再赌某个单一超父，而是把所有支撑量足够大的超父对应的 SPODE 模型加起来：

$$\hat{P}(c_k\mid\mathbf{x})\;\propto\; \sum_{i:\;n_i\geq m} P(c_k)\,P(x^{(i)}\mid c_k)\prod_{j\neq i} P(x^{(j)}\mid c_k,\,x^{(i)}).$$

其中 $n_i$ 是训练集中取值 $x^{(i)}$ 出现的样本数，阈值 $m$（通常设为 30）用来过滤掉支撑不足、估计不可靠的超父。

为什么有效。

  1. 无需结构搜索。 模型固定，只需统计计数。
  2. 降低方差。 对 $d$ 个相关性较高的 SPODE 求平均，效果类似 bagging。
  3. 实现简单。 和 SPODE 用相同的统计量，只是对所有 $i$ 再求一次和。

import numpy as np
from sklearn.naive_bayes import BernoulliNB
from sklearn.model_selection import cross_val_score

# 三个二值特征，给定类别后 x1、x2 都依赖 x0，因此 NB 的独立性假设被破坏
np.random.seed(42)
N = 500
y  = np.random.binomial(1, 0.5, N)
x0 = np.random.binomial(1, 0.8 * y + 0.2 * (1 - y), N)
x1 = np.random.binomial(1, 0.7 * x0 + 0.1 * (1 - x0), N)  # 依赖 x0
x2 = np.random.binomial(1, 0.6 * x0 + 0.2 * (1 - x0), N)  # 依赖 x0
X  = np.column_stack([x0, x1, x2])

print("Naive Bayes 5 折 CV:",
      cross_val_score(BernoulliNB(), X, y, cv=5).mean().round(4))
# 在这种数据上 AODE 通常会稳稳地高出 NB 几个百分点。

3. TAN：树增广朴素贝叶斯

3.1 每个特征自己选父节点

SPODE 强制所有特征共享一个超父节点，而 TAN 更灵活。它允许每个特征单独选择一个额外的父节点，但这些选择必须构成一棵树：

$$P(\mathbf{x}\mid c_k) \;=\; \prod_{j=1}^{d} P(x^{(j)}\mid c_k,\,x^{(\pi_j)}),$$

其中 $\pi_j$ 是特征 $j$ 在增广树中的唯一父节点。如果某个特征是树的根节点，则没有父节点，对应项退化为 $P(x^{(j)}\mid c_k)$。

3.2 用 Chow-Liu 算法学习这棵树

定理（Chow & Liu 1968；Friedman, Geiger, Goldszmidt 1997）。 在所有树形增广结构中，最大化条件对数似然的那棵树，就是以条件互信息为权重的最大生成树。权重公式如下：

$$w_{jk} \;=\; I(X^{(j)};X^{(k)}\mid Y) \;=\; \sum_{y,x_j,x_k} P(x_j,x_k,y)\, \log\frac{P(x_j,x_k\mid y)}{P(x_j\mid y)\,P(x_k\mid y)}.$$

如何理解这个权重？ 它衡量的是「在类别已经解释之后，$X^{(j)}$ 和 $X^{(k)}$ 之间残留的相关性」。权重越大的边，说明它们之间的依赖关系越值得建模。

算法步骤：

  1. 在 $d$ 个特征上构建完全图。
  2. 将边 $(j,k)$ 的权重设为 $I(X^{(j)};X^{(k)}\mid Y)$。
  3. 使用 Kruskal 或 Prim 算法求解最大生成树，复杂度为 $O(d^{2}\log d)$。
  4. 选择根节点，通常取 $\arg\max_j I(X^{(j)};Y)$。
  5. 从根节点出发，将无向树定向为有向树。

import numpy as np
from itertools import combinations

def conditional_mutual_info(X, y, j, k):
    """从数据计算 I(X_j ; X_k | Y)，自然对数。"""
    cmi = 0.0
    for c in np.unique(y):
        mask = (y == c); p_c = mask.mean()
        Xc = X[mask]
        for vj in np.unique(X[:, j]):
            for vk in np.unique(X[:, k]):
                p_jk = ((Xc[:, j] == vj) & (Xc[:, k] == vk)).mean()
                p_j  = (Xc[:, j] == vj).mean()
                p_k  = (Xc[:, k] == vk).mean()
                if min(p_jk, p_j, p_k) > 0:
                    cmi += p_c * p_jk * np.log(p_jk / (p_j * p_k))
    return cmi

在前面提到的合成数据中，依赖对 $(x_0,x_1)$ 的 CMI 明显大于独立对 $(x_0,x_2)$。Kruskal 算法正是利用这种差异来优先连接高权重的边。

4. 贝叶斯网络

NB 和 TAN 都是更通用模型的特例：贝叶斯网络。现在，我来从整体视角看看图模型的全貌。

4.1 DAG 分解

贝叶斯网络 是一个有向无环图 $\mathcal{G}=(V,E)$，每个节点都带有一张条件概率表（CPT）。联合分布可以按图结构分解为：

$$P(X_1,\dots,X_n) \;=\; \prod_{i=1}^{n} P\bigl(X_i\mid \mathrm{Pa}(X_i)\bigr).$$

为什么分解重要？ 上图中四个二值变量的联合分布需要 $2^4 - 1 = 15$ 个自由参数，而分解后只需要 $1+2+2+4 = 9$ 个。如果扩展到 $n=20$ 个二值变量，且每个节点平均有 3 个父节点，联合分布会有约 $10^{6}$ 项，而分解形式只需约 $20 \cdot 2^{3} = 160$ 项。

我们已经见过的两种特例：

  • 朴素贝叶斯：星形图，$Y$ 是所有 $X_j$ 的父节点，$X_j$ 之间没有边。
  • TAN：在星形图的基础上，再加一棵连接 $X_j$ 的树。

4.2 d-分离

DAG 中的条件独立性由图形化准则 d-分离 决定。任何复杂情况都可以归结为以下三种基本结构：

  • 链 $A\to B\to C$： 如果不观测 $B$，$A$ 和 $C$ 不独立；如果条件化 $B$，路径被阻断，$A\perp C\mid B$。
  • 叉 $A\leftarrow B\to C$： 行为与链类似：共同原因 $B$ 导致 $A$ 和 $C$ 相关，观测 $B$ 后相关性消失。
  • 对撞 $A\to B\leftarrow C$： 行为相反。如果不观测 $B$，$A\perp C$；一旦观测 $B$，两个原本独立的原因会被耦合起来——这就是著名的 explaining away（因因相消） 效应。

判定规则：如果路径上的任意链或叉节点被观测，路径被阻断；或者路径上的任意对撞节点未被观测且其后代也未被观测，路径同样被阻断。两个集合在给定 $Z$ 时被 d-分离，当且仅当它们之间的所有路径都被 $Z$ 阻断。

4.3 马尔可夫毯

马尔可夫毯 $\mathrm{MB}(X)$ 是使得「给定它之后 $X$ 与外界条件独立」的最小集合。在贝叶斯网络中，它正好是：

$$\mathrm{MB}(X) \;=\; \mathrm{Pa}(X)\;\cup\;\mathrm{Ch}(X)\;\cup\;\bigl\{\,X\text{ 的子节点的其他父节点}\,\bigr\}.$$

那些「共同父节点」很容易被忽略，但不可或缺——根据对撞规则，条件化一个子节点会激活通向其他父节点的路径。马尔可夫毯也是 Gibbs 采样中重新采样 $X$ 所需的最小变量集合，从因果角度看，它是用网络其余部分预测 $X$ 的最小充分统计量。

4.4 推断：变量消除

计算诸如 $P(D\mid \text{evidence})$ 的后验时，暴力求和的时间复杂度是 $n$ 的指数级。变量消除 每次只消去一个变量，并复用中间因子：

  1. 确定消除顺序，例如查询 $D$ 时按 $A,B,C$ 顺序；
  2. 把所有涉及 $A$ 的因子相乘后对 $A$ 求和，得到新因子 $\tau_1(B)$；
  3. 重复：将涉及 $B$ 的因子相乘并求和，得到 $\tau_2(C)$，依此类推。

对于链结构，复杂度从 $O(S^{n})$ 降到 $O(nS^{2})$；对于一般图，复杂度由图的树宽决定——即消除过程中产生的最大中间因子的大小。这也是为什么稠密图即使做了消除依然难以处理。

4.5 联合树

联合树 算法将变量消除全局化，让所有边缘分布都能通过两次消息传递计算出来：

  1. 道德化： 在每对共同父节点之间添加无向边，然后去掉所有箭头方向；
  2. 三角化： 添加弦边，使所有长度 $\geq 4$ 的环至少有一条弦。这一步控制树宽，最优三角化是 NP-hard，实践中常用 minimum fill-in 等启发式方法；
  3. 提取极大团；
  4. 构建团树： 相邻团共享一个 separator 集，并满足 running-intersection 性质：任何变量出现的团构成一个连通子树；
  5. 沿树传消息： 先 collect 再 distribute。两轮之后，每个团都持有自己变量上的正确边缘分布。

总开销与最大团大小（即树宽 + 1）成指数关系——与变量消除复杂度相同，但所有查询共享一次预计算。

4.6 结构学习

DAG 本身也可以从数据中学习。主要有两大类方法：

  • 打分 + 搜索： 在 DAG 空间中爬山（增/删/反一条边），用「似然 + 罚项」的打分函数评价候选。最经典的是 BIC：
$$\mathrm{BIC}(\mathcal{G}) \;=\; \log p(\mathcal{D}\mid\hat{\theta},\mathcal{G}) \;-\; \tfrac{|\theta|}{2}\log N.$$

第一项奖励拟合，第二项惩罚参数量；BIC 在标准假设下是相合的（$N\to\infty$ 时收敛到真实图）。

  • 基于约束： （PC、FCI 算法）。先做条件独立性检验得到骨架，再用对撞规则对 v 形结构定向。

一个硬事实：精确的结构学习是 NP-hard 的（Chickering, Heckerman & Meek 2004）。实际算法通常限制每个节点的最大父节点数、加入随机重启，或者借助先验固定某些边的方向。

5. 贝叶斯网络的推断复杂度

结构分解 $P(\mathbf{x}) = \prod_i P(x_i \mid \mathrm{pa}(x_i))$ 让存储变得紧凑，但推断——也就是计算 $P(X_q \mid \mathbf{x}_e)$——通常是 #P 难的。复杂度取决于网络在消元顺序下的分解质量。

变量消元。 我先选定一个消元顺序 $\sigma$，覆盖所有非查询、非证据变量。每一步消去一个变量，把涉及它的所有因子相乘后求和：

$$\phi'(\mathbf{Y}) = \sum_{x_k} \prod_{f \in \mathcal{F}_k} f(x_k, \mathbf{Y}).$$

单步计算的成本随 width 指数增长，这里的 width 是指消元过程中遇到的最大范围 $\mathbf{Y} \cup \{x_k\}$ 的大小。最优消元顺序对应最小的 induced treewidth，但找到这个顺序本身是 NP 难的。实际中，min-fill 和 min-neighbour 启发式方法在数百节点的网络上表现不错。

对于树状网络（道德图中没有无向环），treewidth 为 1，精确推断的时间复杂度是 $O(N \cdot K^2)$（$N$ 个变量，每个变量有 $K$ 个取值）。这正好是 HMM 的前向-后向算法的复杂度，HMM 就是链式结构的特例。但在像全连接医疗诊断 BN 这样的密集网络中，treewidth 随 $N$ 线性增长，当 $N \approx 30$ 时，精确推断就变得不可行了。

近似方法。 当 treewidth 太大时，常用的替代方法有三种：loopy BP（不管有没有环，直接跑 BP，希望它收敛；很多真实网络上效果不错，但没有理论保证）、Gibbs 采样（轮流遍历变量，从其他变量当前值的条件分布中采样）以及变分方法（详见第 14 篇）。对一个二值变量、treewidth 为 25 的密集 BN，单条 Gibbs 链跑 10 万样本通常能得到边缘概率的 2-3 位有效精度——虽然慢，但至少能跑出结果。

6. 结构学习：真正的难点在哪里

给定固定结构 $G$，条件概率表的极大似然估计（MLE）非常简单——数一数再除一下就行。真正困难的是从数据中学习 $G$ 本身。目前主要有两大方法。

基于评分的搜索。 定义一个评分函数 $\mathrm{score}(G, \mathcal{D})$，在拟合优度和模型复杂度之间做权衡，然后在有向无环图（DAG）空间中进行搜索。常用的评分标准有两个：BIC 和 BDeu。BIC 的公式如下：

$$\mathrm{BIC}(G, \mathcal{D}) = \log P(\mathcal{D} \mid \hat\theta_G, G) - \frac{|\theta_G|}{2}\log N,$$

而 BDeu 是均匀 Dirichlet 先验下的边缘似然。这两种评分都具有 可分解性——评分可以按每个局部结构 $(x_i, \mathrm{pa}(x_i))$ 分解。因此，局部边的增删只需重新计算受影响的部分。即便如此，$n$ 个节点的 DAG 空间是超指数级的（$\sim n! \cdot 2^{\binom{n}{2}}$），所以搜索算法通常是贪心的：比如带随机重启的爬山法、禁忌搜索（tabu search），或者当 $n \lesssim 25$ 时使用 Silander-Myllymäki 提出的动态规划精确算法。

基于约束的搜索。 通过条件独立性检验来确定边的方向。PC 算法从完全图开始，如果找到一个分隔集 $\mathbf{Z}$ 满足 $X \perp Y \mid \mathbf{Z}$，就删除边 $(X, Y)$；然后用 Meek 规则对碰撞结构（collider）定向。这种方法理论上更干净（在忠实性假设下能恢复马尔可夫等价类），但对有限样本条件下独立性检验的 I 类错误非常敏感。我只在一些玩具数据集上见过 PC 算法表现良好，而在真实噪声数据上，基于评分的爬山法通常更胜一筹。

TAN 的优势再探。 TAN 将搜索空间限制为以类别节点为根的树状结构，这种问题可以用 Chow-Liu 算法精确求解，时间复杂度为 $O(n^2 N)$，核心是基于条件互信息的最大权生成树。这正是 TAN 在实际应用中的价值所在——它几乎保留了朴素贝叶斯（Naive Bayes）的鲁棒性，同时具备接近完整贝叶斯网络（BN）的表达能力，而且结构学习的时间复杂度是多项式级别。

7. 练习题

练习 1 —— SPODE 参数估计

总共有 100 个样本，其中类别 $c_1$ 占了 60 个。在 $c_1$ 中，30 个样本满足 $x^{(p)}{=}1$，而这 30 个样本里又有 18 个满足 $x^{(j)}{=}1$。使用拉普拉斯平滑，取 $\alpha=1$，且 $|x^{(j)}|=3$，计算如下：

$$\hat{P}\bigl(x^{(j)}{=}1\mid c_1, x^{(p)}{=}1\bigr) \;=\; \frac{18+1}{30+1\cdot 3} \;=\; \frac{19}{33} \;\approx\; 0.576.$$

练习 2 —— TAN 参数数量

假设特征是二值的，类别数为 $K$：

  • 朴素贝叶斯：每个 (类别, 特征) 组合对应 1 个参数，总共 $K\cdot d$ 个；
  • TAN：每个非根节点的特征除了类别外还多了一个二值父节点，因此每个 (类别, 特征) 对需要存储 2 项，总共 $2Kd$。

TAN 的参数量是朴素贝叶斯的两倍，但换来的是对成对依赖关系的建模能力。如果特征是 $S$ 元的，倍数会从 2 变成 $S$。

练习 3 —— Chow-Liu 边选择

假设 $I(X_1;X_2\mid Y)=0.8$，$I(X_1;X_3\mid Y)=0.5$。Chow-Liu 最大生成树算法优先选择权重更大的边，因此 $X_1\!-\!X_2$ 会先进入树中，而 $X_1\!-\!X_3$ 只有在不形成环的情况下才会被加入。

练习 4 —— d-分离

在结构 $A\to C\leftarrow B$ 中：

  • (a) $A\perp B$？是。 $C$ 是对撞节点，未观测时路径被阻断；
  • (b) $A\perp B\mid D$（$D$ 与 ABC 无关）？是。 $D$ 不是 $C$ 的后代，对撞节点仍然阻断路径；
  • (c) $A\perp B\mid C$？否。 条件化对撞节点会激活路径，导致 $A$ 和 $B$ 耦合。

练习 5 —— 数据有限时 AODE 还是 TAN

  • $d$ 大、数据充足：TAN 的预测复杂度是 $O(d)$，优于 AODE 的 $O(d^{2})$；
  • $N$ 小：AODE 的平均操作自带正则化效果，通常比 TAN 更稳健——TAN 在这种情况下容易因噪声 CMI 估计导致 MST 边过拟合。

实际策略建议：先用 AODE；当样本量 $N$ 明显超过需要估计的 CMI 格子数时，再切换到 TAN。

Q&A

为什么不直接用完整的贝叶斯网络？

结构学习是 NP 难问题，DAG 的搜索空间是超指数级的。TAN 和 AODE 通过限制结构（比如树形或平均超父）让学习变得可行，同时保留了最重要的依赖关系。

有向边表示因果关系吗？

不是。多个 DAG 可以表达相同的条件独立结构（称为等价类），并且在统计上无法区分。要推断因果关系，需要额外的干预假设——比如 Pearl 的 do-演算或者工具变量。

TAN 如何处理连续特征？

有三种方法：(i) 离散化（等宽、等频或基于熵的方法）；(ii) 假设条件高斯分布（条件高斯贝叶斯网络）；(iii) 使用核密度估计来建模局部条件概率分布。

AODE 和 TAN 怎么选？

AODE 更简单，小样本时更稳健，不需要结构学习。TAN 在 $d$ 很大时预测更快，能为不同特征选择不同的父节点。在文本分类任务中，AODE 是更常见的默认选择；而在稠密数值表中，TAN 通常表现更好。

深度生成模型和这些有什么关系？

现代隐变量模型（比如 VAE、流模型、PixelCNN 这样的自回归模型）本质上就是连续型贝叶斯网络，只是把条件概率表换成了神经条件密度。因子分解的思想完全一样，变化的只是 $P(X_i\mid \mathrm{Pa}(X_i))$ 的参数化形式。
