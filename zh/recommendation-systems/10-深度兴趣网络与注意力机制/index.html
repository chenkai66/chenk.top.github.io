好厨师不会给每位客人做同一道菜。她会观察你进门时的状态，记住你点的酒，瞥一眼你盯着黑板的目光，然后才决定今晚的招牌菜是牛排还是烩饭。你过去的到访记录很重要，但只有符合 当下 心情的部分才会被考虑进去。

以前的推荐模型就像个蹩脚的厨师：把用户点击过的所有内容平均成一个向量，然后给所有人端上同一道菜。你上周浏览的复古皮衣和半年前随手点过的手机充电器权重相同，完全不管你现在正在看什么。

Deep Interest Network (DIN) 让模型学会了“看人下菜”。它的思路简单得让人惊讶：在给候选商品打分时，根据每个历史行为与 这个 候选商品的相关性来加权。同一个用户，面对不同商品会有不同的表征——就像主厨根据不同心情准备不同的菜品。

这篇文章带你梳理基于注意力机制的 CTR 模型家族：DIN（目标注意力）、DIEN（用 GRU + AUGRU 建模兴趣演化）、DSIN（会话感知）、BST（在行为序列上跑 Transformer）。我会让数学推导严谨、代码可运行、直觉清晰明了。

你将学到

  • 直接对用户历史取平均会丢失关键信息，注意力机制能解决这个问题
  • DIN——用 Local Activation Unit 实现目标注意力
  • DIEN——用 GRU + AUGRU 和辅助损失建模兴趣演化
  • DSIN——捕捉会话级别的浏览模式
  • BST——在行为序列和候选商品上应用 Transformer
  • 工程实践技巧：Dice 激活函数、小批量感知正则化、序列截断

1. 从平均池化到注意力机制

平均池化的问题在哪里？

假设一个用户点击了 5 部动作片、3 部爱情片、2 部纪录片和 1 部恐怖片。现在要为一部新的动作片打分，显然那 5 次动作片点击应该占主导地位。但简单平均会把这 11 次点击一视同仁，结果恐怖片这个离群点反而会干扰用户的表征，偏离推荐目标。

用公式表达，传统方法会计算一个固定的用户向量：

$$\mathbf{v}_u = \frac{1}{T} \sum_{j=1}^{T} \mathbf{e}_{b_j}$$

其中 $\mathbf{e}_{b_j}$ 是行为 $b_j$ 的嵌入向量。这个向量完全忽略了候选商品。无论推荐的是动作片还是纪录片，用户的表征都是一样的。

注意力机制如何解决这个问题？

注意力机制为每个历史行为 $b_j$ 计算一个与候选商品 $i$ 的相关性分数 $\alpha_j$：

$$\alpha_j = \text{score}(\mathbf{e}_{b_j}, \mathbf{e}_i)$$

用户表征变成 加权求和：

$$\mathbf{v}_u(i) = \sum_{j=1}^{T} \alpha_j \, \mathbf{e}_{b_j}$$

现在 $\mathbf{v}_u(i)$ 会随着 $i$ 变化。如果推荐动作片，动作类点击会被突出；如果推荐爱情片，爱情类点击会占主导。同样的历史行为，模型读出了不同的重点。上图清楚地展示了这一点：同一位用户的 10 次点击，针对两个候选商品，注意力权重分布完全不同。模型没变，问题变了。

如何选择打分函数？

按表达能力从弱到强，有三种常见选择：

  • 点积——$\text{score}(\mathbf{q}, \mathbf{k}) = \mathbf{q}^\top \mathbf{k}$。计算简单，但能力有限。
  • 缩放点积——除以 $\sqrt{d}$ 稳定数值范围，Transformer 就是这么做的。
  • 加性 MLP——$\mathbf{v}^\top \tanh(\mathbf{W}_q \mathbf{q} + \mathbf{W}_k \mathbf{k} + \mathbf{b})$。表达能力最强，DIN 选择了这种方案。

DIN 还做了进一步优化：不只是简单拼接 $\mathbf{q}$ 和 $\mathbf{k}$，而是把四样东西一起输入 MLP——query、key、query−key、query⊙key。相减捕捉 差异，逐元素相乘捕捉 交互，MLP 在这四者基础上学习一个非线性的兼容性函数。

2. Deep Interest Network (DIN)

阿里巴巴在 2018 年提出了 DIN（Zhou 等，KDD'18）。这是基于注意力机制的点击率（CTR）模型的开山之作，至今仍是工业界的经典工具。它的核心组件是 Local Activation Unit——一个小型多层感知机（MLP），用来为每个历史行为与候选商品的匹配程度打分。

DIN 的工作原理

假设用户的行为序列为 $[b_1, b_2, \ldots, b_T]$，候选商品为 $i$：

  1. 嵌入：将行为、候选商品、用户特征和上下文映射为稠密向量。
  2. 打分：通过激活单元计算每个历史行为与候选商品的相关性分数。
  3. 加权求和：对行为嵌入进行加权求和，生成"激活后"的用户表示。
  4. 拼接 + MLP：将用户表示与其他特征拼接，输入 MLP 预测 CTR。

激活单元的打分公式如下：

$$\text{score}(\mathbf{e}_{b_j}, \mathbf{e}_i) = \text{MLP}\big([\,\mathbf{e}_{b_j};\ \mathbf{e}_i;\ \mathbf{e}_{b_j} - \mathbf{e}_i;\ \mathbf{e}_{b_j} \odot \mathbf{e}_i\,]\big)$$

一个容易被忽略但非常重要的细节是：DIN 原版没有使用 softmax。作者发现，不对权重归一化可以保留兴趣的 强度 信息——如果用户的历史行为中有大量强相关行为，就应该生成更大的用户向量；而只有少量弱相关行为时，用户向量会更小。代码中我会同时展示两种实现方式。

import torch
import torch.nn as nn
import torch.nn.functional as F

class LocalActivationUnit(nn.Module):
    """DIN 的 Local Activation Unit。

    使用一个小 MLP 消化四种交互视角：行为本身、候选本身、二者之差、
    二者逐元素乘积，从而为每个历史行为与候选商品的匹配度打分。
    """

    def __init__(self, embedding_dim, hidden_dims=(80, 40), use_softmax=False):
        super().__init__()
        # 输入 = [行为; 候选; 行为-候选; 行为*候选]
        in_dim = embedding_dim * 4
        layers = []
        for h in hidden_dims:
            layers += [nn.Linear(in_dim, h), nn.PReLU()]
            in_dim = h
        layers.append(nn.Linear(in_dim, 1))
        self.mlp = nn.Sequential(*layers)
        self.use_softmax = use_softmax

    def forward(self, behaviors, candidate, mask=None):
        """
        behaviors: (B, T, D)   历史行为嵌入
        candidate: (B, D)      候选商品嵌入
        mask:      (B, T)      真实位置为 1，填充位置为 0
        Returns:
            user_repr: (B, D)
            weights:   (B, T)
        """
        B, T, D = behaviors.shape
        cand = candidate.unsqueeze(1).expand(B, T, D)

        # 四种交互视角
        feats = torch.cat([behaviors, cand, behaviors - cand, behaviors * cand], dim=-1)
        scores = self.mlp(feats).squeeze(-1)              # (B, T)

        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)

        if self.use_softmax:
            weights = F.softmax(scores, dim=1)
        else:
            # DIN 默认：保留原始权重，不做归一化
            weights = scores
            if mask is not None:
                weights = weights * mask

        user_repr = torch.bmm(weights.unsqueeze(1), behaviors).squeeze(1)
        return user_repr, weights

class DIN(nn.Module):
    """用于 CTR 预估的 Deep Interest Network。"""

    def __init__(self, item_dim=64, user_dim=32, ctx_dim=16,
                 mlp_hidden=(200, 80), dropout=0.5):
        super().__init__()
        self.activation = LocalActivationUnit(item_dim)

        in_dim = item_dim + item_dim + user_dim + ctx_dim
        layers = []
        for h in mlp_hidden:
            layers += [nn.Linear(in_dim, h), nn.PReLU(), nn.Dropout(dropout)]
            in_dim = h
        layers += [nn.Linear(in_dim, 1)]
        self.mlp = nn.Sequential(*layers)

    def forward(self, user_feats, behaviors, candidate, ctx_feats, mask=None):
        user_repr, attn = self.activation(behaviors, candidate, mask)
        x = torch.cat([user_repr, candidate, user_feats, ctx_feats], dim=1)
        logit = self.mlp(x).squeeze(-1)
        return logit, attn

训练目标与阿里的工业化技巧

DIN 使用 logits 上的二分类交叉熵作为损失函数：

$$\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \big[ y_i \log \sigma(\hat{y}_i) + (1 - y_i) \log(1 - \sigma(\hat{y}_i)) \big]$$

论文将大部分性能提升归功于以下三个技巧：

  • Dice 激活函数——一种数据自适应的 PReLU，拐点会根据当前 batch 的分布动态调整（见第 7 节）。
  • 小批次感知正则化——不直接对所有 embedding 做 L2 正则化（千万级商品，本 batch 中绝大多数未出现），而是只对当前 batch 中出现的 embedding 进行正则化，并按其频次加权。效果相近，但计算成本大幅降低。
  • 梯度裁剪——长行为序列在训练初期容易导致梯度爆炸，裁剪能有效缓解这一问题。

3. Deep Interest Evolution Network (DIEN)

DIN 把用户历史行为当成一个无序集合，完全忽略了时间维度。但用户的兴趣是会变化的——上个月你还在研究笔记本电脑，这周已经开始关注配件，下周可能又迷上了人体工学椅。

这张图清楚地说明了 DIN 的局限性：用户的兴趣不是一个静态向量，而是一个时序信号，不同兴趣点按一定规律此起彼伏。DIN 只能看到所有兴趣点的总和，而 DIEN（Zhou 等，AAAI'19）通过两层结构专门建模了这种动态变化。

一张图看懂架构

$$\mathbf{h}_t = \text{GRU}(\mathbf{e}_{b_t}, \mathbf{h}_{t-1})$$


每个隐藏状态 $\mathbf{h}_t$ 就代表用户在 $t$ 时刻的兴趣。

$$\tilde{u}_t = a_t \cdot u_t \qquad \mathbf{h}'_t = (1 - \tilde{u}_t) \odot \mathbf{h}'_{t-1} + \tilde{u}_t \odot \tilde{\mathbf{h}}_t$$

简单来说：如果某个历史兴趣与候选商品高度相关，就让它主导演化；如果不相关，就冻结状态，避免噪声干扰信号。图中箭头粗细反映了 $a_t$ 的大小：粗箭头传递更多信息，细箭头则基本保持前一状态不变。

辅助损失的小技巧

$$\mathcal{L}_{\text{aux}} = -\frac{1}{T-1}\sum_{t=1}^{T-1} \Big[ \log \sigma(\mathbf{h}_t^\top \mathbf{e}_{b_{t+1}}^+) + \log\big(1 - \sigma(\mathbf{h}_t^\top \mathbf{e}_{b_{t+1}}^-)\big)\Big]$$

用大白话解释：如果 $t$ 时刻的隐藏状态既能预测 $t+1$ 时刻的真实点击（正样本），又无法预测随机采样的负样本，那它就学到了有用的信息。

最终目标函数是 $\mathcal{L} = \mathcal{L}_{\text{ctr}} + \lambda \cdot \mathcal{L}_{\text{aux}}$，其中 $\lambda$ 通常取值范围是 $[0.1, 1.0]$。

class AUGRUCell(nn.Module):
    """更新门被注意力加权的 GRU 单元。"""

    def __init__(self, input_dim, hidden_dim):
        super().__init__()
        self.W_ir = nn.Linear(input_dim, hidden_dim)
        self.W_hr = nn.Linear(hidden_dim, hidden_dim)
        self.W_iz = nn.Linear(input_dim, hidden_dim)
        self.W_hz = nn.Linear(hidden_dim, hidden_dim)
        self.W_in = nn.Linear(input_dim, hidden_dim)
        self.W_hn = nn.Linear(hidden_dim, hidden_dim)

    def forward(self, x_t, h_prev, a_t):
        """
        x_t:    (B, input_dim)   t 时刻的兴趣（来自 GRU 层）
        h_prev: (B, hidden_dim)  上一步演化后的状态
        a_t:    (B, 1)           相对候选商品的注意力权重
        """
        r = torch.sigmoid(self.W_ir(x_t) + self.W_hr(h_prev))
        z = torch.sigmoid(self.W_iz(x_t) + self.W_hz(h_prev))
        n = torch.tanh(self.W_in(x_t) + r * self.W_hn(h_prev))
        z_tilde = a_t * z                       # ← AUGRU 的关键改动
        h_t = (1 - z_tilde) * h_prev + z_tilde * n
        return h_t

class DIEN(nn.Module):
    def __init__(self, item_dim=64, user_dim=32, ctx_dim=16,
                 hidden_dim=64, mlp_hidden=(200, 80), dropout=0.5):
        super().__init__()
        self.gru = nn.GRU(item_dim, hidden_dim, batch_first=True)
        self.attn = LocalActivationUnit(hidden_dim, use_softmax=True)
        self.augru = AUGRUCell(hidden_dim, hidden_dim)

        in_dim = hidden_dim + item_dim + user_dim + ctx_dim
        layers = []
        for h in mlp_hidden:
            layers += [nn.Linear(in_dim, h), nn.PReLU(), nn.Dropout(dropout)]
            in_dim = h
        layers += [nn.Linear(in_dim, 1)]
        self.mlp = nn.Sequential(*layers)

    def forward(self, user_feats, behaviors, candidate, ctx_feats):
        # 第 1 层：兴趣提取
        interest, _ = self.gru(behaviors)               # (B, T, H)

        # 计算与候选商品的注意力权重
        _, attn = self.attn(interest, candidate)        # (B, T)

        # 第 2 层：用 AUGRU 演化兴趣
        B, T, H = interest.shape
        h = torch.zeros(B, H, device=behaviors.device)
        for t in range(T):
            h = self.augru(interest[:, t, :], h, attn[:, t:t+1])
        final_interest = h                              # (B, H)

        x = torch.cat([final_interest, candidate, user_feats, ctx_feats], dim=1)
        return self.mlp(x).squeeze(-1), interest        # interest 用于辅助损失

线上部署时，这段逐时间步循环的 Python 代码会被替换为自定义 CUDA kernel，但核心逻辑就是如此。

4. Deep Session Interest Network (DSIN)

用户行为往往是扎堆的。比如午饭时间花十五分钟浏览笔记本，晚上回来快速扫一遍耳机，第二天早上又看跑鞋。每个行为簇内部主题一致，而簇与簇之间通常伴随着兴趣或心境的切换。

DSIN（Feng 等，IJCAI'19）显式建模了这种结构。上图展示了 9 次行为被划分为 3 个会话的完整流程：

  1. 会话切分——如果两次行为的时间间隔超过 30 分钟（原论文的阈值），就切分会话。
  2. 会话内自注意力——在每个会话内部，用多头自注意力捕捉局部模式（哪些物品在这个行为簇中相互关联）。
  3. 会话间 Bi-LSTM——跨会话使用双向 LSTM 建模兴趣从一个会话到下一个会话的漂移过程。
  4. 目标注意力——最后，基于候选物品对各会话向量加权，得到与候选最相关的会话级用户表征。

直觉很简单：会话是模型的“思考单元”。把三十次点击当作一整袋无差别数据丢进去，就会丢失它们其实分成三个连贯小簇的事实；而把每次点击当作独立时间步，又忽略了同一会话内的点击通常主题相近。会话正好是介于两者之间的合适粒度。

class DSINSessionLayer(nn.Module):
    """会话内自注意力 → 平均池化 → 会话向量。"""

    def __init__(self, embed_dim, num_heads=4):
        super().__init__()
        self.attn = nn.MultiheadAttention(embed_dim, num_heads, batch_first=True)

    def forward(self, session_behaviors, mask=None):
        # session_behaviors: (B, S, D)
        attended, _ = self.attn(session_behaviors, session_behaviors, session_behaviors,
                                key_padding_mask=mask)
        return attended.mean(dim=1)                      # (B, D)

def split_sessions(timestamps, gap_seconds=1800):
    """按时间间隔将行为序列切分为多个会话区间。"""
    sessions, start = [], 0
    for i in range(1, len(timestamps)):
        if timestamps[i] - timestamps[i - 1] > gap_seconds:
            sessions.append((start, i))
            start = i
    sessions.append((start, len(timestamps)))
    return sessions

什么时候用哪个模型

模型关键创新适合场景
DIN在扁平行为列表上做目标注意力历史短、时序结构不明显
DIENGRU + AUGRU + 辅助损失历史长、兴趣演化平滑
DSIN会话内自注意力 + 会话间 Bi-LSTM浏览模式有清晰会话边界

5. Behavior Sequence Transformer (BST)

2019 年，Transformer 已经彻底占领了 NLP 领域。阿里巴巴淘宝团队提出了一个大胆的想法：直接把 Transformer 应用到用户行为序列上，是不是就能解决问题？

BST（Chen 等，DLP-KDD'19）的核心思想是将用户行为序列和候选商品拼接成一个 token 序列，然后用 Transformer encoder 处理。多头自注意力机制让每个行为都能关注其他所有行为，同时也能关注候选商品。位置嵌入用来编码时间顺序。

整体架构可以概括为：

$$\mathbf{Z} = \text{TransformerBlock}\big(\,[\mathbf{e}_{b_1} + \mathbf{p}_1,\, \ldots,\, \mathbf{e}_{b_T} + \mathbf{p}_T,\, \mathbf{e}_i + \mathbf{p}_{T+1}]\,\big)$$

接着将 $\mathbf{Z}$ 与侧特征拼接，送入 MLP。在淘宝日志上的实验表明，BST 相比 WDL 基线模型的 AUC 提升了约 7.5%。需要注意的是，BST 没有显式的“目标注意力”步骤。它不需要这个步骤，因为 [历史, 候选] 的自注意力机制已经让候选商品直接访问每个行为，同时也让每个行为直接访问其他行为——这是 DIN 从未建模的部分。

class BST(nn.Module):
    def __init__(self, item_dim=64, max_len=50, num_heads=8, num_layers=2,
                 user_dim=32, ctx_dim=16, mlp_hidden=(1024, 512, 256), dropout=0.2):
        super().__init__()
        self.pos_embed = nn.Embedding(max_len + 1, item_dim)
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=item_dim, nhead=num_heads,
            dim_feedforward=item_dim * 4, dropout=dropout,
            batch_first=True, activation='relu',
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=num_layers)

        in_dim = item_dim * (max_len + 1) + user_dim + ctx_dim
        layers = []
        for h in mlp_hidden:
            layers += [nn.Linear(in_dim, h), nn.PReLU(), nn.Dropout(dropout)]
            in_dim = h
        layers += [nn.Linear(in_dim, 1)]
        self.mlp = nn.Sequential(*layers)

    def forward(self, user_feats, behaviors, candidate, ctx_feats, mask=None):
        # 将候选拼接到行为序列末尾
        seq = torch.cat([behaviors, candidate.unsqueeze(1)], dim=1)   # (B, T+1, D)
        pos = torch.arange(seq.size(1), device=seq.device)
        seq = seq + self.pos_embed(pos).unsqueeze(0)

        z = self.transformer(seq, src_key_padding_mask=mask)          # (B, T+1, D)
        flat = z.reshape(z.size(0), -1)
        x = torch.cat([flat, user_feats, ctx_feats], dim=1)
        return self.mlp(x).squeeze(-1)

6. 这些技巧到底能带来多少收益？

数据来自 DIN/DIEN/DSIN/BST 原论文在 Amazon Books CTR 基准上的实验结果，经过归一化处理以便对比。有两点值得关注：

  • 从 sum/avg pooling 到 DIN 的提升最大。加入注意力机制是单次改动中效果最显著的优化。后续改进都是渐进式的——DIEN 在 DIN 的基础上提升了零点几个百分点，DSIN 再稍微高一点，BST 的表现则视数据集而定，大致与 DSIN 持平或略胜一筹。
  • AUC 提升看似微小，但在大规模场景下意义重大。在淘宝这样的平台上，0.005 的 AUC 提升可以转化为几个百分点的点击率增长，以及数亿的 GMV 增量。这就是为什么推荐系统团队会持续优化那些在外人看来像是“噪声”的指标。

除了准确性，每个模型的成本特性也不同：DIN 的推理成本最低，因为注意力机制只是为每个行为增加了一个 MLP；DIEN 的序列化 AUGRU 最慢；BST 在 GPU 上运行快，但显存占用高；DSIN 的会话切分逻辑（实时会话化）则是运维中的头疼问题。

我的经验建议是：先用 DIN。它能以 20% 的工程投入拿到 80% 的收益。如果用户行为序列较长且话题顺序重要（比如订阅类产品、有明确升级路径的兴趣爱好），可以考虑 DIEN。如果会话特征明显且频繁（如短视频应用、电商浏览场景），DSIN 是更好的选择。如果希望用一个统一的模型覆盖所有场景，并且服务栈已经对 Transformer 友好，那就选 BST。

7. 真正能提升效果的工业化技巧

Dice——数据自适应激活函数

PReLU 的拐点固定在 $x = 0$，如果激活值以 0 为中心还好，但分布一旦偏移就显得很僵硬。Dice（Data-adaptive Activation）用基于 batch 均值的平滑 sigmoid 替代了硬切换：

$$\text{Dice}(x) = p(x) \cdot x + (1 - p(x)) \cdot \alpha x, \qquad p(x) = \sigma\!\left(\frac{x - \mathbb{E}[x]}{\sqrt{\text{Var}[x] + \epsilon}}\right)$$

拐点现在会随着数据分布动态调整。图中右侧展示了三个均值不同的 batch，Dice 的拐点随之漂移，而 PReLU 的拐点始终钉在零点。不同层、不同分布、不同有效激活函数——完全免费。

class Dice(nn.Module):
    def __init__(self, dim, eps=1e-8):
        super().__init__()
        self.alpha = nn.Parameter(torch.zeros(dim))
        self.bn = nn.BatchNorm1d(dim, eps=eps, affine=False)

    def forward(self, x):
        x_norm = self.bn(x)
        p = torch.sigmoid(x_norm)
        return p * x + (1 - p) * self.alpha * x

小批次感知正则化

对 1 亿条 item embedding 每次都做 L2 正则太浪费了，梯度里 99.99% 都是零。我只对当前 batch 中出现的 embedding 做正则，并按它们在 batch 中的频次缩放：

$$\mathcal{L}_{\text{reg}} = \frac{\lambda}{2} \sum_{j \in \mathcal{B}} \frac{n_{j,\mathcal{B}}}{n_j} \|\mathbf{e}_j\|^2$$

其中 $n_{j,\mathcal{B}}$ 是 item $j$ 在 batch $\mathcal{B}$ 中的出现次数，$n_j$ 是它的全局出现次数。效果几乎一样，但开销小了好几个数量级。

变长序列处理

真实用户的历史行为长度差异很大。我的做法是统一 pad 到固定最大长度，然后用 mask 标记有效部分：

def pad_and_mask(sequences, max_len, pad_value=0):
    padded, masks = [], []
    for seq in sequences:
        seq = seq[-max_len:]                                    # 只保留最近的行为
        pad_len = max_len - len(seq)
        padded.append([pad_value] * pad_len + list(seq))
        masks.append([0] * pad_len + [1] * len(seq))
    return torch.LongTensor(padded), torch.FloatTensor(masks)

在注意力机制中，先把被 mask 的位置打到 $-10^9$ 再做 softmax，这样它们的权重就会被压到零。

大流量场景下的服务优化

面对百万 QPS 的场景，我总结了几招实用技巧：

  • 离线预计算并缓存 item embedding：item 表相对静态，每晚重新计算一次即可。
  • 截断到最近 N 条行为：N = 50–100 通常已经能捕捉绝大部分信号，成本却低得多。
  • 量化：FP16 或 INT8 能把模型体积压缩 2–4 倍，AUC 几乎无损。
  • 批量推理：GPU 在 batch size 达到 64 以上时性能最佳。
  • 用自定义 CUDA 算子替换 AUGRU 的 Python 循环：如果 DIEN 必须上线，这一步很关键。

8. 常见问题

为什么 DIN 选择目标注意力而不是自注意力？
目标注意力解决的问题是：“用户过去的哪些行为和 当前 候选商品相关？” 自注意力只关注历史行为内部的关系（比如"笔记本和手机都属于电子产品"），虽然有用，但它不依赖候选商品。而候选商品才是推荐的核心条件。后来 BST 证明，用 Transformer 可以同时兼顾两者。

为什么 DIN 不用 softmax？
作者发现，softmax 会抹掉兴趣的 强度 信息。一个有很多强匹配行为的用户，和一个只有一个弱匹配行为的用户，经过 softmax 归一化后，向量看起来可能完全一样。去掉 softmax 后，用户向量的模长可以直接反映兴趣的强弱。

辅助损失真的有帮助吗？
确实有帮助，尤其是在长序列场景下。没有辅助损失时，GRU 容易退化成一些平凡状态，这些状态虽然能降低 CTR 损失，但无法有效表达用户兴趣。DIEN 论文提到，在 Amazon 数据集上，仅辅助损失就能提升约 0.3% 的 AUC。

计算成本如何？
注意力机制的时间复杂度是 $O(T^2 \cdot d)$，其中 $T$ 是序列长度。当 $T \le 100$ 时还能接受，再长就很难处理。对于长历史序列，常用方法包括：截断（最常见）、稀疏/线性注意力、两阶段检索（比如 SIM 硬搜索 → DIN）。

冷启动用户怎么处理？
可以回退到用户画像特征（如人口统计、地理位置、设备信息）和品类级别的先验知识。基于内容的 item embedding（从标题、图片提取）在用户行为数据或商品行为数据稀疏时也能派上用场。

注意力权重是否真的可解释？
基本可以，但需要注意一些细节。它能告诉我模型在某次推荐中更依赖用户过去的哪些行为，这对调试和建立信任很有帮助。不过，经过 softmax 归一化的权重是 相对值——权重高并不意味着绝对相关性高，只是相对于序列中的其他行为更高而已。

总结

深度兴趣网络（Deep Interest Network）给推荐系统带来了一个经久不衰的核心思想：用户的历史行为并非同等重要，模型需要每次动态判断哪些行为真正关键。

后续的工作都是围绕这个核心思想展开的：

  1. DIN —— 根据候选物品的相关性，对用户行为加权。
  2. DIEN —— 捕捉用户兴趣随时间的变化过程。
  3. DSIN —— 将用户行为分组为会话，保留会话结构信息。
  4. BST —— 交给 Transformer 自行挖掘所有行为模式。

优秀的厨师不会对每位顾客都做同一道菜。有了 DIN，好的推荐系统也不会再这样做。
