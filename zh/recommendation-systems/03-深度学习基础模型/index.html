推荐系统（三）—— 深度学习基础模型

从 MLP 到 Embedding，再到 NeuMF、YouTube DNN、Wide & Deep —— 用渐进的方式讲清深度学习推荐系统的每一块基石，附经过原文核对的架构图和可直接运行的 PyTorch 代码。

2016 年 6 月，Google 发表了一篇只有一页的论文，悄悄改变了推荐系统的格局。这篇论文介绍了 Wide & Deep Learning，当时这个模型已经在拥有十亿用户的 Google Play 应用商店中使用。短短一年内，各大科技公司纷纷将深度学习模型部署上线；到 2019 年，行业标准彻底改变，矩阵分解从核心系统降级为基线方法。

为什么会有这样的变化？多层神经网络带来了经典方法无法实现的四大能力。

  • 学到的表征。Embedding 层取代了 one-hot 向量，直接从点击数据中端到端地学出稠密、语义化的隐向量。
  • 非线性交互。两层带 ReLU 的 MLP 能拟合 XOR，而点积做不到。
  • 多模态融合。文本、图像和行为序列可以通过同一个梯度流优化。
  • 端到端优化。不再需要手工设计特征交叉，损失函数会自行决定。

本文带你从 dot(p_u, q_i) 出发，一路走到 NeuMF、 YouTube DNN 和 Wide & Deep。所有架构都经过原论文验证，每一步都配有可以直接运行的 PyTorch 代码。

你将对以下内容建立直观理解#

  • MLP 的核心直觉：为什么堆叠线性层和 ReLU 能成为通用的交互引擎。
  • 嵌入（Embedding）：不只是如何使用 nn.Embedding，而是梯度为何会把相似的 ID 拉到一起。
  • NeuMF（He 等，WWW 2017）：两条路径，一个优化目标。
  • YouTube DNN（Covington 等，RecSys 2016）：如今所有大规模推荐系统都在使用的两阶段流程。
  • Wide & Deep（Cheng 等，DLRS 2016）：记忆与泛化的经典结合。

前置知识#

  • 熟悉 PyTorch 的基础用法，包括 nn.Module、 autograd 和 DataLoader。
  • 读过本系列第二篇 ，掌握矩阵分解以及隐式和显式反馈的区别。
  • 掌握基本线性代数，如点积和矩阵向量乘法。

一、为什么选择深度学习，为什么是现在#

经典方法的天花板#

这些数字来自公开的 CTR 数据集（Criteo、Avazu、隐式反馈版的 MovieLens-1M），非常典型。结论很明确：每增加一个非线性来源，就能提升几个 AUC 百分点，而这几个点对 GMV 的影响非常大。

要理解原因，得先看每种经典方法能表达什么。

矩阵分解用点积预测评分：

$$\hat{r}_{ui} = \mathbf{p}_u^\top \mathbf{q}_i$$

简单说：每个用户和物品是一个短向量，预测就是它们的匹配程度。优雅，但线性的——无法捕捉"我同时喜欢科幻和动作，但单独不喜欢"这种复杂关系。

$$\hat{y}(\mathbf{x}) = w_0 + \sum_i w_i x_i + \sum_{i 它是 MF 的超集，但也只到二阶。像"年轻用户 × 周五晚上 × 惊悚片"这样的三阶效应，还得靠手动设计交叉特征。

协同过滤完全绕过建模，直接找相似用户或物品。在矩阵稀疏之前效果不错——但在生产环境中，矩阵总是稀疏的。

三种方法的共同短板是：最多支持二阶交互，并且都需要人工设计特征。

深度学习带来的改变#

理论上，带非线性的单隐藏层神经网络是通用函数逼近器。实际应用中，这意味着：

  • 在 Embedding 上加一个 MLP，可以拟合数据所需的任意阶交互。
  • 同一套骨干网络能处理图像（CNN）、文本（Transformer）、序列（RNN）——还能联合训练。
  • 冷启动有了突破口：新物品只要有内容（标题、图片），预训练编码器就能生成一个合理的初始向量。

代价是更高的计算成本、更低的可解释性和更多的超参数。第七节会讲如何通过工程实践让这笔交易划算。

二、 MLP 的直觉：从点积到非线性交互#

在了解具体模型之前，先搞清楚 MLP 为什么比点积强。

点积 $\mathbf{p}^\top \mathbf{q} = \sum_k p_k q_k$ 是把每个维度的乘积累加起来。它对称、线性，没法表达“特征 A 只有在特征 B 同时激活时才有意义”这种逻辑。

$$f(\mathbf{p}, \mathbf{q}) = \mathbf{w}^\top \, \text{ReLU}\!\big(\mathbf{W} [\mathbf{p}; \mathbf{q}] + \mathbf{b}\big)$$

ReLU 根据输入维度的激活组合，决定每个隐藏单元是开还是关。隐藏单元足够多时，这就是通用逼近定理的结果。交互不再是固定的公式，而是学出来的。

用 MLP 替代点积这一招，就是 NeuMF、 YouTube DNN 和 Wide & Deep 的起点。

三、 Embedding：从稀疏 ID 到语义表征的桥梁#

One-hot 是敌人#

假设一个推荐系统有 1000 万用户和 100 万物品。如果用 one-hot 编码，每个用户会变成一个 1000 万维的向量，只有一个位置是 1。这种表示方法直接带来以下三个问题：

  1. 存储和计算开销巨大：单个用户输入就占用 40 MB 的浮点向量。
  2. 信息密度极低： 99.99999% 的向量值都是零。
  3. 所有距离都相等：$\|\mathbf{e}_i - \mathbf{e}_j\|_2 = \sqrt{2}$，无论 $i$ 和 $j$ 是否接近。用户 42 和用户 43 的距离，跟用户 42 和用户 9,999,999 的距离完全一样。

Embedding 层一次性解决了这些问题。它把每个 ID 映射到一个稠密向量，比如 64 维。训练完成后，兴趣相似的用户会在 64 维空间中靠近彼此。

类比： one-hot 像一本电话簿，每个人住在一座孤岛上，岛与岛之间的距离完全相等。而 Embedding 就像一位地理学家，重新排列这些岛屿，让相关的人住得更近。这样一来，“找相似用户”就变成了一个最近邻搜索问题。

Embedding 到底是什么#

数学上， Embedding 层是一个可学习的矩阵 $\mathbf{P} \in \mathbb{R}^{m \times d}$，其中第 $i$ 行就是用户 $i$ 的向量。“查表”操作 $\mathbf{p}_i = \mathbf{P}[i, :]$ 等价于 $\mathbf{P}^\top \mathbf{e}_i$，但实际实现时通过行索引加速。

梯度如何赋予 Embedding 语义#

Embedding 初始化为随机向量，其语义结构完全由梯度更新形成。

以点击预测损失为例：当用户 $u$ 点击物品 $i$ 时，损失函数告诉优化器“让 $\mathbf{p}_u^\top \mathbf{q}_i$ 变大”。反向传播会将 $\mathbf{p}_u$ 微微推向 $\mathbf{q}_i$，反之亦然。经过数百万次点击后，这条简单的规则会产生惊人的结构：受众重叠的物品靠拢，常被一起购买的物品靠拢，同一位艺术家的歌曲靠拢。

我从未告诉模型“流派”是什么，但梯度的几何结构自己发现了它。

可视化 Embedding 学到了什么#

把训练好的物品 Embedding 矩阵用 t-SNE 投影到二维，通常会看到这样的画面：同类物品紧密聚集成簇，混合类型物品位于相关簇之间（比如“科幻动作”介于科幻和动作之间），长尾物品则散落在外围。这是判断模型是否学到东西的最佳诊断工具。

如何选择维度#

物品规模推荐 $d$备注
<10 万8 – 32再大容易过拟合
10 万 – 100 万32 – 64多数场景的甜点区
100 万 – 1 亿64 – 128超过 128 收益递减
网级128 – 256交互量上百亿才划算

YouTube 论文和后续研究反复验证了一条经验法则：从 $d = 32$ 开始，逐步翻倍，直到验证集 AUC 提升不到 ~0.5%，然后停止。 内存和服务延迟随 $d$ 线性增长，但模型质量的增长是凹的。

import torch
import torch.nn as nn

class IdEmbedding(nn.Module):
    """带 Xavier 初始化和可选 padding 的 Embedding 层。"""

    def __init__(self, vocab_size: int, dim: int, padding_idx: int | None = None):
        super().__init__()
        self.emb = nn.Embedding(vocab_size, dim, padding_idx=padding_idx)
        nn.init.xavier_uniform_(self.emb.weight)
        if padding_idx is not None:
            with torch.no_grad():
                self.emb.weight[padding_idx].zero_()

    def forward(self, ids: torch.LongTensor) -> torch.Tensor:
        return self.emb(ids)

对于多字段类别特征（如用户 ID、物品 ID、类目、城市等），每个字段单独建一张表，再堆叠起来：

class MultiFieldEmbedding(nn.Module):
    def __init__(self, field_dims: list[int], dim: int):
        super().__init__()
        self.tables = nn.ModuleList([IdEmbedding(v, dim) for v in field_dims])

    def forward(self, x: torch.LongTensor) -> torch.Tensor:
        # x: [batch, num_fields] -> [batch, num_fields, dim]
        return torch.stack([t(x[:, i]) for i, t in enumerate(self.tables)], dim=1)

这个 [batch, num_fields, dim] 的张量，就是后续所有模型期望的输入格式。

四、神经协同过滤（NCF 与 NeuMF）#

核心思想：用更智能的方式替代点积#

He、 Liao、 Zhang、 Nie、 Hu 和 Chua 在 WWW 2017 上提出了 Neural Collaborative Filtering。他们的想法简单直接：矩阵分解其实是网络的一种特例，把它泛化后可以做得更好。

论文中提出了三个模型变体：

  • GMF（Generalized Matrix Factorization）：加权可学习的点积版本。
  • MLP：纯深度拼接，不带内积归纳偏置。
  • NeuMF：融合 GMF 和 MLP，每条路径使用独立的嵌入。

实际应用中， NeuMF 是最值得关注的。以下是它的架构图，完全忠实于原论文。

NeuMF 架构#

从下往上看：

  1. 每边两张嵌入表。 GMF 和 MLP 不共享嵌入——论文强调这一点很重要。每条路径学习适合自己的表示。
  2. GMF 路径：用户和物品嵌入的元素积 $\mathbf{p}_u^\text{GMF} \odot \mathbf{q}_i^\text{GMF}$。再加上一个可学习权重，这就是标准点积的推广。
  3. MLP 路径：拼接 $[\mathbf{p}_u^\text{MLP}; \mathbf{q}_i^\text{MLP}]$，然后通过 2 到 3 层 Dense + ReLU （典型尺寸：$128 \to 64 \to 32$）。
  4. 融合：将两条路径的输出拼接起来，通过 sigmoid 投影为标量： $$\hat{y}_{ui} = \sigma\!\left(\mathbf{h}^\top \begin{bmatrix} \mathbf{p}_u^\text{GMF} \odot \mathbf{q}_i^\text{GMF} \\ \mathbf{z}_L^\text{MLP} \end{bmatrix}\right)$$ 对于隐式反馈（点击、播放、购买），损失函数是二元交叉熵： $$\mathcal{L} = -\sum_{(u, i) \in \mathcal{D}^+ \cup \mathcal{D}^-} \big[ y_{ui} \log \hat{y}_{ui} + (1 - y_{ui}) \log(1 - \hat{y}_{ui}) \big]$$

负样本集 $\mathcal{D}^-$ 通过采样生成——通常每个正样本配 4 个负样本。

import torch
import torch.nn as nn

class NeuMF(nn.Module):
    """NeuMF：融合 GMF 和 MLP，两条路径独立嵌入（He 等，2017）。"""

    def __init__(
        self,
        num_users: int,
        num_items: int,
        embedding_dim: int = 32,
        mlp_layers: tuple[int, ...] = (128, 64, 32),
        dropout: float = 0.0,
    ):
        super().__init__()
        # 两条路径独立嵌入——这是 NeuMF 的关键设计。
        self.user_gmf = nn.Embedding(num_users, embedding_dim)
        self.item_gmf = nn.Embedding(num_items, embedding_dim)
        self.user_mlp = nn.Embedding(num_users, embedding_dim)
        self.item_mlp = nn.Embedding(num_items, embedding_dim)

        # MLP 塔，输入是拼接后的用户/物品向量。
        layers, in_dim = [], embedding_dim * 2
        for out_dim in mlp_layers:
            layers += [nn.Linear(in_dim, out_dim), nn.ReLU()]
            if dropout > 0:
                layers.append(nn.Dropout(dropout))
            in_dim = out_dim
        self.mlp = nn.Sequential(*layers)

        # 输出头接收 [GMF 向量；MLP 向量]。
        self.head = nn.Linear(embedding_dim + mlp_layers[-1], 1)

        for emb in (self.user_gmf, self.item_gmf, self.user_mlp, self.item_mlp):
            nn.init.normal_(emb.weight, std=0.01)

    def forward(self, users: torch.LongTensor, items: torch.LongTensor) -> torch.Tensor:
        gmf = self.user_gmf(users) * self.item_gmf(items)           # [B, d]
        mlp_in = torch.cat([self.user_mlp(users), self.item_mlp(items)], dim=-1)
        mlp_out = self.mlp(mlp_in)                                  # [B, mlp_layers[-1]]
        logits = self.head(torch.cat([gmf, mlp_out], dim=-1)).squeeze(-1)
        return torch.sigmoid(logits)

# 冒烟测试
model = NeuMF(num_users=10_000, num_items=5_000, embedding_dim=32, dropout=0.2)
users = torch.randint(0, 10_000, (8,))
items = torch.randint(0, 5_000, (8,))
labels = torch.randint(0, 2, (8,)).float()
preds = model(users, items)
loss = nn.BCELoss()(preds, labels)
print(f"preds={preds.detach().numpy().round(3)}  loss={loss.item():.4f}")

NeuMF 实战训练技巧#

论文里提到三个容易被忽略的细节：

  • 分别预训练两条路径。先单独训练 GMF，再单独训练 MLP，最后用两者的权重初始化 NeuMF。论文显示，仅这一步就能提升约 2% 的 AUC。
  • 负样本比例要合适。“1 正 4 负"是经典默认值； 1:1 容易欠拟合， 1:10 浪费计算资源。
  • 偏置项不做 L2 正则。对偏置项加 weight decay 会降低性能，优化器参数组里需要把它们排除掉。

五、 YouTube DNN：支撑半个互联网的两阶段推荐系统#

Covington、 Adams 和 Sargin （RecSys 2016）是深度学习时代被引用最多的工业推荐论文。它的“召回 + 排序”两阶段设计，成为后来几乎所有大规模推荐系统的模板：抖音、 Spotify、 Pinterest、 Instagram、淘宝。

为什么需要两阶段？#

每次请求都给十亿个视频打分不现实。实时用丰富特征处理十亿个物品也不现实。解决方法很简单：

  1. 召回阶段用轻量模型和简单特征，把候选集从百万级缩小到几百个。
  2. 排序阶段用复杂模型和丰富特征，对这几百个候选精排，选出 top-K。

架构#

召回被建模为一个极端多分类问题：“给定用户状态，预测他接下来会看哪个视频。”类别数等于候选视频总数（百万级）。用户塔对最近观看的视频 Embedding 做平均池化，拼接人口学特征，经过三层 ReLU （$1024 \to 512 \to 256$），输出一个 256 维的用户向量。训练时用 sampled softmax 损失；上线时，视频 Embedding 存入 ANN 索引（HNSW 或 ScaNN），用户向量直接做最近邻查询——十亿规模也只需个位数毫秒。

排序是一个更深的前馈网络（$1024 \to 512 \to 256 \to 128$），特征更丰富：曝光视频的 Embedding、同频道历史观看的 Embedding、距离上次观看的时间、在 feed 中的位置、语言匹配等。关键在于输出头用了带权重的 logistic 回归，目标是预测期望观看时长，而不是点击率。论文证明，这个目标比单纯优化 CTR 更能提升长期满意度。

YouTube 论文里值得借鉴的设计#

三个设计选择至今仍然非常经典：

  • 平均池化用户近期行为。成本低、并行化、效果强，是很好的基线。序列模型（第六篇）只有在数据足够多时才能超越它。
  • 召回当作分类任务，而非回归任务。 Sampled softmax + ANN 上线检索，已经成为行业主流范式。
  • 标签选择要慎重。“观看时长”、“长点击”（停留 >30 秒）、“完播”都比裸点击泛化能力更强。损失函数就是产品目标。

import torch
import torch.nn as nn
import torch.nn.functional as F

class YouTubeCandidateTower(nn.Module):
    """召回塔：生成用于 ANN 检索的用户向量。"""

    def __init__(
        self,
        num_videos: int,
        num_searches: int,
        num_geos: int,
        embedding_dim: int = 64,
        user_dim: int = 256,
    ):
        super().__init__()
        self.video_emb = nn.Embedding(num_videos, embedding_dim, padding_idx=0)
        self.search_emb = nn.Embedding(num_searches, embedding_dim, padding_idx=0)
        self.geo_emb = nn.Embedding(num_geos, embedding_dim)

        in_dim = 3 * embedding_dim + 2  # 视频池化 + 搜索池化 + 地域 + 年龄 + 性别
        self.tower = nn.Sequential(
            nn.Linear(in_dim, 1024), nn.ReLU(),
            nn.Linear(1024, 512), nn.ReLU(),
            nn.Linear(512, user_dim), nn.ReLU(),
        )

    def forward(self, watched, searched, geo, age, gender):
        # watched, searched: [B, L]，padding 过；对非 padding 的位置做 mean pool。
        v = self._masked_mean(self.video_emb(watched), watched != 0)
        s = self._masked_mean(self.search_emb(searched), searched != 0)
        g = self.geo_emb(geo)
        x = torch.cat([v, s, g, age.unsqueeze(-1), gender.unsqueeze(-1)], dim=-1)
        return F.normalize(self.tower(x), dim=-1)  # 归一化的用户向量，适合 cosine 检索

    @staticmethod
    def _masked_mean(emb: torch.Tensor, mask: torch.Tensor) -> torch.Tensor:
        m = mask.unsqueeze(-1).float()
        return (emb * m).sum(1) / m.sum(1).clamp(min=1.0)

训练时，用户向量和采样的候选视频 Embedding 一起过 softmax；上线时，用户向量直接对 ANN 索引做点积检索。这一座塔，就是现代工业级召回的核心组件。

六、 Wide & Deep：记忆与泛化的结合#

核心洞察#

Cheng 等人（DLRS 2016）发现，深度模型虽然擅长泛化，但有时会“过度推荐”——因为嵌入向量过于平滑，导致推荐的物品看似合理，但并不准确。而带交叉特征的线性模型能完美记住具体的共现关系，却无法外推。

类比：记忆像是那个对你说“你喜欢《盗梦空间》，那一定喜欢《信条》”的朋友——具体、精准，但从不冒险。泛化则是那个说“你喜欢烧脑悬疑片，试试《Primer》”的朋友——视野更广，偶尔出错，但能带来惊喜。一个好的推荐系统需要同时具备这两种特质。

他们的解决方案是：联合训练，将两部分的分数在 sigmoid 前直接相加。

架构设计#

Wide 部分：一个线性层，输入原始稀疏特征和手动构造的交叉特征 $\phi(\mathbf{x})$，例如 installed_app=Pandora AND impression_app=YouTube。输出为 $\hat{y}^w = \mathbf{w}^\top [\mathbf{x}, \phi(\mathbf{x})] + b$。

Deep 部分：每个类别字段分别嵌入，拼接后通过三层 ReLU 层（$256 \to 128 \to 64$）。输出为 $\hat{y}^d$。

联合输出头：$\hat{y} = \sigma(\hat{y}^w + \hat{y}^d)$。两边共享同一个损失函数的梯度，优化器动态调整各自的贡献。

论文中特意采用了双优化器策略：Wide 端用 FTRL+L1（稀疏、可解释、自动选择特征），Deep 端用 AdaGrad 或 Adam（稠密、平滑）。这种双优化器设置很关键——如果只用一个优化器，效果会打折扣。

import torch
import torch.nn as nn

class WideAndDeep(nn.Module):
    """Wide & Deep（Cheng 等，2016），用于二分类 CTR 预估。"""

    def __init__(
        self,
        wide_dim: int,                  # 稀疏特征 + 交叉特征向量的维度
        field_dims: list[int],          # Deep 端各字段的词表大小
        embedding_dim: int = 32,
        deep_layers: tuple[int, ...] = (256, 128, 64),
        dropout: float = 0.0,
    ):
        super().__init__()
        # Wide 端：一个大的稀疏线性层。
        self.wide = nn.Linear(wide_dim, 1)

        # Deep 端：每字段嵌入 + MLP。
        self.embeddings = nn.ModuleList([nn.Embedding(v, embedding_dim) for v in field_dims])
        in_dim = len(field_dims) * embedding_dim
        layers = []
        for out_dim in deep_layers:
            layers += [nn.Linear(in_dim, out_dim), nn.ReLU()]
            if dropout > 0:
                layers.append(nn.Dropout(dropout))
            in_dim = out_dim
        layers.append(nn.Linear(in_dim, 1))
        self.deep = nn.Sequential(*layers)

    def forward(self, x_wide: torch.Tensor, x_deep: torch.LongTensor) -> torch.Tensor:
        wide_score = self.wide(x_wide).squeeze(-1)
        deep_in = torch.cat([emb(x_deep[:, i]) for i, emb in enumerate(self.embeddings)], dim=-1)
        deep_score = self.deep(deep_in).squeeze(-1)
        return torch.sigmoid(wide_score + deep_score)

# 双优化器训练——很多教程忽略了这个生产级实现细节。
model = WideAndDeep(wide_dim=10_000, field_dims=[50_000, 10_000, 100, 20])
opt_wide = torch.optim.Adagrad(model.wide.parameters(), lr=0.05)
opt_deep = torch.optim.Adam(
    [p for n, p in model.named_parameters() if not n.startswith("wide.")],
    lr=1e-3, weight_decay=1e-5,
)

实践中，也可以用单个 Adam 优化器完成训练，模型依然能跑通。但 Wide+Deep 的核心在于联合损失，这才是它名字的由来。它不是两个独立训练模型的简单集成。 Wide 和 Deep 参数之间能看到彼此的梯度，这种交互才是关键。

七、训练纪律：决定这些方法是否真的有效#

正确的架构是必要的，但远远不够。以下细节经常对离线 AUC 的影响比模型选择更大。

负采样#

对于隐式反馈，每个用户都有成千上万个“负样本”（未见过的商品）。以下是三种负采样策略，按复杂度排序：

  • 均匀随机采样。这是默认方法，出人意料地难以超越，尤其在召回阶段表现稳健。
  • 热度加权采样。热门商品被采样的概率更高——如果用户忽略了爆款商品，这就是一个很强的负向信号。
  • Batch 内 / 难负样本。将同一批次中其他用户的正样本作为当前用户的负样本；或者定期从当前模型中抽取高分负样本。这种方法提升了判别能力，但需要仔细调整温度参数。

import numpy as np

def sample_negatives(user_history: set[int], catalog_size: int, k: int = 4) -> list[int]:
    """均匀随机采样负样本，排除用户历史中的商品。"""
    out = []
    while len(out) < k:
        cand = int(np.random.randint(catalog_size))
        if cand not in user_history:
            out.append(cand)
    return out

def train(model, train_loader, val_loader, *, epochs=50, patience=8, device="cuda"):
    model = model.to(device)
    bce = nn.BCELoss()
    opt = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-5)
    sched = torch.optim.lr_scheduler.ReduceLROnPlateau(opt, mode="min", patience=3, factor=0.5)

    best_val, bad = float("inf"), 0
    for epoch in range(1, epochs + 1):
        model.train()
        train_loss = 0.0
        for users, items, labels in train_loader:
            users, items, labels = users.to(device), items.to(device), labels.to(device)
            opt.zero_grad()
            preds = model(users, items)
            loss = bce(preds, labels)
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), 5.0)
            opt.step()
            train_loss += loss.item() * len(labels)
        train_loss /= len(train_loader.dataset)

        model.eval()
        val_loss = 0.0
        with torch.no_grad():
            for users, items, labels in val_loader:
                users, items, labels = users.to(device), items.to(device), labels.to(device)
                val_loss += bce(model(users, items), labels).item() * len(labels)
        val_loss /= len(val_loader.dataset)
        sched.step(val_loss)

        if val_loss < best_val - 1e-4:
            best_val, bad = val_loss, 0
            torch.save(model.state_dict(), "best.pt")
        else:
            bad += 1
            if bad >= patience:
                print(f"第 {epoch} 轮早停；最佳验证损失={best_val:.4f}")
                break
        print(f"轮次 {epoch:>3}  训练损失={train_loss:.4f}  验证损失={val_loss:.4f}")

场景主指标实际衡量的内容
CTR 预测AUC模型能否将正样本排在负样本前面
CTR 预测LogLoss预测概率是否校准
Top-K 召回Recall@K、HitRate@K正确商品是否出现在前 K 个结果中
Top-K 排序NDCG@K正确商品的排名有多靠前
评分预测RMSE预测评分的均方误差

如果用 RMSE 衡量 CTR，或者用 AUC 衡量评分预测，那就是拧错了优化的方向。一开始就选定指标，写下来，永远不要偷偷更换。

八、常见问题，认真回答#

Q： Embedding 维度应该设多大？
物品规模小于 100 万时，从 32 开始；超过 100 万，从 64 开始。逐步翻倍，直到验证集 AUC 提升不到约 0.5%。维度 $d$ 超过 256 后，几乎都会过拟合，除非交互数据达到百亿级别。

Q：是不是 NeuMF 一定比 MF 好？
不是。在交互数据量低于 100 万时，正则化调好的 MF 往往优于 NeuMF——深度模型容易过拟合。 NeuMF 的优势通常在交互数据量超过 1000 万且有丰富 side feature 时才会显现。

Q： Wide & Deep 模型能不能只用 Deep 部分？
可以，很多人确实这么做了（比如 DeepFM、 DCN）。 Wide 部分的作用是精确记忆高基数共现模式，比如“安装了应用 X 的用户也安装了应用 Y”。如果业务依赖这些具体模式的精准捕捉，就保留 Wide 部分。否则，使用自动化的交叉网络（如 DeepFM、 DCN）通常是更好的选择。

Q：只有几百万用户的场景适合用 YouTube DNN 吗？
物品规模小于 10 万时，两阶段模式显得多余。直接用一个排序模型对全量目录打分即可。当全量打分无法满足延迟要求时，再考虑引入两阶段模式。

Q：全新物品（没有交互记录）怎么处理？
按效果从高到低推荐以下方法：

  1. 基于内容初始化——用预训练编码器（如 BERT、 CLIP）从物品的文本或图像生成 Embedding。
  2. 类目均值初始化——取同类目下其他物品 Embedding 的平均值。
  3. Bandit 探索——将新物品曝光给一小部分流量，收集初始信号。

Q：如何防止深度推荐模型过拟合？
采用分层防御策略：

  • 在 Embedding 上加 $10^{-5}$ 的 weight decay。
  • 在 MLP 中使用 0.2 到 0.3 的 dropout。
  • 根据验证集 AUC 进行 early stopping。
  • 如果以上方法都无效，减小维度 $d$。
    复杂度只在验证指标能随之提升时才增加。

Q：如何加速训练过程？
80/20 法则：

  1. 首选 GPU，速度提升 10 到 100 倍。
  2. 增大 batch size，提高 GPU 利用率。
  3. 使用混合精度训练（torch.cuda.amp），速度提升约 2 倍。
  4. 设置 DataLoader(num_workers > 0)。
    所有特征离线预计算，避免在训练循环中进行 join 操作。

九、走到这里，我看到了什么#

深度学习没有发明推荐系统，但它改变了三个关键约束：

  • 表征是学出来的，不是设计出来的。 Embedding 替代了 one-hot，梯度能发现我手写代码无法实现的结构。
  • 交互是任意阶的，不再局限于二阶。 MLP 能拟合数据需要的任何复杂形式。
  • 流水线是端到端的，不再是分段拼接。损失从预测直接回传到原始 ID。

NeuMF 证明了“学出来的交互”比“固定内积”更强。 YouTube DNN 展示了如何在十亿级物品规模下拆分召回和排序。 Wide & Deep 表明，记忆和泛化最好一起学，而不是分开学再组合。
