强化学习（十一）：层次化强化学习与元学习

层次化强化学习（Options、MAXQ、Feudal Networks、目标条件策略）与元强化学习（MAML、FOMAML、RL²）的系统讲解：时序抽象、半马尔可夫过程、Manager-Worker 架构、二阶元梯度与循环式元学习器，附带 PyTorch 实现。

#强化学习#层次化RL#元学习#MAML
Read in English →

标准强化学习把每个问题都看成一连串原子级别的决策：观察状态，选动作，拿奖励，然后重复。这种方法在时间跨度短、奖励密集的简单任务上还能应付，但遇到人类轻松解决的任务就彻底失效了。比如“做早餐”这件事，根本不是一次决策就能搞定的，而是一个由多个子任务组成的树状结构——煮咖啡、煎蛋、烤面包、装盘上桌——每个子任务本身就是一个小策略。层次化强化学习（HRL） 的核心思想是把宏动作当成基本单位，让智能体能在不同时间尺度上思考和行动。

标准强化学习还有一个致命缺陷：每次面对新任务都要从零开始学。一个会骑自行车的人，花一下午就能学会骑摩托车，而不是需要一千万次环境交互。元强化学习（Meta-RL） 解决的就是这个问题：通过在一个任务分布上训练，使得智能体在面对新任务时只需要少量回合，甚至只需通过一个循环神经网络（RNN）的一次前向传播，就能快速适应。

这两条思路其实殊途同归：层次化是在时间维度上做抽象，元学习是在任务维度上做抽象，二者都能降低学习问题的有效维度。近年来的一些代表性工作，比如 FuN、HIRO、MAML 和 RL²，往往同时结合了这两种方法，效果显著。

你将学到什么

  • Options 框架——半马尔可夫决策过程与 intra-option Q-learning
  • MAXQ——按任务层级分解值函数
  • Feudal RL（FuN、HIRO）——基于 Manager-Worker 架构的连续子目标方法
  • 目标条件 RL——通用值函数与 HER（事后经验回放）
  • MAML / FOMAML——学习一个能快速适应的初始化，只需几次梯度更新
  • RL²——将内层 RL 算法嵌入 RNN 的隐藏状态
  • 可运行代码——Four Rooms 环境中的 intra-option Q-learning 和 2D 导航任务中的 MAML 策略梯度

前置知识

  • Q-learning、策略梯度和值函数（第 1 到 6 部分 ）
  • 熟悉 RNN 的展开操作和二阶自动微分
  • 掌握 PyTorch 框架

一、层次化：Options 框架

时序抽象为什么重要

扁平策略每走一步环境就做一次决策，因此在长度为 $T$ 的任务中，信用分配路径的长度是 $T$，探索树的叶子数达到 $|\mathcal{A}|^T$。这两者都呈指数增长。如果引入平均长度为 $\bar k$ 的宏动作，决策次数会减少到 $T/\bar k$，探索树规模缩小到 $|\mathcal{O}|^{T/\bar k}$，其中 $\mathcal{O}$ 是一个通常很小的选项集合。下图对比了这两种视角。

除了理论上的复杂度分析，层次化方法还有三个实际好处：

  1. 模块化——在一个任务上学到的选项可以直接迁移到其他任务。比如，“走到门口”这个选项可以通用于各种导航问题。
  2. 可解释性——高层策略规模小，容易检查；决策点也集中在语义上有意义的关键位置。
  3. 奖励塑形——子目标天然提供密集的内在奖励，即使外部奖励稀疏也能有效学习。

Option 三元组

根据 Sutton、Precup 和 Singh（1999）的定义，一个 Option 是一种时序扩展动作：

$$o = \langle \mathcal{I}, \pi_o, \beta \rangle,$$

其中 $\mathcal{I} \subseteq \mathcal{S}$ 是允许启动该选项的状态集合，$\pi_o(a \mid s)$ 是选项的内部策略，$\beta(s) \in [0, 1]$ 是终止概率。引入选项后，原始 MDP 就变成了半马尔可夫决策过程（semi-MDP）：高层策略 $\mu(o \mid s)$ 选择一个选项，选项运行直到 $\beta$ 触发，然后才会进行下一次高层决策。

Intra-Option Q-Learning

最简单的做法是等到选项结束再用累积奖励更新 $Q(s, o)$，但这很浪费。一个选项可能运行几十步，中间的所有转移都被丢弃了。Intra-option Q-learning（Sutton 等，1999）在每一步都更新 $Q(s, o)$，核心思想是：同一个转移 $(s, a, r, s')$ 对所有“在状态 $s$ 下会选择动作 $a$”的选项都是有效的训练样本：

$$Q(s, o) \leftarrow Q(s, o) + \alpha \big[r + \gamma U(s', o) - Q(s, o)\big],$$

其中延续值定义为：

$$U(s', o) = (1 - \beta(s'))\, Q(s', o) + \beta(s')\, \max_{o'} Q(s', o').$$

延续值的设计非常巧妙：如果选项继续运行，就沿用它的 Q 值；如果选项在 $s'$ 处终止，就把控制权交还给高层策略，选择当前最优的下一选项。

import numpy as np
from collections import defaultdict

class IntraOptionQLearning:
    """离散选项集合上的 intra-option Q-learning。

    每一条原始转移都会更新：
    (1) 当前选项的高层 Q(s, o)；
    (2) 该选项的内部 Q 表，从而隐式修正其内部策略。
    """

    def __init__(self, env, options, alpha=0.5, gamma=0.99, epsilon=0.1):
        self.env, self.options = env, options
        self.alpha, self.gamma, self.epsilon = alpha, gamma, epsilon
        self.Q = defaultdict(lambda: np.zeros(len(options)))     # Q(s, o)
        self.Q_prim = defaultdict(lambda: np.zeros(env.n_actions))

    def _select_option(self, state):
        available = [i for i, o in enumerate(self.options)
                     if o.can_initiate(state)]
        if not available:
            return None
        if np.random.rand() < self.epsilon:
            return int(np.random.choice(available))
        q = self.Q[state][available]
        return int(available[np.argmax(q)])

    def train_episode(self, max_steps=1000):
        state = self.env.reset()
        total_reward = 0.0
        for _ in range(max_steps):
            oid = self._select_option(state)
            if oid is None:
                break
            option = self.options[oid]

            while not option.should_terminate(state):
                action = option.act(state, self.epsilon)
                next_state, reward, done = self.env.step(action)
                total_reward += reward

                # 计算延续值 U(s', o)
                if option.should_terminate(next_state):
                    U = np.max(self.Q[next_state])
                else:
                    U = self.Q[next_state][oid]

                # 更新高层 Q 值
                self.Q[state][oid] += self.alpha * (
                    reward + self.gamma * U - self.Q[state][oid])

                # 更新内部策略（基于原始动作的扁平 Q 表）
                td = (reward + self.gamma
                      * np.max(self.Q_prim[next_state])
                      - self.Q_prim[state][action])
                self.Q_prim[state][action] += self.alpha * td
                option.policy[state] = int(np.argmax(self.Q_prim[state]))

                state = next_state
                if done:
                    return total_reward
        return total_reward

在经典的 Four Rooms 基准测试中，使用四个手工设计的“穿过门口”选项后，intra-option Q-learning 比扁平 Q-learning 收敛速度快 3 到 5 倍，而且学到的选项能直接迁移到新的目标位置。

MAXQ：沿任务树分解值函数

Options 把层次结构隐藏在选项集合里，而 MAXQ（Dietterich, 2000）则明确地将其表达出来。智能体被赋予一棵子任务的有向无环图，对每个复合任务 $i$ 和它的子任务 $a$，值函数分解为：

$$Q_i(s, a) = V_a(s) + C_i(s, a),$$

其中 $V_a(s)$ 是完成子任务 $a$ 的价值，$C_i(s, a)$ 是完成函数——子任务结束后继续完成父任务 $i$ 的额外价值。由于 $V_a$ 只依赖于 $a$，所有调用 $a$ 的父任务都能共用同一份 $V_a$，这正是样本效率提升的原因。

代价是：MAXQ 只能保证递归最优，而不是全局最优。也就是说，它只在给定的任务分解下最优。如果你的任务图无法表达真正的最优策略，MAXQ 就找不到它。

二、Feudal RL：用连续子目标替代离散选项

离散选项在连续控制或像素输入场景下很难扩展。手工列举出有用的选项集合几乎不可能。Feudal Networks（FuN，Vezhnevets 等，2017） 和 HIRO（Nachum 等，2018） 提出了一个新思路：用高层 Manager 输出的连续目标向量取代离散选项。

Manager 根据状态 $s_t$，每隔 $c$ 步输出一个目标 $g_t \in \mathbb{R}^d$（FuN 中 $c = 10$）。Worker 是一个目标条件策略 $\pi_\phi(a \mid s, g)$，它通过朝目标方向移动获得内在奖励：

$$r^{\text{int}}_t = \cos\!\big(s_{t+c} - s_t,\, g_t\big),$$

而 Manager 的训练则基于环境提供的外部奖励。这种解耦设计是 Feudal 架构的核心亮点：Worker 在稠密且几何化的内在奖励上学习运动控制，Manager 则专注于长时序信用分配，有效时序长度缩短为 $T/c$。

HIRO 的子目标重标注技巧

Feudal 训练面临一个棘手问题：Worker 还在学习，行为是非平稳的，回放池中存储的旧目标已经无法反映 Worker 当前的实际能力。HIRO 通过子目标重标注解决了这个问题。具体来说，当从回放池采样到 $(s_t, g_t, a_{t:t+c}, s_{t+c})$ 时，将 $g_t$ 替换为最能解释实际动作序列的目标：

$$\tilde g_t = \arg\max_{g} \log \pi_\phi(a_{t:t+c} \mid s_{t:t+c},\, g).$$

这一步确保 Worker 的训练数据始终与当前参数保持一致，显著提升了 Manager 离线学习的稳定性。

三、目标条件 RL 与 HER

目标条件策略 $\pi(a \mid s, g)$ 需要单独讲一节，因为它连接了分层强化学习和元学习。同一个网络可以完成多个目标，而目标 $g$ 只是额外的一个输入。

经典方法是 Universal Value Function Approximators（UVFA，Schaul 等，2015）：学习 $V(s, g)$ 或 $Q(s, a, g)$，而不是传统的 $V(s), Q(s, a)$。但如果不加任何技巧，UVFA 在稀疏奖励的情况下表现会很差。大多数目标在探索时根本无法到达，导致奖励信号几乎总是零。

Hindsight Experience Replay（HER，Andrychowicz 等，2017） 是解决这个问题的标准方法。当一个回合没能达到目标 $g$ 时，我会把轨迹重新标记为它实际到达的目标（通常是最终状态），把失败转化为另一个任务的成功演示：

$$(s_t, a_t, r, s_{t+1}, g) \;\longrightarrow\; (s_t, a_t, r', s_{t+1},\, g' = s_T).$$

结合离线策略算法（DDPG、SAC），HER 把稀疏奖励下的目标到达问题从“几乎不可能”变成了“常规操作”。

四、Meta-RL：学会学习

Meta-RL 不再假设单一的 MDP，而是引入了一个任务分布 $p(\mathcal{T})$。在元训练阶段，智能体会接触到许多任务 $\mathcal{T}_i \sim p(\mathcal{T})$；到了元测试阶段，它会面对一个全新的任务，并且需要尽可能少的交互次数完成适应。

目前主流方法分为两大类：

  • 基于优化（MAML、Reptile、ANIL）：适应 = 在测试任务上跑几步梯度。
  • 基于上下文 / 循环（RL²、PEARL）：适应 = 数据流入时更新 RNN 的隐藏状态，测试时无需梯度计算。

MAML——寻找好的初始化

Model-Agnostic Meta-Learning（MAML，Finn 等，2017） 的核心目标是找到一组参数 $\theta$，使得在任意任务 $\mathcal{T}_i$ 上执行一步（或几步）内层 SGD 后，模型能快速达到高性能：

$$\theta_i' = \theta - \alpha \nabla_\theta \mathcal{L}_{\mathcal{T}_i}(\theta), \qquad \theta \leftarrow \theta - \beta \nabla_\theta \sum_{i} \mathcal{L}_{\mathcal{T}_i}(\theta_i').$$

外层梯度需要穿过内层更新进行反向传播，因此包含 Hessian 项 $\nabla^2 \mathcal{L}$。这正是 MAML 计算开销大的原因，也促使了 FOMAML 的出现——直接忽略二阶项。实验表明，FOMAML 每次外层更新快了约 $10\times$，最终回报损失不到 5%。

左图给出了最直观的理解：元训练的目标不是找一个对单个任务最优的初始化，而是找一个“甜区”——从这里出发，一步梯度就能接近每个任务的特定最优解。

import torch
import torch.nn as nn

class GaussianPolicy(nn.Module):
    """连续控制下的对角高斯策略"""

    def __init__(self, state_dim=2, action_dim=2, hidden=64):
        super().__init__()
        self.trunk = nn.Sequential(
            nn.Linear(state_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
        )
        self.mean = nn.Linear(hidden, action_dim)
        self.log_std = nn.Parameter(torch.zeros(action_dim))

    def sample(self, state):
        h = self.trunk(state)
        mean = torch.tanh(self.mean(h))
        std = self.log_std.exp().clamp(1e-3, 1.0)
        dist = torch.distributions.Normal(mean, std)
        a = dist.rsample()
        return a, dist.log_prob(a).sum(-1)

class MAML:
    """REINFORCE 风格策略梯度的一阶或二阶 MAML"""

    def __init__(self, policy, inner_lr=0.1, outer_lr=1e-3,
                 first_order=False):
        self.policy = policy
        self.inner_lr = inner_lr
        self.first_order = first_order
        self.opt = torch.optim.Adam(policy.parameters(), lr=outer_lr)

    def _rollout(self, env, max_steps=50):
        state = env.reset()
        rewards, log_probs = [], []
        for _ in range(max_steps):
            s = torch.as_tensor(state, dtype=torch.float32).unsqueeze(0)
            a, lp = self.policy.sample(s)
            state, r, done = env.step(a.detach().numpy()[0])
            rewards.append(r)
            log_probs.append(lp)
            if done:
                break
        return rewards, log_probs

    @staticmethod
    def _returns(rewards, gamma=0.99):
        G, out = 0.0, []
        for r in reversed(rewards):
            G = r + gamma * G
            out.insert(0, G)
        out = torch.as_tensor(out, dtype=torch.float32)
        return (out - out.mean()) / (out.std() + 1e-8)

    def _loss(self, rewards, log_probs):
        returns = self._returns(rewards)
        return -(torch.stack(log_probs) * returns).mean()

    def adapt(self, env):
        """在 env 上做一步内层更新，返回适应后的参数列表"""
        rewards, log_probs = self._rollout(env)
        loss = self._loss(rewards, log_probs)
        grads = torch.autograd.grad(
            loss, self.policy.parameters(),
            create_graph=not self.first_order,
        )
        return [p - self.inner_lr * g
                for p, g in zip(self.policy.parameters(), grads)]

    def meta_step(self, task_envs):
        meta_loss = 0.0
        for env in task_envs:
            adapted = self.adapt(env)
            # 暂存原参数，将适应后的参数装入网络进行评估前向
            backup = [p.detach().clone() for p in self.policy.parameters()]
            for p, a in zip(self.policy.parameters(), adapted):
                p.data = a.data
            rewards, log_probs = self._rollout(env)
            meta_loss += self._loss(rewards, log_probs)
            for p, b in zip(self.policy.parameters(), backup):
                p.data = b
        meta_loss /= len(task_envs)

        self.opt.zero_grad()
        meta_loss.backward()
        self.opt.step()
        return meta_loss.item()

上面的实现通过“换权重再前向”的方式完成元评估，而不是使用真正的函数式前向传播。这是最简单易读的版本，对小规模网络已经足够。如果要做研究级的 MAML，建议改用 higher 或 torch.func.functional_call 这类函数式 API。

RL²——把算法折叠进 RNN

RL²（Duan 等，2016；Wang 等，2016）走了另一条路：测试时冻结智能体参数，所有适应都由 RNN 的隐藏状态完成。循环策略接收的输入被扩展为：

$$x_t = (s_t,\, a_{t-1},\, r_{t-1},\, d_{t-1}),$$

即状态加上前一步的动作、奖励和终止标志。在一个元试验内的多个回合中，隐藏状态 $h_t$ 不断累积当前任务的信息——本质上是在隐式地执行贝叶斯后验更新。元训练阶段，外层优化器（PPO 或 A2C）调整 RNN 权重，使得这套“内置的 RL 算法”在 $p(\mathcal{T})$ 上具有高效的样本利用率。

RL² 有两个显著优势：(i) 测试时无需梯度计算，每步适应只需一次前向传播；(ii) 适应过程是学出来的，原则上能在训练分布上超越任何手工设计的优化器。代价是元训练阶段的信用分配极其困难——梯度需要穿过几十甚至上百步、横跨多个回合的循环展开。

五、常见问题

Options 框架为什么能加速学习？
三个原因叠加：(i) 有效时序长度从 $T$ 缩短到大约 $T/\bar k$；(ii) 高层分支因子 $|\mathcal{O}|$ 通常远小于 $|\mathcal{A}|$；(iii) intra-option 学习让每条原始转移都能更新所有兼容选项的价值，而不仅仅是当前控制的那个选项。

层次化最优和全局最优有什么区别？
层次化最优策略是在给定分解（选项集合或任务图）下的最佳策略；全局最优策略是底层扁平 MDP 上的最佳策略。只要分解无法表达真正的最优策略，两者就会出现分歧。比如，一个至少运行 10 步才能终止的选项，无法实现每 3 步切换行为的策略。

MAML 为什么需要二阶梯度？FOMAML 的损失有多大？
元损失是 $\mathcal{L}_{\mathcal{T}_i}(\theta_i')$，其中 $\theta_i' = \theta - \alpha \nabla_\theta \mathcal{L}_{\mathcal{T}_i}(\theta)$。对 $\theta$ 求导后得到 $(I - \alpha \nabla^2_\theta \mathcal{L}_{\mathcal{T}_i})\, \nabla_{\theta_i'} \mathcal{L}_{\mathcal{T}_i}(\theta_i')$，这里包含 Hessian 矩阵。FOMAML 直接丢掉 $-\alpha \nabla^2 \mathcal{L}$ 这个因子，把 $\theta_i'$ 当作 stop-gradient 处理。原论文报告，在标准小样本基准上，性能损失不到 5%。

什么时候该用 MAML，什么时候该用 RL²？
如果适应时能计算梯度，且任务分布广到固定策略无法覆盖所有情况，就选 MAML。如果任务族窄、偏向贪婪利用（比如 arms 会变化的多臂老虎机），或者测试时无法计算梯度，就选 RL²。PEARL 用独立编码器推断任务嵌入，通常是不错的折中方案。

HER 为什么只能用于离线策略算法？
HER 修改了某条转移对应的目标，这违反了在线策略算法的假设——数据是由"当前策略 + 当前目标"生成的，一旦目标被修改，这个假设就不成立了。离线策略算法（DQN、DDPG、SAC）只需要在更新时知道 $\pi(a \mid s, g')$，而这个值是可以计算的，所以重标注不会造成问题。
