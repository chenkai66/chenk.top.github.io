强化学习（九）：多智能体强化学习

系统讲透多智能体强化学习：马尔可夫博弈、非稳态与信用分配、CTDE 范式、值分解（VDN/QMIX）、反事实基线（COMA）、MADDPG，以及 AlphaStar 与 OpenAI Five 的联赛训练，附可运行的 QMIX 混合网络 PyTorch 实现。

#Reinforcement Learning#Multi-Agent#QMIX#MADDPG#Game Theory
Read in English →

单智能体强化学习（RL）有一个隐含但极其重要的假设：环境是静态的，转移核不会因为智能体的学习而改变。然而，一旦出现第二个学习者，这个假设就彻底失效了。每个智能体面对的环境动力学会随着其他智能体的更新而变化，奖励在智能体之间相互交织，联合动作空间更是呈组合爆炸式增长。这些问题不是工程上的小麻烦，而是多智能体强化学习（MARL）需要全新算法的根本原因，而不是简单地“并行跑 n 个 DQN”。

解决这些问题的回报非常可观。AlphaStar 在《星际争霸 II》天梯中达到了宗师段位，OpenAI Five 在《Dota 2》中击败了世界冠军。合作型 MARL 逐渐成为仓储机器人调度、交通信号灯控制以及多 LLM 智能体系统的核心技术。本章会深入讲解概念框架——马尔可夫博弈、CTDE、值分解、反事实信用分配，并最终给出一个简洁的 QMIX 混合网络实现，你可以直接嵌入自己的训练流程中。

你将学到什么

  • MARL 的四大核心难题：非平稳性、信用分配、部分可观测性、可扩展性
  • 必须掌握的博弈论术语：马尔可夫博弈、纳什均衡、Pareto 前沿、社会困境
  • CTDE（集中训练、分散执行）为何成为主流方法
  • 值分解：VDN 和 QMIX，以及确保贪心策略有效的单调性约束
  • 多智能体 Actor-Critic：MADDPG（每个智能体配备一个集中式评论家）和 COMA（反事实基线）
  • 通信机制：广播、稀疏通信和基于注意力的消息传递分别在什么情况下有效
  • AlphaStar 联赛的设计与 OpenAI Five 的大规模分布式训练工程实践

前置知识

  • Q-Learning 和 DQN（第 2 篇 ）
  • Actor-Critic 和策略梯度（第 3 篇 、第 6 篇 ）
  • 熟悉期望和条件概率；博弈论部分只涉及最基础的内容

1. 为什么 MARL 比单智能体 RL 更难

奖励结构几乎决定了所有其他因素。在合作型游戏中，所有智能体共享团队奖励——比如足球配合、群体物流、星际争霸中的微操。竞争型游戏是零和博弈：一个智能体的收益就是另一个的损失，这种场景下自对弈（self-play）是最常用的工具。最有趣的是中间地带——混合动机（一般和）游戏，比如市场、交通、谈判、拥塞控制。短期个人利益往往会破坏长期集体福利，囚徒困境就是经典的 2×2 案例，图中第三个收益矩阵也展示了这一点。

无论在哪种场景下，以下四个障碍都会让单智能体 RL 的工具失效。

非平稳性。从智能体 $i$ 的角度看，状态转移是 $P(s' \mid s, a_i, a_{-i})$，其中 $a_{-i}$ 表示其他智能体的动作。训练过程中，其他智能体的策略 $\pi_{-i}$ 不断更新，导致 $i$ 看到的有效转移核持续变化。像 DQN 这样的离策略方法假设数据生成分布固定，在 MARL 中很容易发散或震荡。

信用分配。一局合作下来，团队只得到一个奖励 $r$，到底是谁的贡献？如果简单地共享梯度，就会鼓励搭便车行为——什么都不做的智能体会获得和努力工作的智能体相同的奖励。COMA 的反事实基线（第 5 节）和值分解（第 4 节）是两种有效的解决方法。

部分可观测性。每个智能体只能看到自己的观测 $o_i$，执行时没有马尔可夫状态可用。常见的做法是用循环网络加上信念状态风格的表示，但更根本的解决方法是设计架构，让集中式评论家在训练时看到比执行时更多的信息。

可扩展性。假设有 $n$ 个智能体，每个智能体有 $|A|$ 个动作，联合动作空间就是 $|A|^n$。如果不利用问题结构，即使是规模适中的合作任务也会变得不可解。

这四个问题的统一解决方案——也是本章的核心概念——是 CTDE。

2. CTDE：训练用全量信息，执行靠局部观察

CTDE 的核心思想是不对称的。训练时，算法可以看到全局状态和每个智能体的动作——比如一个集中式的评论家、混合器或价值函数。但到了部署阶段，只保留基于局部观测的单个智能体策略。执行接口是分散的，优化接口却是集中的。

CTDE 能奏效，是因为让多智能体强化学习（MARL）困难的因素——非稳态、信用分配、部分可观测性——都出现在学习阶段。而让 MARL 实用的限制条件——带宽有限、异步控制、隐私保护——则集中在执行阶段。CTDE 把两类问题分开处理。如今几乎所有主流的合作型 MARL 算法——VDN、QMIX、MADDPG、COMA、MAPPO——都属于 CTDE 方法。它们的区别仅在于集中对象是什么，以及如何分解这个对象。

3. 马尔可夫博弈与纳什均衡：够用就好

马尔可夫博弈把 MDP 扩展到 $n$ 个智能体：

$$\langle \mathcal{N}, \mathcal{S}, \{A_i\}_{i\in\mathcal{N}}, P, \{r_i\}_{i\in\mathcal{N}}, \gamma \rangle.$$

每个智能体 $i$ 都有自己的动作集 $A_i$ 和奖励函数 $r_i(s, a_1, \ldots, a_n)$。纳什均衡是一个联合策略 $\pi^* = (\pi_1^*, \ldots, \pi_n^*)$，没有任何一个智能体能通过单方面改变策略来获得更高回报：

$$V_i(\pi_i^*, \pi_{-i}^*) \;\geq\; V_i(\pi_i, \pi_{-i}^*) \qquad \forall \pi_i,\, \forall i.$$

纳什均衡有两个特点让它变得难以捉摸。第一，它通常不是唯一的；第二，它不一定是 Pareto 最优的。但在完全合作的情况下（$r_1 = \cdots = r_n$），问题变得简单多了——目标退化为最大化团队的整体回报，纳什均衡和 Pareto 最优重合。因此，本章的大多数算法设计都优先针对合作场景。竞争和混合动机的场景则在此基础上引入了自我对弈、对手建模和正则化等额外机制。

4. 值分解：VDN 和 QMIX

值分解是合作场景下的一种 CTDE 方法。执行时，每个智能体用各自的值函数 $Q_i(o_i, a_i)$；训练时，用一个合并的 $Q_\text{tot}(s, \mathbf{a})$。通过约束二者关系，确保在 $Q_i$ 上分散贪心选择的结果等价于在 $Q_\text{tot}$ 上联合贪心选择的结果。这就是 Individual-Global-Max（IGM） 性质：

$$\arg\max_{\mathbf{a}} Q_\text{tot}(s, \mathbf{a}) \;=\; \big(\arg\max_{a_1} Q_1(o_1, a_1),\; \ldots,\; \arg\max_{a_n} Q_n(o_n, a_n)\big).$$

VDN：加性分解

Sunehag 等, 2017 ↗ 采用了最简单的充分条件：

$$Q_\text{tot}(s, \mathbf{a}) \;=\; \sum_{i=1}^{n} Q_i(o_i, a_i).$$

求和天然对每个被加项单调，因此 IGM 自然成立。代价是表达能力有限——VDN 完全无法表示智能体之间的非加性交互。

QMIX：单调混合

QMIX ↗ 保留了 IGM，但将“加性”扩展为“单调”。它使用一个神经混合网络，把 $Q_i$ 合成 $Q_\text{tot}$，并满足以下约束：

$$\frac{\partial Q_\text{tot}}{\partial Q_i} \;\geq\; 0 \qquad \forall i.$$

实现上，通过构造混合器权重为非负来强制满足约束——比如对权重取绝对值或通过 softplus 函数。关键在于，这些权重由一个以全局状态 $s$ 为输入的超网络生成。不同状态下，智能体可以被赋予不同的权重，混合器也能表达纯求和无法捕捉的交互。图右侧展示了由此生成的 $Q_\text{tot}$ 曲面：对每个 $Q_i$ 单调，因此联合 argmax 与各自 argmax 的组合一致。

import torch
import torch.nn as nn

class QMixer(nn.Module):
    """QMIX 混合网络，权重由超网络生成并保持非负。"""

    def __init__(self, n_agents: int, state_dim: int, hidden_dim: int = 32):
        super().__init__()
        self.n_agents = n_agents
        self.hidden_dim = hidden_dim

        # 超网络：state -> 混合权重（取 abs 强制非负）
        self.hyper_w1 = nn.Linear(state_dim, n_agents * hidden_dim)
        self.hyper_b1 = nn.Linear(state_dim, hidden_dim)
        self.hyper_w2 = nn.Linear(state_dim, hidden_dim)
        # 末层标量偏置无符号约束
        self.hyper_b2 = nn.Sequential(
            nn.Linear(state_dim, hidden_dim), nn.ReLU(),
            nn.Linear(hidden_dim, 1),
        )

    def forward(self, agent_qs: torch.Tensor, state: torch.Tensor) -> torch.Tensor:
        """agent_qs: (B, n_agents)   state: (B, state_dim)   ->  (B, 1)"""
        B = agent_qs.size(0)
        agent_qs = agent_qs.view(B, 1, self.n_agents)

        w1 = torch.abs(self.hyper_w1(state)).view(B, self.n_agents, self.hidden_dim)
        b1 = self.hyper_b1(state).view(B, 1, self.hidden_dim)
        hidden = torch.nn.functional.elu(torch.bmm(agent_qs, w1) + b1)

        w2 = torch.abs(self.hyper_w2(state)).view(B, self.hidden_dim, 1)
        b2 = self.hyper_b2(state).view(B, 1, 1)
        q_tot = torch.bmm(hidden, w2) + b2
        return q_tot.view(B, 1)

训练方式与 DQN 完全相同：从回放池采样转移，在 $Q_\text{tot}$ 上构造 TD 目标，再将梯度反传到混合器及每个智能体网络。执行时，每个智能体只运行自己的 $Q_i$，完全不涉及混合器。

一个工程细节：QMIX 无法表示所有满足 IGM 的可分解函数（单调性是充分条件，但不是必要条件）。QTRAN ↗ 和 QPLEX ↗ 进一步拓展了这一边界，但代价是更复杂的损失函数和工程实现。

5. 多智能体 Actor-Critic：MADDPG 与 COMA

MADDPG：每个智能体一个集中式评论家

MADDPG ↗ 是 DDPG 在 CTDE 框架下的自然延伸。每个智能体 $i$ 包含两部分：

  • 执行者 $\mu_i(o_i)$，只依赖自身的局部观测；
  • 评论家 $Q_i(s, a_1, \ldots, a_n)$，基于全局状态和所有智能体的动作。

执行者通过确定性策略梯度更新，利用自己的评论家；评论家则用所有智能体的目标执行者生成 TD 目标：

$$y \;=\; r_i \;+\; \gamma \, Q_i^{\text{tgt}}\!\left(s', \mu_1^{\text{tgt}}(o_1'), \ldots, \mu_n^{\text{tgt}}(o_n')\right).$$

因为评论家把 $a_{-i}$ 也作为输入，环境对评论家来说是稳定的——它的输入已经包含了变化的部分。执行者保持分布式设计，部署时无需改动。MADDPG 适用于合作、竞争以及混合动机场景，回放池只需存储联合轨迹。

COMA：反事实信用分配

COMA ↗ 直接解决了信用分配问题。它使用一个集中式评论家 $Q(s, \mathbf{a})$，并为每个智能体 $i$ 构造了一个反事实基线：固定其他智能体的动作，仅对 $i$ 的动作按其策略求平均：

$$A_i(s, \mathbf{a}) \;=\; Q(s, \mathbf{a}) \;-\; \sum_{a_i'} \pi_i(a_i' \mid o_i) \, Q\!\left(s, (a_i', a_{-i})\right).$$

仔细看这个公式：第一项是实际发生的价值；第二项是在其他人动作不变的情况下，$i$ 如果选择其他动作的期望价值。两者之差就是 $i$ 的边际贡献。在团队奖励共享时，这是策略梯度应该优化的目标——它去掉了 $a_{-i}$ 带来的噪声，同时不引入偏差。图中展示了四动作的情况：只有被选中的动作得到了非零的优势值。

让 COMA 实际可行的关键在于：集中式评论家一次性输出智能体 $i$ 所有候选动作的 $Q$ 值，因此基线计算只需一次策略概率与 $Q$ 向量的内积，完全不需要额外的环境交互。

6. 通信：智能体之间该交流多少？

有时候，完全分散执行太局限了。测试时，智能体确实需要共享信息。设计空间是一个连续的谱。

广播——每个智能体向其他所有智能体发送固定大小的消息。思路简单清晰，但消息量随智能体数量按 $O(n^2)$ 增长，超过几十个智能体就撑不住了。

稀疏 / k-NN——每个智能体只和最近的 $k$ 个邻居通信（可以是空间上的邻居、图中的邻居，或者角色上的邻居）。固定 $k$ 时，复杂度是线性的。CommNet、IC3Net 和 TarMAC 都属于这一类。

注意力——软路由，可学习。每个智能体发出一个查询，其他智能体用键值响应，消息权重通过 softmax 计算。注意力得分的计算复杂度是 $O(n^2)$，但消息可以大幅剪枝（比如 top-$k$ 注意力），而且整个结构端到端可微。这是现代 MARL 的主流选择，也是通往多 LLM 智能体系统的自然桥梁。注意力机制的路由方式可以直接映射到工具使用和智能体编排模式上。

具体选哪种方式要看任务需求。紧密协作的机器人团队，密集通信没问题；几千个微服务或车辆，稀疏通信或者注意力剪枝几乎是唯一可行的设计。

7. MARL 工业级训练：AlphaStar、OpenAI Five 与联赛训练

AlphaStar ↗ （星际争霸 II）和 OpenAI Five ↗ （Dota 2）的成功秘诀，在于把训练种群设计当作核心工程问题来解决。简单自我对弈容易陷入循环——A 学会打败 B，B 又学会克制 A 的新策略，却忘了自己当初怎么赢的 A，如此反复。AlphaStar 联赛明确划分了三类种群：

  • 主智能体——最终要部署的智能体，是优化的核心目标。
  • 主剥削者——专门寻找主智能体的弱点，逼它补齐短板。
  • 联赛剥削者——定期重置，维持策略多样性，避免联赛风格过于单一。

一个保存历史冻结快照的对手池通过优先虚拟自我对弈（PFSP）与当前学习者匹配。PFSP 倾向于选择那些学习者刚好能勉强战胜的对手。这些机制共同作用，有效避免了简单自我对弈中的灾难性遗忘问题。

OpenAI Five 的思路有所不同。每个英雄由独立的 LSTM 策略控制，参数共享；团队协作则交给手工设计的共享奖励函数和海量算力——每天相当于 180 年的人类自我对弈时间。两套系统的共同启示是：在工业规模下，训练课程的重要性不亚于单智能体算法。

这些思想正在走出游戏领域。RLHF（第 12 篇 ）和工具调用 agent 编排，越来越像多种群训练——一群策略、一群评估者，再加上两者之间的优先匹配机制。

8. 多智能体强化学习中的信用分配：一个实例分析

在合作型多智能体强化学习（MARL）中，信用分配才是真正的难点。它要解决的问题是：团队奖励到底是由哪个智能体的动作导致的？这个问题值得深入探讨。举个例子：一个 4 智能体的网格任务，只有当所有 4 个智能体同时站在目标格子上时，团队才会获得 +10 的奖励。如果直接用朴素的独立 Q-learning 训练，你会发现智能体永远在震荡。原因很简单：每个智能体在更新自己的 $Q$ 值时，把其他三个智能体当成环境的一部分。这样一来，“我应该朝目标移动”这个梯度信号就被队友动作的噪声淹没了。

根据复杂度，解决方案可以分为三类：

差分奖励

$$D_i = R(s, a) - R(s, (a_{-i}, c_i)),$$

其中 $c_i$ 是某个默认动作，比如“什么都不做”。每个智能体的有效奖励就是“团队奖励中有多少是因为我的实际选择”，通过基线进行边缘化处理。差分奖励早在深度强化学习出现前 15 年就已被提出（Wolpert & Tumer, 2002）。如果你能快速计算反事实（比如在支持回滚的模拟器中），这种方法依然非常有效。

反事实基线（COMA）

$$A_i(s, a) = Q(s, a) - \sum_{a_i'} \pi_i(a_i' | \tau_i)\, Q(s, (a_{-i}, a_i')),$$

这里对智能体 $i$ 的备选动作进行边缘化，而其他智能体的动作保持不变。它的巧妙之处在于，边缘化的 $Q$ 值使用了集中式 critic——训练时所有智能体的观测数据都可用，因此基线是明确定义的。而在测试时，每个智能体仍然只依赖自己的历史信息行动。

基于势函数的奖励塑形

如果上述两种方法都不可行，那就退而求其次，使用基于势函数的奖励塑形。具体做法是增加一个额外的奖励项 $F(s, s') = \gamma \Phi(s') - \Phi(s)$，其中 $\Phi$ 是手工设计的势函数，比如到目标距离的负值。Ng-Harada-Russell 定理保证了最优策略不会受到影响。在 MARL 中，为每个智能体单独添加这一项，往往能达到与复杂信用分配方法类似的效果，而且实现起来非常简单，30 行代码就能搞定。

一个真实踩过的坑

在一个多机器人仓库拣货任务中，我们最初用 QMIX 训练，奖励是稀疏的“所有包裹都被拣完”。跑了 100 M 环境步后，策略依然不够理想，因为 4 号机器人几乎一直在闲着。它的个体 $Q$ 值几乎没有变化，原因是它的动作很少能单独决定团队的成功或失败。后来我们引入了差分奖励（每个机器人的奖励 = 团队奖励 $-$ 它不动时的团队奖励），策略在 12 M 步内就收敛了。结果证明，那篇复杂的信用分配论文，效果还不如一个 30 行的奖励塑形函数。

9. MARL 中的奖励塑形与课程设计

稀疏团队奖励在 MARL 中很难收敛。与其换算法，不如试试两个工程技巧，效果更好。

按团队规模逐步训练

先用 2 个智能体训练，再加到 3 个，最后扩展到 4 个。$n=2$ 时联合动作空间小，随机探索偶尔能碰到奖励，梯度信号更强。等策略初步成型后，再增加智能体数量——新加入的智能体用已学到的最佳单体策略初始化，然后全队一起继续训练。AlphaStar 用了这种思路的极端版本（基于 league 的群体训练），但你不需要那么复杂的机制，基础方法就够用了。

冻结对手的自我博弈

在竞争型 MARL 中，标准做法是虚拟自我博弈（fictitious self-play）：每次迭代时，当前学习者与过去多个版本的自己混合对战。这样可以避免“今天的策略赢了昨天的，却输给上周的”循环（剪刀石头布陷阱）。OpenAI Five 训练时，80% 的时间对最新 checkpoint，20% 对早期随机快照。那 20% 是保持群体多样性、实现泛化的关键。

超参数敏感性注意事项

MARL 对超参数的敏感性远高于单体 RL。同一个 QMIX 实现，所有参数默认值，在 StarCraft II 微操任务上某个学习率能收敛，但在 multi-particle 环境中，学习率放大 2 倍就发散了。社区的经验是，初始学习率大约取单体 PPO 的一半，并且熵奖励要保持更长时间。因为在 MARL 中，探索是一种“公共品”：一个智能体停止探索，其他智能体都会损失信息。

常见问题

为什么 QMIX 需要单调性约束？

为了让分散贪心执行和集中联合优化保持一致。如果 $\partial Q_\text{tot}/\partial Q_i$ 可能为负，智能体在自己的 $Q_i$ 上贪心反而会降低团队价值。单调性确保每个智能体的 argmax 和联合 argmax 一致。

MARL 系统什么时候会陷入次优均衡？

当最优联合动作需要协调探索时——两个智能体必须同时尝试高风险动作才能看到收益。如果随机探索几乎不可能产生这个联合动作，系统就会收敛到一个更安全但被 Pareto 支配的均衡。解决方法包括联合探索（committed-exploration）、显式通信和对手建模。

样本复杂度如何随智能体数量变化？

对于适用值分解的合作任务，样本复杂度大致随 $n$ 线性增长——这正是 QMIX 的核心意义。如果没有值分解，直接学习 $Q(s, \mathbf{a})$ 在联合动作空间上的复杂度是指数级的。

独立学习者 vs CTDE，CTDE 总是有优势吗？

如果智能体数量很少，或者全局状态确实不可用，独立 Q-learning 也能工作，而且实现起来简单得多。一旦信用分配变得重要，或者团队规模超过 3–4 个智能体，CTDE 方法的优势就显现出来，并且差距会迅速拉大。

这些思想能迁移到多 LLM agent 系统吗？

可以。集中评论家对应外层评估器，分散执行者对应子 agent，注意力通信对应工具路由，联赛训练对应"评估者种群"模式——后者已经成为 agentic RLHF 中越来越主流的设计。
