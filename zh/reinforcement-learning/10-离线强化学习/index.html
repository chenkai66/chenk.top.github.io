强化学习（十）：离线强化学习

离线强化学习从固定数据集学习策略，无需任何在线交互。本文系统讲解分布偏移、外推误差，以及 CQL、BCQ、IQL、Decision Transformer 四类主流方法，配有完整的 CQL PyTorch 实现与 D4RL 基准对比。

到目前为止，我研究过的所有算法都有一个相同的核心循环：行动、观察、更新。这个循环让强化学习能够运转，但也是阻碍它落地的主要原因。自动驾驶系统不可能通过撞车来练习路口通行。临床决策支持模型不能在真实病人身上测试随机策略。工厂里的机械臂也没法在生产线上尝试上万次抓取。

不过，这些场景中有一个共同点——数据日志非常丰富：数百万小时的人类驾驶记录、几十年的匿名患者病历、TB 级的行为克隆数据。离线强化学习（Offline RL，也叫 Batch RL）要解决的问题是：能否在完全不与环境交互的情况下，从固定的数据集中提取出一个高效的策略？

答案是“可以，但必须非常谨慎”。之所以要小心，是因为本文讨论的核心问题：生成数据的行为策略和想要优化的学习策略之间存在分布偏移。

你将学到什么

  • 直接把标准 off-policy 算法用到离线场景为什么会失败：外推误差、价值高估，还有“死亡螺旋”。
  • CQL（Conservative Q-Learning）：通过悲观正则化，把 Q 值限制在真实价值的下界。
  • BCQ（Batch-Constrained Q-Learning）：用生成模型提出动作，确保策略始终在数据流形内。
  • IQL（Implicit Q-Learning）：用分位数回归近似 max，完全避免查询 OOD 动作。
  • Decision Transformer：把 RL 转换为基于目标回报的条件序列建模问题。
  • D4RL 基准上的具体表现，帮你快速判断该优先选择哪种算法。
  • 一份完整的、可直接运行的 CQL PyTorch 实现。

前置知识

  • 掌握 Q-learning、目标网络和 Actor-Critic（第 2 部分 到 第 6 部分 ）。
  • 熟悉 Bellman 最优算子和重要性采样。
  • 熟练使用 PyTorch 和 Gym/Gymnasium API。

1. 离线 RL 到底难在哪

在线 RL 中，策略和数据分布是绑定的。策略一旦高估某个动作，下一轮采样就会暴露问题，Bellman 更新会立刻修正它。但离线 RL 不一样，数据集是固定的，没有第二次机会。模型对未见过的动作产生的任何错误都会永远留在 Q 函数里，而 argmax 操作会毫不犹豫地利用这些错误。

1.1 分布偏移

假设行为策略 $\pi_\beta$ 生成了数据集 $\mathcal{D}=\{(s_i,a_i,r_i,s_i')\}_{i=1}^N$，我要学习的策略是 $\pi_\theta$。两者的状态-动作分布通常不同：

$$d_{\pi_\theta}(s,a)\neq d_{\pi_\beta}(s,a).$$

当 $\pi_\theta$ 想要选择一个 $\pi_\beta(a\mid s)\approx 0$ 的动作时，问题就来了。Q 网络从未见过这个动作的目标值，它的输出完全依赖神经网络的外推能力——而这种外推往往不靠谱。

1.2 外推误差与死亡螺旋

$$Q(s,a)\leftarrow r + \gamma\,\max_{a'}Q(s',a').$$

max 操作专门挑最乐观的外推值。如果 Q 函数对某个 OOD 动作高估了 $\epsilon$，这个偏差会成为前一时刻的 bootstrap 目标，再往前传播一步，依此类推。在标准基准测试中，这种发散通常在几千步梯度更新内就会显现（Fujimoto 等, 2019 ↗ ）。上图右侧的画面很关键：绿色曲线是真实情况，红色曲线是网络的预测，策略会沿着 OOD 动作的虚假峰值一路走向悬崖。

下面介绍的三类算法都在解决这个问题，区别在于它们约束的对象不同：

类别核心思想约束对象
策略约束（BCQ、BEAR、TD3+BC）“只模仿数据中的动作”$\pi_\theta$
价值悲观（CQL、MOPO）“对陌生的 Q 值保持怀疑”$Q$
样本内学习（IQL、AWAC）“避免查询 OOD 动作”损失函数本身
序列建模（Decision Transformer 等）“彻底绕开 Bellman 方程”问题的定义方式

2. CQL：保守 Q-Learning

CQL（Kumar 等, 2020 ↗ ）是目前离线强化学习中最常用的基线方法。它不改动网络结构、actor 或数据流水线，只在损失函数中增加了一项。

2.1 核心思想一句话概括

先把策略可能选择的动作的 Q 值压低，再把数据中实际出现的动作的 Q 值拉回来。 最终效果是一个在缺乏证据的地方表现悲观的 Q 函数。

2.2 目标函数

$$ \mathcal{L}_{\mathrm{CQL}} \;=\; \alpha\,\Big[\,\underbrace{\log\sum_{a}\exp Q(s,a)}_{\text{压低所有动作}} \;-\; \underbrace{\mathbb{E}_{a\sim\mathcal{D}}\big[Q(s,a)\big]}_{\text{拉回数据动作}}\Big] \;+\; \mathcal{L}_{\mathrm{TD}}. $$

logsumexp 是对动作的“软最大值”。最小化它会把所有动作的 Q 值往下拉。减去数据分布下的期望值后，数据中的动作会被重新抬高，而 OOD 动作只会感受到向下的压力。

$$ \hat{Q}^{\pi}(s,a)\;\leq\;Q^{\pi}(s,a)\quad\forall (s,a). $$

这正是我们需要的：一个真实价值的下界。任何最大化这个下界的策略都不会选择灾难性动作，因为这种动作对它来说没有任何吸引力。

左图展示了普通 SAC critic 在离线数据上的表现：argmax 直接跑到了 OOD 的虚假高峰上。右图是 CQL 的修正结果：橙色阴影表示悲观惩罚，新的 argmax 安稳地落在数据支撑集内。

2.3 实现细节

  • 连续动作空间中无法精确计算 logsumexp，通常用 n_random 个均匀采样动作和 n_actor 个当前策略采样动作来近似，10～20 个样本就足够。
  • 原论文中的 “CQL($\mathcal{H}$)” 变体引入了一个拉格朗日项，自动调整 $\alpha$ 以达到目标差值。这是 d3rlpy 和 JaxRL 默认实现的版本。
  • 实验表明，CQL 在 D4RL 的 medium-replay 和 medium 数据集上表现稳健，但在 medium-expert 上略显保守，此时 IQL 或 DT 通常表现更好。

import torch
import torch.nn as nn
import torch.nn.functional as F

class QNetwork(nn.Module):
    """孪生 Q 网络（Clipped Double Q），用于抑制高估。"""

    def __init__(self, state_dim, action_dim, hidden=256):
        super().__init__()
        self.q1 = self._mlp(state_dim + action_dim, hidden)
        self.q2 = self._mlp(state_dim + action_dim, hidden)

    @staticmethod
    def _mlp(in_dim, hidden):
        return nn.Sequential(
            nn.Linear(in_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, 1),
        )

    def forward(self, state, action):
        sa = torch.cat([state, action], dim=1)
        return self.q1(sa), self.q2(sa)

class GaussianPolicy(nn.Module):
    """SAC 风格的 tanh-squashed 高斯策略。"""

    def __init__(self, state_dim, action_dim, hidden=256):
        super().__init__()
        self.trunk = nn.Sequential(
            nn.Linear(state_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
        )
        self.mean = nn.Linear(hidden, action_dim)
        self.log_std = nn.Linear(hidden, action_dim)

    def sample(self, state):
        x = self.trunk(state)
        mean = self.mean(x)
        log_std = self.log_std(x).clamp(-20, 2)
        std = log_std.exp()
        normal = torch.distributions.Normal(mean, std)
        z = normal.rsample()
        action = torch.tanh(z)
        # tanh 变换的雅可比项
        log_prob = (normal.log_prob(z)
                    - torch.log(1 - action.pow(2) + 1e-6)).sum(1, keepdim=True)
        return action, log_prob

class CQLAgent:
    """在 SAC 基础上添加 CQL 正则项。cql_weight 对应公式中的 alpha。"""

    def __init__(self, state_dim, action_dim, lr=3e-4,
                 gamma=0.99, tau=0.005, alpha=0.2,
                 cql_weight=1.0, n_random=10):
        self.gamma, self.tau = gamma, tau
        self.cql_weight, self.n_random = cql_weight, n_random
        self.action_dim = action_dim

        self.q_net = QNetwork(state_dim, action_dim)
        self.target_q = QNetwork(state_dim, action_dim)
        self.target_q.load_state_dict(self.q_net.state_dict())
        self.policy = GaussianPolicy(state_dim, action_dim)

        self.q_opt = torch.optim.Adam(self.q_net.parameters(), lr=lr)
        self.pi_opt = torch.optim.Adam(self.policy.parameters(), lr=lr)
        self.alpha = alpha

    # ---------- 与普通 SAC 唯一不同的地方 ----------
    def _cql_penalty(self, states, q1_data, q2_data):
        """对动作做 logsumexp，再减去数据上的 Q 平均值。"""
        b = states.size(0)
        # (1) [-1, 1] 上的均匀随机动作
        rand_a = torch.empty(b * self.n_random, self.action_dim).uniform_(-1, 1)
        # (2) 当前策略在状态 s 下采样的动作
        rep_s = states.repeat_interleave(self.n_random, 0)
        pi_a, pi_lp = self.policy.sample(rep_s)

        def _q(s, a):
            return self.q_net(s, a)

        q1_rand, q2_rand = _q(rep_s, rand_a)
        q1_pi,   q2_pi   = _q(rep_s, pi_a)

        # 重要性加权 logsumexp（Kumar 等, Eq. 4）
        log_pi_uniform = -float(self.action_dim) * torch.log(torch.tensor(2.0))
        cat1 = torch.cat([q1_rand - log_pi_uniform,
                          q1_pi   - pi_lp.detach()], 0).view(b, -1)
        cat2 = torch.cat([q2_rand - log_pi_uniform,
                          q2_pi   - pi_lp.detach()], 0).view(b, -1)

        lse1 = torch.logsumexp(cat1, 1, keepdim=True)
        lse2 = torch.logsumexp(cat2, 1, keepdim=True)
        return ((lse1 - q1_data) + (lse2 - q2_data)).mean()

    def update(self, states, actions, rewards, next_states, dones):
        # 1. Bellman 目标（clipped double Q + 熵奖励）
        with torch.no_grad():
            next_a, next_lp = self.policy.sample(next_states)
            tq1, tq2 = self.target_q(next_states, next_a)
            target_q = torch.min(tq1, tq2) - self.alpha * next_lp
            target = rewards + self.gamma * (1 - dones) * target_q

        q1, q2 = self.q_net(states, actions)
        td_loss = F.mse_loss(q1, target) + F.mse_loss(q2, target)
        cql = self._cql_penalty(states, q1, q2)
        q_loss = td_loss + self.cql_weight * cql

        self.q_opt.zero_grad()
        q_loss.backward()
        self.q_opt.step()

        # 2. Actor 更新（标准 SAC）
        a, lp = self.policy.sample(states)
        q1_a, q2_a = self.q_net(states, a)
        pi_loss = (self.alpha * lp - torch.min(q1_a, q2_a)).mean()
        self.pi_opt.zero_grad()
        pi_loss.backward()
        self.pi_opt.step()

        # 3. 目标 Q 的 Polyak 平均
        with torch.no_grad():
            for p, tp in zip(self.q_net.parameters(),
                             self.target_q.parameters()):
                tp.data.mul_(1 - self.tau).add_(self.tau * p.data)

        return {"td": td_loss.item(),
                "cql": cql.item(),
                "pi":  pi_loss.item()}

3. BCQ：生成动作候选的策略约束方法

CQL 约束的是价值函数，而 BCQ（Fujimoto 等, 2019 ↗ ）直接对策略本身进行约束。它的核心思想很简单：永远不要让 Q 函数去评估行为策略不可能产生的动作。

整个架构分为三个部分：

  1. 条件 VAE $G_\omega(s)$：在数据集 $\mathcal{D}$ 上通过标准 ELBO 训练，建模行为策略 $\pi_\beta(a\mid s)$。从这个模型采样的动作天然位于行为流形上。
  2. 扰动网络 $\xi_\phi(s,a)\in[-\Phi,\Phi]^{\dim(a)}$：对每个 VAE 采样结果施加一个小幅、有界的修正。$\Phi$ 很小（通常为 0.05），确保修正后的动作不会偏离数据分布；当 $\Phi=0$ 时，BCQ 就退化为加权模仿学习。
  3. 孪生 Q 网络：对 $N$ 个修正后的候选动作打分，最终通过 argmax 选出执行的动作。

最巧妙的地方在于，扰动网络的目标是最大化 Q 值。这样一来，BCQ 既能享受策略改进的好处（不只是简单模仿），又避免了脱离数据分布的风险（动作不会跑出行为流形）。代价是整体流程更复杂——需要四个网络协同工作，而且 VAE 必须训练得足够好，否则候选动作本身就可能带偏整个系统。

4. IQL：彻底避开 Bootstrap 问题

IQL（Kostrikov 等, 2022 ↗ ）是这三种方法中最优雅的一个。它的核心观察很简单：离线强化学习的所有问题都来自 Bellman 目标中的 $\max_{a'}Q(s',a')$，因为 OOD 动作就是从这里混进来的。那干脆不用它。

$$\mathcal{L}_V \;=\; \mathbb{E}_{(s,a)\sim\mathcal{D}}\big[L_2^{\tau}\big(Q(s,a)-V(s)\big)\big],\qquad L_2^{\tau}(u)=\big|\tau-\mathbb{1}(u<0)\big|\,u^{2}.$$

当 $\tau=0.5$ 时，这就是普通的均方误差（MSE），$V$ 学到的是均值。当 $\tau\to 1$ 时，损失函数对低估的惩罚远大于高估，因此 $V$ 收敛到 $Q(s,a)$ 在数据中动作分布上的上分位数。

$$y \;=\; r + \gamma\,V(s'),$$$$\mathcal{L}_\pi \;=\; -\mathbb{E}_{(s,a)\sim\mathcal{D}}\big[\exp\big(\beta\,(Q(s,a)-V(s))\big)\,\log\pi_\theta(a\mid s)\big].$$

IQL 是现代离线强化学习算法中设计最简洁的一个，在 D4RL 的 AntMaze 和 Adroit 环境中表现尤为突出。这些环境中高质量数据稀疏，使用 max 进行引导最容易出问题，而 IQL 完全避开了这一点。

5. Decision Transformer：将强化学习视为序列建模

既然已经决定抛弃 Bellman 方程，何必止步于 IQL？Decision Transformer（Chen 等, 2021 ↗ ）干脆连价值函数都不要了，直接把强化学习问题转化为下一个 token 预测。

一条轨迹被展开为 $(\hat{R}_1, s_1, a_1, \hat{R}_2, s_2, a_2, \ldots)$ 的序列形式，其中 $\hat{R}_t = \sum_{t'\geq t} r_{t'}$ 表示从时间步 $t$ 开始的剩余回报（return-to-go）。一个标准的因果 Transformer（类似 GPT 风格）通过交叉熵或 MSE 训练，目标是根据前面的所有 token 预测 $a_t$。

测试时，我把期望的剩余回报设为 $\hat{R}_1$，然后让模型自回归生成动作。$\hat{R}_1$ 就像一个调节旋钮：想要更激进的策略，就给更高的目标回报。

这种方法的优势：

  • 不依赖 bootstrap，因此没有外推误差。
  • 长上下文让模型自然具备基于部分观测历史条件的能力，无需额外设计。
  • 直接享受 Transformer 的所有工程优化成果（LayerNorm、RoPE、FlashAttention、混合精度等）。
  • 一套方法通吃 D4RL、Atari 和大规模多任务场景（比如 Gato）。

代价：

  • 模型无法超越数据中见过的最佳回报。如果 $\mathcal{D}$ 中没有任何轨迹达到过 90 分，要求模型输出 90 分只会得到垃圾结果。
  • 没有显式的因果信用分配——它学的是"在特定回报条件下模仿轨迹"，而不是分析哪个动作导致了结果。
  • 在小规模基准测试中，要追平 CQL/IQL 的分数，通常需要更大的模型和更多的数据。

6. 数据说话：D4RL 基准测试

这张图汇总了 CQL（Kumar 等, 2020 ↗ ）、IQL（Kostrikov 等, 2022 ↗ ）和 DT（Chen 等, 2021 ↗ ）论文中，在 D4RL ↗ MuJoCo 运动控制套件（hopper、halfcheetah、walker2d 取平均）上的归一化分数。100 表示专家水平，0 表示随机策略。

三个关键点：

  1. medium-replay 最需要保守性。 这种数据来自 SAC 训练早期的检查点——动作质量差，但包含大量从错误中恢复的转移。CQL 和 IQL 的得分大约是 BC 的两倍；DT 因为缺乏价值引导，无法拼接出高质量的子轨迹，所以表现较差。
  2. medium-expert 是 DT 的强项。 当数据中已经包含接近最优的轨迹时，序列建模的简洁性让它占优。
  3. IQL 最稳定。 它很少在某个数据集上拔得头筹，但也几乎不会跌出前三，而且训练过程在四种方法中最平稳。

如果不确定选哪个：优先用 IQL（最稳）或 CQL（最简单）；当数据接近专家水平时，试试 DT。

7. 常见问题

Q: 离线 RL 在什么情况下会完全失效？
三种典型情况：(i) 数据覆盖不足——只有专家轨迹，没有错误恢复的样例，策略学不会应对自己的失误；(ii) 数据质量太差——长时任务中随机策略生成的数据；(iii) 测试环境变化——评估 MDP 的转移动力学和生成 $\mathcal{D}$ 的环境不一致。

Q: CQL 的 $\alpha$ 参数怎么调？
大致范围是：专家数据用 $\alpha=0.5$-$1.0$，中等质量数据用 $1.0$-$5.0$，随机或 replay 数据用 $5.0$-$10.0$。更好的办法是用原论文里的拉格朗日变体，它会自动调整 $\alpha$，让 $\mathbb{E}_{a\sim\pi}Q - \mathbb{E}_{a\sim\mathcal{D}}Q$ 接近目标值（通常设为 5-10）。

Q: 离线 RL 和模仿学习到底有什么区别？
行为克隆（BC）忽略奖励信号，只能逼近示范者，无法超越它。离线 RL 利用奖励信号，可以拼接不同轨迹中的优质片段，生成一个比 $\mathcal{D}$ 中任何单条轨迹都强的策略。经典例子是：A 轨迹笨拙地到达状态 $s$，但后续表现优秀；B 轨迹高效到达 $s$，但后续表现糟糕。离线 RL 能学到“B 的开头 + A 的结尾”，而 BC 永远做不到。

Q: 是不是该直接用基于模型的离线方法？
在高维连续控制任务中，MOPO、MOReL 和 COMBO 表现不错，但它们需要额外学习一个动力学模型，并引入悲观项。这些方法在数据量少时有优势（模型提供归纳偏置），但工程复杂度更高。2025 年，模型无关的 CQL 和 IQL 仍然是首选起点。

Q: 离线到在线微调怎么做？
过去两年，IQL 在这个方向上表现突出。像 AWAC、Cal-QL 和 RLPD 这些方法，先用离线策略初始化，再用少量在线数据继续训练。CQL 初始化往往过于保守，不适合微调；相比之下，IQL 或 AWAC 初始化效果更好。
