工程师 · 写作者

陈锴。工程师，写作者。以数学和机器学习为骨，这几年深耕于云计算与 Agent 系统的长跑之中。于无界处收边，是我学会的生存法则。

若问所为何事，不过是几线并行：在 AI4Marketing 中，我只与控制器对话，余下的创意与分发交由子 Agent 各司其职；在 AI4Science 中，搭建控制平面，维持一套自动化科研系统的昼夜运转。横贯其间的是 llm-elevator——我反复打磨的性能增强层：prompt 模板、Agent harness、对抗式评测，把同一颗 Frontier 模型在真实任务里的完成率往上抬一档，让上游的每一个项目都吃到这份红利。中间夹杂着记忆系统与技能进化的驯化，以及这个站点的日拱一卒。

万象归一，我关心的核心始终未变：长跑系统如何在出错、迁移、降本与换模型的扰动中，保持韧性。落地即是拆解：令牌预算的分配、失败经验的压缩复用、共享记忆的类型收敛，以及从演示跑通到业务压测之间那道必须跨越的鸿沟。

写作亦是如此。chenk.top 上的四百多篇双语长文，不是翻译，而是重塑。中文练收，英文练放，同一件事，在不同的语境与节奏中重生。

岁月渐深，愈发笃定几件事：工具易逝，判断力长青；文档重于代码；过早抽象是工程师最昂贵的本能。一个能稳跑三十天的系统，远比只能演示三十分钟的 Demo 珍贵。不把 Agent 当作会说话的 Prompt，而是视作有成本、会故障的真实系统——这是我所有项目的底色。

若你也在路上，欢迎来信。二十四小时内，静候回音。
