此刻

最近这阵子在做什么

01

AI4Marketing — 一句话变一场跨境营销

热点追踪 → AI 文案 → 配图生成 → 短视频（口播 / 解说 / 短剧）→ TTS 配音 → 多平台定时发布，全流程自动化。还有内容原子化拆解、多视角 AI 辩论选稿、故事板编排、内容日历、模板复用与数据看板，集成 11 款阿里云产品。

02

AI4Science — 不停歇的科研 Agent 循环

丢一个科研问题进去，Agent 自己读论文、抽方法、跑实验、写报告，整套循环以天为单位连续运转；共享记忆持久化每一轮 session，harness 把失败的尝试压成下一轮的起点。

03

MiniGameForge — 30 分钟把想法变成可上架小游戏

选模板、选风格、点生成，AI 全自动产出代码、美术、过场视频。底层是 elevator agent 三段式（planner → executor → verifier）+ 7 类异常护栏 + 视觉自调试（看运行画面发现错误自己修），6 家 17 把 key 自适应路由。已上线 llm4marketing.asia。

04

DaaS — 把整家公司压成一个 Skill

给它一个产品文档目录，它读完所有内容，自动写两类 Skill：detail skill 管"每个能力怎么做"，intent skill 管"同一个目标该走哪条路"。一行 Skill 都不用人来写。

05

chenk.top — 中英双语长文

你现在看的这块地方。中英两版各自从零起笔——非翻译，中文重凝练，英文重展开。截至目前 30+ 主题系列、700+ 篇深稿、1500+ 张原创图表，全部由我一个人写、配图、审校。

一个反复回到的问题

长周期系统如何在出错、模型切换、成本压缩与环境迁移的扰动下保持韧性？

落地路径很具体：跨 Provider 的动态预算分配、把失败压缩成可复用 Skill、共享记忆的类型收敛、从 demo 到上线之间可观测、可回放、可降级的真实交付鸿沟。

信奉

写代码时遵循的几条朴素准则

  1. i

    工具会过时，判断力不会。

  2. ii

    文档比代码更值得投入时间。

  3. iii

    过早抽象，是工程师最昂贵的本能。

  4. iv

    稳跑三十天的小系统，胜过演示三十分钟的炫 Demo。

  5. v

    把 Agent 当真实系统对待——有显性成本、明确故障域、需运维——而不是"会说话的 Prompt"。
