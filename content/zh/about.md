陈锴。工程师，写作者。以数学与机器学习为根基，近年聚焦于云计算与长周期 Agent 系统的工程落地。

当前主线项目：
- **AI4Marketing**：全栈 AI 营销平台，支持从单句指令到跨渠道营销活动（邮件、广告创意、短视频、TTS）的端到端自动生成。我负责控制平面设计与子 Agent 协同架构，让控制器成为唯一人机接口。
- **AI4Science**：自主科研 Agent 系统，可接收科学问题、检索论文、规划实验、执行代码、生成结构化报告。系统按天级持续运行，依赖共享记忆架构与基于 harness 的技能进化机制。
- **llm-elevator**：内部使用的性能增强层（不对外），包含 prompt 模板引擎、Agent 运行时 harness、对抗式评测框架。它不改变模型本身，而是提升同一颗前沿模型在真实业务任务中的成功率与稳定性——所有上游项目均直接受益于此。
- **DaaS（文档即 Skill）**：将非结构化技术文档自动转化为可调用、可验证、带上下文感知的 Agent 技能模块。
- **chenk.top**：个人技术博客，已发布 249 篇中文 + 249 篇英文原创长文（共 498 篇），覆盖 21 个系列。每篇均为双语独立创作，非翻译；中文重凝练，英文重展开。

我始终关注一个核心问题：**长周期系统如何在出错、模型切换、成本压缩与环境迁移的扰动下保持韧性？** 落地路径很具体：令牌预算的动态分配、失败经验的可复用压缩、共享记忆的类型收敛、从 demo 到压测之间可观测、可回放、可降级的真实交付鸿沟。

信奉几条朴素准则：
- 工具会过时，判断力不会；
- 文档比代码更值得投入时间；
- 过早抽象是工程师最昂贵的本能；
- 能稳跑三十天的系统，远比只能演示三十分钟的 Demo 有价值；
- 不把 Agent 当作"会说话的 Prompt"，而视作有显性成本、明确故障域、需运维的真实系统。

若你也在构建长期可用的智能系统，欢迎来信。二十四小时内必复。
