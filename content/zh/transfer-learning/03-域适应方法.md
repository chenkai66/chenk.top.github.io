我的自动驾驶模型在加州晴天的高速公路上表现得堪称完美。但一到西雅图，天开始下雨，Top-1 准确率直接从 95% 掉到 70%。模型本身没出问题——是数据分布变了。训练集里根本没有教它傍晚湿沥青是什么样子。

这就是域适应要解决的典型问题：源域有大量标注数据，目标域只有未标注数据，而模型必须在目标域上表现出色。这篇文章从理论基础讲起，最后给出一个能跑通的 DANN 实现。

## 1. 分布偏移的三种类型

**域（domain）** 是特征空间 $\mathcal{X}$ 和边缘分布 $P(X)$ 的组合；**任务（task）** 是标签空间 $\mathcal{Y}$ 和条件分布 $P(Y \mid X)$ 的组合。域适应研究的是源域和目标域在这些方面不一致时该怎么处理。

| 设置 | 源域 $\mathcal{D}_S$ | 目标域 $\mathcal{D}_T$ | 目标 |
|---|---|---|---|
| 数据 | 大量带标注 $(x_i, y_i)$ | 多数无标注 $x_j$ | 学习一个在 $\mathcal{D}_T$ 上有效的函数 $f: \mathcal{X} \to \mathcal{Y}$ |

![源域与目标域的特征对齐](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/03-域适应方法/fig1_distribution_shift.png)

这张图说明了一切：适应之前，源域训练出的边界穿过目标域几乎没有数据的空白区域；适应之后，两个域共享同一个特征流形，同一条边界对两边都适用。

### 1.1 协变量偏移——输入分布发生了变化

$$P_S(X) \neq P_T(X), \qquad P_S(Y \mid X) = P_T(Y \mid X)$$
标注规则没变，只是观测到的数据不同了：

- 用 2020 年邮件训练的垃圾邮件分类器在 2026 年部署：主题变了，但“什么是垃圾邮件”的判断标准没变。
- 西门子 CT 扫描仪训练的模型在 GE 扫描仪上推断：成像特性不同，但放射科医生的诊断标准是一样的。

**常用方法——重要性加权。** 给每个源样本乘上密度比 $w(x) = P_T(x) / P_S(x)$，加权后的源域 ERM 就能估计**目标域**的风险：
$$\mathbb{E}_{P_T}[\ell(f(X), Y)] = \mathbb{E}_{P_S}\!\left[\frac{P_T(X)}{P_S(X)}\,\ell(f(X), Y)\right].$$
高维下直接估计密度很难，所以实践中直接估计**比值**——可以用 KLIEP、uLSIF，或者训练一个分类器区分源域和目标域，贝叶斯最优分类器的 logit 等价于这个比值。

### 1.2 标签偏移——类别先验发生了变化
$$P_S(Y) \neq P_T(Y), \qquad P_S(X \mid Y) = P_T(X \mid Y)$$
类别条件下的样本分布没变，只是类别比例不同了：

- ICU 病人数据训练的模型部署到门诊：疾病流行率差异很大。
- 在年轻用户中训练的推荐模型推广到全年龄段用户。

**常用方法。** 用 EM 算法（BBSE / RLLS）在无标注目标数据上估计 $P_T(Y)$，然后按 $P_T(y) / P_S(y)$ 对源域模型的输出概率重新加权并归一化。

### 1.3 概念偏移——规则本身发生了变化
$$P_S(Y \mid X) \neq P_T(Y \mid X)$$
这是最难的一种情况。“sick”在音乐评论里是褒义（“这段 beat 太 sick 了”），在产品评论里是贬义——同一个词，不同的标签。如果没有目标域标签，任何方法都无法解决这个问题。概念偏移至少需要少量目标域标注数据（半监督 DA 场景）。
## 2. 理论：Ben-David 上界

为什么适配是可行的？经典答案来自 Ben-David 等人 (2010) 提出的上界。对于假设类 $\mathcal{H}$ 中的任意假设 $h$：
$$
\epsilon_T(h) \;\leq\; \epsilon_S(h) \;+\; \tfrac{1}{2}\, d_{\mathcal{H}\Delta\mathcal{H}}(\mathcal{D}_S, \mathcal{D}_T) \;+\; \lambda^{*}.
$$
| 项 | 含义 | 我能做什么 |
|---|---|---|
| $\epsilon_S(h)$ | 源域误差 | 在源域上训练得更好 |
| $d_{\mathcal{H}\Delta\mathcal{H}}$ | 域间对称差散度 | **这是域适应要降低的部分** |
| $\lambda^{*}$ | 最优联合预测器的误差 | 不可减少——如果它很大，任何方法都无能为力 |

两个关键点：

1. **适配受一个理论上限约束。** 如果源域和目标域任务本质上差异很大（$\lambda^*$ 很大），再复杂的损失函数也救不了你。你需要的是新标注数据，而不是更花哨的方法。
2. **域散度有一个可操作的代理指标。** 训练一个二分类器来区分源域和目标域特征。如果它的准确率接近 50%，说明你的特征已经实现了域不变性。这正是 DANN 自动完成的工作机制。

## 3. DANN：一次反向传播完成的对抗对齐

**域对抗神经网络**（Ganin 等，2016）是最具影响力的对抗方法，也是“最小化域散度代理”思想最简洁的实现。

![DANN 架构与梯度反转层](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/03-域适应方法/fig2_dann_architecture.png)

### 3.1 三个子网，一个共享主干

| 子网 | 功能 | 训练数据 |
|---|---|---|
| **特征提取器** $G_f$ | 将输入 $x$ 映射为特征 $f = G_f(x)$ | 源域 + 目标域 |
| **标签预测器** $G_y$ | 根据特征 $f$ 预测标签 $\hat{y}$ | 源域标签 |
| **域判别器** $G_d$ | 判断特征 $f$ 来自源域还是目标域 | 源域 + 目标域 |

目标函数是一个 minimax 问题：
$$
\min_{G_f,\, G_y}\; \max_{G_d}\quad \mathcal{L}_y(G_y \circ G_f) \;-\; \lambda\, \mathcal{L}_d(G_d \circ G_f).
$$
$G_d$ 的任务是区分两个域，而 $G_f$ 则要欺骗 $G_d$，同时确保 $G_y$ 能正确分类源域数据。

### 3.2 梯度反转层（GRL）

直接优化 minimax 问题需要交替训练，这种方法既脆弱又难调（想想早期的 GAN）。DANN 的核心贡献是通过**梯度反转层**（GRL），让整个系统在**一次反向传播**中完成训练：
$$
\text{前向传播：}\; \text{GRL}(x) = x, \qquad
\text{反向传播：}\; \frac{\partial\,\text{GRL}}{\partial x} = -\lambda\, I.
$$
GRL 插在特征到域判别器的路径上。在反向传播时，判别器的梯度在传递到 $G_f$ 之前会翻转符号。因此，同一次 `loss.backward()` 调用可以：

- 用正常梯度更新 $G_y$，提升分类性能；
- 用正常梯度更新 $G_d$，增强判别能力；
- 用翻转后的梯度更新 $G_f$，让 $G_d$ 更难区分域，同时仍帮助 $G_y$ 提升分类效果。

不需要交替训练，也不需要多个优化器或手动冻结参数。

### 3.3 对抗权重调度

DANN 并不会一开始就将 $\lambda$ 设为最大值——那样会破坏早期学习。它采用了一条 sigmoid 曲线来逐步增加 $\lambda$：
$$\lambda_p = \frac{2}{1 + \exp(-\gamma p)} - 1, \qquad \gamma \approx 10,$$
其中 $p \in [0, 1]$ 表示训练进度。在训练初期（$\lambda \approx 0$），模型专注于学习高质量的源域特征；随着训练推进（$\lambda \to 1$），域对齐逐渐发挥作用。忽略这个调度策略，是导致“DANN 训练成功但表现不如仅用源域模型”的最常见原因，没有之一。
## 4. MMD：在 RKHS 中对齐均值

对抗式方法虽然强大，但训练过程不稳定。非对抗式的替代方案是直接定义分布之间的显式距离，并最小化它。**最大均值差异**（Gretton 等，2012）是这一领域的标准选择。

![最大均值差异：核均值嵌入](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/03-域适应方法/fig3_mmd_kernel.png)

### 4.1 核心思想

核函数 $k(x, y) = \langle \phi(x), \phi(y) \rangle_{\mathcal{H}}$ 将每个样本隐式映射到一个（可能是无限维的）再生核希尔伯特空间 $\mathcal{H}$。分布 $P$ 的**核均值嵌入**是其特征的平均值：
$$\mu_P = \mathbb{E}_{X \sim P}[\phi(X)] \;\in\; \mathcal{H}.$$
对于**特征核**（高斯 RBF 是典型例子），映射 $P \mapsto \mu_P$ 是单射的——两个分布相等当且仅当它们的核均值相等。因此，可以通过核均值在 RKHS 中的距离来衡量两个分布的差异：
$$\text{MMD}^2(P_S, P_T) = \|\mu_{P_S} - \mu_{P_T}\|_{\mathcal{H}}^2.$$
图示清晰地展示了这一点：即使原始直方图有部分重叠，核均值嵌入也能明确显示差距，阴影部分正好对应 $\text{MMD}^2$。

### 4.2 实际计算的估计量

由于嵌入是隐式的，展开平方范数后，内积就变成了核函数的评估：
$$
\widehat{\text{MMD}}^2 = \frac{1}{n_s^2}\sum_{i,j} k(x_i^s, x_j^s) + \frac{1}{n_t^2}\sum_{i,j} k(x_i^t, x_j^t) - \frac{2}{n_s n_t}\sum_{i,j} k(x_i^s, x_j^t).
$$
这个公式对特征可微，可以直接作为额外损失项加入深度网络：
$$\mathcal{L} = \mathcal{L}_{\text{task}} + \lambda \cdot \widehat{\text{MMD}}^2\!\big(G_f(X_S),\, G_f(X_T)\big).$$
这就是 **DAN / DDC**（Long 等，2015；Tzeng 等，2014）。

### 4.3 实用技巧

- **使用多核 MMD。** 多个不同带宽的高斯核线性组合 $k = \sum_u \beta_u k_{\sigma_u}$ 对带宽误设更鲁棒。
- **带宽选择用中位数启发法。** 取一个 batch 内成对距离的中位数——简单、鲁棒，绝大多数情况下足够好。
- **对深层特征应用 MMD。** 底层特征包含域特定纹理；需要对齐的是顶层的抽象表示。

### 4.4 MMD 与 DANN 快速对比

| | MMD | DANN |
|---|---|---|
| 距离 | 核 RKHS 范数 | Jensen–Shannon（通过判别器） |
| 优化 | 直接最小化 | 对抗 minimax（GRL） |
| 稳定性 | 非常稳定 | 偶尔震荡 |
| 表达力 | 受核选择限制 | 更灵活 |
| 适合场景 | 中小差距、数据少 | 差距大、数据足 |

推荐的工作流：先试 MMD；如果效果停滞，再切换到 DANN。

## 5. CORAL：对齐二阶统计量

对齐均值是个好主意，但同时对齐均值和协方差往往效果更好。**CORAL**（Sun & Saenko，2016）就是这么干的。

![CORAL 协方差对齐](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/03-域适应方法/fig4_coral_covariance.png)

假设源域和目标域的特征协方差矩阵分别是 $C_S$ 和 $C_T$，那么 CORAL 的损失函数定义为：
$$\mathcal{L}_{\text{CORAL}} = \frac{1}{4 d^2} \|C_S - C_T\|_F^2.$$

**直观理解——白化加重新着色。** 把源域特征乘以 $C_S^{-1/2} C_T^{1/2}$，先去掉源域的协方差特性，再叠加上目标域的协方差特性。Deep CORAL 直接把这个损失加到深度网络里，靠梯度下降隐式完成同样的操作。

CORAL 非常高效，每个 batch 只需要计算一个矩阵和一个 Frobenius 范数，完全确定性，而且在轻度分布偏移的情况下表现相当出色。在尝试 MMD 或 DANN 之前，它是一个非常值得参考的基线方法。

## 6. AdaBN：永远该先尝试的免费午餐

最简单的域适应方法：**在目标数据上重新计算 BatchNorm 的统计量**。

标准 BN 在测试时用的是源域训练过程中累积的 running mean 和 variance。如果目标域分布不同，这些统计量就不对了，而它们偏偏卡在每个卷积层和下一个非线性激活之间。AdaBN（Li 等，2017）的做法是：

1. 正常在源域上训练模型。
2. 冻结权重后，用无标注的目标数据跑一遍前向传播，重新计算每一层 BN 的 $\mu_T, \sigma_T^2$。
3. 部署时，直接用目标域的统计量替换源域的统计量。

成本：几分钟。代码改动：改几个 `BatchNorm` 的 running stats。效果：在协变量偏移的情况下，通常能挽回 2–10 个点的准确率。任何复杂方法之前，我都会先试试这个。

## 7. 基于 GAN 和像素级的适配方法

有时候，差距非常明显——比如合成图像和真实图像、白天和夜晚的场景。这种情况下，仅仅对齐**特征**已经不够了，我需要直接转换输入本身。

- **CycleGAN** 学习两个生成器 $G: \mathcal{X}_S \to \mathcal{X}_T$ 和 $F: \mathcal{X}_T \to \mathcal{X}_S$，并加入循环一致性约束 $F(G(x)) \approx x$。先把源域图像转换为目标域风格，然后用原始标签训练分类器。需要注意的是，循环一致性并不能保证语义完全保留，建议加上感知损失或身份损失来提升安全性。
- **ADDA** 将源域和目标域的编码器分开处理。第一阶段：正常训练源编码器和分类器。第二阶段：用源编码器初始化一个**目标**编码器，通过对抗域判别器进行适应，同时保持分类器固定不动。第三阶段：测试时，目标域输入经过目标编码器和源分类器处理。这种不对称设计让 ADDA 比 DANN 具有更强的表达能力，但代价是多了一个训练阶段。

## 8. 自训练：为目标域生成标签

对抗对齐和统计对齐把目标域看作一个没有区别的整体。**自训练**（也叫伪标注）更进一步：用当前模型为目标域生成标签，然后用这些标签继续训练。

![自训练 / 伪标注](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/03-域适应方法/fig6_self_training.png)

流程如下：

1. 在源域上训练 $f$。
2. 对每个目标样本预测，只保留置信度最高的结果中满足 $\max_y f(x)_y > \tau$ 的样本（$\tau$ 是高置信度阈值）。
3. 把保留下来的 (输入, 预测) 对当作新标注数据，重新训练模型。
4. 重复以上步骤。

自训练效果强大，但经常被低估。它有一个臭名昭著的失败模式：**确认偏差**。错误但高置信度的预测会被反复加入训练，导致问题放大。常见的解决方法包括：

- 提高阈值 $\tau$（通常设为 0.9 或更高）；
- 按类别平衡选择样本（限制每类保留的样本数量）；
- 使用数据增广的一致性正则化（类似 FixMatch 的方法）；
- 每轮训练都从源模型重新开始，而不是基于上一轮的自训练模型。

## 9. 决策树：方法怎么选，何时用？

```
1. 目标域有没有标注？
   ├─ 有 → 半监督 DA：微调 + 重要性加权
   └─ 没有 → 看第 2 步

2. 数据分布的偏移在哪？
   ├─ 只是 P(X) 不同         → 看第 3 步
   ├─ 只是 P(Y) 不同         → 标签偏移修正（BBSE / EM）
   └─ P(Y|X) 不同（概念偏移） → 需要少量目标域标注

3. 视觉或特征差距有多大？
   ├─ 极小   → AdaBN（优先尝试）
   ├─ 小     → AdaBN + Deep CORAL
   ├─ 中     → MMD（DAN）或 DANN
   ├─ 大     → DANN / CDAN，或者像素级方法（CycleGAN、ADDA）
   └─ 巨大（仿真→真实）→ CycleGAN + ADDA + 自训练
```

实际项目中，效果好的 pipeline 通常会**组合多种方法**：先用 AdaBN 拿下容易的部分，再用 MMD 或 DANN 对齐特征，最后跑一轮自训练收尾。

## 10. 基准测试：这到底有多大用？

![Office-31 和 DomainNet 基准](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/03-域适应方法/fig7_office31_benchmark.png)

这些数字是基于 ResNet-50 主干的文献平均值。有两点值得关注：

- **从“什么都不做”到“做点什么”，提升最明显。** 即使只用 AdaBN，也能弥补相当一部分差距。关键是动手做，而不是纠结于选完美的方法。
- **DomainNet 比 Office-31 难得多。** 在 DomainNet 上达到 40% 的准确率已经算很强了——这个数据集包含 345 个类别，横跨 6 种风格差异极大的视觉类型。读 DA 准确率时，永远要和 source-only 基线对比，不能只看绝对值。

## 11. 域适应真正发挥作用的地方

- **医学影像**：西门子和GE的设备、1.5T和3T MRI、A医院和B医院的数据。
- **自动驾驶**：晴天到雨天、城市A到城市B、仿真环境到真实场景。
- **推荐系统**：不同国家之间、不同年份之间、网页端到移动端。
- **NLP**：电影评论到商品评论、新闻到社交网络、正式语言到口语化表达。
- **Sim-to-real**：机器人和自动驾驶领域，从合成数据迁移到真实传感器数据。

共同点很明显：**源域标注数据充足，目标域标注要么很贵，要么根本拿不到，但模型还是得上线。**

## 13. 完整实现：DANN

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.utils.data import DataLoader, TensorDataset
from torch.autograd import Function
import numpy as np
from sklearn.metrics import accuracy_score

class GradientReversalFunction(Function):
    """前向传播不变，反向传播时翻转梯度。"""

    @staticmethod
    def forward(ctx, x, lambda_):
        ctx.lambda_ = lambda_
        return x.clone()

    @staticmethod
    def backward(ctx, grad_output):
        return grad_output.neg() * ctx.lambda_, None

class GradientReversalLayer(nn.Module):
    def __init__(self):
        super().__init__()
        self.lambda_ = 1.0

    def set_lambda(self, val):
        self.lambda_ = val

    def forward(self, x):
        return GradientReversalFunction.apply(x, self.lambda_)

class FeatureExtractor(nn.Module):
    def __init__(self, input_dim=28 * 28, hidden_dim=256):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim), nn.ReLU(), nn.Dropout(0.5),
            nn.Linear(hidden_dim, hidden_dim), nn.ReLU(), nn.Dropout(0.5),
        )

    def forward(self, x):
        return self.net(x.view(x.size(0), -1))

class LabelPredictor(nn.Module):
    def __init__(self, feature_dim=256, num_classes=10):
        super().__init__()
        self.fc = nn.Linear(feature_dim, num_classes)

    def forward(self, x):
        return self.fc(x)

class DomainDiscriminator(nn.Module):
    def __init__(self, feature_dim=256):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(feature_dim, 256), nn.ReLU(), nn.Dropout(0.5),
            nn.Linear(256, 256), nn.ReLU(), nn.Dropout(0.5),
            nn.Linear(256, 1), nn.Sigmoid(),
        )

    def forward(self, x):
        return self.net(x)

class DANN(nn.Module):
    """域对抗神经网络。"""

    def __init__(self, input_dim=28 * 28, hidden_dim=256, num_classes=10):
        super().__init__()
        self.feature_extractor = FeatureExtractor(input_dim, hidden_dim)
        self.label_predictor = LabelPredictor(hidden_dim, num_classes)
        self.domain_discriminator = DomainDiscriminator(hidden_dim)
        self.grl = GradientReversalLayer()

    def forward(self, x, alpha=1.0):
        features = self.feature_extractor(x)
        class_logits = self.label_predictor(features)
        self.grl.set_lambda(alpha)
        domain_logits = self.domain_discriminator(self.grl(features))
        return class_logits, domain_logits

class DANNTrainer:
    def __init__(self, model, source_loader, target_loader, test_loader,
                 num_epochs=100, lr=1e-3, device="cpu", gamma=10.0):
        self.model = model.to(device)
        self.source_loader = source_loader
        self.target_loader = target_loader
        self.test_loader = test_loader
        self.num_epochs = num_epochs
        self.device = device
        self.gamma = gamma
        self.optimizer = torch.optim.Adam(model.parameters(), lr=lr)
        self.class_criterion = nn.CrossEntropyLoss()
        self.domain_criterion = nn.BCELoss()

    def _adaptive_lambda(self, epoch):
        # sigmoid 曲线控制对抗权重，训练初期小，后期接近 1
        p = epoch / self.num_epochs
        return 2.0 / (1.0 + np.exp(-self.gamma * p)) - 1.0

    def train_epoch(self, epoch):
        self.model.train()
        source_iter = iter(self.source_loader)
        target_iter = iter(self.target_loader)
        n_batches = min(len(self.source_loader), len(self.target_loader))
        total_loss = 0.0
        lambda_p = self._adaptive_lambda(epoch)

        for _ in range(n_batches):
            try:
                src_x, src_y = next(source_iter)
            except StopIteration:
                source_iter = iter(self.source_loader)
                src_x, src_y = next(source_iter)
            try:
                tgt_x, _ = next(target_iter)
            except StopIteration:
                target_iter = iter(self.target_loader)
                tgt_x, _ = next(target_iter)

            src_x = src_x.to(self.device)
            src_y = src_y.to(self.device)
            tgt_x = tgt_x.to(self.device)

            # 前向计算：源域和目标域都过两个分支
            src_class_logits, src_dom_logits = self.model(src_x, lambda_p)
            _, tgt_dom_logits = self.model(tgt_x, lambda_p)

            # 源域分类损失
            class_loss = self.class_criterion(src_class_logits, src_y)
            # 域判别损失：源域标记为 1，目标域标记为 0
            d_loss_s = self.domain_criterion(
                src_dom_logits, torch.ones_like(src_dom_logits))
            d_loss_t = self.domain_criterion(
                tgt_dom_logits, torch.zeros_like(tgt_dom_logits))
            domain_loss = d_loss_s + d_loss_t

            loss = class_loss + domain_loss
            self.optimizer.zero_grad()
            loss.backward()
            self.optimizer.step()
            total_loss += loss.item()

        return total_loss / n_batches

    @torch.no_grad()
    def evaluate(self):
        self.model.eval()
        preds, labels = [], []
        for x, y in self.test_loader:
            x = x.to(self.device)
            logits, _ = self.model(x, alpha=0.0)
            preds.extend(logits.argmax(dim=1).cpu().numpy())
            labels.extend(y.numpy())
        return accuracy_score(labels, preds)

    def train(self):
        best = 0.0
        for epoch in range(self.num_epochs):
            loss = self.train_epoch(epoch)
            acc = self.evaluate()
            if (epoch + 1) % 10 == 0:
                lam = self._adaptive_lambda(epoch)
                print(f"epoch {epoch + 1:3d}  loss={loss:.4f}  "
                      f"target_acc={acc:.4f}  lambda={lam:.3f}")
            best = max(best, acc)
        print(f"best target accuracy: {best:.4f}")

def main():
    N, D, C = 10000, 28 * 28, 10
    # 模拟源域和目标域数据分布偏移
    src_x = torch.randn(N, 1, 28, 28)
    src_y = torch.randint(0, C, (N,))
    tgt_x = torch.randn(N, 1, 28, 28) + 0.5     # 目标域分布偏移
    tgt_y = torch.randint(0, C, (N,))           # 训练时不使用
    test_x = torch.randn(2000, 1, 28, 28) + 0.5
    test_y = torch.randint(0, C, (2000,))

    BS = 128
    src_loader = DataLoader(TensorDataset(src_x, src_y), BS, shuffle=True)
    tgt_loader = DataLoader(TensorDataset(tgt_x, tgt_y), BS, shuffle=True)
    test_loader = DataLoader(TensorDataset(test_x, test_y), BS)

    model = DANN(D, 256, C)
    trainer = DANNTrainer(model, src_loader, tgt_loader, test_loader,
                          num_epochs=100, lr=1e-3)
    trainer.train()

if __name__ == "__main__":
    main()
```

### 这段代码的核心逻辑

| 组件 | 功能 |
|---|---|
| `GradientReversalLayer` | 前向传播不变，反向传播翻转梯度，将对抗训练简化为一次反向传播。 |
| `_adaptive_lambda` | 使用公式 $\frac{2}{1+e^{-\gamma p}} - 1$ 控制对抗权重，先学特征再加对抗。 |
| `class_loss` | 标准交叉熵损失，仅用源域标签，目标域无标注。 |
| `domain_loss` | BCE 损失，源域标记为 1，目标域标记为 0，用于训练域判别器。 |
| GRL + 域分支 | 反向传播时梯度翻转回 $G_f$，迫使特征提取器隐藏域信息。 |
| `evaluate(alpha=0)` | 测试时关闭 GRL（$\lambda=0$），只用分类头预测。 |

## 总结

域适应解决的是迁移学习中最实际的问题：训练数据和部署数据来自不同的分布。以下是工具箱，按实现成本从低到高排序：

- **AdaBN** —— 在目标域上重新计算批归一化统计量；零成本、无需重训、永远优先尝试。
- **CORAL** —— 对齐源域和目标域的**协方差矩阵**；开销小、结果确定。
- **MMD（DAN）** —— 匹配核均值嵌入；稳定且有理论依据，默认使用多核方法。
- **DANN** —— 通过梯度反转层实现对抗域对齐；一次反向传播即可完成。
- **CDAN / ADDA** —— 更灵活的变体，适合处理更大的域间差距。
- **CycleGAN** —— 当特征对齐不够时，用像素级转换来补足。
- **自训练** —— 使用置信度阈值筛选伪标签；榨干最后几个点的精度。

Ben-David 界限告诉你什么是可能的：只要联合最优误差足够小，压低源域误差和域散度，目标域误差自然会下降。如果联合最优误差本身就大，再多的对齐也无济于事；这时你需要的是标注数据。

接下来是 [第 4 篇——Few-Shot Learning](/zh/transfer-learning/04-few-shot-learning/)，我会彻底抛弃“源域数据充足”这个假设，研究每类只有 1–5 个样本时如何学习。
