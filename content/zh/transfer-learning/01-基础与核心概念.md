我刚花了两周时间，用了一整柜 GPU 训练好了一个 ImageNet 分类器。周一早上，团队负责人让我开发一个胸片肺炎识别模型——手头的标注数据只有 200 张。难道我要再排队等两周 GPU，从零开始重新训练吗？

当然不会。我会直接利用 ImageNet 模型已经学到的边缘、纹理和形状知识，替换掉最后一层，在 X 光片数据上微调。两小时后，该模型的性能就能显著超越任何仅用同样少量数据从随机初始化训练出的模型。这就是**迁移学习**——它正是大多数实际深度学习项目能在几天内交付，而非耗时数月的关键所在。

这篇文章是整个系列的基础。在深入更专业的主题之前，我需要先讲清楚七件事：

1. 为什么从头训练往往不现实；
2. 域、任务、源域和目标域的正式定义；
3. 迁移学习的分类体系：归纳、直推、无监督；
4. 深度网络中每一层到底迁移了什么；
5. 负迁移：借来的知识反而帮倒忙的情况；
6. Ben-David 界与 MMD：如何判断迁移是否可行；
7. 一个完整的、可运行的 MMD 特征对齐实现。

**前置知识：** 掌握基本机器学习术语（损失函数、梯度下降、分类），并能看懂 Python 代码。

## 1. 为什么需要迁移学习

### 从头训练的困境

教科书中的监督学习范式默认满足三个前提，但在实际工业场景中，这些前提往往并不成立：

- **海量标注数据。** 现代深度网络要泛化得好，通常需要几万到几百万条标注样本，但真实业务团队很少能有这样的资源。
- **充裕的算力。** 从随机初始化训练一个 ResNet-50 需要几百个 GPU 小时；而从零开始训练一个 Transformer 可能需要上万个 GPU 小时。
- **知识无法复用。** 即使是高度相关的任务，如胸片 X 光和胸片 CT，也得从头再来。

现实中的医学项目则截然不同：罕见病样本往往仅有数百例，标注工作必须由具备执业资格的医生完成，且留给建模的时间通常只有几周。迁移学习提供了一个简单的解决方案：**拿一个在大规模通用数据上训练好的模型，低成本地适配到你的小数据任务上。**

下图直观呈现了“小数据 + 分布偏移”的典型形态：两类数据分布本应一致，但目标域在特征空间中发生了旋转与平移——这正是 ImageNet 自然图像向医学影像迁移或跨医院扫描设备迁移时最常见的协变量偏移（covariate shift）的简化示例。

![源域与目标域的分布偏移](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig1_domain_shift.png)

### 直觉
人类天然具备知识迁移能力：
- 会骑自行车的人，一下午就能学会骑摩托车，不需要一年。
- Python 程序员看 Java 代码，一眼就能认出类、循环和异常机制。
- 见过家猫的人，第一次看到狮子也会立刻判断为“某种猫科动物”。

深度网络也有类似的特性。视觉模型靠前的卷积层学到的是几乎通用的原始特征——有方向的边缘、色块、简单纹理，这些对几乎所有视觉任务都有用。靠后的层学到的是更专门的概念（如毛皮花纹、眼睛形状），但在相关领域之间也可以复用。迁移学习的核心在于最大化利用这种知识重叠。

### 核心思想一句话总结

> 给定一个**有大量标注数据的源域**和一个**几乎没有标注的目标域**，迁移学习将源域的知识搬过来，让目标模型表现得比单独训练时更好。

其基本前提在于：源域与目标域之间需具备一定程度的相关性。它们不必共享同一特征空间、同一标签集，甚至不必同属一个模态——但共享得越多，可迁移的内容就越多。

## 2. 形式化定义

要精确讨论迁移学习，必须先区分两个新手容易混淆的概念：**域**关注的是“输入是什么样子”，**任务**关注的是“需要预测什么”。Pan 和 Yang 在 2010 年的综述中将这一区分确立为整个领域的核心。

### 域

域是一个二元组 $\mathcal{D} = \{\mathcal{X}, P(X)\}$，其中 $\mathcal{X}$ 是特征空间，$P(X)$ 是其边缘分布。

- **源域：** ImageNet 自然 RGB 图像，$\mathcal{X} = \mathbb{R}^{224 \times 224 \times 3}$，$P_S(X)$ 表示自然场景的像素统计规律。
- **目标域：** 胸部 CT 切片，维度相同，但 $P_T(X)$ 的灰度直方图、结构先验和噪声特性完全不同。

### 任务

任务是另一个二元组 $\mathcal{T} = \{\mathcal{Y}, f(\cdot)\}$，包括标签空间和需要学习的预测函数。在监督学习中，任务隐含地固定了条件分布 $P(Y \mid X)$。

- **任务 1：** ImageNet 1000 类分类，$|\mathcal{Y}| = 1000$。
- **任务 2：** 肺炎二分类，$|\mathcal{Y}| = 2$。

### 源域与目标域

|       | 源域                                         | 目标域                                       |
| ----- | -------------------------------------------- | -------------------------------------------- |
| 域    | $\mathcal{D}_S$，对应任务 $\mathcal{T}_S$    | $\mathcal{D}_T$，对应任务 $\mathcal{T}_T$    |
| 标注  | 充足                                         | 稀缺或缺失                                   |
| 分布  | $P_S(X), P_S(Y\|X)$                          | $P_T(X), P_T(Y\|X)$                          |

迁移学习并不要求源域与目标域完全一致，而这恰恰是其价值所在。两者可以在以下任意方面不同：特征空间（$\mathcal{X}_S \neq \mathcal{X}_T$）、边缘分布（$P_S(X) \neq P_T(X)$）、标签空间（$\mathcal{Y}_S \neq \mathcal{Y}_T$）或条件分布（$P_S(Y\mid X) \neq P_T(Y\mid X)$）。每种差异组合都对应一个不同的子问题。

### 形式化的目标

> 给定源域 $\mathcal{D}_S$ 及其任务 $\mathcal{T}_S$，目标域 $\mathcal{D}_T$ 及其任务 $\mathcal{T}_T$，迁移学习的目标是利用 $\mathcal{D}_S$ 和 $\mathcal{T}_S$ 的知识改进目标预测函数 $f_T(\cdot)$，其中 $\mathcal{D}_S \neq \mathcal{D}_T$ 或 $\mathcal{T}_S \neq \mathcal{T}_T$。

记不使用迁移时的目标域错误率为 $\epsilon_0$，使用迁移后的错误率为 $\epsilon_T$，那么“成功”意味着
$$\epsilon_T < \epsilon_0$$
或者等价地：用更少的目标域标注达到相同的精度。第二种说法在实际应用中通常更有说服力——迁移学习很少能提升已经饱和的模型性能，但在饱和点之前，它几乎总能显著降低标注成本。
## 3. 迁移学习的分类体系

迁移学习的方法看似杂乱，但如果按照“目标侧缺少什么”来整理，就能清晰地分成三类。

![迁移学习分类体系：归纳、直推、无监督](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig2_taxonomy.png)

### 归纳迁移

- 源任务和目标任务**不同**（$\mathcal{T}_S \neq \mathcal{T}_T$）。
- 目标域有**少量标注数据**，通常很少。
- 常用方法：预训练后微调、多任务学习、自训练。
- 典型例子：用 ImageNet 预训练的 ResNet 主干网络，换上新的分类头，用来做胸部 X 光分类，标注数据只有几百张。这是大多数医学影像论文的标准做法。

### 直推迁移

- 任务**相同**，但源域和目标域**不同**（$\mathcal{T}_S = \mathcal{T}_T$，$\mathcal{D}_S \neq \mathcal{D}_T$）。
- 目标域**完全没有标注数据**。
- 常用方法：域适应（特征对齐、对抗对齐）、样本重加权。
- 典型例子：在 GTA5 游戏画面数据上训练的语义分割模型，直接部署到 Cityscapes 真实驾驶数据上，全程没有新标注。自动驾驶研究中常见这种情况。

### 无监督迁移

- **源域和目标域都没有标注数据**。迁移的是结构本身——表示、聚类、流形。
- 常用方法：自监督预训练（MoCo、SimCLR、MAE）、深度聚类。
- 典型例子：在通用语料上训练的 Word2Vec 或 BERT，作为特征提取器服务于任意下游 NLP 任务。这也是现代基础模型流水线的起点。

实际应用中，这三类方法往往界限模糊：典型的基础模型流程会先进行**无监督**预训练，然后通过**归纳**迁移到下游任务；如果部署环境与标注训练集存在差异，还会叠加一层**直推**域适应。这套分类体系更像是一个词汇表，而不是严格的划分。
## 4. 每一层迁移的是什么

深度网络不是一个单一的知识单元，而是由逐层递增的特定表示堆叠而成。Yosinski 等人在 2014 年做了一个经典实验：用 ImageNet 的一半数据训练一个 CNN，冻结前 $k$ 层，在另一半数据上重新训练剩余部分，同时滑动 $k$ 值观察准确率的变化。结果如下图所示，这是关于迁移学习最有用的一个经验事实。

![CNN 各层特征的可迁移性](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig3_layer_transferability.png)

这张图讲了三件事：

1. **底层特征（`conv1`–`conv3`）是通用的。** 边缘和纹理在视觉任务中几乎通用。冻结这些层几乎没有损失，甚至在小目标数据集上，迁移它们的效果往往比从头训练更好。
2. **高层特征（`conv5` 及以后）是特定的。** 针对 ImageNet 物体类别优化的滤波器组，与目标任务的类别不匹配。冻结这些层会带来显著的性能下降——橙色“冻结”曲线急剧下滑。
3. **微调可以免费恢复特定性。** 蓝色“微调”曲线保持平稳：只要允许高层适应目标任务分布，就能既保留底层的通用先验，又匹配目标分布。这就是“低层冻结、高层微调”成为默认策略的原因。

这张图解释了后续大部分实操建议：哪些层需要冻结，每组学习率如何设置，以及为什么 LoRA 这类参数高效方法只针对深层进行修改。
## 5. 负迁移

迁移学习不是免费的。如果源域和目标域差别太大，继承的权重反而会拖后腿，而不是帮忙。这种情况叫**负迁移**：迁移后的模型表现比直接在目标域上训练的模型还差。
$$\epsilon_T > \epsilon_0$$

下图清楚地展示了这个现象。随着源域和目标域差异增大，迁移的准确率会跌破从头训练的基线。过了这个交叉点，你就是在花钱引入错误的归纳偏置。

![负迁移：随域散度增长的正/负迁移区域](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig4_negative_transfer.png)

### 为什么会发生

1. **输入分布差异大。** 照片的纹理对手绘草图毫无用处，两者的高频统计特性完全不同。
2. **任务冲突。** 源任务的最优解可能离目标任务的最优解很远，梯度下降不一定能跳出源任务的局部最优。
3. **源域过拟合。** 预训练模型可能记住了数据集特有的噪声，比如 ImageNet 图片角落里固定的水印，这种“知识”会误导目标域的预测。

### 如何避免

- **先测一下。** 用 MMD 或者域分类器检查原始特征或浅层特征的差异。如果差距太大，要么减少迁移力度，要么换个源数据集。
- **选择性迁移。** 冻结通用的底层网络，重新训练特定的高层网络。前面提到的可迁移性曲线就是你的参考。
- **正则化微调。** 加一个 $L_2$ 惩罚项，把参数拉向预训练值，比如用 *L2-SP* 方法。这样可以限制参数漂移的范围。
- **集成模型保底。** 如果拿不准，就把迁移模型和从头训练的模型做个平均。大多数情况下，集成模型的表现不会比两者中较差的那个更差。

图中的交叉点不是理论假设，而是每个迁移学习实践者实际面对的工作曲线。我的任务是在上线之前搞清楚自己站在哪一边。
## 6. 量化迁移可行性

有两个工具可以帮你**在训练之前**判断迁移是否可行。

### Ben-David 界

对于假设空间 $\mathcal{H}$ 中的任意假设 $h$，目标域错误率可以分解为：
$$\epsilon_T(h) \;\leq\; \epsilon_S(h) \;+\; \tfrac{1}{2}\, d_{\mathcal{H}\Delta\mathcal{H}}(\mathcal{D}_S, \mathcal{D}_T) \;+\; \lambda^{*}.$$
这三项彼此独立，分别对应三个优化方向：

- $\epsilon_S(h)$：源域错误率，通过**更好的训练**来降低。
- $d_{\mathcal{H}\Delta\mathcal{H}}$：源域和目标域分布之间的散度，通过**域适应**来缩小。
- $\lambda^{*}$：最优假设在两个域上的联合错误率下限。这个值由问题本身决定。如果不存在一个假设能在两个域上都表现良好，再复杂的对齐方法也无济于事。

这个界的实际意义很残酷：如果散度或者 $\lambda^{*}$ 太大，直接**放弃对齐，换一个源域**，别浪费计算资源。

### 最大均值差异（MMD）

在实践中，MMD 是一种估计分布散度的方法。它将源域和目标域映射到再生核希尔伯特空间（RKHS），然后比较它们的均值距离：
$$\mathrm{MMD}(\mathcal{D}_S, \mathcal{D}_T) \;=\; \big\lVert \mathbb{E}[\phi(X_S)] - \mathbb{E}[\phi(X_T)] \big\rVert_{\mathcal{H}}.$$
使用 RBF 核时，MMD 有一个无偏闭式估计器，可以直接从单个 mini-batch 计算出来，并加入损失函数：
$$\mathcal{L} \;=\; \mathcal{L}_{\mathrm{task}} \;+\; \lambda \cdot \mathrm{MMD}^{2}.$$
最小化第二项的效果是拉近源域和目标域的特征分布，这正是 Ben-David 界要求你做的事情。

预训练后可以用一个经验规则：嵌入空间中的 MMD 低于 0.1 是正迁移的强烈信号；高于 0.5 基本就是负迁移的地盘了。

## 7. 整合起来：标准配方

### 预训练主干 + 新任务头

解决生产问题的九成方案其实非常简单：

![预训练主干 + 任务专用新头](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig5_backbone_new_head.png)

找一个在大规模通用数据集上训练好的卷积或 Transformer 主干，去掉原来的分类头，换成适合你标签空间的新头，然后微调。逐层决定是冻结（省钱、保守）还是放开（费钱、灵活）。对大多数预算有限的情况，最佳策略是：**先冻结主干训几个 epoch，再用极小学习率放开微调**。

### 为什么在小数据场景下特别有效

这张"数据效率曲线"是迁移学习最有说服力的理由。只有 10 个标注时，从头训练几乎和瞎猜差不多，但迁移已经能给出一个可用模型；100 个标注时，差距非常明显；到一万个标注时，两条曲线逐渐收敛——这正是**为什么迁移学习对那些标不起一万条样本的团队最有价值**。

![数据效率：目标域准确率 vs. 标注量](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig6_data_efficiency.png)

这张图可以当成合同看：水平切片告诉你迁移能省多少标注，垂直切片告诉你迁移能提升多少准确率。两种视角完全等价，而且差距经常是以数量级计算的。

### 无监督场景：域适应

如果目标域**完全没有标注**，就无法按传统方式微调。主流方法（系列第三篇会详细讲）是训练一个**共享编码器**，把源域和目标域特征映射到同一子空间，然后在（有标注的）源域上训练分类器，直接用到（已经对齐的）目标域。

![域适应问题设定：共享编码器](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/01-基础与核心概念/fig7_domain_adaptation.png)

训练目标就是前面提到的那个损失函数：源域分类损失 + 拉近源域和目标域特征的 MMD 惩罚。下一节会展示，用 PyTorch 实现这套方法不到 100 行代码。

## 迁移学习与相关概念

迁移学习和几个相邻领域有交集，用词上需要区分清楚：

| 维度       | 迁移学习                  | 多任务学习                  |
| ---------- | ------------------------- | --------------------------- |
| 目标       | 提升目标域性能            | 同时优化所有任务            |
| 训练方式   | 先源域后目标域（顺序）    | 源域和目标域一起训练（并行）|
| 数据假设   | 源域和目标域可以不同      | 任务之间存在关联            |
| 常见模式   | 预训练-微调               | 共享编码器 + 多个任务头     |

| 维度       | 迁移学习                  | 元学习                      |
| ---------- | ------------------------- | --------------------------- |
| 目标       | 迁移特定知识              | 学习“如何快速掌握新任务”    |
| 训练数据   | 单个或少数源域            | 大量多样化任务              |
| 适应方式   | 通常需要微调              | 快速小样本更新              |

| 维度       | 迁移学习                  | 域泛化                      |
| ---------- | ------------------------- | --------------------------- |
| 测试信息   | 可以访问目标域数据        | 目标域未知                  |
| 方法       | 域适应                    | 学习域不变特征              |

搞清楚这些区别，才知道该看哪些论文、跑哪些 benchmark、用哪些工具。
## 完整实现：基于 MMD 的特征迁移

下面是一个完整的代码示例，展示了本文描述的工作流。我模拟了一个 2D 域偏移场景，并比较了三种策略：在目标域上从头训练、直接用源域模型、以及通过 MMD 对齐进行特征迁移。

```python
"""
特征迁移与域适应示例
方法：特征提取 + MMD 对齐 + 微调
"""

import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset
from sklearn.metrics import accuracy_score
from sklearn.svm import SVC

np.random.seed(42)
torch.manual_seed(42)

# --- 数据生成：模拟域偏移 ---------------------------------------------------

def generate_source_domain(n_samples=1000):
    """源域：两个分得很开的高斯簇。"""
    X0 = np.random.randn(n_samples // 2, 2) * 0.5 + np.array([-2, -2])
    X1 = np.random.randn(n_samples // 2, 2) * 0.5 + np.array([2, 2])
    X = np.vstack([X0, X1])
    y = np.hstack([np.zeros(n_samples // 2), np.ones(n_samples // 2)])
    return X, y

def generate_target_domain(n_samples=200):
    """目标域：旋转 45 度并平移（协变量偏移）。"""
    theta = np.pi / 4
    rotation = np.array([[np.cos(theta), -np.sin(theta)],
                         [np.sin(theta),  np.cos(theta)]])
    X0 = (np.random.randn(n_samples // 2, 2) * 0.6 + np.array([-1, -1])) @ rotation.T
    X1 = (np.random.randn(n_samples // 2, 2) * 0.6 + np.array([ 1,  1])) @ rotation.T
    X = np.vstack([X0, X1])
    y = np.hstack([np.zeros(n_samples // 2), np.ones(n_samples // 2)])
    return X, y

X_source,        y_source        = generate_source_domain(1000)
X_target_train,  y_target_train  = generate_target_domain(50)   # 少量标注数据
X_target_test,   y_target_test   = generate_target_domain(200)  # 测试集

print(f"源域: {X_source.shape}  |  目标训练: {X_target_train.shape}  "
      f"|  目标测试: {X_target_test.shape}")

# --- 方法 1：仅在少量目标样本上从头训练 -----------------------------------

clf_scratch = SVC(kernel="rbf", gamma="auto").fit(X_target_train, y_target_train)
acc_scratch = accuracy_score(y_target_test, clf_scratch.predict(X_target_test))
print(f"[从头训练]   准确率: {acc_scratch:.4f}")

# --- 方法 2：在源域训练，直接套用到目标域（无适应） -----------------------

clf_direct = SVC(kernel="rbf", gamma="auto").fit(X_source, y_source)
acc_direct = accuracy_score(y_target_test, clf_direct.predict(X_target_test))
print(f"[直接迁移]   准确率: {acc_direct:.4f}")

# --- 方法 3：特征迁移 + MMD 对齐 + 微调 -----------------------------------

class FeatureExtractor(nn.Module):
    def __init__(self, input_dim=2, hidden_dim=32, output_dim=16):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim), nn.ReLU(),
            nn.Linear(hidden_dim, output_dim), nn.ReLU(),
        )

    def forward(self, x):
        return self.encoder(x)

class Classifier(nn.Module):
    def __init__(self, input_dim=16, num_classes=2):
        super().__init__()
        self.fc = nn.Linear(input_dim, num_classes)

    def forward(self, x):
        return self.fc(x)

def compute_mmd(x_source, x_target, gamma=1.0):
    """计算两个 batch 之间的 RBF 核 MMD（有偏估计，SGD 下足够用）。"""
    xx = torch.sum(x_source ** 2, dim=1, keepdim=True)
    yy = torch.sum(x_target ** 2, dim=1, keepdim=True)
    K_ss = torch.exp(-gamma * (xx + xx.t() - 2 * x_source @ x_source.t()))
    K_tt = torch.exp(-gamma * (yy + yy.t() - 2 * x_target @ x_target.t()))
    K_st = torch.exp(-gamma * (xx + yy.t() - 2 * x_source @ x_target.t()))
    n_s, n_t = x_source.size(0), x_target.size(0)
    return K_ss.sum() / n_s ** 2 + K_tt.sum() / n_t ** 2 - 2 * K_st.sum() / (n_s * n_t)

def train_with_mmd(X_source, y_source, X_target_unlabeled,
                   X_target_labeled, y_target_labeled,
                   epochs=100, lambda_mmd=0.5):
    """两阶段训练：源域分类 + MMD 对齐，然后在目标域上微调。"""
    X_s   = torch.FloatTensor(X_source)
    y_s   = torch.LongTensor(y_source.astype(int))
    X_t_u = torch.FloatTensor(X_target_unlabeled)
    X_t_l = torch.FloatTensor(X_target_labeled)
    y_t_l = torch.LongTensor(y_target_labeled.astype(int))

    feat = FeatureExtractor()
    clf  = Classifier()
    optimizer = optim.Adam(list(feat.parameters()) + list(clf.parameters()), lr=1e-3)
    criterion = nn.CrossEntropyLoss()

    loader = DataLoader(TensorDataset(X_s, y_s), batch_size=32, shuffle=True)

    # 阶段 1：源域分类 + 在无标注目标上做 MMD 对齐
    for _ in range(epochs):
        for X_batch, y_batch in loader:
            optimizer.zero_grad()
            loss_cls = criterion(clf(feat(X_batch)), y_batch)
            loss_mmd = compute_mmd(feat(X_batch), feat(X_t_u))
            (loss_cls + lambda_mmd * loss_mmd).backward()
            optimizer.step()

    # 阶段 2：在少量标注目标样本上做轻量微调
    for _ in range(50):
        optimizer.zero_grad()
        criterion(clf(feat(X_t_l)), y_t_l).backward()
        optimizer.step()

    return feat, clf

feat, clf_mmd = train_with_mmd(
    X_source, y_source,
    X_target_unlabeled=X_target_test,
    X_target_labeled=X_target_train,
    y_target_labeled=y_target_train,
)

feat.eval(); clf_mmd.eval()
with torch.no_grad():
    preds = torch.argmax(clf_mmd(feat(torch.FloatTensor(X_target_test))), dim=1).numpy()

acc_transfer = accuracy_score(y_target_test, preds)
print(f"[特征 + MMD] 准确率: {acc_transfer:.4f}")
print(f"\n相比从头训练提升: "
      f"{(acc_transfer - acc_scratch) / acc_scratch * 100:.1f}%")
```

### 各部分的作用

| 组件                              | 功能                                                  |
| --------------------------------- | ----------------------------------------------------- |
| `generate_source/target_domain`   | 制造一个可控的协变量偏移（旋转 + 平移）               |
| `compute_mmd`                     | 计算两个嵌入 batch 之间的 RBF 核 MMD                  |
| 阶段 1 训练                       | 在源域分类的同时，把目标域嵌入往源域拉                |
| 阶段 2 微调                       | 用 50 个标注样本做轻量适配                            |

**两个关键参数。** `lambda_mmd=0.5` 控制对齐力度——太小会忽略目标域，太大会破坏源域分类精度。100/50 epoch 的划分把大部分预算留给对齐，再用一个小阶段做精化。

把上面的合成 2D 数据替换成任何真实数据对（Office-31、DomainNet、VisDA），代码结构都不需要改动——这正是这个练习的意义所在。
## 常见问题

### 迁移学习一定比从头训练好吗？

不一定。这要看领域相关性、目标数据量和任务相似度。我的经验是：如果目标数据不到从头训练所需数据量的 10%，优先考虑迁移学习。超过这个比例，收益会明显下降，工程复杂度可能得不偿失。

### 怎么选一个合适的源域？

选一个规模大、多样性高、模态相同的源域。视觉任务默认用 ImageNet；NLP 任务默认用预训练好的 Transformer 模型（比如 BERT、GPT、Llama）。选完后，用 t-SNE 查看浅层特征的重叠情况，再计算 MMD：低于 0.1 是绿灯，高于 0.5 就别用了。

### 哪些层需要冻结？

先冻结网络底部 30%–50%，微调剩下的部分。NLP 任务通常会微调所有层，但学习率要比预训练时小一个数量级。如果数据量特别少（不到 100 条样本），只微调最后 1–2 层，否则很容易过拟合。

### 怎么判断负迁移？

一定要跑一个从头训练的基线模型。如果迁移模型的表现不如基线，或者微调过程中验证损失不降反升，又或者预训练后源域和目标域嵌入空间的 MMD 仍然高于 0.5，那就是负迁移。解决方法按成本从低到高依次是：减少迁移层数、换源数据集、尝试对抗域适应（第三部分会讲）。

## 小结

我们讲了每个迁移学习项目都离不开的七个关键点：

- **动机**——数据不足、计算成本高、知识复用的价值。
- **定义**——域和任务的区别，源域与目标域的对比，以及它们可能存在的四种差异。
- **分类**——归纳式、直推式、无监督；一个术语体系，三种问题类型。
- **逐层迁移性**——底层特征通用，高层特征专用；冻结底层，微调高层。
- **负迁移**——成因是什么，怎么发现，如何控制损失。
- **理论**——Ben-David 分解，以及 MMD 作为实用的散度估计工具。
- **方法与代码**——预训练主干加新头部，附带一个可运行的 MMD 对齐实现。

迁移学习不是魔法。它是对真实数据集中共享结构的系统化利用。用得好，是现代深度学习工具箱里最高杠杆的技术之一；用得不好，模型会悄悄变差，而你可能毫无察觉。这个系列后面的内容，就是教你怎么把它用好。
