你从没见过斑马。如果我告诉你它外形似马、身披黑白相间条纹，下次在动物园看到斑马时，你就能一眼认出来。无需标注数据或微调，只需一座**语义桥梁**，将你已知的概念（如马、条纹）与未知类别（如斑马）关联起来。

**零样本学习（Zero-Shot Learning, ZSL）** 就是让机器学会这个能力。训练时，使用一组带标签的**已见类（seen classes）** 数据进行监督学习；测试时，要求模型识别一组完全不重叠的**未见类（unseen classes）**，这些类别从未在训练中出现。模型对未见类的唯一线索是它们的语义描述，包括一组属性、类别名称的词嵌入、一段文字或一条 CLIP 风格的提示语。模型的表现，完全取决于它在共享视觉-语义空间中学习到的几何结构。

这篇文章从一个公式出发 —— 图像与类别描述之间的**兼容性函数** $F(x, c)$ —— 依次梳理过去十五年里出现的主要方法：属性原型、双线性和深度兼容性、生成式特征合成、广义零样本学习的校准方法，以及最终将整个领域“规模化”的 CLIP 风格预训练范式。

## 1. 问题定义

设 $\mathcal{C}^s$ 表示**已见类**集合（有标注的训练数据），$\mathcal{C}^u$ 表示**未见类**集合（完全没有标注数据）。 ZSL 的核心约束是：

$$\mathcal{C}^s \cap \mathcal{C}^u = \emptyset.$$

每个类别 $c$，无论是已见还是未见，都对应一个**语义描述向量** $a_c \in \mathbb{R}^M$。这是已见类信息传递到未见类的唯一途径。具体来说，它可以是以下几种形式：

- **二值或连续属性向量**，比如 *有条纹*、*有翅膀*、*水生*；
- 类别名称的**词嵌入**，比如 Word2Vec 或 GloVe；
- 百科描述的**句嵌入**，比如 BERT 的 [CLS] 向量；
- 视觉-语言模型中的**提示语嵌入**，比如 CLIP。

接下来，任务可以分为两种设定：

| 设定 | 测试标签空间 | 现实性 |
|---|---|---|
| **经典 ZSL** | $y \in \mathcal{C}^u$ | 简单基准；假设测试图像一定属于未见类。 |
| **广义 ZSL （GZSL）** | $y \in \mathcal{C}^s \cup \mathcal{C}^u$ | 实际应用场景；难度更高，因为模型容易偏向已见类。 |

贯穿所有方法的核心公式只有一个——**兼容性函数**：

$$F: \mathcal{X} \times \mathcal{C} \to \mathbb{R},
\qquad
\hat{y} = \arg\max_{c \in \mathcal{C}_{\text{test}}} F(x, c; \theta).$$

我先在已见类上学习这个函数，然后依靠语义空间的几何特性将其扩展到未见类。

![有监督、小样本、零样本三种监督方式对比](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig1_zsl_vs_fsl_vs_supervised.png)

## 2. 属性表示与 DAP

属性是最直观的语义描述方式。它们是人类定义的简短谓词，比如 *有条纹*、*四条腿*、*有翅膀*、*水生*。常用的 **AwA2 （Animals with Attributes 2）** 数据集为 50 类动物分别分配了 85 维属性向量；细粒度鸟类数据集 **CUB-200-2011** 则使用了 312 个属性。

把每个类别的属性向量排列起来，就构成了一个 $|\mathcal{C}| \times M$ 的**属性原型矩阵**。例如：

$$a_{\text{斑马}} = (\underbrace{1}_{\text{条纹}}, \underbrace{1}_{\text{四条腿}}, \underbrace{0}_{\text{翅膀}}, \underbrace{1}_{\text{蹄子}}, \ldots).$$

### 直接属性预测（DAP）

Lampert 等人在 2009 年提出了现代 ZSL 问题的定义，并设计了一个简洁的两阶段流程：

**第一阶段**：在已见类图像上，为每个属性训练一个二分类器：

$$\hat{a}_m(x) = P(\text{属性 } m \mid x), \qquad m = 1, \ldots, M.$$

**第二阶段**：测试时，用 $M$ 个属性分类器对输入 $x$ 进行预测，得到属性向量 $\hat{a}(x)$，然后选择与之最接近的未见类原型：

$$\hat{y} = \arg\min_{c \in \mathcal{C}^u} d\bigl(\hat{a}(x),\, a_c\bigr).$$

举个例子：假设查询是一张斑马图片。属性分类器在 *有条纹*、*四条腿*、*有蹄子*、*有鬃毛* 上输出高分，在 *翅膀* 和 *水生* 上输出低分。通过余弦相似度计算，$\hat{a}(x)$ 与原型矩阵匹配后，*斑马* 排名第一，*马* 和 *老虎* 紧随其后。

![基于属性的零样本流程（DAP）](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig2_attribute_classification.png)

**为什么有效**：属性分类器之所以能迁移，是因为属性在不同类别之间是共享的。无论动物是斑马（未见类）还是老虎（已见类），*有条纹* 的含义都是一样的。

**为什么不够好**：

1. **误差累积**。即使属性分类器的准确率达到 90%，预测出的 $\hat{a}(x)$ 仍然可能带有噪声，而最近邻搜索会进一步放大这些噪声。
2. **属性独立性假设**。 DAP 假设在给定类别后，属性之间相互独立，但 *有翅膀* 和 *会飞* 显然高度相关。
3. **标注成本高**。定义和标注属性需要专家知识，构建完整的属性体系代价昂贵。

变体 **IAP （Indirect Attribute Prediction）** 先预测已见类的概率分布，再通过属性矩阵映射到未见类，但它的结构性局限依然存在。

## 3. 兼容性函数：统一视角

别再把属性预测当作中间步骤了，直接端到端学习兼容性函数 $F(x, c)$。

### 双线性（ALE / SJE）

最简单的形式是双线性：

$$F(x, c) = \phi(x)^\top W\, a_c,$$

其中 $\phi(\cdot)$ 是 CNN 主干提取的 $d$ 维视觉特征，唯一的可训练参数是矩阵 $W \in \mathbb{R}^{d \times M}$。用标准的 softmax 交叉熵在已见类上训练：

$$\mathcal{L}(x, y) = -\log \frac{\exp F(x, y)}{\sum_{c \in \mathcal{C}^s} \exp F(x, c)}.$$

Akata 等人在 2013 年提出的 **ALE** 和 2015 年的 **SJE** 把交叉熵换成了结构化排序损失 —— 正确类别的得分必须比所有错误类别高出一个固定间隔。实验表明，这种方法对未见类的泛化能力优于普通的 softmax。

### 深度兼容性 / 双塔

为了捕捉非线性的视觉-语义交互，用两个小 MLP 将两种模态投影到共享的 $d$ 维空间：

$$F(x, c) = \frac{f_v(\phi(x))^\top f_s(a_c)}{\|f_v(\phi(x))\|\, \|f_s(a_c)\|} \cdot \tau,$$

其中 $\tau$ 是可学习的温度参数。这种**双塔**结构正是 DeViSE、 CLIP 和现代稠密检索的核心骨架，区别只在于数据和规模。

![共享语义嵌入空间](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig3_semantic_embedding_space.png)

几何图景是 ZSL 的核心直觉。训练时，已见类的视觉特征会围绕对应的语义原型聚拢（比如 *马*、*猫*、*狗*）。未见类的原型（如 *斑马*、*狮子*、*熊猫*）完全由语义信息决定位置。查询图像被投影到同一空间后，会落在与其语义内容最匹配的未见类原型附近，这就是最终预测结果。

### DeViSE：用词嵌入做语义侧

Frome 等人在 2013 年提出了 **DeViSE**（Deep Visual-Semantic Embedding），它用类别名称的词嵌入（Word2Vec / GloVe）取代了人工定义的属性。架构是双塔模型，使用 hinge 排序损失训练：

$$\mathcal{L} = \sum_{c' \neq y} \max\bigl(0,\; m - F(x, y) + F(x, c')\bigr),$$

推理时，计算查询图像与**所有**类别嵌入的得分，包括从未见过图像的类别。

![DeViSE：视觉编码器 + Word2Vec 文本编码器，共享嵌入空间](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig4_devise_visual_text_embedding.png)

DeViSE 首次证明了 ZSL 可以扩展。在 ImageNet 上训练的模型，能对 WordNet 中数万个零样本类别给出合理预测。它的弱点在于，词嵌入反映的是语言相似性（比如 *猫*、*狗*、*宠物* 聚在一起），而不是视觉相似性（按视觉特性，*斑马*、*老虎*、*美洲豹* 应该聚在一起，因为它们都有条纹）。这种**模态鸿沟**直接推动了后续研究的发展。
## 4. 生成式 ZSL：合成未见类的视觉特征

判别式 ZSL 学习一个固定的兼容性函数，寄希望于它能泛化到未见类。生成式 ZSL 则反其道而行之：**根据语义描述直接生成未见类的视觉特征**，然后用真实已见类和合成未见类的特征一起训练一个普通的有监督分类器。这样一来， ZSL 就重新回归了有监督学习的范畴。

### f-CLSWGAN （Xian et al., 2018）

这个模型是一个带分类器引导的条件 Wasserstein GAN：

- **生成器** $G(z, a_c) \to \tilde{x}$。输入是噪声 $z \sim \mathcal{N}(0, I)$ 和类别语义 $a_c$，输出是合成的 CNN 特征。
- **评论器** $D(x)$ 负责区分真实特征和合成特征（不使用 sigmoid，采用 Wasserstein 损失）。
- **辅助分类器** $\mathrm{cls}$ 基于生成的特征构建，并用真实已见类的特征进行训练。

总目标函数如下：

$$\mathcal{L} = \underbrace{\mathbb{E}[D(\tilde{x})] - \mathbb{E}[D(x)] + \lambda\,\mathrm{GP}}_{\text{WGAN-GP}}
\;+\; \beta \cdot \underbrace{\mathbb{E}\bigl[-\log P(y \mid \tilde{x})\bigr]}_{\text{分类损失}},$$

其中 $\mathrm{GP}$ 是标准的梯度惩罚项 $\mathbb{E}[(\|\nabla_{\hat{x}} D(\hat{x})\|_2 - 1)^2]$。分类损失是关键所在——它不仅要求合成特征看起来真实，还必须具备**类别区分能力**。

**测试阶段**：对每个未见类采样 $\tilde{x}^{(u)} \sim G(z, a_u)$，然后用 $\{(x, y) : y \in \mathcal{C}^s\} \cup \{(\tilde{x}^{(u)}, u)\}$ 训练一个 softmax 分类器即可。仅这一步改进， AwA2 数据集上的 GZSL 调和平均值就从约 22 提升到了约 58。

### f-VAEGAN-D2

Xian 等人（2019）在 f-CLSWGAN 的基础上增加了一个 VAE 编码器来稳定训练，同时引入第二个判别器，利用**未标注的测试图像**（transductive）。这套方法的核心思路——"VAE 稳定训练 + GAN 提升特征质量 + 分类器保证可分性"——如今已成为特征生成式 ZSL 的标配方案。

## 5. 广义零样本学习：偏置问题

传统 ZSL 基准测试把标签空间限制在 $\mathcal{C}^u$，这其实掩盖了真正的难点：实际部署时，我无法提前知道一张图片属于已见类还是未见类。

如果把标签空间扩展到 $\mathcal{C}^s \cup \mathcal{C}^u$，所有只用已见类图像训练的模型都会对已见类产生强烈偏置。已见类的视觉特征落在 $F$ 熟悉的区域内；而未见类的特征稍微偏离分布，几乎总是输在 argmax 上。

标准评估指标是已见类和未见类准确率的**调和平均数**：

$$H = \frac{2 \cdot S \cdot U}{S + U}.$$

这个公式会惩罚那些"顾此失彼"的方法。比如一个模型的 $S=88, U=12$，算出来 $H=21$ —— 非常糟糕。

![GZSL 偏置问题与三种解决方法](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig6_gzsl_vs_zsl.png)

三种解决方法：

**1. 校准堆叠（Chao et al., 2016）。** 给所有已见类的得分减去一个固定值：

$$F_{\text{cal}}(x, c) = F(x, c) - \gamma \cdot \mathbb{1}[c \in \mathcal{C}^s].$$

$\gamma$ 在验证集上调一下就行。简单、有效，是个很强的基线。

**2. 生成式特征合成**（见第 4 节）。一旦能生成未见类的特征， GZSL 就变成了一个数据均衡的有监督分类问题。

**3. OOD 门控。** 训练一个二分类器判断"这张图属于已见类还是未见类"，然后分发给对应的子分类器。整体性能受限于这个 OOD 检测器的能力。

## 6. CLIP 与视觉-语言预训练时代

回头看，**CLIP**（Radford et al., 2021）其实就是第 3 节提到的深度双塔模型 —— 只不过它用了**从网上爬取的 4 亿对图文数据**进行训练，损失函数是双向对比交叉熵：

$$\mathcal{L} = -\sum_i \log \frac{\exp\bigl(I_i \cdot T_i / \tau\bigr)}{\sum_j \exp\bigl(I_i \cdot T_j / \tau\bigr)} \;+\; (\text{文本侧对称项}).$$

零样本推理时，直接用文字动态构建分类器。比如面对 $K$ 类问题，先写一个提示模板，比如 `"a photo of a {class}"`，然后用文本塔编码 $K$ 个提示，得到 $K$ 个文本向量。接着把新图像输入视觉塔，得到图像向量，最后选最接近的那个提示对应的类别。

![CLIP 零样本分类流程](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig5_clip_zero_shot.png)

有两点影响特别值得关注：

1. **告别语义工程**。类别用自然语言描述就行。想加新类？写一句话就搞定。
2. **GZSL 不再是独立问题**。 CLIP 在训练集中见过和没见过的类别上表现差不多，因为它从一开始就没有按类别判别的方式训练。

基准测试结果也印证了这一趋势：

![AwA2 与 CUB 零样本基准的演进](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/transfer-learning/07-零样本学习/fig7_benchmark_results.png)

DAP （2009）→ ALE / DeViSE （2013）：双线性与排序兼容性； SAE （2017）：语义自编码器； f-CLSWGAN （2018）→ CADA-VAE （2019）：生成式特征合成缩小了 GZSL 的差距； CLIP （2021）：大规模对比预训练重新定义了上限。

## 7. 实现

这里是一个精简的 PyTorch 实现，包含深度兼容性模型和特征生成 GAN 的核心结构。完整的训练代码（包括 AwA2 数据加载器、 ZSL 和 GZSL 评估）可以在文末的参考实现中找到。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class DeepCompatibility(nn.Module):
    """DeViSE / ALE / CLIP 使用的视觉-语义双塔兼容性模型。"""

    def __init__(self, visual_dim: int, semantic_dim: int,
                 embed_dim: int = 512, temperature: float = 10.0):
        super().__init__()
        self.vis = nn.Sequential(
            nn.Linear(visual_dim, 1024), nn.ReLU(),
            nn.Dropout(0.5), nn.Linear(1024, embed_dim),
        )
        self.sem = nn.Sequential(
            nn.Linear(semantic_dim, 512), nn.ReLU(),
            nn.Dropout(0.5), nn.Linear(512, embed_dim),
        )
        self.tau = temperature

    def forward(self, x: torch.Tensor, a: torch.Tensor) -> torch.Tensor:
        """为每张图像对每个类别打分，返回 [B, C] logits。"""
        v = F.normalize(self.vis(x), dim=-1)        # [B, d]
        s = F.normalize(self.sem(a), dim=-1)        # [C, d]
        return self.tau * v @ s.t()

class FeatureGenerator(nn.Module):
    """f-CLSWGAN 风格的条件生成器。"""

    def __init__(self, noise_dim: int, semantic_dim: int, feature_dim: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(noise_dim + semantic_dim, 4096), nn.LeakyReLU(0.2),
            nn.Linear(4096, feature_dim), nn.ReLU(),
        )

    def forward(self, z: torch.Tensor, a: torch.Tensor) -> torch.Tensor:
        return self.net(torch.cat([z, a], dim=-1))

def gzsl_evaluate(model: DeepCompatibility, features: torch.Tensor,
                  labels: torch.Tensor, attrs: torch.Tensor,
                  seen_ids: torch.Tensor, unseen_ids: torch.Tensor,
                  gamma: float = 0.0) -> tuple[float, float, float]:
    """在 GZSL 测试集上计算 (S, U, H)，支持校准堆叠。"""
    model.eval()
    with torch.no_grad():
        logits = model(features, attrs)                 # [N, C]
        logits[:, seen_ids] -= gamma                    # 已见类降权
        pred = logits.argmax(dim=-1)
        seen_mask = torch.isin(labels, seen_ids)
        S = (pred[seen_mask] == labels[seen_mask]).float().mean().item()
        U = (pred[~seen_mask] == labels[~seen_mask]).float().mean().item()
    H = 2 * S * U / (S + U + 1e-8)
    return S, U, H
```

`gzsl_evaluate` 中的 `gamma` 参数实现了校准堆叠 —— 这是解决 GZSL 问题时最简单且值得优先尝试的方法。
## 8. Q&A

**Q1：什么时候该用 ZSL 而不是小样本学习或主动学习？**  
三种情况适合选 ZSL：(a) 长尾分布又宽又不稳定，新类别出现的速度比标注还快；(b) 手头已经有丰富的语义信息，比如属性分类体系、产品目录或者文本描述；(c) 可以直接用预训练的视觉-语言模型（CLIP、 SigLIP、 OpenCLIP），完全跳过 ZSL 的专用机制。

**Q2：属性、词嵌入、句嵌入和 CLIP 提示，哪个更好？**  
属性单维区分度最高，但需要专家设计。词嵌入扩展性强，但它反映的是语言相似性，而不是视觉相似性。句嵌入是个折中方案。 CLIP 提示在大多数自然图像基准上表现最强，因为它背后的视觉和文本编码器是联合训练的。

**Q3：为什么 GZSL 不校准就会崩？**  
训练目标只奖励已见类别的正确预测，导致已见类和未见类的 logit 分数差距没有对齐。测试时，已见类分数系统性偏高，总是抢走 argmax。校准堆叠、生成合成和 OOD 门控其实都在解决这个不平衡问题。

**Q4： CLIP 出现后，传统 ZSL 研究还有意义吗？**  
对于自然图像来说，基本没太大意义了。但在开源数据稀缺的领域，比如医学影像、工业缺陷检测、卫星图像，老式的语义属性方法仍然是数据效率最高的冷启动方式。而且 CLIP 本质上是一个深度双塔兼容性函数，本文的概念框架依然适用。

**Q5：如何避免 Hubness 问题？**  
Hubness 是高维空间的一种病态现象：少数“枢纽”原型成了太多查询的最近邻。解决方法有几种：打分前对两边都做 L2 归一化（上面代码里已经提到了）；用基于排名的打分方法（CSLS、 ZestNorm）；或者对每个类别的得分做标准化处理。
