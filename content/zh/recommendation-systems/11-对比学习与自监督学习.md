经典推荐模型只学习一个信号——用户是否点击、观看或购买。虽然这个信号很宝贵，但其稀疏性令人头疼。大部分用户接触的商品不到总目录的 1%，大部分商品被触达的用户也不到 0.1%，新用户和新商品则完全没有记录。直接使用这种稀疏标签优化模型，几乎注定会在头部过拟合，而尾部则毫无反应。

对比学习换了一种思路，它不问“这个样本的标签是什么”，而是问“哪两个样本应该相似，哪两个应该不同”。这个问题的答案可以直接从数据中获得：对同一个用户、商品或序列施加两种不同扰动，得到的两个视图即构成一对正样本。同一批次中的其他样本则全是负样本。模型学习的是嵌入空间的几何结构：语义相似的样本彼此靠近，不相似的则相互远离。一旦学好几何结构，有监督的推荐头只需稍作调整即可。

本文先拆解对比学习的核心组件——InfoNCE、温度系数、增广策略，再详细介绍推荐领域真正实用的四类方法：**SimCLR 风格**的批次内对比、**MoCo 风格**的队列对比、**SGL** 的图增广以及**CL4SRec** 的序列增广。最后，我将介绍一个有点反直觉的结果：**XSimGCL** 完全抛弃了复杂的增广策略，仅在嵌入上加一点噪声，效果反而更好。

## 为什么推荐系统需要对比学习？

### 再谈稀疏性问题

稀疏性不只是“标签太少”这么简单，它本质上是结构性的：

1. **冷启动**：新用户和新物品完全没有交互记录。像矩阵分解、双塔模型、 GNN 这些依赖交互数据生成特征的模型，根本无法为它们在嵌入空间中找到有意义的位置。
2. **头部过拟合**：点击分布是长尾的，损失函数几乎完全被少数热门物品主导。一个记住了这些热门物品的模型，在训练指标上可能表现不错，但在长尾物品上的效果却一塌糊涂。
3. **热度偏差**：即使召回阶段提供了多样化的候选集，排序模型依然会倾向于把热门物品排到前面——因为这正是训练损失优化的方向。最终，整个系统会陷入“千篇一律”的状态。

### 对比学习的核心逻辑

对比学习做了一个关键权衡：以高成本的监督标签信号为代价，换取大量低成本的‘扰动一致性’信号。假设我随机去掉用户行为图中 20% 的边，编码一次；再换一种方式去掉另外 20% 的边，再编码一次。两次视图，同一个用户。规则很直接：同一用户的两个增广视图对应的嵌入向量应尽可能接近，而同一批次中其他用户的嵌入向量则应尽可能远离。

这个单一目标同时带来了三大好处：

- **免费的训练信号**：数量几乎是无限的（任何用户都可以被扰动任意多次）。
- **表示先验**：模型学到的是那些在扰动下依然稳定的特征——按定义来说就是*鲁棒*的特征，而这正是冷启动和长尾物品最需要的。
- **防止塌缩的正则化**：损失函数明确要求将不同用户的嵌入向量分开，避免它们都挤到热门物品附近。

![锚点用户被拉近两个增广视图（正样本），同时被推离 batch 内其他用户（负样本）](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/11-对比学习与自监督学习/fig1_contrastive_pairs.png)

下图直观呈现了对比学习的核心思想：蓝色锚点代表一个用户，两个绿色点是其两个增广视图，损失函数将它们拉近；琥珀色点代表同一批次中的其他用户视图，损失函数将它们推开。对每个用户、每个批次重复该过程，最终可构建起‘语义相似性与嵌入距离成反比’的几何结构。
## InfoNCE：真正干活的损失函数

![推荐系统（十一）—— 对比学习与自监督学习 — visual](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/11-对比学习与自监督学习/illustration_2.png)

几乎所有对比推荐模型都用到了 **InfoNCE** 损失（van den Oord 等， 2018）的某种变体。给定一个锚点 $x$、正样本 $x^+$ 和一组负样本 $\{x_i^-\}$，编码器 $f$，以及相似度函数 $\mathrm{sim}(\cdot,\cdot) = z\cdot z'$（基于 $\ell_2$ 归一化的嵌入向量点积），公式如下：

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp\!\big(\mathrm{sim}(f(x), f(x^+)) / \tau\big)}{\exp\!\big(\mathrm{sim}(f(x), f(x^+)) / \tau\big) + \sum_{i} \exp\!\big(\mathrm{sim}(f(x), f(x_i^-)) / \tau\big)}$$

这本质上是一个 $(N+1)$ 分类问题的交叉熵，目标是让模型预测"正样本"为正确类别。分子的作用是拉近正样本的距离；分母则迫使模型将正样本排在所有负样本之前。这种排序压力避免了所有嵌入向量塌缩到同一个点的平凡解。

### 温度系数：最容易被忽视的关键参数

温度 $\tau$ 决定了 softmax 区分正负样本的锐利程度。它不是一个无关紧要的超参数，而是直接影响损失优化的目标。

![左：不同温度下，InfoNCE 损失随正样本相似度的变化；右：相同情况下损失对正样本相似度的梯度](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/11-对比学习与自监督学习/fig2_contrastive_loss.png)

从右边的图中可以得出两点结论：

- **低温度（$\tau=0.05$）**：梯度在决策边界附近非常陡峭。模型几乎把所有容量都集中在最难的负样本上——那些与正样本相似度接近的样本。这种方式适合细粒度区分，但如果这些困难负样本实际上是误标的数据（比如缺失非随机点击），训练会变得不稳定。
- **高温度（$\tau=1.0$）**：梯度均匀分布在所有负样本上，包括容易区分的样本。优化过程更平滑，但嵌入向量的聚类效果会变差。
- **推荐系统的最佳温度范围大约是 $\tau \in [0.1, 0.2]$**：既能学到有用的对比信息，又不会过度追逐噪声。

一个简单的理解方式是：$1/\tau$ 可以看作"放大倍数"。将 $\tau$ 减半，每对样本的相似度差距会被放大一倍，推动近正样本和近负样本分开的梯度压力也随之翻倍。

### 为什么负样本不可或缺

若仅优化分子项（即单纯拉近正样本距离），所有嵌入向量将坍缩至同一常向量，损失迅速收敛至零。负样本并非可有可无，而是确保学习有效的关键约束。这也解释了为什么在 SimCLR 风格的训练中， batch size 的影响如此显著： batch size 为 256 时，每个锚点有 510 个负样本； batch size 为 4096 时，负样本数量增加到 8190。负样本越多，分母越密集，对比信号也越强。

## SimCLR vs MoCo：负样本从哪里来？

视觉自监督学习的两大范式被直接搬到了推荐系统中：

![SimCLR 使用 batch 内其他视图作为负样本（单一共享编码器）；MoCo 维护一个动量更新的 key 编码器和 FIFO 队列缓存负样本](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/11-对比学习与自监督学习/fig3_simclr_vs_moco.png)

**SimCLR**（Chen 等， ICML 2020）设计简单：一个编码器、一个投影头、每个样本两次数据增强，**同一 minibatch 中的其他视图都作为负样本**。缺点很明显——负样本数量受限于 GPU 显存能容纳的 batch 大小。

**MoCo**（He 等， CVPR 2020）将两者解耦： query 编码器通过 SGD 更新，*动量* key 编码器以指数滑动平均方式更新 $\xi \leftarrow m\xi + (1-m)\theta$，并维护一个 FIFO 队列存储最近 $K$ 个 key 的编码（通常 $K = 65{,}536$）。负样本来自队列，因此 $K$ 和 batch size 没有关系。

在推荐系统中， SimCLR 模式更常见。 CTR/CVR 训练的 batch 本来就大；编码器通常是 GNN，前向计算的主要开销是图遍历，而不是 embedding 查找；维护一个上百万的*用户* key 队列意义不大，因为 embedding 表本身一直在更新，队列里的旧 key 很快就失效了。 MoCo 风格的队列更多出现在长序列检索模型中，这种场景下编码器计算成本高，确实需要分摊到多个锚点上。

### SimCLR 损失的参考实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def info_nce(z1: torch.Tensor, z2: torch.Tensor, tau: float = 0.2) -> torch.Tensor:
    """两视图对称的 SimCLR 风格 InfoNCE。

    z1、z2 形状为 (B, D)，已 L2 归一化。z1[i] 的正样本是 z2[i]，反之亦然；
    拼接后的 2B 大小 batch 中，其他位置都算负样本。
    """
    B = z1.size(0)
    z = torch.cat([z1, z2], dim=0)                  # (2B, D)
    sim = z @ z.T / tau                             # (2B, 2B)
    sim.fill_diagonal_(float("-inf"))               # 排除自身相似性
    # 第 i 行（i<B）的正样本在第 i+B 列；第 i 行（B<=i<2B）的正样本在第 i-B 列
    targets = torch.cat([torch.arange(B, 2 * B), torch.arange(0, B)]).to(z.device)
    return F.cross_entropy(sim, targets)
```

这段十几行的代码里有三个细节值得注意：

1. 输入的 embedding 必须提前归一化。归一化后，点积就是余弦相似度，温度 $\tau$ 才有几何意义。
2. 对角线用 $-\infty$ 屏蔽，而不是 0。`cross_entropy` 在 log-softmax 空间工作，$-\infty$ 不会影响分母；如果用 0，会留下一个 $e^0=1$ 的项，悄悄改变梯度。
3. 损失是对称的：$2B$ 行各贡献一个交叉熵。前 $B$ 行把第二组视图当作目标，后 $B$ 行把第一组视图当作目标。

## SGL：用户-物品图上的对比学习

SGL （Wu 等， SIGIR 2021，*Self-supervised Graph Learning for Recommendation*）是将对比学习引入推荐系统主流的开创性工作。核心思想是在 LightGCN 主干上加一个 InfoNCE 头，把**两个随机扰动后的用户-物品图**作为每个节点的正样本对。

### 三种图扰动方式

![原始用户-物品二部图与 SGL 的三种增广：边丢弃、节点丢弃、随机游走子图](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/11-对比学习与自监督学习/fig4_sgl_augmentations.png)

- **边丢弃（Edge Dropout, ED）**：每条边以概率 $1-p$ 独立保留。实现简单，结构保持良好，是最常用的变体。
- **节点丢弃（Node Dropout, ND）**：每个节点及其所有关联边以概率 $p$ 被移除。增强效果更强，但可能让小度节点不稳定。
- **随机游走子图（Random Walk, RW）**：从每个锚点出发，采样长度为 $L$ 的随机游走路径，生成不同视图。

原论文的消融实验表明，边丢弃在大多数数据集上表现稳定，甚至略优于其他两种方法，且实现最简单。除非有特殊原因，直接用 ED 就行。

### 完整的 SGL 训练步骤

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def edge_dropout(edge_index: torch.Tensor, p: float) -> torch.Tensor:
    keep = torch.rand(edge_index.size(1), device=edge_index.device) > p
    return edge_index[:, keep]

class SGL(nn.Module):
    """LightGCN 主干 + 在两个边丢弃视图上的节点级 InfoNCE。"""

    def __init__(self, n_users, n_items, dim=64, n_layers=3,
                 drop_p=0.1, tau=0.2, lam=0.1):
        super().__init__()
        self.n_users, self.n_items = n_users, n_items
        self.n_layers, self.drop_p, self.tau, self.lam = n_layers, drop_p, tau, lam
        self.user_emb = nn.Embedding(n_users, dim)
        self.item_emb = nn.Embedding(n_items, dim)
        nn.init.normal_(self.user_emb.weight, std=0.1)
        nn.init.normal_(self.item_emb.weight, std=0.1)

    def _propagate(self, x, edge_index):
        """对称归一化的 LightGCN 传播，取各层平均。"""
        row, col = edge_index
        deg = torch.bincount(row, minlength=x.size(0)).clamp(min=1).float()
        norm = deg.pow(-0.5)
        layers = [x]
        for _ in range(self.n_layers):
            msg = x[col] * (norm[row] * norm[col]).unsqueeze(1)
            agg = torch.zeros_like(x).index_add_(0, row, msg)
            x = agg
            layers.append(x)
        return torch.stack(layers, dim=0).mean(dim=0)

    def encode(self, edge_index):
        x = torch.cat([self.user_emb.weight, self.item_emb.weight], dim=0)
        return self._propagate(x, edge_index)

    def cl_loss(self, z1, z2):
        z1, z2 = F.normalize(z1, dim=1), F.normalize(z2, dim=1)
        sim = z1 @ z2.T / self.tau
        targets = torch.arange(z1.size(0), device=z1.device)
        return F.cross_entropy(sim, targets)

    def bpr_loss(self, z, users, pos, neg):
        u, p, n = z[users], z[self.n_users + pos], z[self.n_users + neg]
        return -F.logsigmoid((u * p).sum(-1) - (u * n).sum(-1)).mean()

    def forward(self, edge_index, users, pos_items, neg_items):
        # 两个增广视图，提供对比信号
        z1 = self.encode(edge_dropout(edge_index, self.drop_p))
        z2 = self.encode(edge_dropout(edge_index, self.drop_p))
        # 一次干净前向传播，提供推荐信号
        z = self.encode(edge_index)
        return self.bpr_loss(z, users, pos_items, neg_items) \
             + self.lam * self.cl_loss(z1, z2)
```

几个需要注意的实现细节：

- **三次前向传播，不是两次**。对比视图的边丢弃和 BPR 使用的干净图是独立的。如果让 BPR 共用被丢边的视图，监督信号会被偶然存活的边带偏。
- **对比损失在节点层面计算**，作用对象是用户和物品嵌入的拼接。这样二部图的两边都能获得自监督信号。
- **损失权重 $\lambda$ 很关键**。太小（$<10^{-2}$），对比信号几乎消失；太大（$>1$）， BPR 无法抓住推荐目标。 SGL 论文测试了 $\{0.005, 0.05, 0.1, 0.5, 1.0\}$，发现 $0.1$ 是稳健的默认值——建议从这里开始调参。

## CL4SRec：序列推荐中的对比学习

对于基于序列的推荐模型（如 SASRec、 BERT4Rec、 GRU4Rec 等），核心问题变成了："如何为*同一条行为序列*生成两个视图？" CL4SRec （Xie 等， ICDE 2022）提出了三种数据增强方法，这些方法已经成为默认选择。

![行为序列的三种增强方式： Crop 截取连续片段， Mask 将随机位置替换为 [M]， Reorder 打乱一段连续区间](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/11-对比学习与自监督学习/fig5_cl4srec_augmentations.png)

- **Crop**：保留长度为 $\eta L$ 的连续子序列。这种方式保留了局部顺序，同时让模型学会对"晚开始"或"早结束"保持不变性。
- **Mask**：随机将 $\gamma$ 比例的物品替换为特殊标记 `[M]`。这和 BERT 的 Masked Language Modeling 思路一致——编码器需要从上下文中推断被遮挡的物品。
- **Reorder**：打乱一段长度为 $\beta L$ 的连续区间。这种方法告诉模型，在一个会话中，物品集合本身比它们的*精确位置*更重要。

CL4SRec 每次为每个视图随机选择一种增强方式，因此 (视图 A, 视图 B) 共有 9 种可能的组合。这种随机混合本身就是一种正则化手段，防止编码器过拟合到某一种特定的增强策略。

```python
import random
import torch
import torch.nn as nn
import torch.nn.functional as F

class SeqAugment:
    """CL4SRec 的三种序列增强方法。序列是 LongTensor 类型的物品 ID，
    ID=0 保留用于 padding/mask。"""

    def __init__(self, crop_eta=0.6, mask_gamma=0.3, reorder_beta=0.6, mask_id=0):
        self.crop_eta = crop_eta
        self.mask_gamma = mask_gamma
        self.reorder_beta = reorder_beta
        self.mask_id = mask_id

    def __call__(self, seq: torch.Tensor) -> torch.Tensor:
        op = random.choice(["crop", "mask", "reorder"])
        L = seq.size(0)
        if op == "crop":
            k = max(1, int(L * self.crop_eta))
            s = random.randint(0, L - k)
            return seq[s:s + k]
        if op == "mask":
            out = seq.clone()
            n = max(1, int(L * self.mask_gamma))
            idx = torch.randperm(L)[:n]
            out[idx] = self.mask_id
            return out
        # reorder
        out = seq.clone()
        k = max(2, int(L * self.reorder_beta))
        s = random.randint(0, L - k)
        chunk = out[s:s + k][torch.randperm(k)]
        out[s:s + k] = chunk
        return out
```

编码器可以是你已有的任何序列模型（如 Transformer、 GRU 等）。将最后一层隐藏状态池化后，进行投影和归一化处理，然后丢进之前实现的 `info_nce` 函数中，作为辅助损失加到常规的 next-item 预测任务上即可。
## XSimGCL：图增广本身并不重要

Yu 等人（2022/2023）的一项实验结果令人惊讶，值得单独讨论。他们提出了一个问题： SGL 的性能提升有多少来自*图增广*，又有多少来自对比损失本身？答案是：几乎完全来自对比损失。将图丢边替换为**在传播后的嵌入向量上添加少量均匀噪声**，效果可以持平甚至超越 SGL，同时还省去了图扰动相关的所有复杂操作。

具体方法（SimGCL / XSimGCL，后者是简化版）大致如下：

1. 正常运行 LightGCN 的传播过程。
2. 在每一层，向嵌入向量添加一个小噪声 $\Delta$。噪声方向是一个单位范数的随机向量，缩放比例为小值 $\epsilon$（例如 0.1），并且确保噪声与原向量位于同一半球（避免翻转）。
3. 使用不同的噪声样本重新运行一次传播，得到第二个视图。无需图丢边。

```python
def add_noise(x: torch.Tensor, epsilon: float = 0.1) -> torch.Tensor:
    """SimGCL 的同半球均匀噪声扰动。"""
    noise = torch.rand_like(x)
    noise = F.normalize(noise, dim=-1) * torch.sign(x) * epsilon
    return x + noise
```

SimGCL 的分析揭示了一个更深层次的道理：对比损失实际上同时完成了两件事——

- **对齐（Alignment）**：拉近正样本对的距离（分子的作用）。
- **均匀性（Uniformity）**：推动所有嵌入向量在单位超球面上均匀分布（分母的作用）。

其中，均匀性是打破热度偏差的关键正则化手段，也是**主导效应**。一旦实现了均匀性，生成第二个视图的具体方式（图丢边、嵌入噪声或其他合理方法）就变得无关紧要了。

## 常见问题

### 为什么不用更多数据，而非要用对比学习？

冷启动用户拿不到更多数据——他们“冷”就是因为没数据。长尾物品也一样，它们“长尾”就是因为几乎没人和它们交互。对比学习不需要新的交互数据，而是通过对已有交互做扰动生成训练信号。这和单纯收集更多点击是*完全不同的思路*。

### 图增强和嵌入噪声怎么选？

优先选择嵌入噪声（XSimGCL）：速度快、实现简单，而且近期文献一致表明它的效果不输图增强，甚至更好。只有在需要正则化 GNN 的结构归纳偏置，或者对哪些边/节点信息最有价值有明确先验时，才考虑用图增强（SGL）。

### 温度系数怎么设置？

从 $\tau = 0.2$ 开始。如果最难的负样本基本是真实的负样本（比如有明确的停留时间信号），可以将 $\tau$ 调低到 0.05–0.1 来增强区分度；如果负样本噪声较大（比如高维目录中随机采样的同批次用户），保持 $\tau \geq 0.2$ 避免过拟合假负样本。在小范围内调参即可，损失函数对 $\tau$ 的变化比较平滑。

### 是否需要投影头？

SimCLR / SGL 在 GNN 上使用时需要投影头——但预训练完成后就可以丢弃。投影头的作用是让编码器的中间表示保持通用性，而把对比任务的特化交给投影头。 XSimGCL 是个例外，它直接对比传播后的 GNN 嵌入，没有投影头也能正常工作。

### 对比损失和推荐损失如何结合？

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{rec}} + \lambda \cdot \mathcal{L}_{\text{CL}}$$

初始值设为 $\lambda = 0.1$。如果对结果敏感，可以在验证集上扫 $\{0.01, 0.05, 0.1, 0.5, 1.0\}$。相比 $\tau$，$\lambda$ 的敏感度低得多，别在这上面浪费调参预算。

### 对隐式反馈是否有效？

不仅有效，而且可以说*比显式反馈更合适*。整个框架默认每个交互都是正样本，通过 InfoNCE 结构隐式处理负样本。 SGL、 SimGCL、 XSimGCL 和 CL4SRec 都是为隐式反馈（点击、播放、购买）设计的。

### 数据量少到什么程度还能用？

对比学习在监督信号稀缺时帮助最大。 SGL 在 Yelp2018 （稀疏度约 99.87%）上的提升比在稠密数据集上更显著。低于 ~10K 交互时，随机增强的方差会占主导，可能需要更强的先验（跨域迁移、内容特征）；超过 ~100K 交互时，应该能看到稳定的改进。

### 如何评估对比学习推荐系统？

用标准的 top-K 指标（Recall@K、 NDCG@K、 HR@K）与基线对齐，然后额外关注以下几点：

- **冷启动切片**：按交互次数对用户/物品分桶，分别报告每桶的指标。
- **长尾覆盖**：推荐列表中长尾物品的比例（通常定义为按流行度排序的后 80%）。
- **嵌入诊断**：用 t-SNE/UMAP 可视化，或按 Wang & Isola （2020）计算均匀性（$\log \mathbb{E}\,e^{-2\|z_i - z_j\|^2}$）和对齐性（$\mathbb{E}\,\|z - z^+\|^2$）。均匀性高、对齐性低说明嵌入质量较好。

## 总结

对比学习解决了一个单纯靠增加数据无法解决的问题：它让模型能够从稀疏交互中学习*几何结构*，方法是要求模型在扰动下保持一致性。 2024 年的技术方案已经非常成熟：

1. **选择一种数据增广方式**，或者干脆不用（比如 XSimGCL 的噪声）。
2. **使用 InfoNCE**，设置温度参数 $\tau \approx 0.2$。
3. **将其作为辅助损失加入模型**，权重设为 $\lambda \approx 0.1$，叠加在现有的有监督目标上。

对于基于图的推荐系统，直接从 XSimGCL 开始——这是最简单的有效方案。如果需要更易解释的基线， SGL 加上边丢弃仍然是一个不错的选择。对于序列推荐系统， CL4SRec 提供的三种增广方式已经成为标配。无论哪种场景，最大的收益通常都来自冷启动和长尾部分。

## 离线与在线指标的差异

这是我见过很多团队在尝试上线对比学习推荐系统时最容易踩的坑。

SGL/CL4SRec 的离线表现非常一致：在 Yelp、 Amazon、 MovieLens 数据集上， NDCG@20 提升 5%-15%， Recall@20 提升 8%-20%。论文写得漂亮，结果看起来很棒。但线上 A/B 测试时， CTR 只涨了 +0.3%， 95% 置信区间是 $\pm 0.4\%$。离线效果碾压的对比模型，到了线上几乎没动静。

这是为什么？

**离线评测基于留出的点击数据**。这些点击数据本身就有偏差——它们来自一个已经偏向热度的推荐系统，通常是流行度倾斜的 CF 基线模型。当你的对比模型通过更锐利的嵌入向量召回更多长尾物品时，离线评测反而会惩罚它，因为这些物品在历史日志中根本没有机会被点击。离线 NDCG 实际上奖励的是“预测前一个系统已经展示过的内容”。

**对比学习的收益集中在冷启动和长尾部分**。 SGL 提升了嵌入空间的均匀性（Wang & Isola 2020）。交互少的物品得到了与热门物品更可区分的隐向量。但在离线评测中，这些长尾物品几乎没有正样本标签。在线上 A/B 测试中，它们以新曝光的形式出现，获得了一些点击，但整体 CTR 仍然由头部物品主导，而 SGL 对头部物品的影响微乎其微。

**位置偏差主导短期指标**。一个新模型把前 5 个推荐结果重排一位，对 CTR 的影响可能比修复冷启动问题的模型更大。在大多数推荐场景中，位置 1 和位置 2 的 CTR 比例大约是 1.6 倍。

怎么办？

1. 离线评测改用反事实估计（IPS 加权 NDCG），而不是普通的 NDCG。这能弥补大部分差距。
2. 按物品流行度五分位拆分 A/B 测试结果。即使大盘指标持平， SGL 在最低两个五分位上通常能带来 +3%-8% 的 CTR 提升。
3. 实验至少跑 14 天。冷启动的收益会逐步累积：今天拿到曝光的新物品会产生互动数据，这些数据会优化下一轮训练。

如果只能汇报一个数字，那就报**首周活跃物品的提升**（最近 7 天上线的物品）。这才是对比学习真正发挥作用的地方。
## 推理成本：对比学习的轻量一面

CL 的训练成本大家都知道——InfoNCE 加上大批量负采样，计算开销很大。但推理成本却出乎意料：对比学习在线上服务时几乎不增加额外负担。

数据增强、投影头和对比损失都只在训练阶段用到。推理时， SGL 和 LightGCN 在结构上完全一致——相同的 GNN 层数、相同的嵌入维度、相同的点积评分逻辑。 QPS、延迟和内存占用也都一样。

真正的开销在于**训练成本**，这直接影响模型的更新频率。对比学习的训练增加了以下内容：
- 每种数据增强多一次前向传播（通常 2 种增强 → 前向计算翻倍）
- InfoNCE 分母需要一个大小为 (batch, batch) 的负样本矩阵 → 计算复杂度与 batch size 成平方关系

在 1 亿条交互数据的场景下，原版 LightGCN 单卡 A100 跑完训练大约需要 3 小时；而 SGL 使用 2 种增强、 batch size 设置为 4096 时，训练时间延长到约 9 小时。如果生产环境每 4 小时重训一次模型，这个成本是不可忽视的。优化方案有两种：一是用 momentum queue （类似 MoCo 的方法）缩小 batch size，二是直接去掉数据增强（XSimGCL——对嵌入加随机噪声）。

XSimGCL 才是大多数团队应该选择的方案。它能带来和 SGL 相同的 uniformity 提升效果，性能只比原版 LightGCN 慢约 30%，而不是 3 倍。
