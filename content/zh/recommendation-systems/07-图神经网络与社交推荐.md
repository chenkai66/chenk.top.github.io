## 为什么用 GNN 做推荐？

### 推荐数据天生就是一张图

![用户-物品二部图：高亮的边显示 Alice 与 Bob 共享两部电影，构成隐式的协同信号](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig1_bipartite_graph.png)

随便找一个推荐系统，仔细看看它的交互矩阵：行是用户，列是物品，每个非零值代表一次点击、观看、购买或评分。将这个矩阵重新画成 **二部图**：

- **节点** 是用户和物品。
- **边** 将用户连接到他们交互过的所有物品。
- **协同信号** 藏在图的结构中：交互过相似物品的用户通过这些共享物品被隐式关联。

如图所示， Alice 和 Bob 并无直接交互，但通过共同交互的两部电影在图中建立了隐式关联。 GNN 可以沿着图传播这个信号："Bob 喜欢《银翼杀手》； Bob 和 Alice 因为共享电影在图上接近；所以 Alice 很可能也喜欢《银翼杀手》。" 矩阵分解无法建模这种跨用户-物品路径的推理——它仅基于交互矩阵的单个元素进行优化。

### 传统方法漏掉了什么？

| 方法 | 捕捉了什么 | 漏掉了什么 |
|---|---|---|
| **矩阵分解** | 用户/物品的隐向量，点积得分 | 每个嵌入独立学习，没有“邻居的邻居”概念 |
| **神经协同过滤 / 自编码器** | 非线性交互 | 仍然按 (user, item) 对处理，无法沿图传播 |
| **GNN** | 节点嵌入由其 $L$ 跳邻域塑造 | （扩展性和过平滑问题——后面会解决） |

一句话总结：**传统方法孤立地学习节点嵌入，而 GNN 在图结构上下文中联合学习节点嵌入。**

### GNN 的核心能力

- **显式建模图结构** —— 直接操作拓扑。
- **邻域聚合** —— 协同信号作为架构的副产品自然传播。
- **多跳推理** —— 堆两层就能捕捉“喜欢与你喜欢的物品相似的物品的用户”。
- **归纳式（GraphSAGE）** —— 不用重训也能给新用户/新物品生成嵌入。
- **异构图** —— 把社交关系、知识图谱、属性融入同一张图。

## 图神经网络基础

### 图的基本概念

图 $G=(V,E)$ 包含顶点集合 $V=\{v_1,\dots,v_n\}$ 和边集合 $E\subseteq V\times V$。**邻接矩阵** $A\in\{0,1\}^{n\times n}$ 中，当且仅当 $v_i$ 和 $v_j$ 之间有边时，$A_{ij}=1$。

推荐系统中常见的三种图类型如下：

| 类型 | 描述 | 示例 |
|---|---|---|
| **二部图** | 两种节点类型，边只连接不同类型 | 用户 ↔ 物品 |
| **加权图** | 边权重表示强度 | 评分、点击次数 |
| **属性图** | 节点带有特征向量 | 用户画像、物品类目 |

### 一张图讲明白消息传递

![消息传递的三步：邻居生成消息 → 目标节点聚合 → 更新自身状态](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig2_message_passing.png)

不管包装得多复杂，所有 GNN 都遵循同一个三步流程：

1. **生成消息。** 每个邻居计算要发送的消息。
2. **聚合消息。** 目标节点汇总收到的消息（求和、取平均、注意力加权等）。
3. **更新状态。** 目标节点将聚合结果与自身上一状态结合。

形式化地，在第 $l$ 层，节点 $v$ 的更新公式为：

$$\mathbf{m}_v^{(l)} = \mathrm{AGGREGATE}^{(l)}\!\bigl(\{\mathbf{h}_u^{(l-1)} : u\in\mathcal{N}(v)\}\bigr)$$$$\mathbf{h}_v^{(l)} = \mathrm{UPDATE}^{(l)}\!\bigl(\mathbf{h}_v^{(l-1)},\; \mathbf{m}_v^{(l)}\bigr)$$
简单来说就是：*"看看邻居知道什么，总结一下，再和自己已知的信息混合。"* 堆叠 $L$ 层后，每个节点就能感知到它的 $L$ 跳邻居范围。

**电话游戏的比喻。** 想象一个电话游戏：每个人同时悄悄告诉所有朋友自己的信息。一轮后，你知道了朋友的想法；两轮后，你知道了朋友的朋友的想法。 GNN 通过可学习、可微分的神经网络函数实现了这一过程。
## 图卷积网络（GCN）

![推荐系统（七）—— 图神经网络与社交推荐 — visual](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/illustration_2.png)

### 核心思想

GCN （Kipf & Welling， 2017）在图上定义了一种类似卷积的操作。 CNN 通过聚合局部像素完成特征提取，而 GCN 则通过聚合节点的邻居特征向量来实现。

### GCN 层

单层 GCN 的计算公式如下：
$$\mathbf{H}^{(l+1)} = \sigma\!\Bigl(\tilde{D}^{-1/2}\, \tilde{A}\, \tilde{D}^{-1/2}\, \mathbf{H}^{(l)}\, \mathbf{W}^{(l)}\Bigr)$$
乍一看公式复杂，但拆解后就很清晰了：

| 符号 | 含义 |
|---|---|
| $\tilde{A} = A + I$ | 增加自环后的邻接矩阵（每个节点也考虑自身） |
| $\tilde{D}$ | $\tilde{A}$ 的度矩阵 |
| $\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}$ | 对称归一化，避免高连接度节点主导信息传播 |
| $\mathbf{H}^{(l)}$ | 第 $l$ 层节点特征矩阵（每行是一个节点的嵌入） |
| $\mathbf{W}^{(l)}$ | 可学习权重矩阵 |
| $\sigma$ | 非线性激活函数，通常用 ReLU |

对于单个节点 $v$，公式展开为：
$$\mathbf{h}_v^{(l+1)} = \sigma\!\left(\sum_{u\in\mathcal{N}(v)\cup\{v\}} \frac{1}{\sqrt{d_v\,d_u}}\; \mathbf{h}_u^{(l)}\, \mathbf{W}^{(l)}\right)$$
1. **收集**：获取所有邻居（包括自己）的嵌入。
2. **归一化**：按 $1/\sqrt{d_v\cdot d_u}$ 调整贡献，热门节点的影响被削弱。
3. **变换**：通过共享权重矩阵 $\mathbf{W}$ 进行线性变换。
4. **激活**：使用 ReLU 激活函数。

对称归一化为什么重要？如果不归一化，高连接度节点会在每一层放大数值，导致梯度爆炸或消失。$\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}$ 将边权重调整为 $1/\sqrt{d_v d_u}$，均衡了热门节点的信息收发，同时对应图拉普拉斯的谱分解，理论上更稳定。

### 实现： GCN 层

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch_geometric.utils import add_self_loops, degree

class GCNLayer(nn.Module):
    """单层图卷积。"""

    def __init__(self, in_channels, out_channels):
        super().__init__()
        self.linear = nn.Linear(in_channels, out_channels)

    def forward(self, x, edge_index):
        # 1. 添加自环，让节点也关注自身
        edge_index, _ = add_self_loops(edge_index, num_nodes=x.size(0))

        # 2. 计算对称归一化系数
        row, col = edge_index
        deg = degree(col, x.size(0), dtype=x.dtype)
        deg_inv_sqrt = deg.pow(-0.5)
        deg_inv_sqrt[deg_inv_sqrt == float('inf')] = 0
        norm = deg_inv_sqrt[row] * deg_inv_sqrt[col]

        # 3. 线性变换
        x = self.linear(x)

        # 4. 使用 scatter-add 聚合归一化后的邻居特征
        out = torch.zeros_like(x)
        out.scatter_add_(0, col.unsqueeze(1).expand_as(x), norm.unsqueeze(1) * x[row])
        return out
```

### 多层 GCN

堆叠多层可以扩大感受野：两层覆盖两跳邻居，三层覆盖三跳。但要注意 **过平滑问题**：层数过多时，所有节点的嵌入会趋于一致。推荐场景中， 2~3 层是最佳选择。

```python
class GCN(nn.Module):
    """多层 GCN。"""

    def __init__(self, num_nodes, in_channels, hidden_channels,
                 out_channels, num_layers=2, dropout=0.5):
        super().__init__()
        self.num_layers = num_layers
        self.dropout = dropout

        # 如果节点没有原始特征，用 ID 嵌入代替
        self.embedding = None
        if in_channels == 0:
            self.embedding = nn.Embedding(num_nodes, hidden_channels)
            in_channels = hidden_channels

        self.convs = nn.ModuleList()
        if num_layers == 1:
            self.convs.append(GCNLayer(in_channels, out_channels))
        else:
            self.convs.append(GCNLayer(in_channels, hidden_channels))
            for _ in range(num_layers - 2):
                self.convs.append(GCNLayer(hidden_channels, hidden_channels))
            self.convs.append(GCNLayer(hidden_channels, out_channels))

    def forward(self, x, edge_index):
        if self.embedding is not None:
            x = self.embedding.weight
        for i, conv in enumerate(self.convs):
            x = conv(x, edge_index)
            if i < self.num_layers - 1:
                x = F.relu(x)
                x = F.dropout(x, p=self.dropout, training=self.training)
        return x
```
## 图注意力网络（GAT）

### 为什么需要引入注意力机制？

GCN 默认所有邻居节点同等重要（最多通过度归一化调整）。但实际情况并非如此——比如，我最好的朋友的电影品味显然比一个普通熟人更有参考价值。 GAT （Veličković 等， 2018）通过为每条边学习 **自适应的注意力权重** 来解决这个问题。

### GAT 的工作原理

对于从节点 $j$ 到节点 $i$ 的每条边， GAT 首先计算一个注意力得分：
$$e_{ij} = \mathrm{LeakyReLU}\!\bigl(\mathbf{a}^\top [\mathbf{W}\mathbf{h}_i \,\|\, \mathbf{W}\mathbf{h}_j]\bigr)$$
其中，$\mathbf{W}$ 将节点特征映射到共享空间，$\|$ 表示拼接操作，$\mathbf{a}$ 是可学习的注意力向量。然后，通过 softmax 对邻居节点的得分进行归一化：
$$\alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k\in\mathcal{N}(i)} \exp(e_{ik})}$$
更新后的节点嵌入是邻居特征的加权和：
$$\mathbf{h}_i^{(l+1)} = \sigma\!\left(\sum_{j\in\mathcal{N}(i)\cup\{i\}} \alpha_{ij}\, \mathbf{W}^{(l)}\, \mathbf{h}_j^{(l)}\right)$$
简单来说：*"对每个邻居，先学一个相关性分数，再用这个分数加权平均它们的特征。"*

### 多头注意力机制

和 Transformer 类似， GAT 使用多头注意力来稳定训练过程。每个注意力头独立计算注意力权重和聚合结果，中间层将多个头的结果拼接起来，最后一层则取平均：
$$\mathbf{h}_i^{(l+1)} = \big\|_{k=1}^{K}\; \sigma\!\left(\sum_{j\in\mathcal{N}(i)\cup\{i\}} \alpha_{ij}^{(k)}\, \mathbf{W}^{(k)}\, \mathbf{h}_j^{(l)}\right)$$
### 实现： GAT 层

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import add_self_loops, softmax

class GATLayer(MessagePassing):
    """带多头注意力的 GAT 层。"""

    def __init__(self, in_channels, out_channels, heads=1,
                 dropout=0.0, concat=True):
        super().__init__(aggr='add', node_dim=0)
        self.in_channels = in_channels
        self.out_channels = out_channels
        self.heads = heads
        self.concat = concat
        self.dropout = dropout

        self.lin = nn.Linear(in_channels, heads * out_channels, bias=False)
        self.att = nn.Parameter(torch.empty(1, heads, 2 * out_channels))
        nn.init.xavier_uniform_(self.lin.weight)
        nn.init.xavier_uniform_(self.att)

    def forward(self, x, edge_index):
        edge_index, _ = add_self_loops(edge_index, num_nodes=x.size(0))
        # 投影到多头空间：[N, heads, out_channels]
        x = self.lin(x).view(-1, self.heads, self.out_channels)
        out = self.propagate(edge_index, x=x)
        return out.view(-1, self.heads * self.out_channels) if self.concat else out.mean(dim=1)

    def message(self, x_i, x_j, index):
        # x_i 是目标节点特征，x_j 是源节点特征
        alpha = (torch.cat([x_i, x_j], dim=-1) * self.att).sum(dim=-1)
        alpha = F.leaky_relu(alpha, negative_slope=0.2)
        alpha = softmax(alpha, index)  # 按目标节点归一化
        alpha = F.dropout(alpha, p=self.dropout, training=self.training)
        return x_j * alpha.unsqueeze(-1)
```
## GraphSAGE：可扩展的归纳学习

### GCN 的问题

GCN 是 **直推式** 的：训练时需要整张图，无法为训练后出现的新节点生成嵌入。如果明天有新用户注册， GCN 必须重新训练才能处理。

GraphSAGE （Hamilton 等， 2017）通过 **归纳学习** 解决了这个问题。它不再死记每个节点的固定嵌入，而是学习 *如何聚合邻居信息*。当新节点出现时， GraphSAGE 对其邻居运行相同的聚合逻辑，实时生成嵌入。

### 采样与聚合

GraphSAGE 的核心思想有两个：

1. **邻居采样**：不用所有邻居（计算开销大），每层固定采样 $k$ 个。
2. **可学习的聚合**：用可训练的函数（均值、 LSTM、最大池化）处理采样的邻居。

对节点 $v$，第 $l$ 层公式如下：
$$\mathbf{h}_{\mathcal{N}(v)}^{(l)} = \mathrm{AGGREGATE}^{(l)}\!\bigl(\{\mathbf{h}_u^{(l-1)} : u\in\mathcal{N}_{\text{sampled}}(v)\}\bigr)$$$$\mathbf{h}_v^{(l)} = \sigma\!\bigl(\mathbf{W}^{(l)}\cdot[\mathbf{h}_v^{(l-1)} \,\|\, \mathbf{h}_{\mathcal{N}(v)}^{(l)}]\bigr)$$
简单来说就是："采几个邻居，汇总它们的特征，拼上自己的特征，再过一层线性变换。"

| 聚合器 | 工作原理 |
|---|---|
| **Mean** | 邻居嵌入取平均：$\tfrac{1}{|\mathcal{N}|}\sum_u \mathbf{h}_u$ |
| **Max-pool** | 共享 MLP 后逐元素取最大：$\max(\sigma(\mathbf{W}_{\text{pool}}\mathbf{h}_u + \mathbf{b}))$ |
| **LSTM** | 把邻居当作序列输入 LSTM （对顺序敏感） |

### 三种方法对比

![GCN vs GAT vs GraphSAGE：等权 / 学习注意力 / 采样后聚合](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig3_gcn_gat_sage.png)

GCN 给所有邻居同等权重（按度归一化）； GAT 学习重要邻居并提升它们的权重； GraphSAGE 直接采样一部分邻居，在大规模图上也能高效运行。

### 实现： GraphSAGE 层

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import add_self_loops

class SAGEConv(MessagePassing):
    """采用均值聚合的 GraphSAGE 层。"""

    def __init__(self, in_channels, out_channels, normalize=False):
        super().__init__(aggr='mean')
        self.normalize = normalize
        self.lin_self = nn.Linear(in_channels, out_channels)   # 自身特征
        self.lin_neigh = nn.Linear(in_channels, out_channels)  # 邻居特征

    def forward(self, x, edge_index):
        edge_index, _ = add_self_loops(edge_index, num_nodes=x.size(0))
        neigh_agg = self.propagate(edge_index, x=x)
        out = self.lin_self(x) + self.lin_neigh(neigh_agg)
        return F.normalize(out, p=2, dim=1) if self.normalize else out

    def message(self, x_j):
        return x_j
```

能为新节点实时生成嵌入，是 GraphSAGE 在生产环境中流行的主要原因。 Pinterest 的 PinSage 就是它的直接改进版。
## PinSage： Pinterest 的十亿级推荐系统

PinSage （Ying 等， 2018）是 Pinterest 生产环境中的 GNN 模型，运行在 **30 亿节点、 180 亿边** 的大规模图数据上。它的高效运行得益于三个关键创新：

### 1. 随机游走采样

PinSage 不再均匀采样邻居节点，而是从每个节点出发执行短随机游走，保留访问频率最高的 top-$k$ 邻居。这种方法关注的是最重要的邻居，而不仅仅是最近的邻居。

### 2. 重要性加权聚合

邻居特征根据随机游走的访问次数进行加权：
$$\mathbf{h}_v^{(l)} = \sigma\!\Bigl(\mathbf{W}^{(l)}\cdot \bigl[\mathbf{h}_v^{(l-1)} \,\big\|\, \mathrm{AGG}\bigl(\{\alpha_{uv}\, \mathbf{h}_u^{(l-1)} : u\in\mathcal{N}_{\text{top-}k}(v)\}\bigr)\bigr]\Bigr)$$
### 3. 困难负样本挖掘

训练时， PinSage 会采样 *hard negatives* —— 这些样本得分较高但并非真正的正样本。这种策略迫使模型做出更精细的区分，而不是简单地将明显无关的样本与正样本分开。

### 实现： PinSage 卷积

```python
import torch
import torch.nn as nn
import numpy as np
from collections import Counter, defaultdict

class PinSageConv(nn.Module):
    """带重要性加权聚合的 PinSage 卷积模块。"""

    def __init__(self, in_channels, out_channels):
        super().__init__()
        self.aggregator = nn.Sequential(
            nn.Linear(in_channels, out_channels),
            nn.ReLU(),
            nn.Linear(out_channels, out_channels),
        )
        self.combine = nn.Linear(in_channels + out_channels, out_channels)

    def forward(self, x, edge_index, importance_weights=None):
        row, col = edge_index
        neighbor_feats = x[col]
        if importance_weights is not None:
            neighbor_feats = neighbor_feats * importance_weights.unsqueeze(1)

        aggregated = torch.zeros(x.size(0), x.size(1), device=x.device)
        aggregated.scatter_add_(
            0, row.unsqueeze(1).expand_as(neighbor_feats), neighbor_feats
        )
        aggregated = self.aggregator(aggregated)
        return self.combine(torch.cat([x, aggregated], dim=1))

def compute_random_walk_importance(adj_list, num_nodes, num_walks=10,
                                   walk_length=5, top_k=10):
    """通过短随机游走计算邻居重要性。"""
    visit_counts = Counter()
    for start in range(num_nodes):
        if start not in adj_list:
            continue
        for _ in range(num_walks):
            current = start
            for _ in range(walk_length):
                neighbors = adj_list.get(current, [])
                if not neighbors:
                    break
                current = np.random.choice(neighbors)
                visit_counts[(start, current)] += 1

    per_source = defaultdict(list)
    for (src, tgt), cnt in visit_counts.items():
        per_source[src].append((tgt, cnt))

    scores = {}
    for src, neighbors in per_source.items():
        neighbors.sort(key=lambda t: t[1], reverse=True)
        top = neighbors[:top_k]
        total = sum(c for _, c in top)
        for tgt, cnt in top:
            scores[(src, tgt)] = cnt / total if total > 0 else 0
    return scores
```

## LightGCN：少即是多

### 一个令人意外的发现

LightGCN （He 等， 2020）提出了一个大胆的问题：*"如果把 GCN 中所有可学习权重矩阵、非线性激活函数、自环都去掉，只保留邻居聚合，会怎么样？"*

答案是：在协同过滤任务上，**效果反而更好**。图结构本身就包含了足够的信息，而标准 GCN 的特征变换和激活函数实际上是在 *添乱*——它们增加了不必要的复杂性，还容易导致过拟合。

![标准 GCN 保留自环、权重、ReLU；LightGCN 只保留聚合和层组合](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig4_lightgcn_simplify.png)

### 架构设计

LightGCN 使用了最简单的聚合方式：
$$\mathbf{e}_u^{(l+1)} = \sum_{i\in\mathcal{N}(u)} \frac{1}{\sqrt{|\mathcal{N}(u)|\cdot|\mathcal{N}(i)|}}\; \mathbf{e}_i^{(l)}$$$$\mathbf{e}_i^{(l+1)} = \sum_{u\in\mathcal{N}(i)} \frac{1}{\sqrt{|\mathcal{N}(u)|\cdot|\mathcal{N}(i)|}}\; \mathbf{e}_u^{(l)}$$
简单来说：*"对邻居的嵌入向量按度归一化后取平均值。没有激活函数，也没有权重矩阵。这就是一层。"*

最终的嵌入向量是各层的加权组合：
$$\mathbf{e}_u = \sum_{l=0}^{L} \alpha_l\, \mathbf{e}_u^{(l)}, \qquad \mathbf{e}_i = \sum_{l=0}^{L} \alpha_l\, \mathbf{e}_i^{(l)}$$
通常取 $\alpha_l = \tfrac{1}{L+1}$（等权重）。第 0 层是原始嵌入，第 1 层是一跳邻居，第 2 层是两跳邻居……组合起来形成多尺度表示，避免了单一深度的局限性。

### 为什么有效？

- **平滑效应。** 聚合操作让相似节点的嵌入向量更接近，这正是协同过滤需要的效果。
- **层组合。** 混合各层信息，既保留了直接信号（第 0 层），又引入了高阶协同信号（深层），同时缓解了过平滑问题。
- **简洁即正则。** 参数少，模型在稀疏交互数据上更不容易过拟合。

### 实现： LightGCN

```python
import torch
import torch.nn as nn
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import degree

class LightGCNLayer(MessagePassing):
    """LightGCN 的一层：归一化邻居聚合，仅此而已。"""

    def __init__(self):
        super().__init__(aggr='add')

    def forward(self, x, edge_index):
        row, col = edge_index
        deg = degree(col, x.size(0), dtype=x.dtype)
        deg_inv_sqrt = deg.pow(-0.5)
        deg_inv_sqrt[deg_inv_sqrt == float('inf')] = 0
        norm = deg_inv_sqrt[row] * deg_inv_sqrt[col]
        return self.propagate(edge_index, x=x, norm=norm)

    def message(self, x_j, norm):
        return norm.unsqueeze(1) * x_j

class LightGCN(nn.Module):
    """完整的 LightGCN 协同过滤模型。"""

    def __init__(self, num_users, num_items, embedding_dim=64, num_layers=3):
        super().__init__()
        self.num_users = num_users
        self.num_items = num_items
        self.num_layers = num_layers

        self.user_embedding = nn.Embedding(num_users, embedding_dim)
        self.item_embedding = nn.Embedding(num_items, embedding_dim)
        nn.init.normal_(self.user_embedding.weight, std=0.1)
        nn.init.normal_(self.item_embedding.weight, std=0.1)

        self.convs = nn.ModuleList([LightGCNLayer() for _ in range(num_layers)])

    def forward(self, edge_index):
        x = torch.cat([self.user_embedding.weight, self.item_embedding.weight], dim=0)
        layer_embeddings = [x]  # 第 0 层 = 原始嵌入
        for conv in self.convs:
            x = conv(x, edge_index)
            layer_embeddings.append(x)

        # 各层等权组合
        final = torch.stack(layer_embeddings, dim=0).mean(dim=0)
        return final[:self.num_users], final[self.num_users:]

    def predict(self, user_emb, item_emb, user_ids, item_ids):
        return (user_emb[user_ids] * item_emb[item_ids]).sum(dim=1)
```
## NGCF：神经图协同过滤

### 和 LightGCN 的区别

NGCF （Wang 等， 2019）走的是另一条路。它在消息传递中显式加入特征变换和用户-物品交互项，认为这样能学到更丰富的嵌入。 LightGCN 做减法， NGCF 却做加法。

### 消息构造

每层的消息包含一个 **特征交互** 项：
$$\mathbf{m}_{u\leftarrow i} = \frac{1}{\sqrt{|\mathcal{N}(u)|\cdot|\mathcal{N}(i)|}}\;\bigl(\mathbf{W}_1\, \mathbf{e}_i^{(l)} + \mathbf{W}_2\, (\mathbf{e}_i^{(l)} \odot \mathbf{e}_u^{(l)})\bigr)$$
$\odot$ 是逐元素乘法，用来捕捉用户和物品特征的交互。比如，用户在“动作片”维度得分高，物品在同一维度得分也高，相乘后信号会被放大。

聚合之后：
$$\mathbf{e}_u^{(l+1)} = \mathrm{LeakyReLU}\!\bigl(\mathbf{m}_{u\leftarrow u} + \sum_{i\in\mathcal{N}(u)} \mathbf{m}_{u\leftarrow i}\bigr)$$
### 实现： NGCF

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import degree

class NGCFLayer(MessagePassing):
    """带特征变换与交互的 NGCF 层。"""

    def __init__(self, in_channels, out_channels, dropout=0.0):
        super().__init__(aggr='add')
        self.W1 = nn.Linear(in_channels, out_channels, bias=False)
        self.W2 = nn.Linear(in_channels, out_channels, bias=False)
        self.W_self = nn.Linear(in_channels, out_channels, bias=False)
        self.dropout = dropout

    def forward(self, x, edge_index):
        row, col = edge_index
        deg = degree(col, x.size(0), dtype=x.dtype)
        deg_inv_sqrt = deg.pow(-0.5)
        deg_inv_sqrt[deg_inv_sqrt == float('inf')] = 0
        norm = deg_inv_sqrt[row] * deg_inv_sqrt[col]

        self_emb = self.W_self(x)
        neighbor_emb = self.propagate(edge_index, x=x, norm=norm)
        out = F.leaky_relu(self_emb + neighbor_emb, negative_slope=0.2)
        return F.dropout(out, p=self.dropout, training=self.training)

    def message(self, x_i, x_j, norm):
        # W1 * 邻居 + W2 * (用户 . 物品) —— 特征交互
        msg = self.W1(x_j) + self.W2(x_i * x_j)
        return norm.unsqueeze(1) * msg

class NGCF(nn.Module):
    """神经图协同过滤。"""

    def __init__(self, num_users, num_items, embedding_dim=64,
                 num_layers=3, dropout=0.1):
        super().__init__()
        self.num_users = num_users
        self.num_items = num_items
        self.user_embedding = nn.Embedding(num_users, embedding_dim)
        self.item_embedding = nn.Embedding(num_items, embedding_dim)
        nn.init.normal_(self.user_embedding.weight, std=0.1)
        nn.init.normal_(self.item_embedding.weight, std=0.1)

        self.convs = nn.ModuleList([
            NGCFLayer(embedding_dim, embedding_dim, dropout)
            for _ in range(num_layers)
        ])

    def forward(self, edge_index):
        x = torch.cat([self.user_embedding.weight, self.item_embedding.weight], dim=0)
        for conv in self.convs:
            x = conv(x, edge_index)
        return x[:self.num_users], x[self.num_users:]
```

### LightGCN vs NGCF：怎么选？

| | LightGCN | NGCF |
|---|---|---|
| **参数量** | 只有嵌入 | 嵌入 + 权重矩阵 |
| **效果** | 稀疏数据上通常更好 | 特征丰富时表现更强 |
| **过拟合风险** | 低 | 较高（参数多） |
| **训练速度** | 快 | 慢一些 |
| **推荐场景** | 大多数 CF 任务 | 特征密集的任务 |

**默认选择 LightGCN。** 只有在用户/物品特征足够丰富、数据量也大到能支撑额外参数时，才考虑 NGCF。
## 为什么要堆叠层数？多跳聚合

![多跳邻域：目标用户先从交互过的物品收集信息，再从相似用户收集信息，最后从候选物品收集信息](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig5_multihop.png)

画出每一跳的传播路径后，推荐系统为什么需要深度就变得很清楚了：

- **第 1 层** —— 目标用户从自己交互过的物品中提取特征。
- **第 2 层** —— 信号传递到相似用户，这就是经典的协同过滤（CF）那一跳。
- **第 3 层** —— 信号进一步传递到相似用户喜欢的物品，这些就是候选推荐物。

两层网络已经能捕捉“喜欢相似物品的用户”这种矩阵分解（MF）无法表达的模式。三层在稀疏图上偶尔会带来提升，但容易陷入过平滑问题。如果想增加深度又避免性能下降，可以参考 LightGCN 的层组合方法。

## 社交推荐

### 核心思想

朋友会影响你买什么、看什么、听什么。社交推荐在用户-物品图中引入了 **社交边**（好友关系、关注关系、信任关系）。核心假设是：社交关系紧密的用户往往有相似的兴趣。

### 图结构

社交推荐图包含两种边：

- **交互边** $(u_i, i_j)$：用户 $u_i$ 和物品 $i_j$ 之间发生了交互。
- **社交边** $(u_i, u_j)$：用户 $u_i$ 和用户 $u_j$ 是好友。

### 两种机制

**同质性（Homophily）。** 朋友兴趣相似，是因为他们本身背景和文化相近。

**社会影响（Social Influence）。** 朋友会因为彼此的影响，逐渐变得更相似。

对模型来说，这两种机制最终归结为同一个目标：沿着社交边传播用户的偏好。

### 实现： Social GCN

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch_geometric.nn import MessagePassing

class SocialGCNLayer(MessagePassing):
    """同时处理用户-物品边和社交边的 GCN 层。"""

    def __init__(self, in_channels, out_channels):
        super().__init__(aggr='mean')
        self.linear = nn.Linear(in_channels, out_channels)

    def forward(self, x, edge_index_ui, edge_index_social, num_users):
        x_ui = self.propagate(edge_index_ui, x=x)
        x_social = self.propagate(edge_index_social, x=x[:num_users])
        x_combined = x_ui.clone()
        x_combined[:num_users] = x_combined[:num_users] + x_social
        return self.linear(x_combined)

class SocialGCN(nn.Module):
    """基于 GCN 的社交推荐模型。"""

    def __init__(self, num_users, num_items, embedding_dim=64, num_layers=2):
        super().__init__()
        self.num_users = num_users
        self.user_embedding = nn.Embedding(num_users, embedding_dim)
        self.item_embedding = nn.Embedding(num_items, embedding_dim)
        self.convs = nn.ModuleList([
            SocialGCNLayer(embedding_dim, embedding_dim)
            for _ in range(num_layers)
        ])

    def forward(self, edge_index_ui, edge_index_social):
        x = torch.cat([self.user_embedding.weight, self.item_embedding.weight], dim=0)
        for conv in self.convs:
            x = conv(x, edge_index_ui, edge_index_social, self.num_users)
            x = F.relu(x)
        return x[:self.num_users], x[self.num_users:]
```

实证研究表明，当社交信号有价值时， NDCG 指标通常能提升 **5–15%**，这在生产环境中意义重大。但失败的情况也很常见：噪声大的社交关系（比如随机加的 Facebook 好友）不仅没帮助，反而会拖累效果。解决方法有两种：一是用注意力机制降低无关社交边的权重；二是在训练前通过交互重叠过滤掉低质量的社交关系。
## 图采样实现大规模训练

### 问题

全批量 GNN 训练要求每次前向传播时，把整张图加载到内存，并为所有节点计算嵌入。对 MovieLens （10 万次交互）来说没问题，但 Pinterest （180 亿条边）根本跑不动。

![完整图 vs 采样小批：选定一个目标节点，采样少数一跳邻居，再采样少数二跳邻居，只有这些节点参与一次前向传播](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig6_minibatch_sampling.png)

### 采样策略

| 策略 | 原理 | 示例 |
|---|---|---|
| **邻居采样** | 每层为每个节点采固定数量的邻居 | GraphSAGE：第一跳 10 个，第二跳 5 个 |
| **节点采样** | 随机采一批节点，用它们的邻域 | FastGCN |
| **子图采样** | 采样连通子图 | Cluster-GCN、 GraphSAINT |
| **随机游走采样** | 用随机游走找到重要邻居 | PinSage |

邻居采样是主力方法：限制每个节点的感受野，显存占用可控；每轮重新采样还能减少方差。

### 实现：邻居采样器

```python
import numpy as np
import torch

class NeighborSampler:
    """GraphSAGE 风格的邻居采样器，用于小批量训练。"""

    def __init__(self, edge_index, num_nodes, num_neighbors=(10, 5)):
        """初始化采样器。
        
        Args:
            num_neighbors: (第一跳采样数, 第二跳采样数, …)
        """
        self.num_nodes = num_nodes
        self.num_neighbors = num_neighbors

        self.adj = {i: [] for i in range(num_nodes)}
        row, col = edge_index.cpu().numpy()
        for r, c in zip(row, col):
            self.adj[r].append(c)

    def sample(self, target_nodes):
        """为目标节点采样多跳邻居。"""
        current_nodes = set(target_nodes.tolist())
        all_layers = []

        for num_nbrs in self.num_neighbors:
            edges, next_nodes = [], set()
            for node in current_nodes:
                neighbors = self.adj[node]
                if not neighbors:
                    continue
                if len(neighbors) > num_nbrs:
                    sampled = np.random.choice(
                        neighbors, num_nbrs, replace=False
                    ).tolist()
                else:
                    sampled = neighbors
                for nbr in sampled:
                    edges.append([nbr, node])
                    next_nodes.add(nbr)

            if edges:
                all_layers.append(torch.tensor(edges).t().contiguous())
            else:
                all_layers.append(torch.empty((2, 0), dtype=torch.long))
            current_nodes = next_nodes
        return all_layers
```
## 训练技巧

### BPR 损失（贝叶斯个性化排序）

隐式反馈推荐系统中最常用的损失函数：
$$\mathcal{L}_{\text{BPR}} = -\sum_{(u,i,j)} \ln\, \sigma(\hat{r}_{ui} - \hat{r}_{uj}) + \lambda \|\Theta\|^2$$

简单来说：*"对每个用户 $u$，让正样本 $i$（交互过的物品）的得分高于负样本 $j$（未交互的物品）。通过 sigmoid 和 log 函数将目标变得光滑且可微。"*

- $(u,i,j)$：(用户, 正物品, 负物品) 三元组  
- $\hat{r}_{ui} = \mathbf{e}_u^\top \mathbf{e}_i$：点积得分  
- $\lambda \|\Theta\|^2$： L2 正则化  

### 负采样策略

| 策略 | 描述 | 使用场景 |
|---|---|---|
| **均匀采样** | 随机选择未交互的物品 | 基线方法 |
| **基于流行度** | 按物品流行度比例采样 | 解决长尾问题 |
| **困难负样本** | 选择高分负样本 | 后期精调模型 |

### 训练循环

```python
import torch
import torch.nn as nn
import torch.optim as optim

class BPRLoss(nn.Module):
    """带 L2 正则化的 BPR 损失。"""

    def __init__(self, reg_lambda=1e-4):
        super().__init__()
        self.reg_lambda = reg_lambda

    def forward(self, pos_scores, neg_scores, *embeddings):
        bpr = -torch.log(torch.sigmoid(pos_scores - neg_scores) + 1e-10).mean()
        reg = self.reg_lambda * sum(emb.norm(2).pow(2) for emb in embeddings)
        return bpr + reg

def train_lightgcn(model, edge_index, train_pairs, num_items,
                   num_epochs=100, lr=0.001, batch_size=1024):
    """LightGCN 的训练循环。train_pairs: [user, pos_item] 张量。"""
    optimizer = optim.Adam(model.parameters(), lr=lr)
    criterion = BPRLoss()

    for epoch in range(num_epochs):
        model.train()
        perm = torch.randperm(len(train_pairs))
        total_loss, num_batches = 0.0, 0

        for start in range(0, len(train_pairs), batch_size):
            batch = train_pairs[perm[start:start + batch_size]]
            user_ids, pos_items = batch[:, 0], batch[:, 1]
            neg_items = torch.randint(0, num_items, (len(batch),))

            optimizer.zero_grad()
            user_emb, item_emb = model(edge_index)

            pos_scores = (user_emb[user_ids] * item_emb[pos_items]).sum(1)
            neg_scores = (user_emb[user_ids] * item_emb[neg_items]).sum(1)

            loss = criterion(pos_scores, neg_scores,
                             user_emb[user_ids],
                             item_emb[pos_items],
                             item_emb[neg_items])
            loss.backward()
            optimizer.step()
            total_loss += loss.item()
            num_batches += 1

        if (epoch + 1) % 10 == 0:
            print(f"Epoch {epoch+1}/{num_epochs}, Loss: {total_loss/num_batches:.4f}")
```

### 评估指标

```python
import numpy as np
import torch

def evaluate_topk(model, edge_index, test_user_items, train_user_items, k=10):
    """计算 Recall@K 和 NDCG@K。"""
    model.eval()
    recalls, ndcgs = [], []

    with torch.no_grad():
        user_emb, item_emb = model(edge_index)
        for user_id, test_items in test_user_items.items():
            scores = (user_emb[user_id] * item_emb).sum(dim=1)

            # 排除训练集中的物品
            train_items = train_user_items.get(user_id, [])
            scores[train_items] = -float('inf')

            _, top_k = torch.topk(scores, k)
            top_k = set(top_k.cpu().numpy())

            hits = len(top_k & set(test_items))
            recalls.append(hits / len(test_items) if test_items else 0)

            # NDCG：奖励排名靠前的命中
            dcg = sum(1.0 / np.log2(idx + 2)
                      for idx, item in enumerate(top_k) if item in test_items)
            idcg = sum(1.0 / np.log2(idx + 2)
                       for idx in range(min(k, len(test_items))))
            ndcgs.append(dcg / idcg if idcg > 0 else 0)

    return np.mean(recalls), np.mean(ndcgs)
```
## 冷启动：图模型的用武之地

![矩阵分解无法为新用户提供 embedding；GraphSAGE 风格的聚合通过一次交互就能生成 embedding](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/recommendation-systems/07-图神经网络与社交推荐/fig7_cold_start.png)

一个新用户来了，只有一条交互记录。矩阵分解派不上用场——没有这个用户的隐向量，加一行就得重新训练。但像 GraphSAGE 这样的归纳式 GNN 不一样，直接对新用户的邻居跑一遍聚合函数就行。哪怕只有一条交互记录，也能生成有意义的嵌入表示，因为聚合会把与该物品相关的其他用户信息都拉进来。

实际工程中处理冷启动用户的几种方法：

1. **归纳式聚合。** 用 GraphSAGE 或 PinSage 模型，只要节点有一个邻居，就能自动生成嵌入表示。
2. **特征初始化。** 在聚合之前，用画像特征（比如人口属性、设备信息、注册渠道）为冷启动节点生成初始嵌入。
3. **相似用户均值兜底。** 在用户交互数据不足时，将新用户的嵌入与同画像用户的均值混合。
4. **元学习（MAML）。** 如果资源允许，可以训练模型快速适应新用户，只需少量交互数据即可。

## 完整示例：在 MovieLens 数据上运行 LightGCN

```python
import torch
import numpy as np
from sklearn.model_selection import train_test_split

# --- 1. 构造模拟的 MovieLens 数据 ---
num_users, num_items = 1000, 2000
num_interactions = 10000

users = np.random.randint(0, num_users, num_interactions)
items = np.random.randint(0, num_items, num_interactions)
train_u, test_u, train_i, test_i = train_test_split(
    users, items, test_size=0.2, random_state=42
)

# 二部图边索引（无向）
edge_src = np.concatenate([train_u, train_i + num_users])
edge_dst = np.concatenate([train_i + num_users, train_u])
edge_index = torch.tensor([edge_src, edge_dst], dtype=torch.long)

# --- 2. 训练 LightGCN 模型 ---
model = LightGCN(num_users, num_items, embedding_dim=64, num_layers=3)
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
criterion = BPRLoss()

train_pairs = torch.tensor(np.stack([train_u, train_i], axis=1), dtype=torch.long)

for epoch in range(50):
    model.train()
    perm = torch.randperm(len(train_pairs))
    for start in range(0, len(train_pairs), 512):
        batch = train_pairs[perm[start:start + 512]]
        user_ids, pos_items = batch[:, 0], batch[:, 1]
        neg_items = torch.randint(0, num_items, (len(batch),))

        optimizer.zero_grad()
        user_emb, item_emb = model(edge_index)
        pos_scores = (user_emb[user_ids] * item_emb[pos_items]).sum(1)
        neg_scores = (user_emb[user_ids] * item_emb[neg_items]).sum(1)
        loss = criterion(pos_scores, neg_scores,
                         user_emb[user_ids],
                         item_emb[pos_items],
                         item_emb[neg_items])
        loss.backward()
        optimizer.step()

    if (epoch + 1) % 10 == 0:
        print(f"第 {epoch+1} 轮，损失：{loss.item():.4f}")

print("训练完成！")
```
## 常见问题

### 为什么 GNN 比矩阵分解更适合推荐？

矩阵分解独立学习每个嵌入，从不关心“我的邻居是什么样”。 GNN 则通过图结构显式传播协同信号，相似用户（通过共享物品连接）自然会得到相似的嵌入，这是架构自带的效果。多层 GNN 还能捕捉高阶模式，比如“喜欢与你兴趣相似用户喜欢的物品”，单靠点积无法表达。

### GCN、 GAT 和 GraphSAGE，什么时候用哪个？

| | GCN | GAT | GraphSAGE |
|---|---|---|---|
| **优点** | 简单高效 | 自适应邻居权重 | 可扩展，支持新节点 |
| **缺点** | 需要全图，权重固定 | 大邻域计算昂贵 | 采样引入方差 |
| **适用场景** | 小/中型静态图 | 异构边 | 大规模动态图 |

经验法则：协同过滤优先选 LightGCN （简化版 GCN）。需要归纳学习时用 GraphSAGE；邻居重要性差异大时用 GAT。

### GNN 应该堆几层？

推荐系统中通常用 2~3 层：

- **1 层** —— 只看直接邻居，缺乏协同模式。
- **2 层** —— 朋友的朋友，多数任务的最佳选择。
- **3 层** —— 三跳关系，收益有限，注意过平滑。
- **4 层及以上** —— 过平滑占主导，嵌入变得难以区分。

想要兼顾多尺度信息，可以用 LightGCN 的层组合方法，避免单层过深。

### 如何处理冷启动用户？

1. **归纳式聚合** —— 只要有至少一个邻居，就能生成嵌入。
2. **特征初始化** —— 用人口统计或设备等侧信息初始化嵌入，或者取相似用户的均值。
3. **元学习** —— 用 MAML 方法，快速适配少量交互数据。
4. **混合模型** —— 结合 GNN 嵌入和基于内容的特征，让冷启动节点也有有效表示。

参考上方冷启动示意图，了解归纳式处理方式。

### 计算成本怎么评估？

| 模型 | 前向计算 | 显存占用 |
|---|---|---|
| GCN / LightGCN | $O(L\cdot|E|\cdot d)$ | $O(|V|\cdot d + |E|)$ |
| GAT | $O(L\cdot|E|\cdot d\cdot H)$ | $O(|V|\cdot d\cdot H + |E|)$ |
| GraphSAGE （采样） | $O(L\cdot|V|\cdot k\cdot d)$ | $O(|V|\cdot d + k\cdot|V|)$ |

其中 $L$ 是层数，$|E|$ 是边数，$d$ 是嵌入维度，$H$ 是注意力头数，$k$ 是采样邻居数。对于十亿级边的图，必须使用采样（GraphSAGE、 PinSage）或子图划分（Cluster-GCN）。

### 社交推荐一定有用吗？

只有当社交关系真实反映共同偏好时才有效。实验证明，高质量社交图能提升 5~15% 的 NDCG；但噪声社交（随机关注列表）反而有害。可以通过注意力机制降低无关边的权重，或者按交互重叠预过滤社交图。

### GNN 能处理动态图吗？

可以，但需要调整：

- **时间感知聚合** —— 按时间远近加权邻居。
- **时间编码** —— 在边上添加时间嵌入。
- **增量更新** —— 在新边上微调，而不是重新训练。
- **专用时序 GNN** —— 比如 TGN，原生支持流式图。

```python
def temporal_aggregate(neighbor_embs, timestamps, current_time, decay_rate=0.1):
    """按时间近度给邻居加权"""
    time_diffs = current_time - timestamps
    weights = torch.exp(-decay_rate * time_diffs)
    weights = weights / weights.sum()
    return (neighbor_embs * weights.unsqueeze(1)).sum(dim=0)
```

### 如何防止过平滑？

1. **限制深度** —— 控制在 2~3 层。
2. **层组合**（LightGCN 风格）—— 各层等权，保留原始嵌入。
3. **残差连接** —— $\mathbf{h}^{(l+1)} = \mathbf{h}^{(l)} + \mathrm{GNN}(\mathbf{h}^{(l)})$。
4. **边 dropout** —— 训练时随机丢弃边，减少对结构的依赖。
## 关键要点

- **推荐数据天生是图结构。** GNN 通过邻域聚合传播协同信号，这是矩阵分解无法做到的。
- **LightGCN 证明简单就是高效。** 把 GCN 精简到只剩聚合操作，在协同过滤任务上往往能胜过更复杂的架构。
- **GraphSAGE 支持归纳学习。** 在生产环境中，用户和物品不断变化，这一点尤为重要——也是解决冷启动问题的最佳路径。
- **PinSage 展现了十亿级规模的可行性。** 随机游走采样和重要性加权让 GNN 在 Pinterest 这种大规模场景下变得实用。
- **社交信号能提升 5–15% 的效果** —— 但前提是社交关系确实反映共同偏好。
- **2 到 3 层是最佳选择。** 更深的网络容易导致过平滑；通过层组合既能增加深度又避免性能下降。
