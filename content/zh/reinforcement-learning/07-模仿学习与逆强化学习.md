之前的所有算法都默认能够获取奖励函数，但在实际项目中，设计奖励函数往往是最大的难点。让模型像老司机一样开车、像裁缝一样叠衣服、像专业编辑一样写摘要——这些任务演示起来容易，但用文字描述清楚却很难。

模仿学习正是抓住了这个痛点：不靠人工设计的标量奖励，而是直接从专家示范数据 $\mathcal{D} = \{(s_t, a_t)\}$ 中学策略。这一章将介绍四种经典方法：行为克隆、 DAgger、最大熵 IRL 和 GAIL/AIRL。我们将这四种方法组织成一个渐进式框架：每前进一步，就放宽一个关键假设，并引入相应机制来缓解该假设缺失带来的局限。
![强化学习（七）：模仿学习与逆强化学习 — visual](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-imitation-learning/illustration_1.png)

## 你将学到什么
- **行为克隆（BC）**：为什么模仿学习在短任务上有效，在长任务上容易失败。
- **DAgger**：通过交互式重新标注，将 BC 的二次误差转化为线性误差，背后的无悔定理是什么。
- **最大熵 IRL**：从演示中恢复一个可解释的奖励函数，其最优解能够复现专家的行为。
- **GAIL 和 AIRL**：通过对抗训练，端到端地匹配专家的占用度量（occupancy measure）。
- **方法选择指南**：根据专家是否可用、环境是否可访问以及是否需要迁移，明确应选择哪种方法。

**前置知识**：策略梯度（[第 5 部分](/zh/reinforcement-learning/05-model-based强化学习与世界模型)）和 PPO （[第 6 部分](/zh/reinforcement-learning/06-ppo与trpo-信任域策略优化)）。代码片段假设你熟悉 PyTorch 基础知识。

## 1. 问题设定

假设有一组专家示范数据：

$$\mathcal{D} = \{(s_1, a_1), (s_2, a_2), \ldots, (s_N, a_N)\},$$

目标是学习一个策略 $\pi_\theta$，让它的行为尽可能接近某个未知的专家策略 $\pi^*$。注意，我无法直接观察到 $\pi^*$，只能看到它的采样结果，并且没有奖励信号。有时可在训练过程中主动向专家查询新状态下的动作（如 DAgger 所采用的方式）；但在多数实际场景中，此类交互并不可行。

| 维度 | 强化学习 | 模仿学习 |
|---|---|---|
| 监督信号 | 奖励 $r(s, a)$ | 专家数据集 $\mathcal{D}$ |
| 是否需要交互 | 必须探索 | 可选（离线 BC） |
| 优化目标 | 最大化 $\mathbb{E}[\sum \gamma^t r_t]$ | 匹配专家行为分布 |
| 典型失败模式 | 奖励操控、探索不足 | 分布偏移、模式坍塌 |
| 典型应用 | 游戏、机器人、 RLHF | 自动驾驶、手术、语言风格 |

接下来要介绍的五种方法在这些维度上各有取舍。先看一张“阶梯图”，后面会逐层展开说明：

## 2. 行为克隆（Behavioral Cloning）

最简单的模仿学习算法，也是实际应用中最常见的：把数据集 $\mathcal{D}$ 当成监督学习的数据，优化目标是最小化损失函数：

$$\mathcal{L}(\theta) \;=\; \mathbb{E}_{(s,a)\sim\mathcal{D}}\big[ \ell\big(\pi_\theta(s),\, a\big) \big],$$

其中，$\ell$ 在离散动作时是交叉熵，在连续动作时是均方误差（MSE）或负对数似然（NLL）。这个方法最早可以追溯到 **ALVINN**（Pomerleau, 1989），它用一个全连接网络从摄像头图像中学习方向盘角度，从而实现面包车的自动驾驶。

```python
import torch
import torch.nn as nn
import numpy as np
from torch.utils.data import DataLoader, TensorDataset

class BehavioralCloning:
    """监督式模仿：最小化策略输出与专家动作之间的误差。"""

    def __init__(self, state_dim, action_dim, hidden_dims=(256, 256),
                 lr=1e-3, continuous=False, dropout=0.1):
        self.continuous = continuous
        layers, prev = [], state_dim
        for h in hidden_dims:
            layers += [nn.Linear(prev, h), nn.ReLU(), nn.Dropout(dropout)]
            prev = h
        if continuous:
            layers += [nn.Linear(prev, action_dim), nn.Tanh()]
            self.criterion = nn.MSELoss()
        else:
            layers += [nn.Linear(prev, action_dim)]
            self.criterion = nn.CrossEntropyLoss()
        self.policy = nn.Sequential(*layers)
        self.optim = torch.optim.Adam(self.policy.parameters(), lr=lr)
        self.state_mean = self.state_std = None

    def fit(self, states, actions, epochs=100, batch_size=64, val_frac=0.1):
        # 标准化状态输入——BC 是一个小模型，未归一化的特征会主导损失。
        self.state_mean = states.mean(0)
        self.state_std = states.std(0) + 1e-8
        S = (states - self.state_mean) / self.state_std

        n_val = int(len(S) * val_frac)
        idx = np.random.permutation(len(S))
        S_tr, S_val = S[idx[n_val:]], S[idx[:n_val]]
        A_tr, A_val = actions[idx[n_val:]], actions[idx[:n_val]]

        S_tr = torch.as_tensor(S_tr, dtype=torch.float32)
        S_val = torch.as_tensor(S_val, dtype=torch.float32)
        if self.continuous:
            A_tr = torch.as_tensor(A_tr, dtype=torch.float32)
            A_val = torch.as_tensor(A_val, dtype=torch.float32)
        else:
            A_tr = torch.as_tensor(A_tr, dtype=torch.long)
            A_val = torch.as_tensor(A_val, dtype=torch.long)

        loader = DataLoader(TensorDataset(S_tr, A_tr),
                            batch_size=batch_size, shuffle=True)
        best, best_state = float("inf"), None
        for epoch in range(epochs):
            self.policy.train()
            for s, a in loader:
                loss = self.criterion(self.policy(s), a)
                self.optim.zero_grad(); loss.backward(); self.optim.step()
            self.policy.eval()
            with torch.no_grad():
                v = self.criterion(self.policy(S_val), A_val).item()
            if v < best:
                best, best_state = v, {k: t.clone() for k, t in
                                       self.policy.state_dict().items()}
        self.policy.load_state_dict(best_state)

    def act(self, state):
        s = (state - self.state_mean) / self.state_std
        s = torch.as_tensor(s, dtype=torch.float32).unsqueeze(0)
        self.policy.eval()
        with torch.no_grad():
            out = self.policy(s)
        return out.squeeze().numpy() if self.continuous else out.argmax(-1).item()
```

有三个工程细节常被忽视，却对性能影响显著：
1. **输入标准化必不可少。** BC 模型很小，未归一化的特征会让某些维度主导损失，导致在罕见状态下输出过于自信的动作。
2. **验证集早停很关键。** 训练太久会拟合专家的噪声，反而加剧分布偏移问题。
3. **动作表示要选对。** 在连续控制任务中，预测高斯分布参数（用 NLL 损失）通常比“Tanh + MSE”效果更好，尤其是当专家行为多模态时。

### 2.1 BC 为什么在长任务上失败

训练时， BC 看到的状态来自专家分布 $d_{\pi^*}$；部署时，策略却运行在自己的状态分布 $d_{\pi_\theta}$ 上。由于策略不完美，这两个分布会逐渐偏离。

![BC 与 DAgger 的状态分布对比](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-模仿学习与逆强化学习/fig1_bc_vs_dagger.png)

经典理论给出了误差累积的清晰描述。如果策略 $\pi_\theta$ 在专家分布上的单步期望误差是 $\varepsilon$，那么在长度为 $T$ 的轨迹上，最坏情况下的总误差是：

$$J(\pi^*) - J(\pi_\theta) \;\le\; \mathcal{O}\!\left( \varepsilon \, T^2 \right),$$

误差随轨迹长度呈二次增长（Ross & Bagnell, 2010）。这一增长看似平缓，但实际影响显著：即便单步动作准确率达 99%， 200 步后轨迹偏差概率已极高。一旦进入专家从未见过的状态，模型就完全失去训练信号，错误迅速放大。

将训练完成的策略部署到环境中运行，可直观观察到这种误差级联现象：

![专家轨迹与学习者轨迹对比](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-模仿学习与逆强化学习/fig2_expert_trajectory_comparison.png)

初期， BC 策略尚能较好地复现专家轨迹；进入中期后，微小偏差逐渐累积，导致策略访问到训练分布之外的状态，进而持续偏离至危险区域。每一步的误差曲线也反映了同样的问题：

![BC 的误差累积](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-模仿学习与逆强化学习/fig5_compounding_error.png)

## 3. DAgger：数据集聚合

DAgger （Ross, Gordon & Bagnell, 2011）的核心思想非常简单：**在学习者实际访问的状态上向专家请教**。每次迭代，都会从当前策略采样出一批 $(s, a^*)$ 数据加入训练集，然后重新训练模型。

**算法**

1. 使用初始的专家示范训练出 $\pi_1$。
2. 对于 $i = 1, \ldots, N$：
   - 按调度 $\beta_i \to 0$ 混合 $\pi_i$ 和专家策略，生成状态轨迹。
   - 在每个访问到的状态上，向专家查询正确动作 $a^*$。
   - 聚合数据：$\mathcal{D} \leftarrow \mathcal{D} \cup \{(s, a^*)\}$。
   - 使用完整的 $\mathcal{D}$ 训练出 $\pi_{i+1}$。

**无悔保证**  
将这一系列策略看作一个在线学习器，面对由访问分布诱导的损失函数族。可以证明：

$$J(\pi^*) - J(\pi_{\hat{i}}) \;\le\; \mathcal{O}\!\left( \varepsilon \, T \right) + \mathcal{O}\!\left( \tfrac{T \sqrt{\log N}}{\sqrt{N}} \right),$$

即复杂度对时间步 $T$ 是**线性的**。直观理解是：每多一次迭代，都能把 BC 方法丢弃的“恢复信息”重新捡回来。

```python
class DAgger:
    """DAgger = BC + 在学习者访问的状态上反复请专家标注。"""

    def __init__(self, state_dim, action_dim, **bc_kwargs):
        self.bc = BehavioralCloning(state_dim, action_dim, **bc_kwargs)
        self.S, self.A = [], []

    def train(self, env, expert, n_iters=10,
              n_init=50, n_per_iter=20):
        # 第 0 轮：纯专家轨迹做 warm start。
        s0, a0 = self._collect(env, expert, n_init, beta=1.0)
        self.S += s0; self.A += a0
        self.bc.fit(np.array(self.S), np.array(self.A))

        for i in range(1, n_iters + 1):
            beta = max(0.0, 1.0 - i / n_iters)   # 专家比例从 1 衰减到 0
            s, a = self._collect(env, expert, n_per_iter, beta=beta)
            self.S += s; self.A += a
            self.bc.fit(np.array(self.S), np.array(self.A))

    def _collect(self, env, expert, n_episodes, beta):
        states, actions = [], []
        for _ in range(n_episodes):
            s, done = env.reset(), False
            while not done:
                # 行动用混合策略；标签始终来自专家。
                a_play = expert(s) if np.random.rand() < beta else self.bc.act(s)
                a_label = expert(s)
                states.append(s); actions.append(a_label)
                s, _, done, _ = env.step(a_play)
        return states, actions
```

**DAgger 的适用场景**  
DAgger 需要专家能在训练时实时提供反馈。这在以下场景中是可行的：

- 专家是一个算法（例如 MPC 控制器、基于搜索的规划器、最优控制器）。
- 专家是一个更强的模型，目标是进行蒸馏（teacher-student RL、模型蒸馏、语言模型自蒸馏）。
- 人类专家在线，愿意按批次标注数据。

它不适用的典型场景是只有静态日志数据——比如一份录制好的驾驶数据。这种纯离线设定需要转向 GAIL。

## 4. 逆强化学习（IRL）

![强化学习（七）：模仿学习与逆强化学习 — visual](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-imitation-learning/illustration_2.png)

BC 和 DAgger 都是模仿专家的动作。 IRL 则更进一步，问了一个更本质的问题：**为什么这个动作是好的？** IRL 假设专家在优化某个未知的奖励函数 $r^*$，然后从演示数据中反推出一个候选奖励 $\hat r$，最后用 $\hat r$ 来运行标准的强化学习。

为什么要绕道奖励函数？主要有两个原因：

- **可解释性。** $\hat r$ 能直接告诉你专家在意什么，方便检查和调整。
- **迁移能力。** 奖励函数对动力学、形态或初始状态分布的变化更鲁棒，而行为级别的模型一换环境就可能失效。

![逆强化学习从行为反推奖励](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-模仿学习与逆强化学习/fig3_irl_recovery.png)

### 4.1 最大熵 IRL

单纯“匹配专家价值”的目标是有问题的——很多不同的奖励函数都能解释同样的行为。**最大熵 IRL**（Ziebart et al., 2008）通过引入熵最大化解决了这个问题。具体来说，专家在轨迹 $\tau$ 上的分布服从 Boltzmann 形式：

$$p_\theta(\tau) \;\propto\; \exp\!\left( \sum_t r_\theta(s_t, a_t) \right).$$

最大化演示数据的对数似然后，可以得到一个简洁优雅的梯度公式：

$$\nabla_\theta \mathcal{L}(\theta) \;=\; \mathbb{E}_{\tau \sim \pi^*}\!\left[\nabla_\theta r_\theta(\tau)\right] \;-\; \mathbb{E}_{\tau \sim \pi_\theta}\!\left[\nabla_\theta r_\theta(\tau)\right].$$

这个公式可以理解为一种对比更新：**专家走过的轨迹提高奖励，当前策略走过的轨迹降低奖励**。当算法收敛时，两个期望相等，策略就能复现专家的行为分布。

代价在于第二个期望的计算。每次更新奖励函数时，都需要解一个 RL 问题（或者采样轨迹），这就形成了一个嵌套循环，限制了传统 IRL 的应用范围，只能处理小规模网格世界。**Guided cost learning**（Finn, Levine & Abbeel, 2016）用重要性采样的轨迹替代内部的 RL 求解，将最大熵 IRL 推广到了连续控制任务。

### 4.2 奖励的模糊性

即使引入了最大熵正则化，$\hat r$ 也只能恢复到一组**整形不变性**（shaping invariance）。换句话说，给奖励函数加上一个势能差 $\Phi(s') - \Phi(s)$ 不会改变最优策略，但会改变奖励值本身。因此， IRL 恢复的奖励更像是状态之间的一种相对排序，而不是绝对尺度。下一节介绍的 AIRL 显式地分离了整形项的影响。
## 5. 对抗式模仿： GAIL 与 AIRL

IRL 的内层 RL 求解成本太高。**GAIL**（Ho & Ermon, 2016）发现，模仿学习其实不需要 $r$——只需要让策略的状态-动作占用度和专家一致就行。于是 GAIL 把“先恢复奖励、再重新求解 RL”替换成了一个对抗博弈。

判别器 $D_\phi(s, a)$ 负责区分专家样本和策略样本；策略则努力骗过判别器。目标是最小化以下公式：

$$\min_\theta \max_\phi \;\;
\mathbb{E}_{(s,a)\sim \pi^*}\!\left[\log D_\phi(s, a)\right]
\,+\, \mathbb{E}_{(s,a)\sim \pi_\theta}\!\left[\log\big(1 - D_\phi(s, a)\big)\right]
\,-\, \lambda H(\pi_\theta).$$

熵项 $\lambda H(\pi_\theta)$ 稳定了生成器，避免模式塌缩。在鞍点处，$\pi_\theta$ 的占用度等于 $\pi^*$ 的占用度——这正是 BC 想做到的，但 BC 因为监督分布和部署分布不一致而无法实现。

![GAIL 判别器-生成器架构](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-模仿学习与逆强化学习/fig4_gail_architecture.png)

实际操作中，判别器的得分直接作为每步奖励，喂给 PPO 更新策略：

$$\hat r(s, a) \;=\; -\log\!\big(1 - D_\phi(s, a)\big) \quad\text{或}\quad \hat r(s, a) \;=\; -\log D_\phi(s, a).$$

两种形式都在文献中出现过。前者是 GAIL 原论文使用的奖励形式；后者在训练初期方差更小，效果更稳定。

```python
class GAIL:
    """对抗式模仿：判别器输出充当策略的每步奖励。"""

    def __init__(self, state_dim, action_dim, hidden_dim=256, continuous=False):
        self.continuous = continuous
        in_dim = state_dim + action_dim
        self.discriminator = nn.Sequential(
            nn.Linear(in_dim, hidden_dim), nn.Tanh(),
            nn.Linear(hidden_dim, hidden_dim), nn.Tanh(),
            nn.Linear(hidden_dim, 1),
        )
        self.disc_optim = torch.optim.Adam(self.discriminator.parameters(),
                                           lr=3e-4)
        # 策略 / 价值网络：任何标准 PPO 实现都行（此处省略）。

    def _sa(self, s, a):
        if not self.continuous:
            a = torch.nn.functional.one_hot(a.long(), self.action_dim).float()
        return torch.cat([s, a], dim=-1)

    def reward(self, states, actions):
        """喂给策略学习器的每步模仿奖励。"""
        with torch.no_grad():
            logits = self.discriminator(self._sa(states, actions))
            d = torch.sigmoid(logits)
            r = -torch.log(1.0 - d + 1e-8)   # 原版 GAIL 的奖励形式
        return r.squeeze(-1)

    def update_discriminator(self, expert_sa, policy_sa):
        # 二元交叉熵：专家=0，策略=1。
        e_logits = self.discriminator(expert_sa)
        p_logits = self.discriminator(policy_sa)
        bce = torch.nn.functional.binary_cross_entropy_with_logits
        loss = bce(e_logits, torch.zeros_like(e_logits)) + \
               bce(p_logits, torch.ones_like(p_logits))
        self.disc_optim.zero_grad(); loss.backward(); self.disc_optim.step()
        return loss.item()
```

**调参要点。** 实际使用 GAIL 时，最常见的三种失败模式如下：

- **判别器赢太快。** 策略侧的奖励梯度直接消失。解决方法：裁剪判别器 logits、用 spectral normalization 或加 Lipschitz 约束（WGAIL）。
- **奖励信号塌缩。** 当 $D \to 0$ 时，奖励退化成接近零。解决方法：对奖励做 batch 内的滑动归一化（running mean / std）。
- **策略模式塌缩。** 增大熵奖励 $\lambda$，或者直接用最大熵 actor （SAC 风格）作为生成器。

### 5.1 “匹配占用度”到底是什么意思？

GAIL 并不是最小化某种动作预测损失，而是最小化专家占用度 $\rho_{\pi^*}(s, a)$ 和学习者占用度 $\rho_{\pi_\theta}(s, a)$ 的 Jensen-Shannon 散度。这个目标比 BC 强得多——因为它考虑了部署分布。代价是：训练时必须能与环境交互（用来采样 $\rho_{\pi_\theta}$），所以原版 GAIL 不适用于纯离线场景。

### 5.2 AIRL：把奖励从整形中分离出来

GAIL 学到的判别器擅长模仿，但不是一个可复用的奖励。**AIRL**（Fu, Luo & Levine, 2018）重新设计了判别器的形式：

$$D_\phi(s, a, s') \;=\; \frac{\exp\big(f_\phi(s, a, s')\big)}{\exp\big(f_\phi(s, a, s')\big) + \pi_\theta(a \mid s)},
\qquad
f_\phi(s, a, s') \;=\; r_\psi(s) + \gamma \Phi_\xi(s') - \Phi_\xi(s),$$

其中 $r_\psi$ 是只依赖状态的奖励，整形项被 $\Phi_\xi$ 吸收。这样训出来的 $\hat r = r_\psi$ 在动力学发生变化时仍然可用——这是 AIRL 论文的核心实验结果。
## 6. 样本效率：模仿学习 vs 强化学习

模仿学习最大的优势是样本效率。几千条专家示范，就能省下数千万次环境交互，尤其在高维控制任务中效果更明显。

![样本效率对比：模仿 vs RL](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/07-模仿学习与逆强化学习/fig7_sample_efficiency.png)

左图的规律在机器人基准测试中非常一致：行为克隆（BC）的表现始终低于专家水平，因为它无法超越训练数据； DAgger 在接触到恢复状态后开始提升；生成对抗模仿学习（GAIL）最终能追上专家水平；而纯强化学习则需要花更长时间才能达到同样的效果。右图把这种权衡关系表达得很清楚：用的专家数据越多，所需的环境步数就越少，反之亦然。

这也是为什么“模仿预训练 + 强化学习微调”成为现代系统的主流方法。 AlphaStar、机器人操作和 InstructGPT 都是先通过模仿学习打基础，再用强化学习进一步优化性能。
## 7. 方法选择

| 方法 | 需要在线询问专家？ | 需要环境交互？ | 样本效率 | 可解释奖励？ | 典型场景 |
|---|---|---|---|---|---|
| BC | 否 | 否 | 高（离线） | 否 | 短任务、日志充足 |
| DAgger | **是** | 是 | 中高 | 否 | 算法/模型专家可用 |
| MaxEnt IRL | 否 | 是（内层循环） | 低 | **是** | 小状态空间、需要迁移 |
| GAIL | 否 | 是 | 中 | 否 | 高维连续控制 |
| AIRL | 否 | 是 | 中 | 是（状态奖励） | 跨任务迁移奖励 |

简单决策规则：

- **静态示范日志，短时序任务** → 直接用 **BC**。记得加早停和标准化；如果想评估不确定性，可以用集成方法。
- **训练时能随时询问专家** → 上 **DAgger**。它的线性误差保证是最划算的改进。
- **需要理解专家意图，或者要迁移到新环境** → 小问题选 **MaxEnt IRL**，连续控制选 **AIRL**。
- **高维连续控制，有静态数据且能与环境交互** → 用 **GAIL**。但对抗训练容易不稳定，得提前留预算。

## 8. 常见问题

**模仿学习能超越专家吗？**  
纯模仿学习做不到，因为最优的模仿者只能达到专家水平。解决办法是把模仿当作初始化：先用 BC 或 GAIL 策略，然后通过 RL 微调，优化你能定义的奖励函数（或者用 RLHF 的偏好奖励）。目前大多数大规模系统都采用这种两阶段方法。

**专家示范有噪声或次优怎么办？**  
可以从三个方面入手：  
1. 按质量评分（比如回报值或人类偏好）给样本加权；  
2. 把交叉熵损失换成鲁棒性更强的损失函数（如 Huber 或 log-cosh），降低离群点的影响；  
3. 在示范数据上应用离线 RL 方法，比如 IQL 或 CQL，这些方法对次优轨迹处理得很好。

**专家行为是多模态的——同一状态下有不同的动作，怎么办？**  
标准的 MSE-BC 会平均多模态分布，导致危险的“中间动作”（比如一半示范向左、一半向右，结果学到的是直冲障碍物）。可以改用混合密度网络（MDN）、离散动作策略、条件 VAE，或者扩散策略（diffusion policy）。 Chi 等人在 2023 年提出的扩散策略在多模态操作任务中表现最佳。

**多收集数据能不能解决 BC 的分布偏移问题？**  
有一定效果，但有限。更多数据能让 $\varepsilon$ 减小，但不会改变 $T^2$ 的指数增长特性。一旦时间跨度足够长，策略可能会超出数据集 $\mathcal{D}$ 的支撑范围。这时需要 DAgger 风格的重新标注、 GAIL 风格的占用匹配，或者保守的离线 RL 方法（如 CQL 或 IQL），显式惩罚分布外的动作。

**RLHF 是一种模仿学习吗？**  
部分是。 RLHF 的监督微调阶段就是基于人类示范的 BC；而偏好建模阶段更接近逆强化学习（IRL）的一种变体——从对比中学到奖励模型，再用 PPO 进行优化。完整流程会在[第 12 部分](/zh/reinforcement-learning/12-rlhf与大语言模型应用)详细说明。
