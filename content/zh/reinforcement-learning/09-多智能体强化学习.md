单智能体强化学习（RL）有一个隐含但极其重要的假设：环境是静态的，转移核不会因智能体的学习而改变。然而，一旦出现第二个学习者，这个假设就彻底失效了。每个智能体面对的环境动力学会随着其他智能体策略的更新而持续变化，奖励信号在智能体之间高度耦合，联合动作空间更是呈组合爆炸式增长。这些问题并非工程上的小麻烦，而是多智能体强化学习（MARL）必须发展专属算法的根本原因——简单地“并行跑 n 个 DQN”是行不通的。

解决这些问题的回报极为丰厚：AlphaStar 在《星际争霸 II》天梯中达到宗师段位，OpenAI Five 在《Dota 2》中击败了世界冠军。如今，合作型 MARL 正日益成为仓储机器人调度、交通信号灯控制以及多 LLM 智能体系统的核心技术。本章将系统梳理其概念骨架——马尔可夫博弈、CTDE、值分解与反事实信用分配——并最终呈现一个轻量级的 QMIX 混合器实现，你可以直接将其嵌入自己的训练循环。

## 为什么 MARL 比单智能体 RL 更难

![合作、竞争和混合动机的多智能体场景](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig1_scenarios.png)

奖励结构几乎决定了所有其他因素。在**合作型**游戏中，所有智能体共享团队奖励——例如足球配合、群体物流或《星际争霸》中的微操。**竞争型**游戏则是零和博弈：一方的收益即另一方的损失，此时自对弈（self-play）是最合适的工具。而最有趣的是中间地带——**混合动机**（一般和）游戏，如市场、交通、谈判或拥塞控制。在这些场景中，短期的个体利益往往会损害长期的集体福祉（囚徒困境就是经典的 2×2 案例，图中第三个收益矩阵也展示了这一点）。

无论在哪种设定下，以下四个障碍都会导致单智能体 RL 工具失效。

**非平稳性**。从智能体 $i$ 的视角看，状态转移为 $P(s' \mid s, a_i, a_{-i})$，其中 $a_{-i}$ 表示其他所有智能体的动作。由于训练过程中其他智能体的策略 $\pi_{-i}$ 不断更新，$i$ 所感知到的*有效*转移核也随之持续漂移。像 DQN 这类离策略方法假设数据生成分布固定，在 MARL 中极易发散或震荡。

**信用分配**。当团队在整个回合结束后只获得一个共享奖励 $r$ 时，究竟谁做出了关键贡献？若简单地共享梯度，会鼓励“搭便车”行为：一个无所作为的智能体与真正付出努力的智能体获得相同的信用。对此，反事实基线（COMA，见[第 5 节](#马尔可夫博弈与纳什均衡：够用就好)）和值分解（见[第 4 节](#CTDE：训练用全量信息，执行靠局部观察)）提供了两种原则性的解决方案。

**部分可观测性**。每个智能体仅能观测到自身的局部信息 $o_i$，执行时无法获取完整的马尔可夫状态。虽然常用循环网络配合信念状态式表示来缓解此问题，但更根本的解法是在训练架构中让*集中式评论家*看到比执行阶段更多的信息。

**可扩展性**。若有 $n$ 个智能体，每个拥有 $|A|$ 个动作，则联合动作空间大小为 $|A|^n$。若不利用问题结构，即便是规模适中的合作任务也会变得不可解。

应对这四大挑战的统一框架——也是本章的概念主干——正是 **CTDE**。

## CTDE：训练用全量信息，执行靠局部观察

![强化学习（九）：多智能体强化学习 — visual](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/en/reinforcement-learning/09-multi-agent-rl/illustration_2.png)

![集中训练、分散执行示意图](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig2_ctde.png)

CTDE 的核心思想在于不对称设计：训练时，算法可以访问全局状态和所有智能体的动作——例如一个*集中式*的评论家、混合器或价值函数；而在部署阶段，仅保留基于局部观测的个体策略。执行接口是分散的，但优化过程却是集中的。

CTDE 之所以有效，是因为 MARL 的难点（非平稳性、信用分配、部分可观测性）主要出现在*学习阶段*，而实际应用的约束（带宽有限、异步控制、隐私保护）则集中在*执行阶段*。CTDE 将这两类问题分而治之。如今几乎所有主流的合作型 MARL 算法——包括 VDN、QMIX、MADDPG、COMA 和 MAPPO——都属于 CTDE 范式，它们的区别仅在于集中对象的具体形式及其分解方式。

## 马尔可夫博弈与纳什均衡：够用就好

**马尔可夫博弈**将 MDP 推广至 $n$ 个智能体：
$$\langle \mathcal{N}, \mathcal{S}, \{A_i\}_{i\in\mathcal{N}}, P, \{r_i\}_{i\in\mathcal{N}}, \gamma \rangle.$$
每个智能体 $i$ 拥有独立的动作集 $A_i$ 和奖励函数 $r_i(s, a_1, \ldots, a_n)$。**纳什均衡**是一个联合策略 $\pi^* = (\pi_1^*, \ldots, \pi_n^*)$，满足没有任何智能体能通过单方面偏离而获益：
$$V_i(\pi_i^*, \pi_{-i}^*) \;\geq\; V_i(\pi_i, \pi_{-i}^*) \qquad \forall \pi_i,\, \forall i.$$
纳什均衡之所以难以驾驭，主要有两点：一是通常不唯一；二是未必 Pareto 最优。但在**完全合作**情形下（即 $r_1 = \cdots = r_n$），问题大幅简化——目标退化为最大化单一团队回报，此时纳什均衡与 Pareto 最优解集重合。正因如此，本章的算法设计大多优先聚焦合作场景；竞争与混合动机情形则在此基础上叠加自对弈、对手建模或正则化等额外机制。

## 值分解：VDN 和 QMIX

![QMIX 架构与单调因式分解](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig3_qmix.png)

值分解是针对合作场景的一种 CTDE 方法。执行时，每个智能体使用局部值函数 $Q_i(o_i, a_i)$；训练时，则引入一个*联合*值函数 $Q_\text{tot}(s, \mathbf{a})$，并通过约束二者关系，确保在 $Q_i$ 上进行分散贪心选择的结果，等价于在 $Q_\text{tot}$ 上进行联合贪心选择。这一性质称为 **Individual-Global-Max（IGM）**：
$$\arg\max_{\mathbf{a}} Q_\text{tot}(s, \mathbf{a}) \;=\; \big(\arg\max_{a_1} Q_1(o_1, a_1),\; \ldots,\; \arg\max_{a_n} Q_n(o_n, a_n)\big).$$
### VDN：加性分解

[Sunehag 等, 2017](https://arxiv.org/abs/1706.05296) 采用了最简单的充分条件：
$$Q_\text{tot}(s, \mathbf{a}) \;=\; \sum_{i=1}^{n} Q_i(o_i, a_i).$$
由于求和运算对每个被加项天然单调，IGM 性质自然成立。代价是表达能力受限：VDN 完全无法捕捉智能体间的非加性交互。

### QMIX：单调混合

[QMIX](https://arxiv.org/abs/1803.11485) 保留了 IGM，但将“加性”推广为“单调”。它通过一个神经**混合网络**将各 $Q_i$ 组合成 $Q_\text{tot}$，并施加如下约束：
$$\frac{\partial Q_\text{tot}}{\partial Q_i} \;\geq\; 0 \qquad \forall i.$$
该约束通过构造混合网络的权重为非负来实现——例如对权重取绝对值或经 softplus 函数处理。关键在于，这些权重由一个以全局状态 $s$ 为条件的**超网络**生成，因此不同状态下可动态调整各智能体的权重，从而表达纯加法无法捕捉的交互关系。图右侧展示了由此得到的 $Q_\text{tot}$ 曲面：对每个 $Q_i$ 单调，因此联合 argmax 与各智能体 argmax 的组合一致。

```python
import torch
import torch.nn as nn

class QMixer(nn.Module):
    """QMIX 混合网络，权重由超网络生成并保持非负。"""

    def __init__(self, n_agents: int, state_dim: int, hidden_dim: int = 32):
        super().__init__()
        self.n_agents = n_agents
        self.hidden_dim = hidden_dim

        # 超网络：state -> 混合权重（取 abs 强制非负）
        self.hyper_w1 = nn.Linear(state_dim, n_agents * hidden_dim)
        self.hyper_b1 = nn.Linear(state_dim, hidden_dim)
        self.hyper_w2 = nn.Linear(state_dim, hidden_dim)
        # 末层标量偏置无符号约束
        self.hyper_b2 = nn.Sequential(
            nn.Linear(state_dim, hidden_dim), nn.ReLU(),
            nn.Linear(hidden_dim, 1),
        )

    def forward(self, agent_qs: torch.Tensor, state: torch.Tensor) -> torch.Tensor:
        """agent_qs: (B, n_agents)   state: (B, state_dim)   ->  (B, 1)"""
        B = agent_qs.size(0)
        agent_qs = agent_qs.view(B, 1, self.n_agents)

        w1 = torch.abs(self.hyper_w1(state)).view(B, self.n_agents, self.hidden_dim)
        b1 = self.hyper_b1(state).view(B, 1, self.hidden_dim)
        hidden = torch.nn.functional.elu(torch.bmm(agent_qs, w1) + b1)

        w2 = torch.abs(self.hyper_w2(state)).view(B, self.hidden_dim, 1)
        b2 = self.hyper_b2(state).view(B, 1, 1)
        q_tot = torch.bmm(hidden, w2) + b2
        return q_tot.view(B, 1)
```

训练过程与 DQN 完全相同：采样经验转移，在 $Q_\text{tot}$ 上构建 TD 目标，并将梯度反向传播至混合器及所有个体网络。执行时，每个智能体仅运行自己的 $Q_i$，完全不接触混合器。

一个实用提示：QMIX 并不能表示所有满足 IGM 的可分解函数（其单调性是充分但非必要条件）。[QTRAN](https://arxiv.org/abs/1905.05408) 和 [QPLEX](https://arxiv.org/abs/2008.01062) 进一步拓展了这一边界，但代价是更复杂的损失函数与工程实现。

## 多智能体 Actor-Critic：MADDPG 与 COMA

### MADDPG：每个智能体一个集中式评论家

![MADDPG 集中式评论家，分布式执行者](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig5_maddpg.png)

[MADDPG](https://arxiv.org/abs/1706.02275) 是 DDPG 在 CTDE 框架下的自然延伸。每个智能体 $i$ 包含：

- 一个**执行者** $\mu_i(o_i)$，仅依赖局部观测；
- 一个**评论家** $Q_i(s, a_1, \ldots, a_n)$，以全局状态和*所有*智能体的动作作为输入。

执行者通过确定性策略梯度更新，梯度经由自身评论家计算；评论家则使用所有智能体的目标执行者构建 TD 目标：
$$y \;=\; r_i \;+\; \gamma \, Q_i^{\text{tgt}}\!\left(s', \mu_1^{\text{tgt}}(o_1'), \ldots, \mu_n^{\text{tgt}}(o_n')\right).$$
由于评论家显式接收 $a_{-i}$ 作为输入，环境对其而言是平稳的——其输入已包含原本导致非平稳性的变量。执行者保持分散设计，部署时无需改动。MADDPG 可应用于合作、竞争及混合动机场景，回放池只需存储联合轨迹。

### COMA：反事实信用分配

![COMA 反事实基线](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig4_coma.png)

[COMA](https://arxiv.org/abs/1705.08926) 直接针对信用分配问题。它使用一个集中式评论家 $Q(s, \mathbf{a})$，并为每个智能体 $i$ 构造一个**反事实基线**：固定其他智能体的动作，对 $i$ 的动作按其策略求期望：
$$A_i(s, \mathbf{a}) \;=\; Q(s, \mathbf{a}) \;-\; \sum_{a_i'} \pi_i(a_i' \mid o_i) \, Q\!\left(s, (a_i', a_{-i})\right).$$
仔细理解：第一项是实际发生的价值；第二项是在其他人动作不变的前提下，$i$ 若采取其他动作的*期望*价值。二者之差即为 $i$ 的边际贡献。当团队奖励共享时，这正是策略梯度应优化的目标——它剔除了 $a_{-i}$ 引入的噪声，且无偏。图中四动作示例显示：仅被选中的动作获得非零优势值。

COMA 的高效性源于一个技巧：集中式评论家一次性输出智能体 $i$ 所有可能动作的 $Q$ 值向量，因此基线计算仅需一次策略概率与该向量的点积，无需额外环境交互。

## 通信：智能体之间该交流多少？

![通信拓扑：广播、稀疏、注意力](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig6_communication.png)

有时，完全分散执行过于严苛——测试阶段智能体确实需要交换信息。设计空间构成一个连续谱系。

**广播**：每个智能体向其余所有智能体发送固定长度消息。概念清晰，但消息量随智能体数按 $O(n^2)$ 增长，超过几十个智能体便难以承受。

**稀疏 / k-NN**：每个智能体仅与 $k$ 个最近邻通信（可基于空间、图结构或角色）。固定 $k$ 时，复杂度为线性。CommNet、IC3Net 和 TarMAC 均属此类。

**注意力**：采用软路由、可学习的机制。每个智能体发出查询，其他智能体以键响应，消息权重通过 softmax 计算。虽注意力得分计算为 $O(n^2)$，但可通过 top-$k$ 注意力大幅剪枝消息，且整体结构端到端可微。这已成为现代 MARL 的主流选择，也是通往多 LLM 智能体系统的自然桥梁——注意力式路由可直接映射到工具调用与智能体编排模式。

具体选择取决于任务：紧密协作的机器人团队可接受密集通信；而数千个微服务或车辆场景下，稀疏或注意力剪枝几乎是唯一可行方案。

## MARL 工业级训练：AlphaStar、OpenAI Five 与联赛训练

![AlphaStar 联赛训练与两大旗舰系统](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig7_league.png)

[AlphaStar](https://www.nature.com/articles/s41586-019-1724-z)（《星际争霸 II》）与 [OpenAI Five](https://arxiv.org/abs/1912.06680)（《Dota 2》）的成功关键，在于将*训练种群设计*视为首要工程问题。朴素自对弈易陷入循环：A 学会击败 B，B 又学会克制 A 的新策略却遗忘旧招，周而复始。**AlphaStar 联赛**明确划分三类种群：

- **主智能体**：最终部署对象，为优化核心目标；
- **主剥削者**：专职寻找主智能体弱点，迫使其补全能力；
- **联赛剥削者**：定期重置，维持*策略多样性*，防止联赛风格趋同。

一个保存*历史冻结快照*的对手池通过优先虚拟自对弈（PFSP）与当前学习者匹配——PFSP 倾向于选择学习者刚好能勉强战胜的对手。这些机制共同避免了朴素自对弈中的灾难性遗忘。

OpenAI Five 则另辟蹊径：每个英雄由独立 LSTM 策略控制（参数共享），团队协作交由手工设计的共享奖励与海量算力支撑——每日相当于 *180 年人类自对弈经验*。两者的共同启示是：在工业规模下，*训练课程*的重要性绝不亚于单智能体算法本身。

这些思想正走出游戏领域。RLHF（[第 12 篇](/zh/reinforcement-learning/12-rlhf与大语言模型应用)）与工具调用智能体编排，越来越呈现出多种群训练的特征——一群策略、一群评估者，辅以优先匹配机制。

## 多智能体强化学习中的信用分配：一个实例分析

在合作型 MARL 中，信用分配才是真正的硬骨头：如何判断团队奖励究竟由哪个智能体的动作引发？这个问题值得深入探讨。考虑一个 4 智能体网格任务：仅当所有智能体同时站在目标格时，团队才获得 +10 奖励。若用朴素独立 Q-learning 训练，策略会永远震荡——因为每个智能体在更新 $Q$ 值时，将其他三个视为环境的一部分，“我应移向目标”的梯度信号被队友随机移动的噪声彻底淹没。

按复杂度递增，有三类解决方案：

### 差分奖励

最简方案是为每个智能体 $i$ 计算**反事实差值**：
$$
D_i = R(s, a) - R(s, (a_{-i}, c_i)),$$其中 $c_i$ 为默认动作（如“静止”）。每个智能体的有效奖励即“因我的实际选择而产生的团队收益增量”，通过基线边缘化得到。差分奖励早在深度 RL 兴起前 15 年就已提出（Wolpert & Tumer, 2002），只要能高效计算反事实（如支持回滚的模拟器），至今仍惊人有效。

### 反事实基线（COMA）

若反事实难以计算，可转而*学习*它。COMA 的优势函数定义为：$$A_i(s, a) = Q(s, a) - \sum_{a_i'} \pi_i(a_i' | \tau_i)\, Q(s, (a_{-i}, a_i')),$$即在固定他人动作的前提下，对 $i$ 的备选动作求期望。其精妙之处在于，该期望使用*集中式评论家*计算——训练时所有观测可用，基线定义明确；而测试时各智能体仍仅依赖自身历史。

### 基于势函数的奖励塑形

若上述方法均不可行，可退而采用势函数奖励塑形：添加额外项 $F(s, s') = \gamma \Phi(s') - \Phi(s)$，其中 $\Phi$ 为手工设计的势函数（如负目标距离）。Ng-Harada-Russell 定理保证最优策略不变。在 MARL 中，为每个智能体单独应用此技巧，常能达到复杂信用分配方法的效果，且实现极为简单——30 行代码足矣。

### 一个真实踩过的坑

在多机器人仓库拣货任务中，我们最初用 QMIX 训练，奖励为稀疏的“所有包裹拣完”。100 M 环境步后策略仍次优，因 4 号机器人几乎闲置——其个体 $Q$ 值几无变化，因其动作极少单独决定团队成败。加入差分奖励（每机器人奖励 = 团队奖励 $-$ 其静止时的团队奖励）后，策略在 12 M 步内收敛。结果证明，那篇复杂的信用分配论文，效果竟不如一个 30 行的奖励塑形函数。

## MARL 中的奖励塑形与课程设计

稀疏团队奖励在 MARL 中极难收敛。与其更换算法，不如尝试以下两个工程技巧，往往事半功倍。

### 按团队规模逐步训练

先用 2 个智能体训练，再增至 3 个，最后扩展至 4 个。$n=2$ 时联合动作空间小，随机探索偶尔能触达奖励，梯度信号更强。待策略初步成型后，新增智能体以最佳单体策略初始化，再全队联合训练。AlphaStar 采用了此思路的极端版本（基于联赛的群体训练），但你无需复杂机制，基础课程设计已足够有效。

### 冻结对手的自我博弈

在竞争型 MARL 中，标准做法是**虚拟自对弈（fictitious self-play）**：当前学习者与过去多个版本的自己混合对战。这可避免“今日策略胜昨日、败上周”的循环（石头剪刀布陷阱）。OpenAI Five 训练中，80% 对最新 checkpoint，20% 对随机历史快照——正是这 20% 维持了种群多样性，确保泛化能力。

### 超参数敏感性注意事项

MARL 对超参数的敏感性远高于单智能体 RL。同一 QMIX 实现，在 StarCraft II 微操任务中某学习率可收敛，但在多粒子环境中，学习率仅放大 2 倍便发散。社区经验表明：初始学习率宜取单智能体 PPO 的一半左右，并延长熵奖励的衰减周期——因为在 MARL 中，探索是一种“公共品”：任一智能体停止探索，其余智能体都将损失信息。

## Centralised Critic, Decentralised Actor (CTDE) — 一个定理

![MADDPG:训练时使用中心化 critic,执行时去中心化。](https://blog-pic-ck.oss-cn-beijing.aliyuncs.com/posts/zh/reinforcement-learning/09-多智能体强化学习/fig09_maddpg.png)

2017 年之后的论文基本都沿用这套设定。$N$ 个智能体，联合状态 $s$，独立动作 $a_i$。联合策略分解为 $\pi(a \mid s) = \prod_i \pi_i(a_i \mid s)$。执行时每个智能体只看自己的局部观测。训练时放开权限，让 critic 看到全局信息。

这套玩法的理论底座来自 [Lowe et al. 2017](https://arxiv.org/abs/1706.02275)。定义集中式 critic $Q^\pi(s, a_1, \dots, a_N)$，输入完整联合动作。智能体 $i$ 的策略梯度为
$$\nabla_{\theta_i} J(\pi_i) = \mathbb{E}\!\left[ \nabla_{\theta_i} \log \pi_i(a_i \mid s) \, Q^\pi(s, a_1, \dots, a_N) \right],$$
这个估计量无偏。哪怕队友策略还在更新，照样成立。一句话概括 CTDE 定理。

证明思路很短，值得过一遍。冻结队友策略 $\pi_{-i}$ 做一次更新。对智能体 $i$ 来说，转移核变成 $P(s' \mid s, a_i, a_{-i})$ 对 $a_{-i} \sim \pi_{-i}$ 求边缘分布。$\pi_{-i}$ 固定，环境平稳。但边缘化会引入方差，且随 $N$ 暴涨。如果显式*条件化* $a_{-i}$，方差直接消失。集中式架构提供了条件化的可能。独立学习器看不到队友动作，自然做不到。

逆命题同样关键。如果只有 $Q_i(s, a_i)$，梯度在队友策略漂移时必然有偏。这就是 IQL 遭遇非平稳性的数学根源。

证明里还藏着一个维度问题。集中式 critic 要拟合 $\mathbb{R}^{|s| + N \cdot |a|}$ 上的函数，维度很高。但它只需要在联合策略的支撑集上学习。联合策略是分散式 actor 的乘积，支撑集远小于动作空间的笛卡尔积。经验表明 critic 在支撑集上泛化得很干净。MADDPG 用中等容量的网络就能跑通，原因在此。

MADDPG 的骨架代码大概长这样。每个智能体一个 actor，共享一个集中式 critic。离散动作空间用 Gumbel-Softmax 配合确定性策略梯度。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class Actor(nn.Module):
    def __init__(self, obs_dim, act_dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, act_dim),
        )

    def forward(self, obs, hard=False, tau=1.0):
        logits = self.net(obs)
        return F.gumbel_softmax(logits, tau=tau, hard=hard)


class CentralisedCritic(nn.Module):
    def __init__(self, state_dim, n_agents, act_dim, hidden=256):
        super().__init__()
        self.q = nn.Sequential(
            nn.Linear(state_dim + n_agents * act_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, 1),
        )

    def forward(self, state, joint_actions):
        return self.q(torch.cat([state, joint_actions], dim=-1)).squeeze(-1)


def maddpg_update(actors, critic, target_actors, target_critic, batch, gamma=0.99):
    s, a, r, s_next, done = batch
    with torch.no_grad():
        a_next = torch.cat([ta(s_next) for ta in target_actors], dim=-1)
        y = r + gamma * (1 - done) * target_critic(s_next, a_next)
    critic_loss = F.mse_loss(critic(s, a), y)

    actor_losses = []
    for i, actor in enumerate(actors):
        a_i = actor(s)  # Gumbel-Softmax 保持可导
        a_joint = a.clone()
        slice_i = slice(i * a_i.size(-1), (i + 1) * a_i.size(-1))
        a_joint[..., slice_i] = a_i  # 替换联合动作中当前智能体的部分
        actor_losses.append(-critic(s, a_joint).mean())
    return critic_loss, actor_losses
```

我常拿来对标的数据是 2D 环面上的四捕食者追一猎物。Independent Q-learning 跑 5 万 episode 卡在 35% 捕获率。捕食者只会追，不会包抄。MADDPG 同样预算跑到 78%，和原论文 Figure 4 一致。差距不在算法多聪明，在于集中式 critic 不被局部信息干扰。

两个坑了我一周的实现细节。Gumbel-Softmax 温度 $\tau$ 很关键：太高梯度全是噪声，太低变成 one-hot，critic 无法求导。我在前 20% 训练步数里从 $\tau = 1.0$ 衰减到 $\tau = 0.5$。其次，target network 必须慢更新。Polyak 系数 $\rho = 0.995$。不加的话 critic 追逐移动靶，actor 跟着掉沟里。

团队奖励到手了，下一步得搞清楚到底谁出了力。

## Counterfactual Reasoning: COMA Baselines

回合结束只给一个标量团队奖励。每个智能体的梯度都会混进别人的噪声。智能体 3 碰巧做了有用操作，智能体 1 同时完成击杀。两人拿一样奖励，梯度平分功劳。跑几百万回合，偏差不会抵消，只会养出一堆摸鱼的智能体。

[COMA](https://arxiv.org/abs/1705.08926) 提出反事实 baseline。定义智能体 $i$ 的优势函数为
$$A_i(s, a) = Q(s, a_1, \dots, a_N) - \sum_{a'_i} \pi_i(a'_i \mid s) \, Q(s, a_1, \dots, a'_i, \dots, a_N),$$
对智能体 $i$ 的可能动作求边缘分布，队友动作固定为实际执行值。第一项是实际发生的情况。第二项是智能体 $i$ 按策略采样时的期望 $Q$。差值就是 $i$ 的边际贡献。

这不是新发明。机制设计里的 difference reward 技巧（Wolpert & Tumer 2002），套上 value-function baseline 的壳。降方差效果实打实。Foerster 等人在 SMAC 上对比共享 state-value baseline，梯度方差降了约 4 倍。

为什么 difference reward 比共享 baseline 强。盯着数学公式看一小时就明白了。$V(s)$ 这种共享 baseline 从每个智能体的 $Q(s, \mathbf{a})$ 里减掉同一个标量。$a_{-i}$ 的梯度噪声直接漏进智能体 $i$ 的更新里。反事实 baseline 减去的量恰好依赖 $a_{-i}$，能把泄漏抵消掉。套路和蒙特卡洛积分里的 control variates 一样。

实现比看起来便宜。集中式 critic 一次性输出智能体 $i$ 整个动作空间的向量。baseline 就是一个点积：

```python
def coma_advantage(critic, state, joint_actions, agent_idx, policy_probs, n_actions):
    """
    critic: 输入状态与其他智能体动作，输出指定智能体所有动作的 Q 值
    policy_probs: (batch, n_actions) — 策略概率 pi_i(.|s)
    """
    # q_all[b, a'] = Q(s, a_{-i}, a'_i = a')
    q_all = critic(state, joint_actions, agent_idx)
    actual_action = joint_actions[:, agent_idx].long()
    q_taken = q_all.gather(1, actual_action.unsqueeze(1)).squeeze(1)  # 提取实际执行动作的 Q 值
    baseline = (policy_probs * q_all).sum(dim=1)  # 计算期望 Q 作为 baseline
    return q_taken - baseline
```

SMAC 3m 地图（3 机枪对 3 机枪）。独立 actor-critic 跑 200 万步胜率 41%。COMA 同样预算 87%。地图小，集中式 critic 塞进一个 MLP 就够。提升不靠网络容量，全靠 baseline。

短板也很明显：COMA 训练时需要完整的联合动作分布。部分可观测环境下，replay buffer 重建不出 $a_{-i}$，只能退回 off-policy 修正，或者砍掉智能体维度用共享 critic。

另一个实操细节：COMA 是 on-policy 算法。优势函数必须基于*当前* $\pi_i$ 计算，不能用 replay 里的旧策略。混入过期样本会把 baseline 想消除的偏差重新引回来。我只用最近 8 个 episode 的小窗口 replay，状态很稳。窗口一拉长就漂移。

前提是智能体看得够多，能配合。看不见的时候，得靠沟通。

## Communication: Differentiable Message Passing (DIAL)

CTDE 有时不够用。智能体观测真正残缺，执行期拼不出联合状态。必须上通信信道。信道参数得自己学。难点在于离散消息没法反向传播。

[DIAL (Foerster et al. 2016)](https://arxiv.org/abs/1605.06676) 用软技巧破局。训练期发连续值消息，梯度直接穿透，当成普通激活值处理。执行期按信道硬件限制离散化（1 bit、1 byte 或 token）。训练期注入的噪声如果匹配离散化粒度，连续到离散的落差就很小。

消息里的反向梯度是核心。智能体 $j$ 发给 $i$ 的消息变成 $i$ 策略的输入。$i$ 的 loss 依赖自身动作。链式法则把 loss 信号推回 $j$ 的消息发射头。$j$ 学会发能帮 $i$ 做决策的消息。哪怕 $j$ 发消息本身没直接奖励。这招确实管用，调不通的时候也确实难查。

CommNet 风格的模块最容易塞进现有代码。每个智能体发射消息向量。消息池化（均值或注意力）。池化结果广播回去，作为额外输入喂给每个 actor。

```python
import torch
import torch.nn as nn

class CommModule(nn.Module):
    def __init__(self, obs_dim, msg_dim, hidden=128, n_heads=4):
        super().__init__()
        self.encoder = nn.Linear(obs_dim, hidden)
        self.msg_head = nn.Linear(hidden, msg_dim)
        self.attn = nn.MultiheadAttention(msg_dim, n_heads, batch_first=True)
        self.policy_head = nn.Linear(hidden + msg_dim, hidden)

    def forward(self, obs):
        # obs: (batch, n_agents, obs_dim)
        h = torch.relu(self.encoder(obs))
        msgs = self.msg_head(h)  # (batch, n_agents, msg_dim)
        # 智能体间自注意力 — 每个智能体关注所有队友
        pooled, _ = self.attn(msgs, msgs, msgs)
        out = torch.relu(self.policy_head(torch.cat([h, pooled], dim=-1)))  # 拼接局部特征与聚合消息
        return out
```

机器人项目里踩过的坑：DIAL 只在部分可观测是瓶颈时有效。全观测环境（包括小规模星际微操），CTDE 的集中式 critic 已经给足协调信号。加通信信道只会多出一堆参数，最后学会忽略自己。上消息机制前先做消融实验。

通信复杂度。稀疏 top-$k$ 注意力随 $n$ 线性增长。均值池化 $O(n)$ 但丢失个体结构。智能体超过 32 个，复杂度不是重点。重点是联合 $Q$ 函数还能不能算。

离散化噪声值得单拎出来。DIAL 训练用连续消息加高斯噪声，标准差 $\sigma$ 对齐离散化步长。测试期消息四舍五入到最近网格点。$\sigma$ 太小，部署策略因为舍入分布外偏移，表现不如训练期。$\sigma$ 太大，训练期学不到有效信号。手动调参时，只有 $\sigma$ 匹配分箱大小这一条最关键。

目前聊的都是合作场景。竞争博弈的坑，得另开一篇填。

## QMIX: Mixing Networks for Cooperative Tasks

CTDE 的扩展瓶颈卡在集中式 critic 本身。$N$ 个智能体，每人 $|A|$ 个动作。联合动作空间大小 $|A|^N$。集中式 $Q$ 网络得在这个空间上学习。执行期还得遍历空间找联合 argmax。$N$ 超过 8 直接算不动。

[QMIX (Rashid et al. 2018)](https://arxiv.org/abs/1803.11485) 把联合动作价值分解成个体分量，用 mixing network 组合：
$$Q_{\text{tot}}(s, \mathbf{a}) = f_{\text{mix}}\!\left(Q_1(s, a_1), \dots, Q_N(s, a_N); s\right),$$
$f_{\text{mix}}$ 是前馈网络。**权重非负**。通过 hypernetwork 用全局状态 $s$ 条件化。非负性保证单调性：
$$\frac{\partial Q_{\text{tot}}}{\partial Q_i} \geq 0 \quad \forall i,$$
这才是核心。单调性保证：
$$\arg\max_{\mathbf{a}} Q_{\text{tot}}(s, \mathbf{a}) = \left(\arg\max_{a_1} Q_1(s, a_1), \dots, \arg\max_{a_N} Q_N(s, a_N)\right),$$
执行期每个智能体按自己的 $Q_i$ 贪心行动。得出的联合动作在集中式 $Q_{\text{tot}}$ 下严格最优。

实现分两块：个体价值头，生成 mixing 权重的 hypernetwork。

```python
import torch
import torch.nn as nn

class QMixer(nn.Module):
    def __init__(self, n_agents, state_dim, mix_hidden=32):
        super().__init__()
        self.n_agents = n_agents
        self.mix_hidden = mix_hidden
        # Hypernetwork: 状态映射为 mixing 权重 (abs 保证非负)
        self.hyper_w1 = nn.Linear(state_dim, n_agents * mix_hidden)
        self.hyper_w2 = nn.Linear(state_dim, mix_hidden)
        self.hyper_b1 = nn.Linear(state_dim, mix_hidden)
        self.hyper_b2 = nn.Sequential(
            nn.Linear(state_dim, mix_hidden), nn.ReLU(),
            nn.Linear(mix_hidden, 1),
        )

    def forward(self, agent_qs, state):
        # agent_qs: (batch, n_agents)
        # state: (batch, state_dim)
        b = agent_qs.size(0)
        w1 = torch.abs(self.hyper_w1(state)).view(b, self.n_agents, self.mix_hidden)
        b1 = self.hyper_b1(state).view(b, 1, self.mix_hidden)
        hidden = torch.relu(torch.bmm(agent_qs.unsqueeze(1), w1) + b1)  # 矩阵乘法计算隐藏层
        w2 = torch.abs(self.hyper_w2(state)).view(b, self.mix_hidden, 1)
        b2 = self.hyper_b2(state).view(b, 1, 1)
        q_tot = torch.bmm(hidden, w2) + b2  # 计算最终 Q_tot
        return q_tot.view(b)
```

SMAC `3s5z` 地图（3 追猎 5 狂热者对同配置）。MADDPG 胜率卡在 10% 左右。联合 critic 泛化不了异构兵种。QMIX 同样预算跑到 92%。分解结构在干实事。这不是常数级加速。

已知缺陷：QMIX 强推的 IGM (individual-global-max) 性质是充分条件，不是必要条件。最优联合动作需要非单调信用分配的任务（一个智能体 $Q$ 上升必须以另一个下降为代价），QMIX 无法精确表示。QTRAN 和 QPLEX 专门解决这个问题。实操中除了人造反例，很少碰到失效场景。

踩坑换来的训练建议：个体 $Q_i$ 网络和 mixer 别共用学习率。hypernetwork 输出被绝对值门控，mixer 看到的有效梯度范数小得多。学习率对齐的话，前 500 万步 mixer 根本学不动。我把 mixer 学习率开到 agent 的 3 倍，节奏就对了。

合作场景的套路基本盘完了。竞争博弈的坑，得另开一篇填。

## Self-Play and Population-Based Training

对抗游戏里，naive self-play 是个坑。智能体只跟当前的自己打，确实会变强，但提升方向完全受限于当前策略暴露的弱点。半年前早就修好的 failure mode，过阵子又会原封不动冒出来。论文里管这叫“石头剪刀布循环”。我亲眼见过一个 Dota bot，明明 20 万步前守塔守得好好的，后来居然把这招忘得一干二净。

业界常用两招破局。**Fictitious self-play** 让当前模型跟历史版本的均匀混合体打。它对标的是平均策略，不是最新版。**League training** 是 AlphaStar 的变种。它把思路往前推一步，引入多个子种群：main agents（最终上线的目标）、main exploiters（专职找 main agents 的漏洞）、league exploiters（定期重置和分叉，负责保住策略多样性）。

看数据最直观。AlphaStar 的公开结果显示，同等算力下，pure self-play 卡在 Diamond 段位就上不去了。上 full league 能冲到 Grandmaster。算力没变，变的是 curriculum。

一个最简 league 实现长这样：

```python
import random
from collections import deque

class League:
    def __init__(self, max_history=200):
        self.main = None
        self.main_exploiters = []
        self.history = deque(maxlen=max_history)

    def sample_opponent(self, learner_role):
        if learner_role == "main":
            # 50% 最新 checkpoint，35% 历史版本，15% exploiter
            r = random.random()
            if r < 0.5 and self.history:
                return random.choice(list(self.history)[-20:])
            if r < 0.85 and self.history:
                return random.choice(self.history)
            return random.choice(self.main_exploiters) if self.main_exploiters else self.main
        if learner_role == "main_exploiter":
            return self.main  # 始终跟当前 main 训练
        if learner_role == "league_exploiter":
            return random.choice(self.history) if self.history else self.main

    def checkpoint(self, agent):
        self.history.append(agent.snapshot())
```

往合作方向看，这套逻辑跟 [PSRO](https://arxiv.org/abs/1711.00832)（Policy-Space Response Oracles）是通的。PSRO 把对抗型 MARL 抽象成 meta-game。动作是策略，收益矩阵是“策略 A 打策略 B 的期望回报”。内层跑环境，外层用 double oracle 或 Nash equilibrium 算 meta-payoff 矩阵。League training 是填充矩阵的一种调度方式，fictitious self-play 是另一种。视角统一之后，事情就清楚了：合作型 CTDE 就是单策略类加可加收益的 PSRO；对抗型 league training 是多策略类加零和收益的 PSRO。剩下的全是工程活。

讲个踩坑的经历。有次我调对抗型 MARL，整整两周模型啥也没学到。reward 曲线一条直线。打最新 checkpoint 的胜率卡在 50%，纯靠运气。查到最后发现，对手采样窗口只开了最近 5 个 checkpoint。这 5 个全是同一个近似最优策略的变体，模型早就背熟了。把采样窗口拉到 200 个 checkpoint 之后，模型立刻从老版本里挖出漏洞，顺势 bootstrap 上去。策略变强，不是算法变聪明了，而是 *opponent distribution* 变宽了。League training 的核心教训，全在这段话里。

用 meta-game 的视角看 MARL，RLHF reward modelling 和 competitive self-play 的界限就会变模糊。接下来，Part 12 会展开聊这块。

## 常见问题

### 为什么 QMIX 需要单调性约束？

为确保分散贪心执行与集中联合优化一致。若 $\partial Q_\text{tot}/\partial Q_i$ 可为负，智能体在自身 $Q_i$ 上贪心反而可能降低团队价值。单调性保证了个体 argmax 与联合 argmax 重合。

### MARL 系统何时会陷入次优均衡？

当最优联合动作需*协调探索*时——例如两个智能体必须同时尝试高风险动作才能显现收益。若随机探索几乎无法产生该联合动作，系统将收敛至更安全但被 Pareto 支配的均衡。解决方法包括联合探索（committed-exploration）、显式通信与对手建模。

### 样本复杂度如何随智能体数量变化？

对适用值分解的合作任务，样本复杂度大致随 $n$ 线性增长——这正是 QMIX 的核心价值。若无分解，直接学习 $Q(s, \mathbf{a})$ 的复杂度将随 $n$ 指数爆炸。

### 独立学习者 vs CTDE，CTDE 总是有优势吗？

若智能体极少或全局状态确实不可得，独立 Q-learning 也能工作且实现简单。但一旦信用分配重要或团队规模超过 3–4 个智能体，CTDE 方法优势显著，且差距迅速拉大。

### 这些思想能迁移到多 LLM agent 系统吗？

完全可以。集中评论家对应外层评估器，分散执行者对应子 agent，注意力通信对应工具路由，联赛训练则对应“评估者种群”模式——后者已成为 agentic RLHF 中的标准设计。
