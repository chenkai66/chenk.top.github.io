¶Spider Technique Description

According to the definition of Spyder Website, spiders are classes which define how a certain site (or a group of sites) will be scraped, including how to perform the crawl (i.e. follow links) and how to extract structured data from their pages (i.e. scraping items). In other words, Spiders are the place where you define the custom behaviour for crawling and parsing pages for a particular site (or, in some cases, a group of sites).

To implement a basic crawler, we need to implement the following steps:

  1. Understand web pages;
  2. Use the requests library to scrape website data;
  3. Parse web pages with Beautiful Soup;
  4. Cleaning and organizing data.

Take the home page of China Travel Network as an example. We need to grab the first information (title and link) of the home page of China Travel Network, and the data appear in the source code in plain text.

¶HTML

HTML is the structure of the entire web page, equivalent to the framework of the entire website. With <", "> symbols are HTML tags, and tags are in pairs.

  • *<html>..</html> indicates that the element in the middle of the tag is a web page
  • <body>..</body> represents user-visible content
  • <div>..</div> represents the frame
  • <p>..</p> means paragraph
  • <li>..</li> means a list
  • <img>..</img> means image
  • <h1>..</h1> indicates the title
  • <a href="">..</a> means a hyperlink

¶CSS

CSS represents style. <style type="text/css"> means that the following appearance is defined in CSS.

¶JavaScript

The interactive content and various special effects are in JScript, which describes various functions in the website.

If the human body is used as an analogy, HTML is the human skeleton and defines where the human mouth, eyes, ears, etc. should grow. CSS is the appearance details of a person, such as what the mouth looks like, whether the eyes are double or single eyelid, whether the eyes are big or small, whether the skin is black or white, etc. JScript represents human skills, such as dancing, singing, or playing an instrument.

¶robots protocal

Almost every website has a document called robots.txt, and of course some websites do not have a robots.txt set. For websites without robots.txt set, data without password encryption can be obtained through web crawlers, that is, all page data of the website can be crawled. If the site has a robots.txt document, it is necessary to determine whether there is data that is prohibited for visitors.

¶Spider Basic

The process of web page request is divided into two parts:

  • Request: Every web page displayed to the user must go through this step, which is to send an access request to the server.
  • Response: After the server receives the user’s request, it will verify the validity of the request, and then send the content of the response to the user (client). The client receives the content of the server’s response and displays the content.
