This post introduces the origin and evolution of kernel functions, their mathematical foundations, and why they became a core tool in machine learning. The focus is on building a clean mental model: what a kernel is, why positive definiteness matters, and how the “kernel trick” enables non-linear learning with linear algorithms.

Origin and development of kernel functions

¶From orthogonal functions to kernels

Orthogonal function systems play a central role in analysis and function spaces. The idea goes back to Fourier analysis: representing periodic functions using orthogonal trigonometric bases (sines and cosines) made many PDE problems tractable. With the development of Hilbert space theory, orthogonal systems were generalized and became indispensable for studying linear operators and integral equations.

¶A short history of kernel methods

Kernel theory grew out of integral equations (late 19th century) and functional analysis (early 20th century). Mercer (1909) provided the foundational expansion above. Later, the concept of a Reproducing Kernel Hilbert Space (RKHS) was formalized (Moore–Aronszajn theory), making the “kernel = inner product” viewpoint precise.

In statistics, kernel density estimation (KDE) became a major non-parametric tool (Rosenblatt, Parzen). In machine learning, Vapnik’s support vector machines popularized the kernel trick, making non-linear classification/regression practical at scale.

¶Why kernels matter in machine learning

Many real-world datasets are not linearly separable in the raw input space. Kernels let you work as if you mapped data into a higher-dimensional feature space—sometimes infinite-dimensional—without explicitly constructing that mapping.

Mathematical foundations

¶Positive definite and positive semidefinite kernels

A kernel is (semi)positive definite if, for any finite set ${x_i}{i=1}^n{c_i}{i=1}^n$,

$$
\sum_{i=1}n\sum_{j=1}n c_i c_j K(x_i, x_j) \ge 0.
$$

Equivalently, the Gram matrix is positive semidefinite. This property is what allows kernels to behave like inner products in some (possibly implicit) feature space.

¶The kernel trick

Instead of explicitly computing in a high-dimensional space, you compute in the input space. This reduces complexity and avoids numerical issues when is high-dimensional or infinite-dimensional.

Common kernels

¶Gaussian / RBF kernel

$$
K(x, y) = \exp\left(-\frac{|x - y|2}{2\sigma2}\right)
$$

is the bandwidth (length-scale). RBF kernels are smooth, expressive, and widely used, but sensitive to hyperparameter choice and potentially expensive for large datasets.

¶Polynomial kernel

It models interactions up to degree and is often used when polynomial structure is expected.

¶Sigmoid kernel

Inspired by neural activations; it can work in some settings but is less robust and has stricter parameter regimes to remain PSD.

¶Matérn kernel

Common in Gaussian processes. controls smoothness and controls length scale.

¶Epanechnikov kernel (KDE)

Often used in density estimation; compact support and optimal variance among certain kernel estimators.

¶Exponential kernel

Captures local structure with a faster decay than RBF.

¶Linear kernel

The baseline choice for large sparse problems (e.g., text), where linear separation is often sufficient.
