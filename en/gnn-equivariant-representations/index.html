Graph Neural Networks for Learning Equivariant Representations of Neural Networks

Neural network parameters live in a space with strong permutation symmetries: you can reorder hidden units without changing the function, yet the raw weight tensors look completely different. If a representation ignores this, it ends up learning spurious differences and struggles to generalize across architectures or widths. This paper proposes representing a neural network as a neural graph (nodes as neurons/bias features, edges as weights) and then using a GNN to produce equivariant representations that respect these symmetries.

1. Why equivariance matters for ‚Äúlearning over networks‚Äù

Many tasks treat an entire neural network as a data point:

  • predict generalization from weights
  • classify networks by behavior
  • retrieve similar networks
  • meta-learning / architecture analysis

But weights have a key nuisance symmetry: in an MLP, you can permute hidden units (and permute the corresponding incoming/outgoing weights) without changing the function. Any representation that is not symmetry-aware will treat equivalent networks as different.

2. Neural graphs: turning weights into a graph

The core idea is to represent a neural network as a graph:

  • Nodes represent neurons (and optionally include bias features).
  • Edges represent connections with weights as edge features.

With this representation, applying a GNN yields a natural path to permutation-equivariant features: GNN message passing is inherently equivariant to node permutations.

3. What ‚Äúequivariant‚Äù means here

Informally:

  • If you permute hidden units in the underlying neural network, the neural-graph node ordering changes.
  • A good encoder should transform in a consistent way, rather than producing unrelated embeddings.

Equivariance is stronger than invariance: you may want node-level embeddings to permute correspondingly, and then optionally produce a graph-level invariant embedding via pooling.

4. Model sketch

A typical pipeline looks like:

  1. Build the neural graph from a network‚Äôs parameters.
  2. Run message passing (node update functions aggregate neighbor messages).
  3. Pool node representations to get a graph embedding.
  4. Train the embedding for a downstream objective (regression/classification/retrieval).

5. Takeaways

  • Flattening parameters loses structure and makes symmetry handling hard.
  • Graph-based representations allow using GNNs to respect permutation symmetries.
  • The approach is particularly appealing when comparing networks across widths/architectures, where naive parameter alignment is not meaningful.
