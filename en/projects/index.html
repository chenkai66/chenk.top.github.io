DaaS — Documentation as a Skill

Infrastructure · Doc pipeline · Agent toolchain

Turn any product's docs into AI-agent-ready Skill files.

Point it at a folder of docs. It reads everything, figures out the product's capability structure, and writes two kinds of Skill: detail skills (auth, endpoints, examples per capability) and intent skills (decision trees that pick the right path among many for the same goal).

company-skill.com is the visual console: Knowledge Base browses distilled Skills, Ingest feeds new docs, Workspaces switch products, Routing Playground tests intent decisions, Telemetry runs adversarial recall evals.

Built for any company — distill an entire organization into one Skill that agents call directly. No human writes a single line of Skill content.

AI4Science

Architecture · Agent control plane
long-running observation

An autonomous research agent loop — relentless and self-sustaining.

Built on the OpenClaw runtime: drop in a research question, the system reads papers, extracts methods, runs experiments, persists intermediate state, and produces a report. Shared memory keeps every session, and a harness compresses failed attempts into the starting point of the next.

Full service runs 15 internal modules — Chat / Overview / Files / Sessions / Logs / API Docs / Costs / Limits / Memory / Live Feed / Security / Sys Security / Config / Docker — covering experiment orchestration, datasets, and cross-provider token & rate-limit monitoring end-to-end.

Public showcase at llm4science.top exposes Chat / Files / Sessions / Memory so visitors can watch the agent loop run live; orchestration logic, private datasets, and provider credentials stay internal.

llm-elevator

Performance layer · Prompt engineering · Eval pipelines

Squeeze more real-world impact from frontier LLMs — where it actually matters.

The same model can land at very different completion rates depending on how it's prompted, what harness wraps it, and what eval loop tunes it. llm-elevator distills that craft into reusable parts: structured prompt templates, agent harnesses (plan / reflect / tool loops), and adversarial eval pipelines aimed at the gap between “the demo works” and “production holds up.”

Every technique ships with a baseline comparison answering three questions: how much it lifts, how much it costs, and how stable the lift is — instead of vibe-tuning prompts.
