Linux feels hard not because commands are complicated, but because you need a clear system map: files and permissions, processes and resources, networking, and the mental model behind it all. This post gives a practical on-ramp: enough concepts and commands to operate a server confidently.

1. Core concepts

  • Everything is a file: devices and many system interfaces appear under /dev, /proc, /sys.
  • Multi-user: ownership and groups are fundamental.
  • Permissions: read/write/execute behave differently for files vs directories.
