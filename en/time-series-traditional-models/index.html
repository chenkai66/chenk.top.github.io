ARIMA (AutoRegressive Integrated Moving Average)

¬∂The Big Picture

ARIMA is the workhorse of univariate time series forecasting. It assumes the current value depends on:

  1. Past values (AutoRegressive, AR)
  2. Past forecast errors (Moving Average, MA)
  3. Differencing to handle trends (Integrated, I)

When ARIMA works best:

  • Univariate data with linear patterns
  • Moderate sample size (50-500 points)
  • Stationary or can be made stationary via differencing
  • Short-term forecasting (1-10 steps ahead)

¬∂Model Components

ARIMA is denoted as ARIMA(p, d, q) where:

  • p: AR order (how many past values to use)
  • d: Differencing order (how many times to difference to achieve stationarity)
  • q: MA order (how many past errors to use)

¬∂Intuition: ARIMA as a ‚ÄúMemory System‚Äù

Think of forecasting tomorrow‚Äôs temperature:

AR (AutoRegressive): ‚ÄúYesterday was 20¬∞C, day before was 22¬∞C ‚Üí tomorrow probably 21¬∞C‚Äù

  • Uses historical values to find patterns

I (Integrated): ‚ÄúTemperature keeps rising (summer coming) ‚Üí raw values don‚Äôt work‚Äù

  • Differencing = look at changes instead of absolute values
  • ‚ÄúToday +2¬∞C vs yesterday, yesterday +1.5¬∞C ‚Üí tomorrow maybe +1.7¬∞C‚Äù

MA (Moving Average): ‚ÄúMy forecast was off by +3¬∞C yesterday, +2¬∞C day before ‚Üí today probably +2.5¬∞C‚Äù

  • Learns from past forecast errors

Combined: ARIMA = ‚ÄúI predict based on past values + trend changes + my past mistakes‚Äù

¬∂Why SARIMA?

Problem: Ice cream sales prediction

  • SARIMA: Also remembers ‚Äúsales this day last year‚Äù (seasonality)

When SARIMA shines:

  • Data with clear periodic patterns (daily, weekly, yearly)
  • Retail sales, website traffic, energy consumption
  • When ‚Äúsame time last cycle‚Äù matters

¬∂The Linkage System

Problem: Stock and bond markets often move together (seesaw effect)

  • Traditional models: Predict each independently
  • VAR: Predict multiple variables jointly, capturing cross-dependencies

When to use VAR:

Kalman Filter

¬∂State-Space Estimation

Kalman Filter is a recursive algorithm for estimating the state of a system from noisy observations.

When to use:

  • Sensor fusion (GPS + IMU)
  • Real-time tracking (object detection)
  • Time series with measurement noise

Key idea: Combine prediction (physics model) with observation (noisy measurement).

¬∂Mathematical Framework

State transition:

$$
\mathbf{x}t = \mathbf{F} \mathbf{x}{t-1} + \mathbf{B} \mathbf{u}_t + \mathbf{w}_t
$$

Observation:

where:

  • : hidden state
  • : observation
  • : process and measurement noise

Two-step update:

  1. Predict:
    $$
    \begin{aligned}
    \hat{\mathbf{x}}{t|t-1} &= \mathbf{F} \hat{\mathbf{x}}{t-1|t-1} \
    \mathbf{P}{t|t-1} &= \mathbf{F} \mathbf{P}{t-1|t-1} \mathbf{F}^\top + \mathbf{Q}
    \end{aligned}
    $$

  2. Update (correct with observation):
    $$
    \begin{aligned}
    \mathbf{K}t &= \mathbf{P}{t|t-1} \mathbf{H}^\top (\mathbf{H} \mathbf{P}{t|t-1} \mathbf{H}^\top + \mathbf{R})^{-1} \
    \hat{\mathbf{x}}    {t|t} &= \hat{\mathbf{x}}_{t|t-1} + \mathbf{K}_t (\mathbf{y}t - \mathbf{H} \hat{\mathbf{x}}{t|t-1})
    \end{aligned}
    $$

¬∂Parameter Selection Rules

ARIMA (p, d, q):

  • d: Use ADF test ‚Üí d = 0 (stationary), 1 (linear trend), 2 (quadratic)
  • p: ACF decays slowly ‚Üí Start with p = 1-3
  • q: PACF cuts off ‚Üí Start with q = 1-3
  • Selection: Grid search with AIC (prediction) or BIC (parsimony)

SARIMA (p,d,q)(P,D,Q,m):

  • m: Seasonal period (12 for monthly, 7 for weekly)
  • P, D, Q: Same logic as p, d, q but at lag m
  • Quick: Use auto_arima with seasonal=True

VAR:

  • Lag: Use AIC/BIC or Granger causality test
  • Max lag: or 5-10 for small datasets

GARCH:

  • p, q: Usually (1,1) or (2,1) sufficient
  • Distribution: Normal (simple) or t-distribution (fat tails)

¬∂Diagnostic Checklist

Before trusting your model, verify:

  1. ‚úÖ Stationarity: ADF test p < 0.05
  2. ‚úÖ Residuals: White noise (Ljung-Box p > 0.05)
  3. ‚úÖ Variance: Constant (ARCH test p > 0.05)
  4. ‚úÖ Information criteria: AIC/BIC reasonable
  5. ‚úÖ Parameters: Significant (p-values < 0.05)
  6. ‚úÖ Forecast accuracy: Test RMSE comparable to train

¬∂Common Mistakes to Avoid

‚ùå Don‚Äôt: Apply ARIMA to non-stationary data without differencing
‚úÖ Do: Check stationarity first, difference if needed

‚ùå Don‚Äôt: Use high p, q on small datasets
‚úÖ Do: Limit complexity: p, q ‚â§ 2 for n < 100

‚ùå Don‚Äôt: Ignore residual diagnostics
‚úÖ Do: Always check residuals are white noise

‚ùå Don‚Äôt: Forecast too far ahead with ARIMA
‚úÖ Do: Use ARIMA for short-term (1-10 steps), Prophet/DL for long-term

‚ùå Don‚Äôt: Use GARCH for non-financial data
‚úÖ Do: GARCH only for volatility clustering (financial returns)
