Vim Essentials (Modal Editing in Practice)

Vim‚Äôs core idea is modal editing: the same keys do different things depending on the mode, which enables extremely fast navigation and editing once the workflow ‚Äúclicks.‚Äù This post breaks down the key modes (normal/insert/visual/command-line), the most useful motions and text objects, and the handful of patterns that make Vim feel efficient instead of overwhelming. If you want a practical path to ‚Äúediting at speed‚Äù rather than memorizing random shortcuts, start here.

1. Essential modes and operations

Vim is built around modes: the same keys do different things depending on the mode. Once you understand how modes combine with motions and operators, editing becomes fast and predictable. The core modes are:

  1. Normal mode: Vim‚Äôs default mode for navigation and editing commands (delete/yank/paste).
  2. Insert mode: enter with i, a, o, etc. for regular typing.
  3. Visual mode: v (character), V (line), or Ctrl+v (block) to select text and apply operations in bulk.
  4. Command-line mode: press : in Normal mode to run Ex commands (save/quit/search/replace, etc.).
  5. Replace mode: press R to overwrite text as you type.

Press Esc to return to Normal mode.

1. File operations

  • :w: write (save).
  • :q: quit.
  • :wq or :x: save and quit.
  • :q!: quit without saving.
  • :e filename: open/edit another file.
  • :saveas filename: save as (you can also do :w /path/to/file).

These are Ex commands, entered after pressing : from Normal mode.

2. Cursor movement and paging

  • h / j / k / l: left / down / up / right by one character.
  • 0: start of line; ^: first non-blank character.
  • $: end of line.
  • w / b: next / previous word.
  • Ctrl+f / Ctrl+b: page down / up.
  • gg: top of file; G: bottom of file.
  • {number}G: jump to a specific line.

3. Delete, yank, paste, undo

  • Delete
    • x: delete character under cursor.
    • dd: delete current line.
    • d{motion}: delete a range defined by a motion (e.g. dw, d$).
    • G: from the current line position to end of file (combined with operators like dG).
  • Yank (copy)
    • yy: yank current line.
    • y{motion}: yank a range (e.g. yw for a word).
  • Paste
    • p: paste after cursor / below.
    • P: paste before cursor / above.
    • For large pastes, consider paste mode: :set paste (and later :set nopaste).
  • Undo / redo
    • u: undo.
    • Ctrl+r: redo.

4. Cut (move text)

  • dd deletes the line and puts it into a register (so it behaves like ‚Äúcut‚Äù).
  • A typical move workflow:
    1. dd (cut a line)
    2. move the cursor
    3. p (paste)

5. Handy command-line commands

  • :set number: show line numbers.
  • :noh: clear search highlights.
  • :map / :verbose map: inspect mappings (useful for debugging keybinding conflicts).
  • :!<command>: run a shell command, e.g. :!ls.

2. Advanced but practical features

1. Splits, windows, and tabs

For large files or multi-file workflows, splits and tabs are extremely useful:

  • :split / :sp: horizontal split.

  • :vsplit / :vsp: vertical split.

  • Ctrl+w then h/j/k/l: move between splits.

  • :close: close the current split.

  • :only: keep the current split, close others.

  • :tabnew: open a new tab.

  • gt / gT: switch between tabs.

  • When the same file is opened by multiple Vim instances (or Vim previously crashed), you may see a swap file (.swp) warning.

    If you don‚Äôt need recovery, delete the swap file. If you do, follow Vim‚Äôs prompt to recover, then clean up the .swp.

Used well, splits and tabs make side-by-side editing and comparison much faster.

2. Macros (record & replay)

Macros are extremely effective for repetitive edits: record a sequence once, then replay it on other lines or blocks.

  • q{register}: start recording into a register (e.g. qa records into register a).
  • perform any edits (move/delete/insert, etc.)
  • q: stop recording.
  • @{register}: replay the macro.
  • @@: replay the last-used macro again.

To repeat N times, use {number}@{register} in Normal mode (e.g. 10@a).

3. Code folding

When working in large files, folding helps you hide what you don‚Äôt need and focus on one block at a time:

  • zR: open all folds.
  • zM: close all folds.
  • za: toggle the fold under cursor.
  • zc: close current fold.
  • zo: open current fold.

This is especially useful for functions, classes, or long configuration blocks.

4. Search & replace

Vim supports flexible substitution with regular expressions. Common patterns:

:s/old/new/g
Replace matches in the current line (add/remove g for all vs first match).

:%s/old/new/g
Replace matches in the whole file.

For a specific line range, prefix with line numbers:

  • :10,20s/foo/bar/g: only lines 10‚Äì20.
  • Regex groups and patterns make substitutions much more powerful.

5. Customization & configuration

Vim can be customized via ~/.vimrc (or ~/.config/nvim/init.vim for Neovim). Common settings include:

  • set number: line numbers.
  • set expandtab, set tabstop=4, set shiftwidth=4: indentation style.
  • syntax on: syntax highlighting (filetype detection also depends on extensions and can be helped by shebang lines like #!/bin/bash).
  • set relativenumber: relative line numbers.
  • autocmd ...: automatic commands (e.g., remove trailing whitespace on save).
